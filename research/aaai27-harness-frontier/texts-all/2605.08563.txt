                                              Abstract. When an LLM agent fails a multi-step tool-augmented task
                                              and retries, the failed attempt typically remains in its context window—
                                              contaminating the next attempt and elevating the per-step error rate
                                              beyond the base level. This context-contaminated restart phenomenon
                                              is widely observed in practice yet entirely lacks formal treatment. We
                                              introduce the Context-Contaminated Restart Model (CCRM): a
                                              chain of T tool-call steps, each failing with base rate ε0 ; after any failed
                                              attempt, the subsequent attempt operates in contaminated context with
                                              elevated error rate ε1 > ε0 . Under this model we derive five main results.
                                              (R1) An exact closed-form formula for P (succeed in ≤ K attempts).
                                              (R2) A cascade-overhead theorem giving the additional attempts ∆K
                                              incurred by contamination versus the clean-restart baseline. (R3) An
                                              optimal budget-allocation theorem identifying the pipeline depth T ∗ that
                                              maximises success probability

                                        1    Introduction
                                        Tool-augmented LLM agents execute tasks by planning and invoking external
                                        APIs, web search engines, code interpreters, and databases [4, 6, 8, 9]. When such
                                        an agent fails a complex task, it typically retries—but the failure is preserved in
                                        its context window as history. Practitioners have documented the consequence:
                                        “early incorrect attempts remained in the conversation history and contaminated
                                        the final response” [25]; agents “repeatedly referenced the same bad endpoint in
                                        future attempts because it had learned from its own mistake” [25]. Datadog’s
                                        2026 engineering survey reports that 5% of all LLM call spans return errors [22],
2       Zhanfu Yang

yet no formal theory quantifies how retry contamination scales with task depth,
error rate, or budget.
    Existing theoretical work on agent reliability studies single-run reliability [26,
28] or empirical failure catalogues [11], but does not model the cross-attempt
contamination that defines real-world retry behaviour. We fill this gap.

Contributions. We introduce the Context-Contaminated Restart Model
(CCRM) and prove:
R1. Exact CCRM formula (Theorem 1). P (succeed in ≤ K attempts) =
    p0 + (1 − p0 )[1 − (1 − p1 )K−1 ] where pi = (1 − εi )T , with complete Markov-
    chain proof.
R2. Cascade overhead (Theorem 2). Closed-form ∆K = KCCRM − KIID ;
    a phase transition in ε1 /ε0 makes ∆K → ∞.
R3. Optimal pipeline depth (Theorem      q 3). For budget B = KT , the unique
    maximiser of P (success) is T ∗ = B · log(1/(1−ε   1 ))
                                             log(1/(1−ε0 )) , proved by minimising
    a log-convex objective.
R4. Le Cam lower bound (Theorem 4). No policy can achieve P (success) ≥
    1 − δ with fewer than KCCRM − O(1) attempts, via a two-hypothesis con-
    struction.
R5. Clean-restart dominance (Theorem 5). Context-clearing before retry
    strictly reduces required attempts; the exact improvement ratio is derived.

Relation to prior work. Tran-Truong and Le [28] fit absorbing Markov chains to
single-run agent traces (within one attempt); they do not model restart contam-
ination, provide no optimal control results,
                                           √ and give no information-theoretic
lower bounds. Fan et al. [26] derive O( T ) martingale bounds on single-run
distortion in MCP pipelines; cross-attempt dynamics are outside their scope.
Patel et al. [27] study consensus voting for reliability, not restart contamination.
Table 1 summarises the differences. To our knowledge, no prior work models or
analyses the cross-attempt cascade studied here.
    Empirically, reliability decay as a function of task depth has been docu-
mented at scale—Khanal et al. [23] show super-linear degradation across 23,392
episodes; Wang et al. [24] diagnose systematic failure modes in long-horizon
agentic tasks, and budget allocation for tool-calling agents has been studied
empirically [15, 20, 21]—all showing that naive retry strategies waste signifi-
cant compute. None of these works provide a formal cascade model, closed-form
success formula, or the information-theoretic lower bound of Theorem 4. LLM
agents benefit from data-efficient post-training [12] and reasoning-aligned pref-
erence optimisation with implicit tree search [16]; such training reduces the base
error rate ε0 , and CCRM’s Theorem 3 shows this has superlinear benefit when
the cascade ratio ε1 /ε0 is large. Test-time compute allocation for multi-stage
tasks [10] and web agents [18] show that adaptive budget distribution outper-
forms uniform allocation; Theorem 3 provides the theoretical optimum for the
cascade-restart setting. Systematic failure diagnostics [11, 17] confirm that retry
contamination is among the most prevalent failure modes in deployed agents.

2     The Context-Contaminated Restart Model

2.1   Pipeline Setup

Definition 1 (Tool-call pipeline). A pipeline of depth T is a sequence of T
tool invocations executed in order. The pipeline succeeds iff all T invocations
succeed; it fails on the first failure.



2.2   Stochastic Model

Definition 2 (CCRM). The Context-Contaminated Restart Model (ε0 , ε1 , T ),
0 < ε0 ≤ ε1 < 1:

 – Contamination state: Zk ∈ {0, 1}, Z1 = 0.
 – Per-attempt success: P (Sk = 1 | Zk = i) = (1 − εi )T =: pi .
 – Transition: Zk+1 = 1[Sk = 0].

Remark 1 (Realism). Failed attempts pollute the LLM context with incorrect
outputs and erroneous reasoning chains [11, 25], degrading subsequent attempts.
The binary state is the minimal non-trivial extension of the IID model.

Assumption 1 (Well-separated rates). p1 := (1 − ε1 )T < p0 := (1 − ε0 )T
(strict, since ε1 > ε0 ).

2.3   Budget and Objective
Budget B = total tool invocations; maximum attempts K = ⌊B/T ⌋. Primary
objective: choose T to maximise P (succeed in ≤ K attempts).

6                       Clean-Restart Dominance

Definition 3 (Clean-restart). Reset Zk = 0 before each attempt (clear context
window). Pclean (EK ) = 1 − (1 − p0 )K .

Theorem 5 (Clean-restart dominance). For any K ≥ 2 and ε1 > ε0 :
Pclean (EK ) > PCCRM (EK ). The improvement ratio is

                                                               Pclean        1 − (1 − p0 )K
                                                                      =                           .                                                                                                         (8)
                                                              PCCRM     1 − (1 − p0 )(1 − p1 )K−1

Proof. Pclean − PCCRM = (1 − p0 )[(1 − p1 )K−1 − (1 − p0 )K−1 ]. Since p1 > p0 :
1 − p1 < 1 − p0 , so (1 − p1 )K−1 < (1 − p0 )K−1 for K ≥ 2, giving the difference
< 0, i.e. Pclean > PCCRM .                                                     ⊔

Corollary 3 (Savings). KCCRM −Kclean ≈ Kclean (log(1−p0 )/ log(1−p1 )−1) ≥
0. For SWE-bench parameters (p0 =0.761, p1 =0.113, K=3): improvement ratio
≈ 1.21—same budget resolves 21% more tasks.

8                        Discussion
Practical implications. (i) Measure ε1 /ε0 via preliminary trials; use Remark 3 to
check feasibility. (ii) Set T ∗ from Theorem 3. (iii) Clear context before retrying
(Theorem 5)—always beneficial.
    A fourth implication: reducing ε0 via efficient fine-tuning [12] or search-
aligned optimisation [16] has superlinear benefit when ε1 /ε0 is large, because
it simultaneously reduces ε1 = rε0 , shrinking KCCRM doubly fast.

Limitations. (L1) Binary contamination; continuous decay is future work. (L2)
Per-step independence within an attempt is a simplification. (L3) Early-exit
pipelines change the budget accounting. (L4) Correlated task difficulty (Sec-
tion 7.2) lies outside CCRM.
       Why Retrying Fails: Context Contamination in LLM Agent Pipelines    9

Connection to prior work. Tran-Truong and Le [28] fit Markov chains within a
single run; CCRM is a Markov chain across runs. Combining both is an impor-
tant open problem.


9   Conclusion
We introduced CCRM as the first formal framework for cross-attempt context
contamination in LLM agent pipelines. Five theorems—exact formula, cascade
overhead, optimal depth, Le Cam lower bound, and clean-restart dominance—
provide both rigorous foundations and immediately actionable principles. Real
SWE-bench data confirm that the IID assumption overestimates pass@3 by 17.4
percentage points, while CCRM fits with error < 0.001.
