A. Extended System Descriptions
This appendix elaborates the four prompt-composed agent systems summarized in Table 1, providing the per-system
descriptions and the maturation pattern referenced from §2.

OpenClaw. OpenClaw (2026) (379K GitHub stars) exemplifies the paradigm at maximum scale. Users compose
personality files (SOUL.md), capability modules (SKILL.md), and orchestration rules (AGENTS.md) that are concatenated
into the system prompt at startup (Steinberger, 2026). Over 10K community-authored skills are installable from a public
registry, with no sandboxing between skill contexts—composition is resolved entirely by the LLM’s attention mechanism.

Maturation pattern. As systems mature, composition migrates out of prompts into code or external services (conventions
→ templates → class inheritance → prompt-management services)—tacit acknowledgment that text-level composition is
fragile. Yet the most widely deployed systems (OpenClaw, AGENTS.md/CLAUDE.md across thousands of repositories)
still compose at the text level, with composition resolved at runtime by the LLM’s attention mechanism rather than by any
explicit isolation primitive.
