Realistic security evaluation testbeds are highly dynamic stateful
environments where actions can trigger destructive side-effects.
Any interaction of the agent with the environment may produce
adverse effects, e.g., invoking an exploit can crash the target system,
enumeration actions can trigger intrusion detection systems, or
running password-based attacks can result in account lock-outs. In
addition, defenses such as automated EDR solutions act as active
adversaries, creating attacker/defender setups.
   To achieve success, agents must cope with these side-effects and
must combine findings into successful attack paths, making our
trajectories inherently multi-step. As the agent can run arbitrary
commands on the execution host (Section 3.1), the action space
is unbound. In addition, penetration-testing tools are fickle and         Figure 1: System and Agent Architecture. Orange components
