                                                                                                Abstract
                                                     Multi-agent systems (MAS) tackle complex tasks by distributing expertise, though
                                                     this often comes at the cost of heavy coordination overhead, context fragmentation,
                                                     and brittle phase ordering. Distilling a MAS into a single-agent skill can bypass
                                                     these costs, but this conversion lacks a principled answer for when and what to
                                                     distill. Instead, the empirical outcome is surprisingly inconsistent: skill lift ranges
                                                     from a 28% improvement to a 2% degradation across metrics of the exact same
                                                     task. In this work, we reveal that skill utility is governed not by the task, but by the
                                                     evaluation metric. We introduce Metric Freedom (F ), the first a priori predictor of
                                                     skill utility. F measures the topological rigidity of a metric’s scoring landscape
                                                     by quantifying how output diversity couples with score variance via a Mantel test.
                                                     Guided by F , we propose AdaSkill, a two-stage adaptive distillation framework.
                                                     Stage 1 acts as a selective extraction mechanism, extracting tools and knowledge
                                                     while discarding restrictive structures on “free” metrics to preserve exploration.
                                                     Stage 2 applies iterative refinement selectively on free metrics, exploiting their
                                                     forgiving scoring landscape to safely maximize remaining headroom. Evaluating
                                                     across 4 tasks, 11 datasets, and 6 metrics, F strongly predicts skill utility (r=−0.85,
                                                     p<0.0001). Strikingly, identical agent trajectories yield diametrically opposite skill
                                                     lifts under rigid versus free metrics, demonstrating that skill utility is fundamentally
                                                     a metric-level property. Driven by this signal, AdaSkill matches or exceeds the
                                                     original MAS while reducing cost up to 8× and latency by up to 15×.


                                         1     Introduction
                                         Large Language Model (LLM) agents have demonstrated remarkable proficiency across complex
                                         domains, from scientific discovery to software engineering [Boiko et al., 2023, Jimenez et al., 2024,
                                         Zhou et al., 2024]. To tackle increasingly difficult tasks, the field has largely converged on Multi-
                                         Agent Systems (MAS), which decompose problems into specialized, interacting sub-agents to capture
                                         domain expertise. However, this distributed approach comes at a steep cost: severe coordination
                                         overhead, context fragmentation, and brittle phase ordering. Consequently, a compelling alternative
                                         has emerged: distilling the expertise of a MAS into a single-agent system equipped with specialized
                                         skills [Xu et al., 2026, Luo et al., 2026]. While recent work shows these single-agent systems can
                                         match MAS performance at a fraction of the cost, empirical results reveal a baffling inconsistency.
                                         Skill distillation can yield massive performance gains (+28pp) or actively degrade performance
                                         (−2pp) on the exact same task.
                                         Existing literature largely treats this variance as an unpredictable byproduct of task complexity or
                                         data scale, proceeding under the assumption that skill augmentation is universally beneficial. Current

approaches focus entirely on how to distill skills: either through rigid structural conversions that map
MAS pipelines into instructions [Li, 2026, Ni et al., 2026, Yim et al., 2026], or through iterative
trajectory optimization [Alzubi et al., 2026, Yang et al., 2026, Xia et al., 2026, Zhou et al., 2026].
Both paradigms suffer from a critical blind spot. Non-adaptive structural conversions routinely over-
constrain agents on simple queries, while iterative methods succumb to trajectory-local overfitting,
bloating the skill library until retrieval accuracy collapses [Li, 2026]. The fundamental question
remains unanswered: when does distilling a MAS into single-agent skills actually help?
In this work, we reveal that the efficacy of skill distillation is not dictated by the task itself, but
by an overlooked property of the evaluation metric. We introduce Metric Freedom (F ), the first
quantifiable, a priori predictor of skill utility. F measures the topological rigidity of a metric’s
scoring landscape by quantifying how tightly output variation couples with score variation via a
Mantel test. On rigid metrics (low F ), structured skill guidance steers the agent through a narrow
success corridor and yields large gains. On free metrics (high F ), diverse paths score equally well, so
the same constraints restrict exploration and degrade performance.
To achieve this insight, we propose AdaSkill, an F -guided two-stage adaptive distillation framework.
Rather than blindly mimicking MAS pipelines, our approach uses Metric Freedom as a principled
sieve to determine exactly what to extract and when to refine it. In Stage 1 (Adaptive MAS
Converter), tools are universally preserved and domain knowledge is distilled into conditional
references rather than rigid, mandatory instructions. Coordination structures are discarded entirely,
and explicit task-decomposition hints are retained only for low-F metrics. Stage 2 (Skill Iterator)
introduces an optional four-agent iterative loop (Explore, Main, Analyzer, Runner) to close the
residual gap left by Stage 1’s conservative extraction. This loop is activated only on mid- and high-F
metrics, where the forgiving landscape permits safe, monotonic improvement. On low-F metrics, the
knife-edge landscape turns iterative fixes into zero-sum oscillations, so Stage 2 is explicitly disabled.
Extensive evaluation across four complex tasks, 11 datasets, and 6 diverse metrics validates our core
hypothesis: Metric Freedom strongly and reliably predicts the utility of skill distillation (r=−0.85,
p<0.0001, n=18, Fout ). Notably, the MAS Compiler, which faithfully converts MAS pipelines
without F -guidance, frequently underperforms even the unskilled Base Agent. By acting on the F
signal alone, AdaSkill matches or exceeds the original MAS while reducing inference cost by up to
8× and latency by up to 15×. Our core contributions are:

    • Metric Freedom (F ): We introduce the first a priori, metric-level criterion for predicting when
      MAS-to-skill conversion is beneficial. F is computable entirely from standard baseline runs,
      requiring no additional skilled-agent inference overhead.
    • AdaSkill (Adaptive Two-Stage Distillation): We design a principled framework that abandons
      the “one-size-fits-all” approach to skill generation. Guided by F , Stage 1 selectively distills
      knowledge and structures. Stage 2 targets refinement toward mid- and high-F metrics where the
      forgiving landscape permits stable, monotonic improvement, and explicitly disables iteration on
      low-F metrics where the knife-edge landscape turns fixes into zero-sum oscillations.
    • Reframing Skill Evaluation: We provide conclusive evidence that skill utility is a metric-
      level property, not a task-level one. By demonstrating that identical agent trajectories can yield
      diametrically opposite skill lifts under rigid versus free metrics, we fundamentally correct a
      prevalent blind spot in how agentic skills are evaluated.


2     Background and Related Work

Multi-Agent Systems. The rise of tool-augmented agents [Schick et al., 2023, Yao et al., 2023]
and frameworks like AutoGen [Wu et al., 2024] and MetaGPT [Hong et al., 2024] has spurred
task-specific Multi-Agent Systems (MAS). Recent MAS excel in specialized domains, such as text-
to-SQL via hypothesis-verification loops (APEX-SQL [Cao et al., 2026]; evaluated on BIRD [Li
et al., 2023]), causal discovery combining agent debate with classical algorithms [Spirtes et al., 2012,
Zheng et al., 2018] (MATMCD [Shen et al., 2025]), causal estimation automation [Imbens and Rubin,
2015] (CAIS [Verma et al., 2025]), and feature engineering (FELA [Ouyang et al., 2025]). However,
these systems rely on rigid phase ordering, inter-agent messaging, and specialized sub-agents. This
structural complexity introduces overhead that may be unnecessary given that capable single models
can perform internal multi-perspective reasoning [Qian et al., 2024].


                                                    2
Single-Agent vs. Multi-Agent. Recent studies challenge the assumption that MAS consistently
outperform single agents. Li [2026] show single agents with skill libraries can match MAS cost-
effectively, though unstructured accumulation causes a performance-degrading phase transition. Xu
et al. [2026] shows that single-agent multi-turn execution (OneFlow) rivals MAS across diverse tasks.
Meanwhile, Luo et al. [2026] extends classical knowledge distillation [Hinton et al., 2015] to compress
multi-agent interactions into a single model (AgentArk), and Aluru et al. [2025] distills multi-agent
debate interaction graphs into a single student model via graph-aware training. Our work differs from
all these: rather than comparing paradigms broadly or distilling interaction patterns, we investigate
which specific MAS components warrant conversion and provide a metric-level criterion (F ) that
predicts when such conversion is beneficial. Addressing the skill-quantity bottleneck [Li, 2026], our
framework constructs task-adapted, F -guided skills instead of indiscriminately accumulating them.
Skill Discovery and Evolution. Skill-level optimization is crucial for transferable capabilities, as
demonstrated by Alzubi et al. [2026] via EvoSkill’s iterative failure analysis and Voyager [Wang et al.,
2023] via open-ended embodied exploration. Similarly, Ni et al. [2026] hierarchically consolidate
agent experiences into declarative skills (Trace2Skill), overcoming manual authoring bottlenecks.
In multi-agent RL, Zhou et al. [2025] use dual distillation modules to compress agent interactions
into a shared policy network, and Wang et al. [2026] propose skill-conditioned self-distillation for
multi-turn LLM agents. While our skill iterator (Stage 2) shares the spirit of iterative refinement, it
uniquely operates on MAS-derived skills rather than discovering them from scratch, and uses Metric
Freedom to decide what and how much to distill rather than applying a fixed distillation strategy.

3     Methodology
3.1   Overview

The core premise of our framework is that distilling MAS on a skill is a double-edged sword: it guides
reasoning, but it inherently constrains exploration. To balance this, we propose that such structural
guidance should only be imposed where the evaluation metric prefers “a single standard answer”.
To operationalize this, we introduce Metric Freedom (F ), a formal measure of a metric’s scoring
landscape. This concept drives our two-stage system: an adaptive MAS converter (Stage 1) that
selectively extracts structure only where F dictates it is beneficial, and a skill iterator (Stage 2) that
performs targeted, automated refinement strictly in higher-F regimes where the scoring landscape
permits safe exploration.

3.2   Metric Freedom: Formalizing the Scoring Landscape

Selection Accuracy (MSA) is all-or-nothing, whereas Mean Relative Error (MRE) rewards proximity
to the ground truth. We formalize this via Metric Freedom (F ), a quantity measuring the topological
decoupling between an agent’s behavioral variation and its score variation.

in Appendix C.2 confirms that this approach primarily enhances sampling efficiency: while metric

variance at low n (e.g., < 5% vs. 20% error at n = 6). Finally, we extract only mixed questions

3.3   Theoretical Foundation

We show that Metric Freedom controls an upper bound on the gain achievable by any skill. Let
s : X → [0, 1] be a Hölder continuous scoring function, P0 and Pπ the output distributions of the
base and skill-augmented agents, and Lift(π) ≜ EPπ [s] − EP0 [s].
Theorem 3.1 (Bound on Skill Lift). Assume s is (α, L0 )-Hölder continuous on (X , d) with α ∈ (0, 1],
and that the metric exhibits non-negative concordance (rM ≥ 0). Then for skill π:

where W̃1 is the Wasserstein-1 distance under the lifted metric d(x, ˜ y) = d(x, y)α , and ∆n ≤

Proposition B.5 for the explicit characterization of ∆n and the origin of the 0.047 constant; formal
proof in Appendix B.1). No absolute continuity (Pπ ≪ P0 ) is required. The Hölder condition is
likewise mild: for discrete metrics such as MSA, L0 = 1 follows automatically (Appendix B.1).
Proof intuition. We decompose s into a component s∥ concordant with the behavioral metric and a
discordant residual s⊥ via L2 -projection onto the Lipschitz subspace. The concordant lift is bounded
by L0 ρm · W̃1 using the HGR variational characterization, and the discordant residual contributes at
most L0 (1 + ρm ) · W̃1 by the triangle inequality for Lipschitz constants. The classical inequality
|rM | ≤ ρm then links the combined bound to F = 1 − rM , yielding a gain ceiling of L0 (1 − F ) up

Design Implications. The gain ceiling vanishes as F → 1: skill distillation is fundamentally limited
on free metrics. When F > 1, the bound reduces to the trivial Lipschitz constant (Remark B.1), yet
the empirical trend extends smoothly across the entire F range (r=−0.85, p<0.0001; Section 4.3).
We further show the bound is tight (Appendix B.2), establish a phase transition explaining why
iterative refinement oscillates in low-F but converges in high-F regimes (Appendix B.3), and prove
formal properties of F including range, monotonicity, stability, and a product rule for multi-metric
tasks (Appendix B.4).

Phases 1 & 2: Analysis and Inventory. The distiller decomposes the MAS architecture into
four component classes: (a) callable tools, (b) domain knowledge, (c) pipeline structure, and (d)
coordination mechanisms.
Phase 3: F -Guided Conversion. Two rules are metric-invariant: coordination mechanisms are
always discarded (a single large model internalizes coordination natively), and callable tools are
always retained (they extend capability without constraining exploration). The treatment of pipeline
structure follows F monotonically: as F increases the effective lift ceiling L0 (1−F )W̃1 decreases
(Theorem 3.1), so the distiller scales structural rigidity accordingly—from mandatory step sequences
for near-zero-F metrics, through conditional hints at intermediate values, to bare-minimum reference
knowledge when F approaches or exceeds 1. The ablation in Section 4.5 empirically validates
this monotone relationship: pipeline structure alone yields r=−0.88 with lift across our metrics
(Figure 6), confirming that structural imposition is the primary driver of harm at high F . For multi-
metric tasks, distillation is applied per metric: structure is imposed in proportion to each metric’s F ,
and the resulting modules are composed into a single skill.
Phase 4: Validation. The distiller automatically verifies tool imports, ensures no sequences leak into
high-F skills, validates neutral reference tones, and tests boundary conditions (Fig. 1, Panel C).

3.5       Stage 2: Skill Iterator

Stage 1 intentionally leaves a gap below the lift ceiling at higher F tiers by conservatively stripping
structure. Stage 2 is an optional refinement module to close this gap.
Crucially, Stage 2 is only activated for Mid and High-F metrics. In Low-F environments, the
sharp landscape causes empirical oscillation (Section F). In higher-F regimes, the forgiving landscape
permits safe, monotonic hill-climbing.
Four-Agent Architecture. The autonomous loop employs four roles (Figure 1, Panel D):
 • Explore Agent runs once to analyze benchmark boundaries and dataset characteristics.
 • Main Agent orchestrates runs, tracks evolution history, and makes stopping decisions.
 • Runner executes the skill on train/val splits, generating verbose reasoning traces.

4                     Experiments
4.1                    Setup

Tasks and datasets. We evaluate on 4 tasks with 11 datasets: Text-to-SQL on BIRD-147 [Li et al.,
2023] and Spider-120[Lei et al., 2025]; Causal Estimation (CE) on QRData, Synthetic, and Real
datasets from CauSciBench[Acharya et al., 2026]; Causal Discovery (CD) on AutoMPG[Quinlan,
1993], DWDClimate[Mooij et al., 2016], Sachs[Sachs et al., 2005] and Child[Spiegelhalter, 1992];
and Feature Engineering (FE) on Taobao[Alibaba Group, 2018] and Dia[Teboul, 2022].
Methods. We compare 5 paradigms: Original MAS (APEX-SQL[Cao et al., 2026], CAIS[Verma
et al., 2025], MATMCD[Shen et al., 2025], FELA[Ouyang et al., 2025]—representing SOTA multi-
agent frameworks), Base Agent (raw single agent, no skill), Adaptive Skill (Ours), Auto Opti-
mization [Alzubi et al., 2026], and MAS Compiler (a baseline that compiles all MAS components
faithfully) [Li, 2026]. MAS baselines follow official implementation. To ensure fairness, all methods
use Claude Sonnet 4.6 as the backbone model, and all single-agent methods use Claude Code backend.
Evaluation protocol. F is computed from n=6 raw-agent runs (without skill) per question and
m = 6 questions per dataset. For per-question tasks (CE, T2SQL), F is the median over mixed
questions. Stage 1 uses full code, paper, and benchmark from each task to build adaptive skill. Stage 2
uses a held-out 10% for training and validation each to track per-iteration gains.

5   Conclusion
We introduced metric freedom (F ) as a principled, a priori criterion for predicting when MAS-to-skill
conversion provides value. Both Fout and Ftrace confirm the same strong negative correlation with
skill lift (r=−0.85 and r=−0.77). The key insight is that freedom is a metric-level property, not a
task-level one: identical agent outputs yield dramatically different skill lifts under different evaluation
metrics. The practical implication is direct: diagnose your metric’s freedom before investing in skill
engineering. AdaSkill, guided by F with optional iteration on mid-to-high-F metrics, matches or
exceeds both base agents and original MAS at 1.4–8× lower cost and up to 15× lower latency.

A     Limitations and Future Work

Limitations. F is currently defined over auto-graded, structured benchmarks where behavioral
distances are precisely computable; extending the framework to human-evaluated or open-ended
metrics (e.g., writing quality, dialogue coherence) is a natural next step. F requires running the base
agent n times on a small question sample—a one-time diagnostic cost ($6.12 at our operating point
of N =6, M =6) that amortizes across all subsequent skill decisions for a given metric; for genuinely
novel tasks with no target benchmark, F must be estimated from proxy benchmarks. Backbone
coverage spans Claude Sonnet 4.6 and GPT-5.1 (r=−0.71; Appendix C.6); broader multi-model
validation would further establish universality.
F as a Thermometer of Knowledge Externalization. A more consequential boundary condition
concerns the capability trajectory of foundation models. The relative advantage of structured skills
may diminish as models become more capable, since a sufficiently powerful model may internalize
domain knowledge that skills currently provide externally. Crucially, F itself would signal this shift:
as a future model internalizes, say, causal method-selection expertise, the raw-agent outputs would
become less diverse (the model confidently selects one method), causing FMSA to rise toward 1—
indicating that the skill has become redundant. In this sense, F functions not only as a static predictor
of skill utility but as a thermometer of knowledge externalization: a low F reveals a knowledge
gap that structured distillation must fill; a rising F across model generations signals that the gap
is closing. Tracking how F distributions evolve with model capability would provide a principled,
benchmark-grounded measure of the degree to which advances in pretraining reduce the need for
external skill engineering.
Future work. Several promising directions emerge. First, extending Metric Freedom to open-ended
and human-evaluated tasks would broaden the framework’s applicability. Second, studying cross-task
skill transfer—where skills distilled from one domain improve performance on related tasks—could
reduce per-task bootstrapping overhead. Third, systematically measuring F across successive model
generations would reveal whether foundation model progress is converging toward universally high-F
regimes, or whether structurally rigid metrics remain stubbornly resistant to internalization [Guo
et al., 2024].

shows this is not an artifact of loose bounding—it reflects genuine landscape geometry.

B.3     Convergence and Oscillation in Iterative Skill Refinement

The empirical behavior of Stage 2 (Section F) exhibits a stark dichotomy: iterative refinement
converges monotonically on mid/high-F metrics but oscillates on low-F metrics. We provide a
formal framework explaining this phenomenon.

Proposition B.13 (Monotonicity under Metric Refinement). Let d1 ⪯ d2 denote that d1 is a

discordant direction; such cases require the finer metric to contradict the scoring function’s ordering
at the resolved points.

C.1    Computation Details

For each (task, dataset, metric) tuple, we compute metric freedom from two complementary perspec-
tives: Fout (structured output distances) and Ftrace (embedding-based reasoning trace distances). Both
are computed using existing benchmark runs without additional cost. We use all available runs from
the “Base Agent” condition, as these provide the most unbiased estimate of variation (skills may
constrain output diversity).

Diversity planner. To ensure sufficient behavioral diversity under low-temperature inference—
where repeated runs often collapse to the same output—each question is seeded by a lightweight
diversity planner. The planner is a single LLM call (same backbone model) that receives the original
problem and generates N methodologically distinct solution strategies (e.g., different statistical
methods, different schema traversal orders). Each strategy is then prepended as a brief hint to
the original problem prompt, and the base agent executes independently per hint. This yields N
behaviorally varied trajectories without any skill-induced bias. The planner prompt enforces that
all approaches must be methodologically sound; fine-grained variations (e.g., same method with
different covariate sets) are explicitly permitted to avoid forcing artificial diversity. No additional
sampling temperature or top-p manipulation is used. The full prompt template is shown below.

                       Listing 1: Diversity planner: approach generation prompt.

Ftrace computation. We instantiate Eq. (2) with X = trace embedding space. Each reasoning

embedded via text-embedding-3-large with persistent disk caching. The behavioral distance is

d(τi , τj ) = 1 − cossim (embed(τi ), embed(τj )). Runs where embedding fails are dropped from the

Bootstrap confidence intervals. We report σF in Table 7, computed via bootstrap resampling
(B=1000) of the run pairs. For per-question metrics (CE, T2SQL), we first compute F per question,
then report the median and its bootstrap standard error.

Exclusion criteria. For per-question tasks (CE, T2SQL), we include only mixed questions where
scores vary across runs—questions where all runs succeed or all runs fail are excluded, as they reflect
task difficulty or task ceiling rather than the metric’s scoring geometry. For per-dataset tasks (CD,

Sensitivity to distance function. We verified that replacing Jaccard distance with edit distance
(for T2SQL) or Hamming distance (for CD) does not qualitatively change the freedom ranking.
The Spearman correlation between freedom and lift remains significant (p < 0.05) under all tested
distance functions. Fout and Ftrace rankings are correlated (r ≈ 0.78) but not identical; divergences
arise when diverse reasoning processes converge to similar outputs, which we discuss in Section 4.3.

C.2   Sensitivity to M and N

A central design decision in computing F is how many questions (M ) and how many runs per
question (N ) to collect. Both quantities directly affect the reliability of the Mantel-test estimate: too
few runs leave pairwise distances noisy; too few questions make the median-F ranking unstable. At
the same time, collecting M ×N runs has a real cost: panel (c) of Figure 8 shows that this cost is
strictly linear in M ×N , reaching $20.4 at the maximum budget of M =N =20.
The full sweep in Figure 8 reveals a clear saturation pattern for both FMSA and FMRE . For FMSA
(panel a), which is near zero and therefore sensitive to small estimation errors, the estimate stabilises
once N ≥ 5: beyond this point the colour map changes by less than two percentage points per
additional run column. For FMRE (panel b), which is intrinsically high and diffuse, even N =3
already places the estimate in the correct high-F regime—additional runs compress the variance but
do not alter the regime assignment. In both panels, increasing M beyond 6 produces only marginal
shifts in cell value, indicating that 6 questions are sufficient to capture the metric’s structural character.
Taken together, the gold operating point at M =6, N =6 ($6.12) lies at the knee of the accuracy–cost
curve: it achieves stable F estimates for both rigid and diffuse metrics while consuming roughly
one-third of the maximum budget. This choice is therefore not a hand-tuned hyperparameter but an
empirically grounded threshold that any practitioner can validate directly from this plot. Cells with
N < 3 are greyed out because fewer than three runs yield only one or two behavioural pairs, which
is insufficient to form a stable pairwise distance distribution for the Mantel test.

Diversity planner ablation. Figure 8 compares F estimation with and without the diversity planner.
Without it, each of the N runs is an independent random execution of the base agent. Because
low-temperature LLM inference tends to collapse to a small number of dominant reasoning paths,
many of these runs produce near-identical trajectories—effectively reducing the number of distinct
behavioral samples. This manifests as a pronounced upward drift in FMSA with increasing N : at
N =6 the estimate is only 42 (vs. the converged value of ∼53), a ∼20% underestimate that could
mislead the framework into over-constraining the skill. With the diversity planner, each run is seeded
by a methodologically distinct strategy, guaranteeing that every run explores a different region of the
behavioral space. This yields accurate estimates (< 5% error) already at N =6, avoiding the need for
costly large-N grids.

C.4           F Value Generalization across Datasets

A natural concern is whether F values measured on one dataset generalize to another dataset within
the same task—i.e., whether one can compute F on an easier or smaller dataset and trust that the
resulting regime assignment (Low/Mid/High) will hold on harder datasets. We address this directly
using Causal Estimation (CE), the task with the widest spread of difficulty across three datasets
(Textbook, Real, Synthetic) and two metrics (MSA, MRE).
Tables 4 and 5 report, respectively, the per-dataset F distributions and the pairwise cross-dataset
comparisons for both F out and F trace .

Distribution. Table 4 shows that, within each metric, the F means across datasets are strikingly
similar. For MSA, F trace ranges from 0.431 to 0.559 across all three datasets (grand mean 0.510, CV
= 13.5%). For MRE, F out ranges from 0.638 to 1.012 (CV = 25.7%), but the two harder datasets
(Real and Synthetic) are nearly identical (0.697 vs. 0.638, ∆µ = 0.059). This consistency means a
practitioner who measures F on any single CE dataset will arrive at the same regime assignment.

Cross-dataset comparison. Table 5 reports pairwise ∆µ (absolute difference of means), 95%
CI overlap, and Mann–Whitney U test p-values (no shared question IDs exist across datasets, so


                                                                                                           27

and p ≥ 0.37—no statistically significant difference across datasets. For MRE/F out , Real and
Synthetic are indistinguishable (∆µ = 0.059, p = 1.00), and even Textbook remains within the same
Mid-to-High regime despite a larger absolute gap. The sole exception is MRE/F trace for Textbook
vs. Synthetic (p = 0.026, CI no overlap), where Textbook’s more open-ended real-world questions

C.6     Backbone Generalization: Replication on GPT-5.1

A potential concern is that all primary experiments use a single backbone (Claude Sonnet 4.6 via
Claude Code), which raises the question of whether the skill-lift pattern and the predictive validity of
Metric Freedom F are specific to that model family rather than a general phenomenon. To address
this, we replicate the full benchmark suite—covering all four tasks, all datasets, and both raw-agent
and skill-augmented conditions—using GPT-5.1 as the backbone, with identical skill definitions,
prompts, and evaluation protocols.
Figure 11 replicates Figure 3 for GPT-5.1. The negative relationship between F and headroom-
normalized lift is preserved under both Fout (r=−0.71, p<0.01) and Ftrace (r=−0.79, p<0.001),
confirming that F is a transferable predictor of skill utility and not an artifact of the Sonnet 4.6
architecture or toolchain. For reference, the Sonnet 4.6 trend lines (dotted blue) are overlaid; the GPT-
5.1 lines (dashed gray) are shallower and intercept lower, reflecting two complementary observations.
First, the absolute skill lifts obtained by GPT-5.1 are generally smaller: a more capable base model

F                Stage 2: Skill Iterator Details

Fig. 13 tracks Stage 2 optimization across all four domains; the trajectories align with our theoretical
bound (Theorem 3.1).
On CE-MSA (F ≈ 0), the iterator quickly reaches 100% validation accuracy by iteration 2 via tool
fixes and knowledge alignment. However, iterations 3–4 exhibit classic oscillation—fixing one failure
case breaks another. The knife-edge landscape turns optimization into a zero-sum game, validating
why Stage 2 is disabled for low-F metrics.
On CD (F = 0.24–0.77) and T2SQL (F = 0.50), the forgiving scoring surface allows the iterator
to surgically inject tool fixes and missing knowledge without displacing correct behaviors. CD
gains +14pp on DWDClimate (F1=0.701) and +15pp on Child (F1=0.952); T2SQL val EX improves
steadily, plateauing at 86.7% from v3 on BIRD-147. Both confirm smooth, monotonic hill-climbing
in mid-freedom regimes.
On FE (F ≈ 0.59–0.97), val AUC on Taobao plateaus after iteration 2 with only marginal gains. The
flat landscape means structural tweaks cannot meaningfully improve the score, as the base agent’s
intrinsic capability ceiling has already been reached.

 if is_rct:

The adaptive skill converts the same logic to conditional-reference prose in SKILL.md:



                          v0 SKILL.md: same knowledge as conditional reference
 ### Step 1 - RCT check (do this first)
 If description contains: randomized / RCT / randomly assigned -> is_rct = True
 - When is_rct = True: matching, PSM, and IPW are FORBIDDEN.
   OLS with covariates is the correct choice.
   Random assignment already balances covariates --
   applying matching on top produces an incorrect selected_method.
 - If an instrument for non-compliance exists -> use IV (encouragement design).




The agent now reasons about the rule in context rather than executing it mechanically. This is the key

Fix 1 (Iteration 1): Staggered DiD. Failing case: panel dataset, 57 unique treatment-start years
(first_sanction). The agent set treatment_period=1982 (the mean), so units treated before
1982 received _post = 0 for their actual treatment periods — a mis-specified TWFE.

Before (v0) — no guidance on staggered adoption:
 ### DiD
 - treatment_group_col: binary column (1 = treated group, 0 = control)
 - treatment_period: the single period when treatment started


After (v1) — explicit staggered TWFE path added:
 ### CRITICAL: Two types of DiD -- choose correctly
 Type B -- Staggered DiD: dataset has BOTH an ever-treated column
   (ever_sanctioned) AND a period-specific column (treat).
   -> Do NOT use estimate_did. Use estimate_ols with treat as treatment
      + unit FE + time FE. Staggered signal: first_treatment_year
      with many unique values.




Fix 2 (Iteration 1): RDD bandwidth. Failing case: Head Start RDD, n=23,599. Agent used
10%-of-range bandwidth → estimate = −0.031; reference = −0.079; MRE = 61%.

Before (v0):
 ### RDD
 - Bandwidth choice: try default (10% of range), then check n_in_bandwidth >= 20


After (v1):
 ### RDD -- bandwidth (critical for accuracy)
 ALWAYS call estimate_rdd with bandwidth=None.
 The tool uses rdrobust MSE-optimal IK bandwidth automatically
 (same algorithm as R reference). Do NOT specify a narrow bandwidth.




Fix 3 (Iteration 2): PSM ATT ̸= ATE. Failing case: social media campaign, max_smd=1.01,
50/50 split. Agent ran PSM (correct method choice), but PSM ATT= 2.23 vs. ATE reference= 1.58;
MRE= 41.3%.

Before (v1) — warning exists, no action:
 ### Matching / PSM
 - Estimates ATT (effect on treated), NOT ATE -- this is critical


After (v2) — mandatory AIPW step added:
