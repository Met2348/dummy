多 AI agents 组成虚拟科研团队，并设计经实验验证的 SARS-CoV-2 nanobodies。

## 背景和核心问题
这篇把 AI agents 放入真实生命科学场景，强调不同 agent 扮演 PI、scientist、critic 等角色，共同设计生物分子。

## 方法拆解
系统通过多角色讨论形成研究计划，围绕 nanobody 设计、筛选和实验验证展开。agent 生成候选方案，人类和实验平台完成验证闭环。

## 证据与实验怎么看
Nature 正式论文报告设计出的 nanobodies 经实验验证，说明 agent 团队不仅能写方案，还能提出可被湿实验检验的候选。

## 局限、风险和批判点
实验成功离不开专家、实验平台和领域先验。多 agent 对话的具体贡献需要从实验记录中细看，避免把组织形式等同科学因果。

## 放在 LLM Auto Research / AI Scientist 图谱中的位置
与 Coscientist、Robin 组成 domain discovery 三件套，代表真实实验验证路线。

## 组会讨论问题
- 多角色 agent 团队是否比单 agent 更好，证据是什么？
- 实验失败的候选如何反馈给系统？
- virtual lab 的贡献应归因于设计策略、筛选工具还是 agent 讨论？

## 可复现或可延伸的 follow-up
做 virtual lab 模板：PI agent 定方向，method agent 提模型，experiment agent 设计验证，critic agent 查 novelty。
