# Q&A —— 逐函数深挖系列的建立(2026-07-07)

## Q:从现在开始你就应该一个函数一个函数地教我,面向 AI 科研场景,写详尽教程,先从 np 开始

后续追加:深度要由浅入深,至少 100 个函数,分批次完成;一口气做完,中途不需要反复确认;还要追加 20 个 Python 高级知识(装饰器/lambda/yield/async 等课堂不讲但很重要的东西),一个知识点一个章节。

## 做了什么

建立了两条并行的"深挖"系列,和已有的 01~04 入门教程互补(入门教程负责建立概念,深挖系列负责系统性地过一遍、建立"看到就认得出"的识别感):

1. **[numpy-deep-dive/](../numpy-deep-dive/00-roadmap.md)** —— 约 120 个 numpy 函数,10 个分类(创建初始化/形状结构/索引选择/逐元素数学/归约统计/线性代数/排序集合/广播ufunc机制/随机数进阶/IO验证工具),由浅入深排列。
2. **[python-advanced/](../python-advanced/00-roadmap.md)** —— 20 个 Python 中高级知识点,4 个分类(函数闭包/迭代器生成器/OOP进阶/类型注解与并发)。

## 怎么做的(方法论,值得记录)

- **规模太大,不可能自己手写完**:14 个文件用并行 subagent 分工写(每个 subagent 负责一个分类),但每个都被要求必须实际跑代码验证 assert,不能凭空写。
- **不能盲信 subagent 的自我报告**:每个文件写完后,我都独立重新提取代码、重新跑一遍验证,而不是直接相信 agent 说"已验证"。过程中还抽查了几处 agent 引用仓库真实文件的行号(比如 `transformers/modeling_utils.py` 第172-174行),确认不是编出来的。
- **进度表要如实反映状态**:一度把 roadmap 提前标成"全部完成",后来改成先标"生成中",真正验证通过才标"已完成(已验证)"。

## 结论

两个系列都已 100% 完成并逐条独立验证。每个函数/知识点固定同一套结构:签名(人话翻译)→ 一句话是什么 → AI 研究场景 → 可运行例子(带 assert)→ 常见坑。这不代表要背下来——[03 教程](../03-how-to-look-up-not-memorize.md)"别死记签名"的心态依然适用,这里是精读一遍建立认知。
