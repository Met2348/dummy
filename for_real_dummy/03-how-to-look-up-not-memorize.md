# 记不住函数签名怎么办 —— 专家也不背这个

> 前提：已看完 [01](01-numpy-for-c-programmers.md)、[02](02-pytorch-basics.md)
> 目标：建立正确的"记忆策略"，而不是死记硬背

---

## 0. 先纠正一个误解

**没有人背下所有 numpy/torch 函数的参数顺序和默认值——包括写这些库的人自己。**

真正的专家技能是两层：

```
第一层：意图 → 大概函数名（模糊记忆，靠高频使用自然形成）
              "我要合并两个数组...是叫 cat 还是 concat 还是 merge？"

第二层：函数名 → 精确签名（5 秒内查到，不用记）
              悬停鼠标 / help() / 官方文档
```

**C 语言类比：** 你写 C 的时候，会背下 `qsort()` 的第三个参数是 `size_t` 还是 `int` 吗？
大概率不会——你记得"排序要用 `qsort`"，具体签名每次查 man page。numpy/torch 也一样，只是查阅工具更方便（IDE 悬停比翻 man page 快多了）。

## 1. 真正值得"肌肉记忆"的核心清单（就这些，不多）

这些高频到你不需要专门背，用 20-30 次自然就记住了：

| 类别 | 函数/写法 |
|------|-----------|
| 查看信息 | `.shape` `.dtype` `.ndim` |
| 创建数组 | `zeros()` `ones()` `randn()` `arange()` |
| 形状变换 | `.reshape(...)` |
| 四则运算 | `+ - * / **`（直接用运算符） |
| 矩阵乘法 | `@` |
| 统计归约 | `.sum()` `.mean()` `.max()` + `dim=`/`axis=` |
| 转置 | `.T` |
| 索引切片 | `a[i, j]` `a[:, 1]` `a[1:3]` |
| （torch）设备 | `.to(device)` |
| （torch）训练 | `requires_grad=True` `.backward()` `.item()` |

**除了这 15 个左右，剩下的都应该查，不该背。** 这不是偷懒，是正确的策略——把大脑容量留给"我现在要解决什么问题"，而不是"这个参数叫 axis 还是 dim"。

## 2. 剩下的：用"意图 → 函数"索引表，而不是死记

这是本文最重要的部分。**不要按函数名字母顺序记，按"我想干什么"来查。**

| 我想要... | 用这个 | 备注 |
|-----------|--------|------|
| 创建全 0/全 1/随机初始化的数组 | `zeros()` / `ones()` / `randn()` / `full()` | |
| 改变形状但不改数据 | `.reshape(...)` | torch 还有 `.view()`，但要求内存连续，不确定就用 `reshape` |
| 去掉/增加一个大小为1的维度 | `.squeeze()` / `.unsqueeze(dim)`（torch） | numpy 用 `np.expand_dims(a, axis)`；两者都支持 `a[:, None]` 加维度的写法 |
| **沿现有维度合并**多个数组 | `torch.cat([a,b], dim=0)` / `np.concatenate(...)` | 不增加新维度 |
| **新增一个维度**堆叠多个数组 | `torch.stack([a,b], dim=0)` | 增加新维度——常和 cat 搞混，见下面"地雷"部分 |
| 沿某维度求和/平均/最值 | `.sum(dim=)` `.mean(dim=)` `.max(dim=)` | dim=0 通常是"跨行"，dim=1 是"跨列"——不确定就打印 shape 验证 |
| 找最大值的位置（不是值） | `.argmax(dim=)` | 分类问题里"预测的类别"就是这个 |
| 逐元素数学函数 | `.exp()` `.sqrt()` `.log()` `.abs()` | |
| 矩阵乘法（不是逐元素！） | `@` 或 `.matmul()` | 见下面"地雷" |
| 按条件选取数据 | `a[a > 0]`（布尔索引） | |
| 按索引列表选取 | `a[[0, 2, 5]]`（花式索引） | |
| 搬到 GPU / 搬回 CPU | `.to('cuda')` / `.cpu()` | |
| tensor 转普通 Python 数字 | `.item()`（仅限单元素 tensor） | |
| tensor 转 numpy 数组 | `.detach().cpu().numpy()` | 见下面"地雷" |
| 推理时不计算梯度（省内存） | `with torch.no_grad(): ...` | 训练外的所有前向传播都该包这个 |
| 冻结某些参数不训练 | `param.requires_grad = False` | |

**用法：** 下次写代码卡住时，先在这张表里找"我想要做的事情"最接近哪一行，再去查那个函数的精确参数。这比翻函数列表快得多。

## 3. 五个新手常踩的地雷（知道这些比背签名更重要）

### 地雷 1：`*` 不是矩阵乘法
```python
A = torch.randn(3, 3)
B = torch.randn(3, 3)

A * B    # 逐元素相乘（Hadamard 积），几乎不是你想要的
A @ B    # 矩阵乘法，线性代数课本里的那个
```
数学课本里 "AB" 看起来像乘法，但代码里 `*` 永远是逐元素。这是从数学转代码最容易踩的坑。

### 地雷 2：`cat` vs `stack`
```python
a = torch.randn(3, 4)
b = torch.randn(3, 4)

torch.cat([a, b], dim=0).shape    # torch.Size([6, 4])  ← 沿已有维度拼接
torch.stack([a, b], dim=0).shape  # torch.Size([2, 3, 4]) ← 新增一个维度
```

### 地雷 3：`torch.max` 返回的是一个元组，不是单个值
```python
x = torch.randn(3, 4)
x.max()            # 单个标量，全局最大值
x.max(dim=1)       # 返回 (values, indices) 两个东西！
values, idx = x.max(dim=1)   # 要这样拆开用
```
numpy 的 `.max(axis=)` 不会返回 indices，这是 numpy 和 torch 少数不一致的地方。

### 地雷 4：tensor 转 numpy 报错
```python
x = torch.randn(3, 3, requires_grad=True).cuda()
x.numpy()   # 报错！不能直接转

x.detach().cpu().numpy()   # 正确写法：先脱离计算图，再搬回CPU，再转numpy
```

### 地雷 5：忘记 `zero_grad()`，梯度会累加
```python
# 训练循环里如果漏掉这一行，梯度会一直累加，训练直接跑飞
optimizer.zero_grad()   # 别忘
loss.backward()
optimizer.step()
```

## 4. 怎么在 5 秒内查到精确签名（VSCode 环境）

1. **悬停提示：** 鼠标停在函数名上不动，VSCode 弹出签名 + 文档说明
2. **参数提示：** 打出左括号 `(` 后，自动弹出参数列表，当前光标对应的参数会高亮
3. **交互式查询：** 在 Python/Jupyter 里输入 `help(函数名)`，或 Jupyter 专用的 `函数名?`
4. **官方文档：** 拿不准就搜 `numpy.org/doc` 或 `pytorch.org/docs`，两边文档质量都很高
5. **兜底：** 直接问我，我可以现查现讲

**核心心态转变：** 不确定用哪个函数时，先 `print(x.shape)` 看一眼数据长什么样，再决定——用"运行 + 观察"代替"凭空回忆"，这是训练直觉最快的方式。

## 6. 练习：意图识别测试

不看上面的表，先凭直觉猜函数名，再用 VSCode 悬停或 `help()` 验证：

1. 我有两个 `(batch, seq_len, hidden)` 的 tensor，想在 batch 维度上合并成一个更大的 batch —— 用哪个函数？
2. 我训练完想看某一层的参数是否还在更新，想知道 tensor 里的具体数字（不是 tensor 对象本身打印出来一堆信息）—— 用什么方法？
3. 我有一个 `(3, 4)` 的矩阵，想知道每一行的最大值是多少，以及最大值在哪一列 —— 用什么，怎么拆解返回值？
