# 终面深水区总纲：追问链应对法 + 默写练习使用法

> 先读这篇再进具体内容。`baguwen-mastery` 练的是"知道"；这里练的是**被连续追问三四层还不垮**，外加"自己的项目被当众拆解"和"闭卷手撕代码"。三件事，三套练法。

## 1. 为什么初面题库在终面不够用

今天核实的 2026 年真实情况：
- 终面/主管面会**以简历为支点**连续追问 25 分钟以上——问的是你项目里的具体决策，而不是教科书定义。
- 前沿 Lab（DeepMind/Anthropic/OpenAI）终面核心是 **Project/Research Deep Dive**：挑一个项目当众 defend，考官盯 originality/rigor/诚实讨论局限性。
- **追问深度正比于你自己说出口的关键词**——你提一个术语，考官就顺着往下挖。这意味着"背对了答案"没用，得练"每句话背后接得住三层追问"。

## 2. 追问链（DeepPoint）怎么练

每个 `DeepPoint` 是一条 `trigger`（考官的开场追问）+ `chain`（3-4 层层层递进的追问，每层带参考答案+采分词）+ `pitfall`（大多数人在哪层开始编）。

练法：
1. 看到 `trigger`，**自己先口头或写下第一层答案**，不要提前看 `chain[0]` 的参考答案。
2. 用 `deep_common.grade_chain(dp, your_answers)` 逐层打分，命中率低的层就是你会在真实终面里垮掉的地方。
3. 对着 `pitfall` 诚实检查：你是不是也在那一层开始"感觉在编"？如果是，回去补那一层背后的原理，而不是记住这个 chain 的答案——**考官不会问一模一样的 chain，但会在同一深度上逼你**。
4. `real_world_link` 非空时，尝试把这条追问接到你自己对应的 capstone 项目上讲一遍——这才是真正的终面场景。

## 3. 项目深挖（project-defense/）怎么练

`project-defense/` 下每篇文档是**脚手架**，不是台词稿。结构固定：背景/目标 → 个人贡献 → 关键技术决策与理由 → 踩过的坑与解决 → 结果与诚实局限 → 追问树。

practice 方法：
1. 合上文档，口头把"背景到结果"这条主线讲一遍（计时，目标 3 分钟内讲完主线）。
2. 翻开追问树，从第一层开始，逐层自问自答，卡住的地方就是要回去补的知识/记忆缺口。
3. **不要背文档原文**——终面会用你自己的语言反问，背台词式的回答一听就是编的。文档只负责帮你把项目细节想全，口头表达必须是你自己的。

## 4. 默写练习（dictation/）怎么练

对该手撕的代码，闭卷重写一遍，而不是"我知道原理就够了"：

1. 打开 `dictation/specs/<name>_spec.py`，读接口约定（docstring 里的边界条件/禁止项，比如"不能调用 F.scaled_dot_product_attention"）。
2. **合上任何参考资料**，把函数体在草稿文件里从零写完。
3. 跑 `python learning/onsite-mastery/src/dictation/harness.py <name>`，看 PASS/FAIL；FAIL 会报具体哪个断言挂了。
4. 只有通过 harness 之后，才打开 `dictation/solutions/<name>_solution.py` 对比实现风格/简洁度——不是对答案，是学"更干净的写法"。
5. 时间自查：终面白板/pair coding 通常给 15-20 分钟一道，练到能在这个时间内闭卷写对。

## 5. 场景判断题（`frontier_deep/sc_engineering_judgment.py`）怎么练——和 DeepPoint 不一样

老手二次审阅后指出：2026 年终面里出现了一类"线上效果突然变差你怎么定位""如何证明新 Prompt 比旧的好"这类**没有标准答案**的场景题，`DeepPoint` 那种"每层都有唯一参考答案"的格式不适合这类题——练法必须不同：

1. 看到 `ScenarioPoint.trigger`，口头或写下你的应对思路，**不要去找"正确答案"**，因为根本不存在。
2. 用 `deep_common.grade_scenario(sp, your_answer)` 自查——这个分数是"你的回答覆盖了 `rubric` 里几个要点"，不是"你答对了没有"。覆盖率低不代表你"错了"，代表你的思路还不够全面。
3. 对着 `trap` 诚实检查：你是不是也只说到了"大多数人会说的那个浅层答案"就停住了？终面考官恰恰是在追问"除此之外呢"的时候把人问倒的。
4. 这类题的正确练法是**口头对着草稿纸练"想全"**，而不是背下 `rubric` 里的几个词——真实终面现场的场景描述不会和这里的 `trigger` 一模一样，练的是遇到任何一个"if this happens"式追问时的思考框架，不是背题。

## 6. 层层通关（`staff_gauntlet/`）怎么练——按顺序过关，不要随机抽题

老手三审后的反馈是"Frontier Lab招人那叫一个难"，要求专门查清楚2026年这些实验室到底考什么。核实后发现：2026年OpenAI/Anthropic/DeepMind的真实面试loop本身就是**分阶段关卡制**——screen（动机匹配）→research coding（论文转代码）→paper critique（论文批判）→系统设计判断→values round（价值观关）→跨团队协作，每一关考的能力维度完全不同。`ai_deep`/`backend_deep`/`frontier_deep` 按知识主题组织、随机抽题练就够了；`staff_gauntlet` 按**面试关卡阶段**组织，练法必须不同：

1. **按 `GATES` 里 `n` 从1到11的顺序过关，不要挑自己喜欢的关卡跳着练**——真实loop本身就是顺序gate，前一关过不了根本进不了下一关，练的就是这种"顺序压力感"，打乱顺序会失去这个训练目的。
2. **第6关（values round）不要背话术，要练"讲真话"**——这一关的 `rubric` 考的是"包含自己出错/不确定的时刻""真实怀疑而非表演式认同"这类回答质地，不是知识点。练法是对着草稿纸诚实回想自己真实的经历，包括让自己不舒服的部分，而不是编一个体面的故事。如果发现自己的故事里自己永远是对的一方，这本身就是要修正的信号。
3. **第4/5关（临场改约束的系统设计判断）练的是"推倒重来"的勇气，不是"打补丁"的熟练度**——面试官会在你设计到一半时突然换一个约束，真正资深的人会直接丢掉原方案从新约束重新设计，而不是在原设计上勉强打补丁。练的时候刻意问自己：如果这个约束是从一开始就给的，我会设计出一个完全不同的方案吗？
4. **第2/7/9关会专门测试"这段代码/这个方向是不是你自己吃透的"**——如果面试官让你现场修改一段"你说是你写的"代码，或者追问一个研究方向具体是谁先提出来的，答不上来恰恰暴露了靠AI agent代写但自己没有深度理解的部分。诚实的做法是练习清楚区分"我自己设计/推导过的部分"和"AI辅助生成、我还没完全吃透的部分"，而不是笼统地把一切都算成"我做的"。
5. **第11关（HPC集群基础设施）要求记住具体数字，不能只背机制名词**——比如Llama 3 405B真实发生过419次意外中断这类真实数据，面试官问到具体量级时，"故障很常见"这种模糊表述是接不住追问的。

## 7. 四个子包/track 的边界（诚实说明）

- `baguwen-mastery`：知识点广度，练"知道 + 30 秒说出关键词"，对应初面/一面。
- `interview-prep/mlcoding`：真 torch 从零手写 9 件，已经是**正确实现**，这里的 `dictation/` 复用同一批函数改成**闭卷默写**版本，练的是"手速+不看资料也写得对"。
- `onsite-mastery`（本 track）：知识**深度**（连续追问，含 2026 前沿补强）+ 项目**深挖**（自己的东西）+ 代码**默写**（闭卷手撕）+ 工程**判断力**（无标准答案场景题）+ 面试**关卡通关顺序感**（`staff_gauntlet/`），五者合起来对应从初筛到终面全流程。
