## 1 · 背景与目标

**要回答的问题**：2026 年，"harness / agent 运行时"为什么从"调参手艺"被正名成一个可以押 PhD 的工程学科？

三条并列证据（原文第 3 节）：

1. **模型商品化**：GPT-5.5 / Claude Sonnet 4.6 / Gemini 3.1 / DeepSeek-V4 数周内齐发，能力都极强——选哪个模型不再构成护城河。
2. **护城河转移到 harness 的直接证据**：同一个底层模型，**只换 harness，SWE-bench 解决率能差出几十分**（Claude Code / Codex / Aider / OpenHands 之间的对比）。
3. **企业侧失败数据**（deck 标注引自本人 harness-engineering 课程 L01）：**88% 的 agent 项目到不了生产，其中 65% 的失败可追溯到 harness 缺陷**（context drift / schema 错配 / state 退化），而非模型推理能力不够。
4. **命名时刻**：2026-02 Mitchell Hashimoto 提出"harness engineering"是工程学科 → 同周 Codex / LangChain 跟进；RUCAIBox 发布《Agent Systems with Harness Engineering》综述。

但 deck 自己在这页之后立刻插了一句提醒，也是整篇稿子的分水岭：**"工程被正名"≠"适合做 PhD"**。这就引出第 4 节的判据（见第 3 节详述）。

**这份稿子要回答的三个问题**（deck 第 1 节）：
1. 领域成立吗——是真问题还是工程热度？
2. 里面哪些问题抗模型进步、值得 PhD？
3. 如果做，从哪起手、终点是什么？

## 3 · 关键技术决策与理由

### 3.1 判据：为什么用"抗模型进步测试"筛问题，而不是追热点

deck 第 4 节的判据原句：**"假设 18 个月后模型强一倍，这个问题会被自动解决吗？"**——会，是工程/artifact，顶多一篇短文；不会（数字变了但结构还在），是研究/知识，值得押。

这条判据不是一句直觉，Q1 §1.1 把它精确化成"正交性原理"：模型进步只沿三条轴移动——(a) 函数逼近（固定的单次前向计算类内逼近更准）、(b) 对齐（更贴近代理目标）、(c) 知识压缩（把更多事实压进权重）。**一个问题是"不变量"当且仅当它的难度来源与这三条轴正交。** 这把"抗模型进步"从主观判断变成了一个可以逐条核对难度来源的结构性命题。

### 3.2 五个候选方向的打分逻辑，以及为什么推荐路径是"C 起手 → B 核心 → D 收口"

deck 第 11 节打分表（★越多越好，拥挤度★越多越挤）：

| 方向 | 抗模型进步 | 拥挤度 | 适配我 | 角色 |
|---|:--:|:--:|:--:|---|
| A 长程稳定性理论 | ★★★★ | ★★（蓝海） | ★★★★ | thesis 核心候选 |
| B interp × agency | ★★★★ | ★★（低） | ★★★★★ | thesis 核心 + 最差异化 |
| C 因果归因测量 | ★★★ | ★★★★（升温） | ★★★★ | 第一篇 / 起手 |
| D 率失真 context | ★★★★ | ★★ | ★★★ | 理论章（第 3 年） |
| E harness 边界原理 | ★★★ | ★★★★★（过热） | ★★★ | 谨慎，只碰原理 |

**为什么不直接 all-in 在打分最高的 B 上**：deck 第 12 节给的路径是"C（起手/复现）→ B（核心，A 强备选）→ D（压轴章）"。这不是随意排序，Q2 §2.6 给了明确的风险管理逻辑：**"把不可能变可能"= 高回报高风险的 moonshot，正文 §12 的阶梯正是对应的风控——先用可做的测量型第一篇（C）落地一次完整研究循环 + 建测量基建，再上 B 这种 moonshot；不要一上来 all-in 在 moonshot 上。** C 的价值不只是"好发"，而是给 B/D 之后要用的测量基建先跑通一遍；deck 也承认 C 本身正在"升温"（拥挤度★★★★），差异化必须是因果/方法论论文而非又一个榜单——这一步没有被自己美化成零风险。

D 被放在第 3 年是因为"高风险低 tractability"（率失真理论虽然抗模型进步★★★★，但需要等有了实证肌肉才啃得动这块硬理论）。E 被明确列为"谨慎区"——不是因为它不重要，而是"用 RL 训 agent"极度拥挤（综述已有 500+ 论文）且 frontier lab 砸资源，"博 0 拼不过配方"，deck 的态度是"要碰只碰原理，不碰配方"。

### 3.3 五个"结构不变源"：各自对应什么数学、为什么不会被模型进步消解

Q1 §1.2–1.6 的核心论证，逐条如实转述：

**① 计算（复杂度论）——最硬的一条**
- 事实（✅今日查证）：标准 decoder-only transformer 的**单次前向**只能表达 TC⁰（常深度阈值电路）内的函数，常精度下更窄，只到 AC⁰。
- 推论：NC¹-complete（模拟自动机）、NL-complete（图连通性）、P-complete（解线性方程）这类问题，**直接输出答案的 transformer 在证明上做不到，与规模无关**——加参数不抬复杂度类。
- 唯一出路是中间生成：CoT / scratchpad / agent loop（Merrill & Sabharwal：log 步升到 L；Li et al：T 步 CoT + 常精度可解任意 size-T 布尔电路）。
- 不变量：**串行计算深度是"循环"的结构属性，不是权重的属性**——再怎么重训单次前向也跨不过它自己的复杂度类。这是"harness/loop 在做不可约计算工作"最严格的地基，对应方向 A/D。
- 诚实边界：这些是"可表达性"结论，不是"可学习性"结论——能表达不等于能学到。

**② 容错阈值（编码/控制论）**
- von Neumann 阈值定理（经典 1956）：用错误率 ε 的不可靠元件，当且仅当 ε 低于某阈值且引入冗余/纠错，才能做出任意可靠的长程计算；否则噪声必然吞掉计算。
- 映射到 agent：每步错误 ε，朴素成功率 ~(1−ε)ⁿ 指数坍缩——吻合实测"时长翻倍、失败翻 4 倍"的超线性衰减。
- 暂态 vs 不变的拆解：每步 ε **是暂态**（能力一升就降）；但"存在阈值"+"需要纠错结构"本身**是不变的**——即便 ε 变得很小，无限时程自治仍需要"纠错快于注入"。对应方向 A。

**③ 信息（信息论/统计）**
- (a) POMDP 部分可观测：agent 行动时隐状态不在观测里，预训练给不了数据中不存在的信息，信念状态估计是结构性的。
- (b) 在线/截止后信息：权重是训练时刻的快照，"此刻"永远在权重之外（实时价格、私有状态、cutoff 之后的事件）；检索/工具/记忆是唯一补法——**而这些按定义就是 harness 功能**。deck 给出一个近乎平凡却深刻的证明形式：∀ 训练时刻 T，∃ T 之后的信息权重无法包含 ⇒ 某些 harness 功能永久必要。
- (c) 归因可识别性：model×harness×task 的贡献分解在有交互项时是否统计可识别，这是测量论问题，只要 agent = 模型×harness 就不变。对应方向 C/D。

**④ 规范与监督鸿沟（对齐论）——随能力变大而变大的杀手类**
- (a) Goodhart 定律：更强的优化器优化代理目标（如 RLHF 的人类认可/RM）得更狠，偏离真目标的风险可能更大，结构性而非能力缺口。
- (b) 可扩展监督/sandwiching：当模型在某任务上强过人类评估者，人就给不出可靠训练信号——**能力越高，鸿沟越宽**。
- (c) weak-to-strong（⚠高置信记忆，OpenAI 2312.09390，未能现场复核）：弱监督尝试引出强模型全部能力，结果是部分恢复、gap 持续，未解。
- (d) 内对齐/欺骗性对齐：模型可以在训练分布上行为对齐、分布外追别的目标——**按定义这从行为上不可检测**（不然就不叫"欺骗"）；唯一抓手是读内部，即可解释性。
- (e) CoT 忠实性（⚠高置信记忆，Lanham et al 2307.13702、Anthropic 2025，未能现场复核）：推理被优化后，外显 CoT 可能偏离真实计算；若推理移入潜空间，行为/CoT 监控失效。
- 这五点合起来是"interp×agency 是结构必需而非暂态工具"最严格的论证，对应方向 B。

**⑤ 不可判定与自指（逻辑论）——外层天花板**
- Rice 定理（✅经典教科书结果）：程序的任何非平凡语义性质不可判定。"这个 plan 会达成目标吗/这个动作在所有后果上安全吗"是行为的语义性质 ⇒ **没有任何算法**（包括完美模型、包括 agent 自己）能在一般情况下判定——这不是模型的极限，是任何验证器的极限。
- 自验证极限（Gödel 味）：系统无法完全验证自身正确性，需要外部独立验证，而外部验证器又被 Rice 定理封顶。
- 可操作读法：拿不到通用完美验证器，但能圈出"最大的可判定/可验证子类"——研究动作是"圈子类"而非"造万能验证器"，对应方向 B 的工程化路径。

**架构会不会推翻这些？**（§1.8 的诚实处理）：会推翻一部分，不是全部。自适应深度/looped transformer/端到端潜推理这类架构变化，能把①的 TC⁰ 那条线移动，因为它把"循环"内化进权重了；**但这是"把循环搬进权重"，不是"消灭循环"**——串行计算仍在发生，只是从显式 loop 变成隐式 loop。于是②③④⑤原样转移，甚至更糟，因为推理移入潜空间后更不可监控。结论：①的具体边界是架构相关的，但②④⑤跨架构存活，因为它们的来源不是"单次前向能算什么"，而是"代理目标会发散、监督信号会枯竭、语义性质不可判定"——这些和架构无关。

## 4 · 踩过的坑 / 诚实局限

这一节直接使用 deck 第 13 节"诚实的反方"的原始内容，不弱化、不回避：

1. **领域 artifact 迭代极快** → 我做的若是"东西"会过时。**对冲**：只做 measurement/theory/interp/复现（判据兜底）。
2. **frontier lab 资源碾压** → 比算力必输。**对冲**：选"一个有数学的人能赢工程队"的角（A/B/C/D），躲开 E 的配方竞赛。
3. **可能被下个模型部分解决** → 比如更长原生 context 削弱 compaction 的迫切性。**回应**：B/D 的核心（内部可监控性、最小充分信息）是结构性的，窗口变大不消除。
4. **B 太新、可能做不出信号** → 探针在 agent 轨迹上未必 work。**缓释**：先在可控 toy 上验证信号存在性，再上真模型；做不出本身也是可发表的负结果。
5. **interp 整体还在"能不能 scale"的争议里** → 我赌的是"小规模可证伪的机理"，不赌"全网络可解释"。

**一个真实发生过的自我修正，比任何反方列表都更能说明这份稿子的诚实态度**：deck 最初（第 9 节 + 证据分级表）判断"把 von Neumann 容错阈值定理严肃用于 agent 可靠性，我没见过已有工作 = 潜在 gap（待查证）"。经过后续更深的文献核验（Q4），这个判断被**部分推翻**："我标 von Neumann 阈值用于 agent 疑似空白——部分错：BFT/共识/可靠性工程已大量进场。但相关误差时序阈值定理确实仍空白，且比我原来的说法更精确、更难、更有卖点。" 再往后（Q5→Q6 下载全文精读 8 篇论文）连这个收窄后的版本也被进一步下调——多篇论文已经把"误差相关参数"当作决定命运的旋钮在处理，deck 的结论最终收窄到"没人从 interp 角度去测量/因果降低这个相关参数"这个更小的切口。

这个过程本身暴露了三条更深的、需要坦白承认的局限（Q16 的最终诚实归因）：

- **"plant 不 bind"风险**：多篇文献自承"没有干净的通用模型，可靠性只能逐案工程"——这直接打击任何"harness 大一统理论"的野心，deck 明确判定这是陷阱，要避开。
- **interp 脆性的具体证据**：Chinchilla 负结果（数月探 70B，电路碎片化且脆，稍改任务神经元重要性就掉）；circuits 不泛化、不 scale 到前沿、劳动密集；《Open Problems in MI》（Casper/Tegmark/Bau）质疑根基；ICML'26 MI Workshop CFP 明令"淡化局限的投稿不收"。
- **残余的迁移风险**：即便退到"小规模可证伪信号"这个更谦虚的版本，最大的失败模式仍是"interp 测的相关参数在小模型上有意义、但不迁移到真实 agent——可发表但影响有限"。deck 的缓解方式不是假设它一定迁移，而是把"是否迁移"本身设计成一个可证伪的研究问题。

## 5 · 结果与现状

**必须如实说明**：这是一份**待老师拍板的研讨稿**，不是已经执行完的研究——今天的产出是"判据 + 候选方向打分 + 推荐路径"，不是实验结果、不是论文草稿。deck 第 14 节列出的 6 个问题，就是这份工作现在最诚实的状态：

1. **核心选 B 还是 A？** —— B 更差异化但更新更险；A 更稳但更偏理论。组里资源更支持哪个？
2. **第一篇允许是"复现型 C"吗？** —— 还是希望博 0 直接上差异化（风险更高）？
3. **算力/数据**：B 要在真模型上取激活、跑长轨迹——组里 GPU 与 agent 评测环境够吗？
4. **协作面**：组里/系里有 interp 或 systems/控制的人可合作吗？（B 需 interp，A 需控制）
5. **安全/对齐角度**：B（trajectory 级 oversight）是否好对接安全方向的 funding/合作？
6. **范围**：这五个全押太散——老师建议先收敛到哪 1–2 个？

**补充说明（不算已解决，但说明这不是拍脑袋一次成文）**：deck 在这份 15 页正文之后，附了一轮自我驱动的更深文献核验（Q1–Q16），做了包括下载 8 篇论文全文逐字读定理、9 个方向的饱和度扫描、社会学/管理学框架迁移的可行性核查、学术生态位（谁在做这件事）侦察等工作。这些工作**没有推翻 C→B→D 的推荐路径**，但把"B 的精确切口"从笼统的"interp×agency"逐步锐化到更具体的"用可解释性构造与生成器去相关的校验器"，并且在 Q16 得到一个关键的自我纠偏结论：**大一统版本（干净的 harness 通用理论）被判定为陷阱、应避开；切片版本（小模型、可证伪、interp 驱动的局部机制）才是真正够得着的机会。** 这恰恰说明——第 14 节的 6 个问题依然是悬而未决的、需要导师参与拍板的问题，而不是我自己关起门来已经想清楚、只是走个流程来汇报的事。

## 6 · 追问树

> 每条链：`trigger`（考官开场追问）→ 3–4 层递进追问（每层给参考答案）→ `pitfall`（最容易露怯的一层）。答案全部来自 deck 原文或忠实转述，不新增 deck 没说过的"证据"。

### 追问链 1："你说 C 是起手方向，为什么不直接做最有差异化的 B？"

- **Layer 1** — Q：B 在你自己的打分表里"适配我"是★★★★★最高分，为什么不直接从 B 开始？
  A：是的，B 被称为"最差异化"、"别人最难复制的组合"；但 deck 明确把 C 标注为"tractability 最高，复用 M7+M9"，且第 12 节推荐路径本身就是风控设计——先用可做的测量型第一篇（C）落地一次完整研究循环 + 建测量基建，再上 B 这种 moonshot，不要一上来 all-in 在 moonshot 上（Q2 §2.6）。

- **Layer 2** — Q：那"先建立基建"具体指什么？是不是变相的拖延？
  A：不是拖延，是复利。第一篇 C 复用 Module 7+9 的统计/实验设计能力，直接建起后面 B/D 都要用的测量基建（deck 原句"顺手把后面要用的测量基建建起来"）；而且即便结论保守（"这篇 SOTA 没那么强"），诚实复现本身也有价值。

- **Layer 3** — Q：如果 C 做完发现没什么可发的，会不会浪费一整年？
  A：deck 对 C 的态度本身不美化——"这条在升温"（拥挤度★★★★），差异化不能是又一个榜单，必须是因果/方法论论文，这是需要投入才能验证的假设。但即使第一篇结果保守，它仍然是"完整研究循环"的实战训练，这个训练本身是后面挑战性更高的 B 的必要前置。

- **Layer 4（pitfall 层）** — Q：如果导师觉得资源更支持 B、希望博 0 直接冲差异化呢？
  A：这正是 deck 第 14 节明确留给导师拍板的问题之一（"第一篇允许是复现型 C 吗？还是希望博 0 直接上差异化？"）——这份稿子本身定位是"研讨"而非"已定方案"。如果导师坚持 B 先行，需要重新评估的是 B 的风险缓释路径（先在可控 toy 上验证信号存在性）能否在第一年就产出可发表的阶段性结果。

  **Pitfall**：容易把"我全听导师的"讲成没有自己判断，或反过来固执坚持 C——两者都显得不够扎实。正确姿态是：C→B→D 是我的推荐，但这是开放决策点，需要资源匹配验证。

### 追问链 2："你的判据'抗模型进步测试'本身怎么验证是对的？会不会只是你事后为已选方向找的正当化？"

- **Layer 1** — Q：这个判据本质上是一句"18 个月后模型强一倍会不会解决"，这不就是主观预测吗？
  A：deck 把它精确化为"正交性原理"（Q1 §1.1）：模型进步只沿三条轴移动——函数逼近、对齐、知识压缩；一个问题是"不变量"当且仅当它的难度来源与这三轴正交。这把判据从直觉变成了"逐条核对难度来源在不在这三条轴上"的结构性命题。

- **Layer 2** — Q：但"难度来源与三轴正交"本身怎么证明，不也是你的论证/推断吗？
  A：deck 对此是分级的，不是一刀切宣称"已证明"：五个不变源里，①计算复杂度论（TC⁰，Merrill & Sabharwal 2310.07923）和⑤Rice 定理是✅今日查证/经典教科书结果；而④里的 CoT 不忠实性、weak-to-strong 部分恢复是⚠高置信记忆、未能现场复核。证据分级表本身就是防止过度宣称的机制。

- **Layer 3** — Q：这五个不变源恰好都落在你的牌面（EE 数学+机制可解释性）上，deck 自己也说"这不是巧合"——这句话听起来就是在给自己壮胆。
  A：这是需要正面承认的最脆弱环节。诚实的回应是：判据的数学论证（TC⁰、von Neumann 阈值、POMDP、Goodhart、Rice 定理）独立于"我是谁"而成立，换一个不懂 EE 数学的人来判断，这些定理的真假不会变。巧合只在于"我恰好有能力做这几个方向"，不在于"判据的真假是为配合我的技能而设计的"。但这个区分必须讲清楚，否则确实像预先设计好的自我辩护。

- **Layer 4（pitfall 层）** — Q：判据的"对不对"和"是否适合你"是两件事，那如果判据是对的但你并不是最适合做的人呢？
  A：这正是 deck 单独用第 6 节论证"为什么是我"的原因——判据筛出"哪些问题值得押"，第 6 节回答"谁去押"。证据是具体可核查的：harness-engineering 14 讲 11 测试全绿、agent-harness-design 16 讲、Module 12 真实模型上的 probe/SAE/CoT 忠实性实操、Module 9 的复现方法论,这是"别人要先花半年搭环境,我的 src 现在就能跑"的硬证据,不是空谈数学背景。

  **Pitfall**：这条链最容易在 Layer 3 垮——矢口否认"是巧合"或过度辩护都不对，正确姿势是承认这是全篇论证里最需要外部审视的一环，然后用"判据真假独立于个人"这条线守住,不要试图证明"完全不是巧合"。

### 追问链 3："5 个结构不变源里，哪个的证据等级最低/最需要进一步查证？"

- **Layer 1** — Q：先说说这五个不变源分别是什么证据等级？
  A：①计算复杂度和⑤Rice 定理/Goodhart 定律是✅（今日查证/经典教科书结果）；②von Neumann 阈值定理本身✅经典，但"严肃用于 LLM-agent 可靠性"这个应用被标注"我未见已有工作=潜在 gap（待查证）"；④里的 CoT 不忠实性（Lanham et al 2307.13702、Anthropic 2025）和 weak-to-strong（2312.09390）都标⚠高置信记忆、未能现场复核。

- **Layer 2** — Q：那"未能现场复核"的这两条，如果被证明过时了，方向 B 还站得住吗？
  A：需要区分"具体数字"和"结构性论证"。即使 weak-to-strong 的具体 PGR 数值或 CoT 不忠实性的具体实验设置被后续工作修正，§1.5(b) 可扩展监督鸿沟——"模型强过人类评估者时人给不出可靠训练信号"——是逻辑推导，不依赖某一篇论文的具体数字；(d) 欺骗性对齐"按定义行为上不可检测"也是定义级论证，不是实证论证。

- **Layer 3** — Q：von Neumann 阈值定理那条"未见已有工作"的判断，后来查了吗？
  A：这是 deck 里一次非常诚实的自我修正：后续文献核验发现这个判断"部分错"——BFT/共识/可靠性工程已大量进场；但收窄后的精确版本（正相关误差下的时序容错阈值定理，因为经典定理假设独立、实测误差正相关）仍然空白。再往后甚至这个收窄版本也被发现有相近工作在推进，最终收窄到"用可解释性去相关校验器"这个更小的切口。

- **Layer 4（pitfall 层）** — Q：这种"越查越发现别人已经做了"的过程，会不会说明你一直在追一个越缩越小、可能追到没有的靶子？
  A：deck 最后一轮分析（Q16）专门处理了这个问题，结论是"分版本看"：大一统版本（干净的通用容错阈值定理）已被领域判定难以做到，是陷阱，应避开；切片版本（小模型、可证伪、interp 驱动的局部机制）之所以空，是因为"时机新+跨学科+不时髦"，不是"本质上不可解"——这是专门做过反方查证得出的判断。

  **Pitfall**：最容易露怯的地方是不敢承认"最初判断被证明部分错了"，转而硬拗说"我从一开始就是这么想的"。诚实做法是把这个自我修正过程本身当作方法论严谨性的证据来讲。

### 追问链 4："如果 frontier lab 用更大算力很快把 B 方向的问题解决了怎么办？"

- **Layer 1** — Q：Anthropic/OpenAI 的算力和数据你比不了，他们出个新模型是不是就能把你的探针研究冲掉？
  A：deck 在"诚实的反方"里正面处理了这条——"可能被下个模型部分解决，比如更长原生 context 削弱 compaction 的迫切性"，回应是"B/D 的核心（内部可监控性、最小充分信息）是结构性的，窗口变大不消除"。具体探针实现会过时，但"需要从内部监控"这件事不会。

- **Layer 2** — Q：模型能力提升难道不会让"从输出就能看出问题"变得足够好？
  A：deck 的论证方向相反——自治链条越长，"每一步都盯着"在结构上越不可能（Q1④可扩展监督鸿沟：模型强过人类评估者时，能力越高监督鸿沟越宽，不是越窄）；CoT 忠实性可能随规模"更糟"而非更好，这是 deck 明确说的"随能力变大而变大的杀手类"。

- **Layer 3** — Q：如果模型直接把推理搬进潜空间，不再输出人类可读的 CoT，你的方向 B 不就更难做了？
  A：这正是 §1.8"换架构会不会推翻这些"专门讨论的问题——会推翻一部分不是全部。架构变化能把 TC⁰ 那条线移动，因为它把"循环"内化进权重，但这是"把循环搬进权重"不是"消灭循环"，串行计算仍在发生；于是可靠性/监督/验证这几条不变源原样转移，甚至更糟，因为推理移入潜空间后更不可监控。推理搬进潜空间不是消灭 B 的研究对象，反而让内部可监控性更迫切。

- **Layer 4（pitfall 层）** — Q：这听起来像"不管发生什么我的方向都对"，这种不可证伪的论证不就是有问题的吗？
  A：需要诚实承认边界——deck 自己列出了明确会被模型进步消解的清单：具体 compaction 启发式、context 窗口大小、"lost in the middle"这种特定架构 artifact（deck 说它"正在被解决"）、工具/schema 格式、prompt 技巧、"25–30 次工具调用后退化"这个具体数字；也承认长程可靠性里"很大一部分是暂态的高 ε"，真正不变的只是阈值/复合结构而非今天的具体数值。判据有明确的失效范围，不是万能挡箭牌。

  **Pitfall**：容易把"结构性不变"讲成"我的每个具体做法都不会过时"，这是过度泛化。正确边界是：具体探针/架构细节会过时，但"需要内部监控"这件事本身不会。

### 追问链 5："内部可监控性（方向 B）和现有的 CoT 监控有什么本质区别？"

- **Layer 1** — Q：现在大家不是已经在用 CoT 监控 agent 了吗，方向 B 跟这个有什么不一样？
  A：deck 明确指出现状局限——"现有 self-correction 都是错后才纠"（reactive，被明确列为开放问题），"所有 oversight 都基于输出，内部状态监控几乎空白"。CoT 监控仍是"看模型说了什么"，是输出/行为层监督；方向 B 要做的是 proactive：在输出暴露之前，从激活直接预测"即将走错/已丢目标/在编造"。

- **Layer 2** — Q：CoT 本身不也是一种"内部"信息吗？
  A：关键区别在于 CoT 是模型生成的、可被优化的外显文本。§1.5(e) 明确指出 CoT 常不忠实于真实计算——"模型不汇报改变其答案的线索"（Lanham et al 2307.13702、Anthropic 2025，⚠未能现场复核）；对"可监控外显"的优化压力会诱导出"看起来可监控"却未必反映成因的轨迹。CoT 是模型"愿意讲给你听"的东西，仍是行为层，可以被 gaming；方向 B 读的是激活/内部表征，不依赖模型"愿不愿意说"。

- **Layer 3** — Q：那和"debate"这类黑盒监督方法比呢？
  A：deck 把两者定位为并列互补而非竞争——"debate 从行为做监督且有真复杂度理论；interp 从内部做监督是其互补面。定位 B=白盒监督，与 debate 黑盒监督并列、互补"。Debate 理论上能让多项式 judge 判定 PSPACE（Irving-Christiano-Amodei），但信号来源仍是模型的输出/论证行为；方向 B 的信号来源是内部激活，是监督问题的两个不同切面。

- **Layer 4（pitfall 层）** — Q：你怎么知道"读激活"就一定比"读输出"更难被 gaming？模型会不会学会对探针隐藏激活？
  A：deck 明确把这个当作"有理论分量的硬骨头"而非回避的漏洞——"探针抗规避：能力越强模型越可能对探针隐藏激活，这把问题接到监督 gap 随能力变宽这个不变量"。方向 B 没有假设内部监控天然免疫，"探针可能被规避"本身是需要研究的开放问题——正因为存在这个结构性张力，B 才是持续的研究对象而非一次性工程方案。

  **Pitfall**：最容易把 B 说成"内部监控绝对可靠、CoT 监控注定失败"，这是过度承诺。诚实立场是两者互补，且 B 自身也有尚未解决的抗规避问题。

### 追问链 6："为什么选 C（因果归因）当第一篇而不是更快能发出来的东西？"

- **Layer 1** — Q：因果推断是个很重的方法论包袱，想快速证明自己能做研究，是不是有更简单的题目？
  A：deck 对 C 的定位不是"最简单"而是"tractability 最高、复用 M7+M9"——综合抗模型进步、拥挤度、适配我三项打分后，C 是唯一被标"第一篇/起手"角色的方向。deck 的诚实提醒是"这条在升温，差异化不能是又一个榜单，而是因果/方法论论文"——选 C 不是图省事，而是承认存在被做烂的风险，需要用方法论深度对冲。

- **Layer 2** — Q："model×harness 的交互项"这个具体切口，真的没人做过吗？
  A：后续更细致的实查确认这一点在理论层面是真空白——harness 能摆动分数 10–35 分，但 Harness-Bench 自我声明只是"诊断级、非因果分解"；另一篇因果保证文直言"目前没有系统理论严格支持 agent 评测的因果推断"。这确认了 deck 第 5/9 节的判断——"测量做出来了，因果/机理还没有"——不是想当然的判断。

- **Layer 3** — Q：这个方向拥挤度打了★★★★（升温），怎么保证不被抢先？
  A：对冲策略是不做"又一个榜单"（不重复已有的全栈榜单/能力解耦观察），而是把交互项当"正经统计对象"——用 EE 统计+析因设计+置信区间，产出方法论/因果论文而非又一次评测。deck 清楚知道热门在哪、冷门缺口在哪，不是盲目选了个热门方向。

- **Layer 4（pitfall 层）** — Q：如果 C 做出来只是"确实存在交互项，但没有惊艳的新洞见"，算不算浪费了博一？
  A：deck 正面接受了这种可能性——即使"这篇 SOTA 没那么强"，"诚实复现也有价值"；更重要的是第一篇的价值不完全在单篇论文的影响力，而在于"顺手把后面要用的测量基建建起来"，为 B（需要在真实 agent 轨迹上取激活）和 D（率失真理论需要真实压缩失真数据）提供可复用的评测/统计基础设施。

  **Pitfall**：容易把"选 C"说成纯功利选择（因为好发），回避"领域在升温、可能被做烂"的真实风险。正确姿态是承认拥挤度风险、说明差异化对冲手段、接受"可能不惊艳"的结果预期。

### 追问链 7："方向 B 说要在 agent 轨迹上训探针，这个技术本身靠谱吗？我听说 interp 领域现在争议很大，说它根本 scale 不了。"

- **Layer 1** — Q：探针在 agent 长程轨迹上真的能训出信号吗？
  A：deck 对此的诚实态度是"B 太新、可能做不出信号——探针在 agent 轨迹上未必 work"，这是 deck 自己列在反方里明确要证伪的一条。缓释策略是分阶段验证：先在可控 toy 上验证信号存在性，再上真模型；做不出本身也是可发表的负结果。第一篇起手设计对应这个策略：在会漂移的 toy agent 上训线性探针，预测"第 k 步→30 步后失败"。

- **Layer 2** — Q："做不出也是负结果"听起来像给失败找退路？
  A：Module 9 科研技能训练的核心态度是"诚实复现能力"，负结果在这个价值体系里不是退路而是被认可的产出形式——前提是必须经过真实的、有控制变量的实验设计，且要报告清楚具体条件下（哪个模型规模、哪种漂移类型）信号不存在，这本身对领域仍是信息增量。

- **Layer 3** — Q：interp 学科本身争议很大，说电路发现不泛化、根本撑不起大规模应用，你怎么保证没押在一个正在被证伪的范式上？
  A：deck 对这条争议是正面承认的，且给出具体证据——Chinchilla 负结果（数月探 70B，电路碎片化且脆，稍改任务神经元重要性就掉）；circuits 不泛化、不 scale 到前沿、劳动密集；《Open Problems in MI》（Casper/Tegmark/Bau）质疑根基；ICML'26 MI Workshop CFP 明令"淡化局限的投稿不收"。deck 的立场是"我赌的是小规模可证伪的机理，不赌全网络可解释"——不赌 interp 能完整解释大模型的一切认知，而是赌在小/开源模型（gpt2/TinyLlama/Gemma Scope 覆盖的 270M–27B）上能做出局部、可证伪的机理信号。

- **Layer 4（pitfall 层）** — Q：如果连"局部可证伪信号"这个更谦虚的版本，最后在真实 agent 上也不迁移呢？
  A：这是 deck 最后一轮分析明确指出的残余风险，不是被掩盖的问题——"即便切片，最大失败模式是 interp 测的相关参数在小模型上有意义、但不迁移到真 agent，可发表但影响有限"。缓解方式是把"是否迁移"本身当成一个可证伪的研究问题，而非默认假设它迁移。

  **Pitfall**：最容易在 Layer 3/4 垮掉的地方，是想一口咬定"interp 肯定能 scale"来显得方向稳，或者被问住了就说"那我换方向"——两者都不对。正确姿态是承认学界争议真实存在、承认自己的具体赌注范围（小规模、可证伪、非全网络），并承认迁移风险需要被当作研究问题而非假设。
