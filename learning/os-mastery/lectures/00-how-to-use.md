# OS Mastery 使用说明

## 为什么这是五专题队列的收官之作

老手指出用户EE转NLP/LLM背景缺少科班CS学生默认已掌握的软件工程/设计模式/数据库/网络/操作系统基础知识,这是五个专题里的最后一个,组织方式沿用前四个专题(`software-engineering-mastery`/`design-patterns-mastery`/`database-mastery`/`networking-mastery`)已验证的"讲解+追问链"混合格式与"难度分层"轴。

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:先建立"程序怎么被操作系统管理起来运行"这套基础认知——进程与线程的基本单位区别、内存怎么被虚拟化和分页管理。这一层的重点是把每个机制"要解决什么具体问题"想清楚,比如虚拟内存要解决的是"让每个进程都以为自己独占了整块内存,同时实现进程间隔离和物理内存的高效复用",不是简单背"虚拟地址映射到物理地址"这句话。
2. **Tier 2(深,`tier2_deep/`)**:机制深水层,三个文件分别对应"CPU时间怎么被公平分配并避免死锁"(调度与同步,CFS/锁/死锁四条件)、"数据怎么被高效存取"(文件系统与IO,inode/epoll演进/零拷贝)、"一台物理机怎么被安全地共享给多个隔离环境"(虚拟化与容器,namespace/cgroups)。这三条线索有内在联系:操作系统的核心职责就是"多路复用硬件资源同时提供隔离",调度解决CPU的多路复用,文件系统与IO解决存储的高效访问,虚拟化与容器解决整机资源的隔离共享,建议练习时留意这条"资源多路复用与隔离"的统一视角。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"生产环境性能问题怎么排查"(CPU飙升/内存泄漏/IO瓶颈/容器OOMKilled),第二个文件考"系统设计里OS层面的选型怎么判断"(高并发IO模型选型/内核参数调优边界/CPU亲和性)。这两个文件是把tier1/tier2的机制知识转化为真实工作场景里的判断力,`explain` 字段建立背景知识框架,不是给出唯一正确答案。

## 一个容易被忽视的陷阱

操作系统的面试追问链经常在"名词定义"这一层很容易答对(比如背出进程三种状态:就绪/运行/阻塞),但第二三层追问"为什么"或者"具体机制"时容易露怯——比如"为什么线程切换比进程切换快,具体省掉了哪一步""select/poll/epoll的演进具体解决了什么问题,时间复杂度怎么从O(n)降到O(1)""容器和虚拟机的隔离性差异具体体现在共享了操作系统的哪一层"。这类"知道名词但说不出机制细节"的陷阱,是这个专题训练的核心目标,`pitfall`/`trap` 字段反复指向的正是这类问题。
