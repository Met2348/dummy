# Networking Mastery 使用说明

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:先建立"数据怎么从一台机器传到另一台机器"的分层认知框架(OSI/TCP-IP)和寻址机制(IP/MAC/DNS),再理解"传输层协议怎么保证可靠性"(TCP三次握手四次挥手)和"应用层协议怎么演化"(HTTP/1.1到HTTP/2)。这一层的重点是把每个机制"要解决什么具体问题"想清楚,比如三次握手要解决的是"避免旧的失效连接请求被误认为新连接",而不只是背"SYN/SYN-ACK/ACK"这三个词。
2. **Tier 2(深,`tier2_deep/`)**:机制深水层,三个文件分别对应"传输层怎么应对网络拥塞"(TCP拥塞控制,慢启动/拥塞避免/快重传快恢复)、"应用层怎么保证安全和效率"(HTTPS与Web安全,TLS握手/HTTP2多路复用)、"多台服务器之间流量怎么调度"(负载均衡与CDN,一致性哈希/CDN缓存策略)。这三条线索覆盖了"单条连接内部优化→协议层安全与效率→多机协同"这样一个从小到大的视角切换,建议练习时留意这个视角变化。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"网络架构选型怎么判断"(Service Mesh vs API Gateway、跨机房设计),第二个文件考"生产网络故障怎么排查"(延迟突增/丢包/连接耗尽)。这两个文件把tier1/tier2的机制知识串成"真实工作场景里怎么用",`explain` 字段是建立判断框架,不是给出唯一正确答案。

## 一个容易被忽视的陷阱

计算机网络的面试追问链经常在"协议流程本身"这一层很容易答对(比如背出三次握手的三个步骤),但第二三层追问"为什么"或者"如果...会怎样"时容易露怯——比如"为什么两次握手会出错,能不能举一个具体场景""为什么HTTP/2解决了应用层队头阻塞但还需要HTTP/3""慢启动为什么是指数增长而拥塞避免是线性增长"。这类"知道流程但说不出机制背后的设计动机"的陷阱,是这个专题训练的核心目标,`pitfall`/`trap` 字段反复指向的正是这类问题。
