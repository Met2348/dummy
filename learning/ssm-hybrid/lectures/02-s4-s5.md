## Slide 1 · S4 motivation

```
长序列建模:
  RNN: vanishing grad
  CNN: 局部
  Attention: O(L²)
  S4: O(L log L) 卷积形式 + HiPPO 长记忆
```

## Slide 2 · S4 完整定义

```
SSM 参数: A (d, d), B (d, 1), C (1, d), D (skip)
连续: dx/dt = Ax + Bu, y = Cx + Du
离散: x_k = A_bar x_{k-1} + B_bar u_k
```

## Slide 3 · A 矩阵 — 关键

HiPPO 初始化：
```
A_{nk} = -√((2n+1)(2k+1))  n>k
         -(n+1)            n=k
         0                 n<k
```

decay polynomial 投影。

## Slide 4 · 卷积形式

```
y = K * u
K = [C B_bar, C A_bar B_bar, C A_bar^2 B_bar, ...]
```

K 是 SSM 的"impulse response"，长度 L。

## Slide 5 · 计算 K — 频域

直接算 K 需 O(L · d²)。S4 用频域：
```
K_k = C (I - A_bar)^{-1} (I - A_bar^L) B_bar
```

FFT 实现 O(L log L)。

## Slide 6 · diagonal A trick

S4 实务用 diagonal A（DPLR/HiPPO-LegS）：
```
A = diag(λ_1, ..., λ_d)
```

K 简化为 weighted exponential。

## Slide 7 · S5 (Simplified)

去掉 S4 的复杂数学：
```
A = diag(λ_i)  (complex)
parallel scan 算 y
```

→ 实现更简单，性能略低 1-2%。

## Slide 8 · parallel scan

```
prefix sum 风格:
  out[i] = A_i · out[i-1] + B_i · u_i
↓
O(log L) depth 并行
```

associative property: (A*x + b) 可拼接。

## Slide 9 · 与 RNN 区别

```
RNN: 任意 cell, 非线性
S4/S5: linear SSM, 线性
↓
S4 用 linear → 可 FFT/scan parallel
RNN 不行
```

## Slide 10 · 训练时

S4 用 K 卷积 (FFT)。
S5 用 parallel scan。
两者 forward O(L log L)。

## Slide 11 · 推理时

S4/S5 用 recurrent 形式 → O(d_state) per step。

## Slide 12 · 与 Transformer 内存

```
attention: KV cache O(L · d)
S4:        state O(d_state) <<<
```

state size 通常 16-64，比 attention 小 100×。

## Slide 13 · S4 论文成绩

```
PG-19 (book) ppl:
  Transformer-XL: 25.5
  S4: 19.9
LRA (Long Range Arena): 86% (S4) vs 65% (Transformer)
```

## Slide 14 · 代码 s4_naive.py

```python
class S4Naive(nn.Module):
    def __init__(self, d, d_state=64):
        self.A = nn.Parameter(hippo_init(d_state))
        self.B = nn.Parameter(torch.randn(d_state, 1))
        self.C = nn.Parameter(torch.randn(1, d_state))
        self.log_delta = nn.Parameter(torch.zeros(d))
    def forward(self, u):
        delta = exp(self.log_delta)
        A_bar, B_bar = discretize(self.A, self.B, delta)
        # convolution 或 recurrent
        ...
```

