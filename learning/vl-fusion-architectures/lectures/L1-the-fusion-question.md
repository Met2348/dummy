# L1 · 核心问题: 视觉 token 到底怎么接进 LLM

> 30-min lecture · 目标: 理解 VLM 设计最关键的问题 —— 视觉如何接入 LLM, 建立三条主流路线的全局地图。

---

## 0. VLM = 视觉塔 + LLM + ??? 

M10.1 给了你一个懂语言的视觉塔, 它吐出视觉 token。你也已有一个文本 LLM。看似只差「拼起来」, 但这一步恰恰是 VLM 设计**最关键、决定一切**的地方:

> **核心问题: 视觉 token (来自 ViT) 和文本 token (LLM 的母语) 怎么融合?** 两者 embedding 空间不同、维度可能不同、谁先谁后、占不占 LLM 的上下文 —— 这些选择定义了 VLM 的架构、成本、能力上限。这是你造 VLM 第一个、也是最该想清的决定。

---

## 1. 三条主流路线 (本模块的地图)

```
   视觉 token ──→ ??? ──→ LLM

   ① 投影 (LLaVA)        视觉 token 投到 LLM 空间, 当普通 token 拼进输入序列
   ② cross-attn (Flamingo) 在 LLM 层间插 cross-attn, 文本「看」视觉; 视觉不占序列
   ③ early-fusion (Chameleon) 视觉和文本从第一层就当同一种 token, 无独立连接器
```

一句话区分:
- **投影**: 视觉变成「LLM 看得懂的额外 token」, 排进队伍。最简单。
- **cross-attn**: 视觉在「旁路」, 文本通过 attention 去查它。视觉不占队伍位置。
- **early-fusion**: 根本不分视觉/文本, 全是 token, 一锅煮。最统一。

---

## 2. 区分它们的两个关键维度

不同路线的差别, 可归到两个问题:

### 维度一: 视觉占不占 LLM 的输入序列?

```
   占 (投影/early-fusion):  [v1 v2 ... v16 | t1 t2 ... t10]  → 序列长 = 16+10
   不占 (cross-attn):        [t1 t2 ... t10]  + 旁路视觉(cross-attn 注入) → 序列长 = 10
```

> 这直接决定**成本**: 视觉占序列 → LLM 上下文变长 → attention 是 $O(n^2)$ → 高分辨率图 (上千 patch token) 会撑爆。cross-attn 把视觉移出序列, 但代价是改 LLM 结构。**这是 VLM 的核心成本-能力张力** (接 M10.1-L4 的 token 数权衡)。

### 维度二: 要不要改 LLM 结构 / 能不能复用纯文本预训练?

- **投影**: 不改 LLM 内部, 直接复用现成的纯文本 LLM (冻结), 只训一个小 MLP。**最省。**
- **cross-attn**: 要在 LLM 层间插新的 cross-attn 层, 改了结构。
- **early-fusion**: 不改结构, 但视觉文本同质 → 通常**得从头训**, 不能直接拿现成纯文本 LLM。

> 这决定**工程门槛**: 想用现成 LLM 快速造 VLM → 投影 (LLaVA 的成功秘诀); 想要统一架构、理解生成一体 → early-fusion (但要从头训, 贵)。

---

## 3. 一张速查表 (本模块要填满的)

| 路线 | 视觉占序列? | 改 LLM 结构? | 能复用文本 LLM? | 代表 | 详见 |
|---|---|---|---|---|---|
| 投影 | 是 | 否 | 是 (冻结) | LLaVA | L3 |
| cross-attn | 否 | 是 | 部分 | Flamingo | L2 |
| early-fusion | 是 | 否 | 否 (从头训) | Chameleon/Fuyu | L4 |

`connectors.param_comparison()` 给的就是这张表的代码版。N1 会让你在同一 tiny setup 上跑三种连接器, 亲眼看序列长度差异。

---

## 4. 为什么没有「最好」, 只有「适配」

新手想问「哪个最强」。答案: **取决于约束。**
- 算力少 + 有现成 LLM + 快速验证 → **投影** (LLaVA, 一个 MLP 起步)。
- 要处理多图/视频/长视觉 + 不想撑爆上下文 → **cross-attn** (Flamingo)。
- 要统一理解+生成 (图也能生成, M10.4) + 愿意从头训 → **early-fusion** (Chameleon)。

> 这呼应你 9.4 的实验设计精神: 架构选择是有**设定**的决定, 没有脱离约束的「最优」。造 VLM 前先问清你的约束 (算力/数据/是否要生成/分辨率), 再选路线。L2-L4 逐个深入, L4 给完整决策树。

---

## 5. 本讲小结 + 通往 L2

- VLM 最关键的决定: **视觉 token 怎么融合进 LLM**。
- 三路线: **投影** (当 token 拼) / **cross-attn** (旁路注入) / **early-fusion** (同质一锅煮)。
- 两个区分维度: **视觉占不占序列** (成本) + **改不改 LLM/能否复用文本预训练** (门槛)。
- 没有最好, 只有适配约束。

> **下一讲 L2「cross-attention 路线」**: 先深入 Flamingo 路线 —— 用 perceiver resampler 把视觉 token 压缩, 再让文本 cross-attend 它, 视觉不占输入序列。这是处理多图/视频的经典方案。
