# L3 · idea pipeline: 从灵感闪现到可立项的 4 道工序

> 35-min lecture · 目标: 建一条让 idea **不漏、不卡、能落地**的流水线, 把「偶尔的灵感」变成「稳定的研究项目供给」。

---

## 0. 为什么 idea 需要「流水线」而不是「灵感」

新手对 idea 有个浪漫误解: 好 idea 是某个天才瞬间「想出来」的。真相相反:

> **好 idea 不是想出来的, 是「养」出来的。** 它先以一个模糊念头出现 (洗澡时、读论文时), 经过捕获、孵化、和别的念头碰撞、筛选, 最后才长成一个可执行的项目。整个过程可能跨几周几个月。

如果没有流水线, 两种失败必然发生:
1. **漏**: 闪过的念头没记, 忘了。(最常见, 也最可惜 —— 你的好点子被别人发出来时你才想起「我当时也想到了」)
2. **卡**: 记了一堆念头, 全停在「想法」阶段, 没有一个推进到「能动手」。

流水线就是治这两个病: **入口宽 (什么念头都接, 不漏), 出口严 (只放行通过筛选的, 不卡)。**

## 1. idea 流水线的四道工序

```
  ① 捕获 CAPTURE        ② 孵化 INCUBATE       ③ 筛选 SCREEN         ④ 立项 COMMIT
  ─────────────        ─────────────        ─────────────        ─────────────
  任何念头秒落地         让它和别的念头/        用三筛 + 优先级        写成正式 idea 卡,
  不评判、不修饰         论文碰撞, 长大         砍掉大部分            进入实验设计 (9.4)
       │                    │                    │                    │
  idea inbox          permanent notes      gap 库 + 打分          idea 卡 (9.3)
  (一行就行)           双链 (L2)           Priority公式           可证伪假设
       │                    │                    │                    │
  ✓ 入口极宽          ✓ 时间发酵           ✓ 出口收窄           ✓ 少而精
```

逐道工序讲清楚 ——

### ① 捕获 (capture): 入口要无摩擦

- **目标**: 任何时刻闪过的念头, **3 秒内**有地方落下。摩擦大一点, 你就懒得记, 念头就漏了。
- **工具**: 一个 `idea-inbox.md` (手机上也能记), 或 Obsidian 的 daily note。一行字就行: 「DPO 在多轮对话里会不会退化?」
- **铁律**: **捕获阶段绝不评判。** 不要在这一刻就想「这个是不是太蠢/已经有人做了」—— 评判是工序③的事。在捕获阶段评判, 会让你的大脑不敢往外冒念头, 入口就堵死了。
- 对应 Zettelkasten 的概念叫 **fleeting note (闪念笔记)**: 短命的、随手的, 等着被处理或丢弃。

### ② 孵化 (incubate): 让念头发酵长大

- **目标**: 一个刚捕获的念头通常太模糊。孵化 = 让它和你的**永久笔记网络**碰撞, 慢慢长出形状。
- **怎么做**: 定期 (weekly review, L4) 翻 inbox, 对每个还活着的念头问: 「它和我哪条永久笔记相关? 哪篇论文碰过? 」然后**双链过去**。
- 一旦一个念头开始累积链接 (这篇论文的 gap + 那条永久笔记 + 我那个复现), 它就从「闪念」升级成「值得认真对待的候选」。
- **为什么要发酵而不是马上做**: 大多数第一反应的念头, 要么已被做过, 要么是死路。发酵期 (几天到几周) 让你的潜意识 + 后续读到的论文帮你过滤。**时间本身是最便宜的筛子。**

### ③ 筛选 (screen): 出口要狠

- **目标**: 孵化出的候选还是太多, 资源 (你的时间) 极有限, 必须狠砍。
- **工具**: 直接复用 9.3 的两把筛子:
  - **优先级公式**: $\text{Priority} = \dfrac{\text{Importance} \times \text{Tractability}}{\text{Cost}}$
    - **Importance (重要性, 1-5)**: 做出来多少人会在意。
    - **Tractability (可做性, 1-5)**: 以你现在的资源能不能真做出来。
    - **Cost (成本, 1-5)**: 要多少算力/时间/数据。
    - 分子是「值得且能做」, 分母是「要花多少」; **高分 = 高杠杆**。
  - **idea 三筛 (一票否决)**: ☐ 没人解决过? ☐ 我能做? ☐ 重要 (so-what)? 任一不过即砍。
- **博0 现实建议**: 第一个项目刻意选 **Tractability 高、Cost 低** 的 (哪怕 Importance 中等)。先跑完一个完整研究循环比一上来挑大题更重要。

### ④ 立项 (commit): 变成可执行项目

- **目标**: 通过筛选的少数 idea, 升级成一张正式 **idea 卡** (9.3 的 `idea-card.md`), 写清:
  - **可证伪假设** (能被实验推翻的具体陈述),
  - **最小验证实验** (一周内出第一个信号),
  - **预期的那张关键图**, 风险, 复用哪个已有专题的代码。
- 立项后, 这个 idea 就**离开知识管理领域, 进入实验设计 (9.4)**。流水线在这里交棒。

## 2. 一张图看懂「漏斗」形状

```
   捕获 (入口宽)      ████████████████████████  20 个闪念/月
                              │ 孵化淘汰 (大多没长大)
   孵化后候选         ████████                  6 个值得想的
                              │ 三筛 + 优先级狠砍
   通过筛选          ███                        2 个能做的
                              │ 立项
   真正动手          █                          1 个项目
```

这个漏斗的关键设计原则:
- **顶部要尽量宽**: 捕获越多越好, 因为你不知道哪个会长大。压制捕获 = 自断供给。
- **底部要尽量窄**: 真正动手的越少越好, 因为你时间极有限。**博士不是 idea 多的人赢, 是把对的 idea 做完的人赢。**
- 中间靠**时间 (孵化)** 和**公式 (筛选)** 自动过滤, 不靠你当场拍脑袋。

> 一个常见反模式: 「我 idea 太多做不完」。这几乎总是出口 (筛选) 太松, 而不是入口太宽。解药不是少捕获, 是**狠筛选** —— 逼自己同一时间只 commit 1 个项目。

## 3. 这条流水线如何接住整个 Module 9

```
   9.2 文献综述 ─┐
   9.3 读论文找gap ┼─→ ① 捕获 (inbox) ─→ ② 孵化 (永久笔记网) ─→ ③ 筛选 (Priority) ─→ ④ 立项 (idea卡) ─→ 9.4 实验设计
   日常闪念 ──────┘
```

- 9.2 (摸领域) 和 9.3 (找 gap) 是这条流水线的**主要进料口**。
- 9.1-L2 的永久笔记网络是**孵化场**。
- 9.3 的优先级公式 + 三筛是**筛选器**。
- 9.4 实验设计是**下游**, 接收立项的 idea 卡。

**整个 Module 9 其实就是一条「知识 → idea → 实验 → 论文」的大流水线, 本讲是其中 idea 这一段。**

## 4. 本讲小结 + 通往 L4

- 好 idea 是「养」出来的, 需要流水线, 不能靠灵感。
- 四道工序: **捕获** (入口无摩擦、不评判) → **孵化** (双链发酵、时间过滤) → **筛选** (Priority 公式 + 三筛, 狠砍) → **立项** (升级 idea 卡, 交棒 9.4)。
- 漏斗原则: 顶宽底窄; 博士赢在「把对的做完」, 不在「想得多」。

> **下一讲 L4「信息流系统」**: 流水线的进料 (捕获) 很多来自读新论文。但 arxiv 每天上百篇, 怎么追而不被淹没? L4 给你一条可持续的 arxiv triage + weekly review 流程 —— 也正是「孵化」和「清 inbox」固定发生的地方。

**动手**: 现在就建一个 `idea-inbox.md` (用 `templates/idea-inbox.md`), 把你此刻脑子里任何一个「我好像可以试试…」的念头扔进去。别评判, 就记下来。这是你 idea 流水线的第一滴水。
