# L24 · 高级字符串算法 —— KMP / 中心扩展 / Rabin-Karp 滚动哈希

> 对应代码：`src/problems/p24_advanced_strings.py`。

## 这几题为什么算"硬核"

Phase 1 的字符串章节（04 · 字符串）处理的题目大多可以用"暴力 + 一点小聪明"（双
指针、滑动窗口）在可接受的复杂度内解决。这一章的几道题不一样——想要做到线性或
近线性复杂度，**必须**依赖专门设计的字符串算法（KMP、Manacher、Rabin-Karp），
单纯的暴力优化到头也甩不掉平方级的复杂度。这是字符串算法里真正需要"背算法"而不是
"临场想技巧"的部分。

## 深挖：KMP 实现 strStr（LC28，Medium）—— 失配时为什么能跳过一大截

**先想清楚朴素做法在浪费什么**：暴力字符串匹配的做法是，对 haystack 的每一个
起点，都从头开始逐字符比较 needle，一旦发现某个字符不匹配，就把起点往后挪一位、
重新从 needle 的第 0 个字符开始比较。这个"退回起点"的动作，正是浪费所在——
匹配失败之前，我们其实已经知道"刚刚被匹配上的这一段"内容是什么（它等于
`needle[0..j-1]`，j 是失配时 needle 指针的位置），但朴素做法直接把这个信息扔掉了。

**KMP 的核心 insight**：如果 `needle[0..j-1]`（刚匹配上的这一段）自身存在一个
"长度为 L 的前缀，同时也是它的后缀"，那么这个前缀必然已经在 haystack 里出现过——
就是刚匹配上的那一段的**开头部分**。既然这部分已经确认匹配，下一次比较完全可以
直接从 `needle[L]` 继续，而不必把 needle 的指针整个退回 0 重新开始。这正是
**lps 数组**（最长相同前后缀长度数组）要存的信息：`lps[i]` 表示 `pattern[0..i]`
这一段里，"最长的、既是真前缀又是真后缀"的长度。

```python
def _build_lps(pattern: str) -> list[int]:
    n = len(pattern)
    lps = [0] * n
    length = 0
    i = 1
    while i < n:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        elif length != 0:
            length = lps[length - 1]
        else:
            lps[i] = 0
            i += 1
    return lps
```

主匹配过程中，一旦发生失配（`haystack[i] != needle[j]` 且 `j != 0`），把 needle
的指针 `j` 跳到 `lps[j-1]`——这一步跳过的，正是"已经证明不可能匹配"的那部分位置，
而 **haystack 的指针 `i` 完全不需要回退**，因为 haystack 里已经扫过的字符不会
因为 needle 指针的调整而需要重新比较。

```python
i = j = 0
while i < len(haystack):
    if haystack[i] == needle[j]:
        i += 1
        j += 1
        if j == len(needle):
            return i - j
    elif j != 0:
        j = lps[j - 1]
    else:
        i += 1
return -1
```

这就是 KMP 相比朴素算法的本质提升：**haystack 的指针只会前进、不会回退**，整个
匹配过程摊还下来是 O(n+m)，而不是朴素算法最坏情况下的 O(n·m)。

**容易出错的地方**：失配时如果图省事直接把 `j` 重置为 0（退化成朴素算法），虽然
结果依然正确，但完全丧失了 KMP 的加速效果——这是初学者写"看起来像 KMP、其实是
暴力"代码的常见误区，必须确认失配分支写的是 `j = lps[j-1]` 而不是 `j = 0`。

## 其余 3 题点睛

- **LC459 重复的子字符串模式**：判断字符串能否由更短的子串重复拼接而成。最简洁
  的写法是 `s in (s+s)[1:-1]`，但更能体现"进阶理解"的是复用 LC28 的 lps 数组：
  取 `s` 自身 lps 数组的最后一个值 `border`（s 的最长相同前后缀长度），若
  `border > 0` 且 `len(s) % (len(s) - border) == 0`，说明 s 存在一个长度为
  `len(s) - border` 的最小循环节——这是因为"最长相同前后缀"和"字符串的最小周期"
  在数学上是等价的概念。`border == 0` 时不能直接套这个公式（`len(s)-border`
  等于 `len(s)` 本身，取模恒为 0，会被误判为存在循环节）。
- **LC647 回文子串**：中心扩展法——每个回文串都有一个对称中心（奇数长度是一个
  字符，偶数长度是相邻字符间的空隙），枚举所有 n 个字符中心和 n 个"空隙"中心，
  从每个中心向两边同时扩展统计能扩展出多少个回文串。复杂度 O(n²)，对面试场景
  已经足够；如果需要 O(n)，要用 Manacher 算法（利用回文串关于中心对称的性质
  复用已算出的信息），但其实现复杂度在真实面试里通常不要求手写，了解存在即可。
- **LC1044 最长重复子串**：二分答案（子串长度）+ Rabin-Karp 滚动哈希。二分
  是因为"是否存在某个长度的重复子串"具有天然的可判定性，可以用二分查找答案；
  对每个固定长度，用滚动哈希把"判断某个长度的子串是否出现过"降到均摊 O(1)——
  窗口滑动一位时，新哈希值可以用旧哈希值通过"减去滑出字符、乘 base、加上滑入
  字符"的公式 O(1) 递推。取模的 mod 必须是足够大的质数（代码里用 Mersenne 质数
  `2^61-1`），且哈希相同不代表子串真的相同，发现哈希匹配后必须用真正的字符串
  切片比较做二次确认，避免哈希碰撞导致误判。
