# L13 · DP 进阶：01 背包 / 双序列比较 / 区间 DP

对应代码：`src/problems/p13_dp_advanced.py`

## 这一类在讲什么

`12-dp-basics`（如果你已经学过）讲的是"一维状态、单序列"的 DP：`dp[i]` 只依赖
`dp[i-1]`、`dp[i-2]` 这种线性历史。这一类是三个进阶方向，思考方式各不相同：

| 方向 | 状态怎么定义 | 代表题 |
|---|---|---|
| 01 背包 | "选或不选某个物品"，每个物品只能用一次 | 分割等和子集 |
| 双序列比较 | 两个下标 `i, j` 同时向前推进，比较两个序列 | 最长公共子序列、编辑距离 |
| 区间 DP | `dp[i][j]` 表示"解决区间 `[i,j]`（或开区间 `(i,j)`）"，枚举分割点/最后一步 | 戳气球 |

三个方向的共同点：都是把一个"看起来要枚举指数级可能性"的问题，通过"定义子问题 +
找到子问题之间的递推关系"压缩成多项式时间。区别在于"子问题怎么切"：01 背包按
"物品"切、双序列按"两个下标"切、区间 DP 按"区间边界"切。

## 深挖 1：编辑距离——双序列 DP 的通用模板

**题目**：`word1` 变成 `word2`，每次可以插入/删除/替换一个字符，求最少操作数。

### 为什么要开二维 dp

只有一个字符串的时候，一维 `dp[i]` 就够了（比如"最长递增子序列"）。但编辑距离
天生要同时跟踪"`word1` 走到哪了"和"`word2` 走到哪了"两条进度——这就是"双序列
DP"的标志：**两个下标 `i, j` 同时出现在状态里**，`dp[i][j]` 表示"把 `word1`
前 `i` 个字符变成 `word2` 前 `j` 个字符，最少要几步"。

### 三个转移分支对应哪个下标动

这是新手最容易卡住的地方——插入、删除、替换到底谁改 `i`、谁改 `j`？把它想象成
"对齐两个字符串"的过程：

- **插入**一个字符到 `word1`：插入的目的是让 `word1` 多出一个字符去匹配
  `word2[j-1]`，也就是说"消耗了 `word2` 的一个字符"，但 `word1` 自己的进度
  没变。所以状态是 `dp[i][j-1] + 1`（`i` 不动，`j` 退一步）。
- **删除** `word1` 的一个字符：直接扔掉 `word1[i-1]`，`word1` 的进度往前挪，
  `word2` 没被消耗。状态是 `dp[i-1][j] + 1`（`i` 退一步，`j` 不动）。
- **替换**：把 `word1[i-1]` 直接改成 `word2[j-1]`，一步同时把两边都往前推。
  状态是 `dp[i-1][j-1] + 1`（`i`、`j` 都退一步）。

记忆口诀：**插入消耗对方、删除消耗自己、替换两边都消耗**。如果 `word1[i-1] ==
word2[j-1]`，两个字符本来就相等，不用花操作，直接 `dp[i][j] = dp[i-1][j-1]`
（成本 0，蹭一下"替换"分支但不加 1）。

### 边界

`dp[i][0] = i`：`word2` 是空串，要把 `word1` 前 `i` 个字符全删光。
`dp[0][j] = j`：`word1` 是空串，要插入 `j` 个字符凑出 `word2`。

这套"两个下标同时推进 + 三个转移分支各对应一个下标的移动"就是双序列 DP 的通用
模板，`LCS`（最长公共子序列）是它的简化版——只有"相等就继承对角线 +1"和"不等取
`max(左, 上)`"两种情况，不需要"替换"这个分支，因为 LCS 不允许"改字符"，只允许
"跳过字符"。

## 深挖 2：戳气球——区间 DP 最反直觉的一步

**题目**：戳破气球 `i` 得到 `nums[left] * nums[i] * nums[right]` 个硬币
（`left/right` 是当前还没被戳破的相邻气球），求戳光所有气球的最大总硬币数。

### 为什么"先戳哪个"这条路走不通

直觉会想："贪心地每次戳系数最大的那个气球"，或者"按顺序 DP，`dp[i]` 表示戳到第
`i` 个气球时的最大值"。但戳破一个气球之后，它左右的气球会变成新的相邻关系，
后续每一步能拿到的硬币数都依赖"之前具体戳了哪些气球、以什么顺序戳的"——子问题之间
纠缠在一起，没法写出干净的递推。

### 反过来想："最后戳破哪个"

区间 DP 的核心技巧是换一个问的角度：**不问"第一步戳谁"，问"这个区间里最后一步戳谁"**。

先在数组首尾各补一个值为 1 的虚拟气球（`nums` 变成 `[1] + nums + [1]`），定义
`dp[i][j]` 为"戳破**开区间** `(i, j)` 内所有气球（不包括 `i`、`j` 本身）能拿到
的最大硬币数"——这里 `i`、`j` 代表"目前还健在、夹住这个区间的两个边界气球"。

枚举 `k`（`i < k < j`）作为这个区间里**最后一个**被戳破的气球。因为 `k` 是最后
戳的，戳它的那一刻，`(i, k)` 和 `(k, j)` 这两段里的气球早就已经全部戳完了——
也就是说戳 `k` 的时候，它左边紧挨着的就是 `i`、右边紧挨着的就是 `j`（中间的都
不在了）。于是戳 `k` 能拿到的硬币数就**固定**是 `nums[i] * nums[k] * nums[j]`，
和"`(i,k)`、`(k,j)` 这两段具体是按什么顺序戳的"完全无关——两段互相解耦，可以
分别递归求解再相加：

```
dp[i][j] = max_{i<k<j} ( dp[i][k] + dp[k][j] + nums[i]*nums[k]*nums[j] )
```

这就是"枚举最后一步"把互相纠缠的子问题拆成两个独立子问题的经典手法——区间 DP
里凡是"戳/删/合并会改变相邻关系"的题（戳气球、石子合并、扎气球类问题），基本都
是靠"反过来想最后一步/最后一次操作"来解耦。

`dp` 要按区间**长度从小到大**递推（先算所有短区间，再算长区间），因为
`dp[i][j]` 依赖的 `dp[i][k]`、`dp[k][j]` 都是更短的区间。

- **LC121 买卖股票的最佳时机**：只能交易一次，一次遍历维护"目前为止最低价"，
  当天卖出利润 = 当天价 - 目前最低价，取遍历过程中的最大值。O(n) / O(1)。
- **LC122 买卖股票的最佳时机 II**：可以多次交易。这题是状态机 DP 的贪心简化版
  ——完整写法是 `dp[i][0/1]` 表示第 i 天不持有/持有股票的最大利润，但因为交易
  次数不受限，可以证明"总利润 = 所有相邻上升区间差价之和"，直接贪心累加正差价
  即可，不需要真的维护状态机。
- **LC416 分割等和子集**：转化成"能否凑出 `sum/2`"的 01 背包问题。**01 背包
  遍历顺序为什么要倒序**：内层循环必须从 `target` 倒序到 `num`，因为如果正序
  遍历，更新 `dp[j]` 时可能已经用到了"这一轮刚更新过的 `dp[j-num]`"，相当于
  同一个物品被选了两次，退化成"完全背包"（物品可重复选）；倒序保证每次用的都是
  "上一轮、还没加入当前物品"的状态——这也是 01 背包和完全背包在代码上唯一的
  区别（完全背包因为允许重复选，遍历顺序反而要正序）。
- **LC1143 最长公共子序列**：双序列 DP 的基础版，详见"深挖 1"最后一段。

### 深挖：LC10 正则表达式匹配 vs LC44 通配符匹配 —— 长得像的两题，`*` 的语义完全不同

这两题几乎是"姐妹题"：状态定义一模一样（`dp[i][j]` 表示 s 前 i 个字符能否被 p 前 j 个
字符匹配），转移方程的骨架也长得很像，**但只要把两题的代码抄串了，几乎必错**——因为
两题里的 `*` 根本不是同一个东西。

**LC10 的 `*` 依附在前一个字符上**：它的语义是"让 `p[j-2]` 这个字符出现零次或多次"，`*`
本身从不单独出现，必须和它前面那个字符（可以是普通字符，也可以是 `.`）绑定解读。所以看到
`p[j-1]=='*'` 时，要往前看一格 `p[j-2]`：

```python
dp[i][j] = dp[i][j - 2]                                   # "字符*" 匹配 0 次，整体跳过
if p[j - 2] == '.' or p[j - 2] == s[i - 1]:
    dp[i][j] = dp[i][j] or dp[i - 1][j]                   # 该字符再匹配 s 的一个字符（重复一次）
```

**LC44 的 `*` 是独立的通配符**，语义是"匹配任意长度（含 0）的任意字符串"，完全不依赖它
前面是什么字符。所以看到 `p[j-1]=='*'` 时，不需要回头看 `p[j-2]`：

```python
dp[i][j] = dp[i - 1][j] or dp[i][j - 1]
# dp[i][j-1]：让这个 * 匹配 0 个字符，直接看 p 去掉 * 之后能不能匹配 s 现在的前缀
# dp[i-1][j]：让这个 * 多吃一个字符（不管吃的是什么），j 不退位因为 * 还能继续吃
```

**一句话对比**：LC10 的 `*` 关心"前一个字符再重复几次"，所以转移要看 `p[j-2]`、且"匹配
零次"要跳过**两个**字符（`j-2`）；LC44 的 `*` 只关心"要不要再吃一个字符"，转移只看
`p[j-1]` 自己，"匹配零次"只需要跳过**这一个** `*`（`j-1`）。写代码前先问自己一句"这里的
`*` 依赖前一个字符吗"，就能避免把两套转移方程搭配错。

`dp[0][j]`（s 是空串）这一行也因此不同：LC10 是 `dp[0][j] = dp[0][j-2]`（当
`p[j-1]=='*'`，"字符*"整体匹配空串）；LC44 是 `dp[0][j] = dp[0][j-1]`（当
`p[j-1]=='*'`，因为 `*` 自己就能匹配空串，不需要参照前一个字符）。

### 其余 11 题点睛

- **LC123 买卖股票的最佳时机 III**：状态机扩展成 `buy1/sell1/buy2/sell2` 四个滚动变量,
  必须按这个顺序更新（后面的状态要用到同一天刚算出的前面状态的新值），这是"允许同一天
  先卖再买"这一等价变形在多状态场景下的自然延伸。
- **LC188 买卖股票的最佳时机 IV**：把 LC123 手写的四个变量推广成两个长度 k+1 的数组
  `buy[j]/sell[j]`，内层 `j` 必须从 1 递增到 k（不能倒序），因为 `sell[j-1]` 需要读到
  当天已经更新过的新值——这个方向和 01 背包"必须倒序防重复选取"正好相反，容易搬错。
- **LC309 含冷冻期**：三个状态 `hold/sold/rest` 里，`sold` 专门表示"今天刚卖出、处于
  冷冻期"，转移必须基于"三个状态今天开始之前"的旧值同时计算，不能用刚更新的 `hold`
  去算 `sold`（那样等于"当天买当天卖"，不合法）。
- **LC714 含手续费**：状态机退回到 `hold/cash` 两态，手续费固定记在"卖出"这一步扣一次
  即可，不需要在买入、卖出两处各扣一次。
- **LC518 零钱兑换 II**：和 LC322 同样是完全背包，但这里求的是**组合数**而不是最少个数,
  **遍历顺序的含义因此完全不同**：必须外层遍历硬币、内层遍历金额，否则同一种组合会因为
  硬币的不同使用顺序被重复计数，把组合数错误地膨胀成排列数。
- **LC115 不同的子序列**：双序列计数 DP，`dp[i][j]` 永远要先继承"不用 `s[i-1]`"这个分支
  （`dp[i-1][j]`），字符相等时再**额外加上**"用它匹配"的方案（`dp[i-1][j-1]`）——是加法
  而不是取 max，这是计数型双序列 DP 和最优化型（如 LCS）最本质的区别。
- **LC97 交错字符串**：`dp[i][j]` 是"或"的关系（来自 s1 的最后一个字符，或来自 s2 的最后
  一个字符，只要有一种能让前缀合法即可），长度校验 `len(s1)+len(s2)==len(s3)` 要放最前面
  避免下标越界。
- **LC1035 不相交的线**：几何上的"连线不相交"翻译成代数语言，就是"下标同步递增地一一
  对应"——这恰好是最长公共子序列（LCS）的定义。**本题本质上就是 Part I 的 LC1143 换了
  一层"画线"的包装**，直接套用 LCS 的转移方程，不需要另起炉灶设计新状态。
- **LC516 最长回文子序列**：区间 DP，`dp[i][j]` 两端字符相等则 `dp[i+1][j-1]+2`，不等则
  舍弃一端取 `max(dp[i+1][j], dp[i][j-1])`；遍历顺序必须 i 倒序、j 正序，保证子区间先算好。
- **LC1039 多边形三角剖分的最低得分**：区间 DP，枚举和边 `(i,j)` 组成三角形的第三个顶点
  k，把区间拆成 `[i,k]` 和 `[k,j]` 两个独立子问题——和"戳气球"（Part I 深挖 2）同源的
  "枚举和某个固定元素一起参与最后一步的第三方"技巧，但这题不需要像戳气球那样补虚拟顶点。
- **LC646 最长数对链**：按每个数对的**结尾**排序后贪心扫描，结尾越小的数对越有利于后续
  接续——和 LIS 优化到 O(n log n) 时"维护最小结尾"是同一个核心直觉的不同应用场景。

### 深挖：LC546 移除盒子——为什么状态里要多加一维"能合并的同色块数量"

**先看为什么最朴素的区间 DP 状态在这题里彻底失效。** 区间 DP 的常规套路是定义
`dp[l][r]`="最优地清空区间 `[l,r]` 能拿到的最大分数"，对"戳气球""石子合并"这类题都够用。
但这题有一个前两讲任何区间 DP 题都没有的效应：**移除中间一段盒子之后，原本不相邻的两段
同色盒子会拼接起来，形成一个更长的连续同色段，未来可以一次性移除拿更高的分。** 换句话说,
"要不要先清空中间这一段、把两侧同色盒子拼在一起"本身就是一个需要被 DP 考虑进去的决策，
而 `dp[l][r]` 这个纯粹由"当前物理区间"决定的状态，根本没有地方安放"我打算把 `boxes[l]`
这个盒子留到以后，和区间外面（或区间内某个更远的位置）的同色盒子拼在一起"这条信息。

**解法是在状态里显式加一维 `k`：** `dp(l, r, k)` 表示"考虑区间 `[l,r]`，并且在 `boxes[l]`
左侧（或者按本讲代码的约定，在决策发生时和 `boxes[l]` 同色、将来注定要拼在一起移除的）
还挂着额外 `k` 个同色盒子"。这一维 `k` 本质上是把"隐式的、跨越了物理区间边界的等待关系"
显式记录了下来——**只要一个操作会"改变元素之间的相邻关系"（移除、合并、交换…），常规
`dp[l][r]` 往往就不够用，需要多加一维去记录"当前还悬而未决、将来才会生效的额外信息"**,
这正是这道题被反复当作"三维区间 DP 教学范例"的原因。

**转移的核心是两种决策的取舍：**

```python
# 决策 A：现在就把 boxes[l] 连同它挂着的 k 个同色搭子一起清空
best = (k + 1) ** 2 + dp(l + 1, r, 0)

# 决策 B：把 boxes[l..l] 这 k+1 个盒子"过继"给后面某个同色的 boxes[m]，
#        中间 [l+1, m-1] 这一段independent地先清空（不携带外层的 k）
for m in range(l + 1, r + 1):
    if boxes[m] == boxes[l]:
        best = max(best, dp(l + 1, m - 1, 0) + dp(m, r, k + 1))
```

决策 B 里 `dp(l+1, m-1, 0)` 的 `k` 必须重置为 0——`[l+1, m-1]` 是被"独立消耗掉"的一段，
它不该继承外层"`boxes[l]` 还挂着 k 个同色盒子"这个上下文，这是实现时最容易搞混、也最
容易被想当然地传错参数的地方。

**一句话总结这道题的通用启示**：区间 DP 遇到"移除/合并会改变相邻关系，从而让原本分离的
部分产生新的相互作用"这类题，第一反应应该是问自己"当前物理区间之外，是否存在需要被状态
记住的、悬而未决的额外信息"，而不是执着于优化 `dp[l][r]` 本身的转移方程——状态设计错了,
转移方程再怎么调整都不可能算对。

### 深挖：LC174 地下城游戏——为什么"正向贪心最大化血量"会失败

这道题另一个值得单独强调的陷阱是方向感：直觉上很容易想从左上角正向递推
`dp[i][j]`="走到 `(i,j)` 时剩余的最大生命值"，每步贪心选择让"当前剩余血量"最大的路径。
**但这个贪心是错的**——"当前血量最大"的路径未必是"全局所需初始血量最小"的路径，因为
后面的格子可能需要在某个特定的血量水位上才能兼顾"不死"和"总体最省初始血量"这两个目标，
局部贪心看不到这一层。正确做法是**反向 DP**：从终点往起点推，`dp[i][j]`="从 `(i,j)`
出发走到终点，至少需要多少初始血量才能全程存活"，转移
`dp[i][j] = max(1, min(dp[i+1][j], dp[i][j+1]) - dungeon[i][j])`，每一步都要用 `max(...,
1)` 兜底血量下限。**这是一个通用的教训：当"局部最优"和"全局目标"的方向不一致时（本题
里是"当前剩余血量"和"未来所需血量下限"这两个量根本不是同一个东西），换成反向定义状态,
往往能让转移方程重新变得干净。**

### 其余 8 题点睛

- **LC1691 堆叠长方体的最大高度**：允许旋转的关键预处理是把每个长方体自己的三条边先
  排序，转化成三维偏序下的 LIS——因为是三维偏序（不是一维严格序），朴素 O(n²) 已经是
  能做到的下界，不能像一维 LIS 那样二分优化到 O(n log n)。
- **LC1770 执行乘法运算的最大分数**：状态可以压缩成二维 `dp[i][left]`——"已经完成 i 次
  操作，其中 left 次取左"就能唯一确定取右的个数，不需要三维 `(left, right, i)`，这是
  "状态数量可以更少"这一直觉在双端选择类题目里的典型应用。
- **LC730 统计不同回文子序列**：`s[i]==s[j]` 时按内部区间是否存在字符 `s[i]`（`lo>hi` /
  `lo==hi` / `lo<hi` 三种情况分别对应 `+2` / `+1` / `-dp[lo+1][hi-1]`）精确去重，是区间
  DP 计数类题目里公认最容易记混系数的一道。
- **LC902 最大为 N 的数字组合**：数位 DP 入门标杆，`dp[i]` 从右往左推，"比当前位小"贡献
  `D^(剩余位数)`、"恰好相等"贡献 `dp[i+1]`（贴合下一位），别忘了加上"长度严格小于 n 位数"
  的方案数。
- **LC1074 元素和为目标值的子矩阵数量**：固定上下两行边界，把中间的列压缩成一维前缀和,
  退化成 LC560「和为 K 的子数组」的哈希计数，是二维前缀和问题最经典的降维技巧。
- **LC828 统计子串中的唯一字符**：不枚举子串，反过来算"每个字符在哪些子串里贡献 1 分"—
  —位置 `i` 的贡献是 `(i-prev)*(next-i)`（`prev`/`next` 是该字符左右最近的同字符位置）,
  这种"逐元素算贡献"的思路是很多"对所有子数组/子串求和"类问题通用的降维手段。
- **LC1000 合并石头的最低成本**：从"每次合并 2 堆"（戳气球式的二叉区间 DP）推广到"每次
  合并恰好 k 堆"，枚举分割点要跳步 `k-1`（保证子段长度满足 `(mid-i)%(k-1)==0` 天然能
  收尾成 1 堆），可行性判断 `(n-1)%(k-1)==0` 必须放在最前面提前拦截无解输入。
- **LC1547 切棍子的最小成本**：把 `0`、`n` 和所有切割点一起排序当作区间 DP 的"虚拟端点",
  和「戳气球」补虚拟气球、「多边形三角剖分」直接用原始顶点是同一类"给区间 DP 补边界"技巧
  的不同变体；转移里"这一刀的代价固定是当前整段长度"，与具体选哪个 `mid` 无关。
