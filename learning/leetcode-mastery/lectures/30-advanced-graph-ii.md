# L30 · 高级图论 II —— Tarjan(桥/割点/SCC) / LCA 倍增法 / 网络流

> 对应代码：`src/problems/p30_advanced_graph_ii.py`。Phase 3 竞赛级新分类。

## 22 类是"带权最短路"，这一类是"图的结构性质"

22 类「高级图论」处理的是"边有权重时怎么求最短路(Dijkstra/Bellman-Ford)"、
"怎么用最少代价连通所有点(MST)"、"怎么判断能否二染色"——这些问题关心的都是
"从 A 到 B 有多好走"。这一类往前推一步，处理三种更硬核的场景：

- **图本身的结构脆弱性**：删掉哪条边、哪个点会让图断开？这是"割点"和"桥"
  要回答的问题；一张图里"互相强连通"的节点天然聚成一簇，这是"强连通分量
  (SCC)"要回答的问题——三者都建立在同一套 Tarjan 式 DFS + low-link 技术上。
- **树上路径查询的加速**：暴力求两个节点的最近公共祖先(LCA)要各自往上爬到根，
  树退化成一条链时是 O(n)；"倍增法"用 O(n log n) 的预处理换来 O(log n) 的
  单次查询，是"空间换时间"的经典范式。
- **网络流**："一条边最多能同时传输多少流量"——这是一类全新的问题，用增广路
  (augmenting path)反复寻找"还能挤出更多流量的路径"来求解。

## 深挖：Tarjan 求桥 —— 从 low-link 的定义到"父边"的坑

**先建立直觉**：一条无向边 (u, v) 是"桥"，当且仅当删除它会让 u、v 无法互相
到达。什么时候会这样？当且仅当 (u, v) **不在任何环上**——如果它在一个环上，
删掉它之后，环的另一侧仍然能把 u、v 连起来；如果不在任何环上，它就是连接
两侧的唯一通道。

**用 DFS 树的语言重新表述**：对图做一次 DFS，会生成一棵 DFS 树，树边之外的
边（还没访问过就被再次碰到的边）叫"返祖边"。一条树边 (parent, child) 是桥，
当且仅当 child 的整棵子树里，**没有任何一条返祖边能跳出这棵子树、连回
parent 或者比 parent 更早访问的祖先**——一旦有这样一条返祖边，说明子树和
外部之间除了 (parent, child) 这条边，还有"另一条路"能绕回去，删掉它就
不会断开。

**low-link 就是在量化"最远能绕多远"**：给每个节点记录 `disc`（DFS 访问它的
时间戳）和 `low`（它自己 + 它子树里所有节点，通过任意一条返祖边能够到达的
**最早**时间戳）。`low[child]` 综合了整棵子树"能不能绕出去、绕多远"的信息；
判定条件就是 `low[child] > disc[parent]`：如果 child 子树里最远能绕到的
地方，都不比 parent 更早，说明真的绕不出去，(parent, child) 就是桥。

```python
for nxt, eidx in it:
    if eidx == parent_edge:
        continue
    if disc[nxt] == -1:
        disc[nxt] = low[nxt] = timer; timer += 1
        stack.append((nxt, eidx, iter(graph[nxt])))
        advanced = True
        break
    else:
        low[node] = min(low[node], disc[nxt])
```

**为什么要跳过的是"边的编号"而不是"父节点"**：无向图的边 (u, v) 在邻接表里
u 那边存一份、v 那边也存一份——DFS 从 parent 走到 child 时，child 的邻接表
里会包含 parent 本身。如果只用"nxt == parent 节点编号"来判断"这是刚走过来的
父边、要跳过"，一旦图里 parent、child 之间有**两条边**（重边），第二条边
明明是一条真正的返祖边（能把 child 重新连回 parent，说明这两条边构成一个
"二元环"，都不是桥），却会被误判成父边而被跳过，导致 `low[child]` 没有被
正确更新，从而把一条本不该是桥的边错误地判成桥。用边的编号 `eidx` 去重，
"跳过父边"就精确对应"跳过刚才走过来的那一条边"，不会误伤重边里的另一条。

割点(`find_articulation_points`)和强连通分量(`tarjan_scc`)是同一个 low-link
框架的变体：割点把桥的判定条件从 `low[child] > disc[u]` 放宽成 `>=`（因为
即使 child 子树能绕回 u 自己，u 仍然是必经点，只是 u-child 这条边不算桥了），
外加"DFS 树根节点至少有两个孩子才算割点"的特判；SCC 则是在有向图上用一个
显式栈，`low[u] == disc[u]` 时把栈里从栈顶到 u 的所有节点弹出，作为一个完整
的强连通分量。三者的实现都在 `p30_advanced_graph_ii.py` 里用手工构造的
"三角环 + 链"图（明确知道哪几条边是桥、哪几个点是割点）和两个独立的环
（明确知道有几个 SCC）做了交叉验证。

## 其余 7 题 + 2 个自实现算法点睛

- **LC1192 关键连接**：`find_bridges` 的直接应用——"删除后某两台服务器无法
  互达的连接"就是桥的定义，不需要额外的算法，只需要把桥找出来。
- **`LCABinaryLifting`（自实现）**：倍增法用"任意距离能分解成 2 的幂之和"这个
  性质，把"跳 k 步"拆成最多 O(log n) 次"跳 2^i 步"；预处理 `up[k][v] =
  up[k-1][up[k-1][v]]`（跳 2^k 步 = 跳两次 2^(k-1) 步）只需要 O(n log n)。
  查询 LCA 分两阶段：先把更深的节点跳到同一深度，再让两个节点一起、从大
  到小尝试跳、但只跳到"跳完两者仍不同"为止——贪心跳到相同会跳过 LCA 本身。
  用两棵不同形状的树（LC1483 官方样例的树 + 退化成一条链的树）对所有节点对
  做穷举式交叉验证，逐一比对暴力法（往上爬到根记录祖先集合）的结果。
- **LC1483 Kth Ancestor of a Tree Node**：`LCABinaryLifting` 的姊妹应用——
  不查询 LCA，而是直接查询"第 k 个祖先"，同样靠倍增表把 k 按二进制分解、
  逐位跳跃，中途跳到 -1（跳出树外）就可以提前终止。
- **`max_flow`（自实现，Edmonds-Karp）**：只要残量网络里还有一条从源点到
  汇点的路径，就沿着它增加"路径瓶颈容量"的流量；用 BFS 而不是任意 DFS 找
  增广路，保证每次找到边数最少的路径，这个限制让算法在多项式次增广内终止。
  每次增广要同时给反向边增加容量——这是"允许后悔、重新分配流量"的关键，
  没有反向边，某些图会因为"正向流量分配的顺序不对"而永远达不到真正的最大流。
  用 CLRS 教科书的经典例题（答案 23）、一个源点容量恰好是瓶颈的"菱形图"
  （答案 5）、以及一条简单链（答案就是链上最小容量 3）三个手工验证的网络
  交叉验证。
- **LC1568 使陆地分离的最少天数**：利用"答案只可能是 0、1、2"这个观察——
  0 天说明本来就不连通；否则暴力试删每个陆地格子看能不能 1 天拆开；都不行
  就是理论上界 2 天。数据规模小（<=30x30）使得这种暴力枚举可以接受。
> LC1489"找出最小生成树的关键边和伪关键边"（同样基于 Kruskal 变体判定）改归入
> [`17-union-find.md`](17-union-find.md)，这里不重复实现。

- **LC1615 网络最大分数**：`degree[i] + degree[j]`，两点间有直连道路就减 1
  去重，枚举所有城市对取最大值即可，不要求整体连通。
- **LC1345 跳跃游戏 IV**：BFS 建图时"用完即弃"——同值下标只在第一次被展开
  时全部连边，之后清空这个值分组，避免相同元素很多时边数退化到 O(n^2)。
- **LC1627 图连通性和阈值**：与其枚举城市对判断有没有公共因数(O(n^2))，
  不如枚举因数、把所有倍数用并查集合并到一起，总操作数是调和级数 O(n log n)。
- **LC2492 两个城市间路径的最小分数**：路径允许重复访问，本质上"能利用
  整个连通分量内的任意一条边"，答案就是包含起点和终点的连通分量内所有边的
  最小值，一次 DFS/BFS 遍历即可，不需要真的搜索"路径"。
