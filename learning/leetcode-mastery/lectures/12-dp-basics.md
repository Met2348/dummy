# L12 · DP 基础/序列

> 对应代码：`src/problems/p12_dp_basics.py`。动态规划是新手公认最难"想到"的分类,
> 但难的从来不是某个固定模板,而是每道题的 `dp[i]` 定义都不一样。本讲先给出一套
> 通用思考顺序,再挑两道题深挖"从暴力到优化"的完整推导过程。

## 通用思考顺序（比背模板更重要）

拿到一道新的 DP 题,按顺序回答这四个问题,而不是急着背"这题是不是背包/是不是
状态机"：

1. **`dp[i]` 表示什么？** 用一句完整的中文把它说清楚（"考虑前 i 个元素时,能
   XXX 的最优值"），含糊的定义会导致第 2 步的递推关系写不对。
2. **`dp[i]` 和更小子问题（`dp[i-1]`、`dp[i-2]`……或者某个 `dp[j]`, j<i）之间
   的递推关系是什么？** 通常来自"对最后一步做的决策"进行分类讨论（选/不选、
   偷/不偷、以谁结尾……）。
3. **初始条件是什么？** `dp[0]` 或 `dp[1]` 该是多少,这是整条递推链的"地基",
   地基错了后面全错。
4. **遍历顺序是什么？** 需要几层循环？内外层顺序能不能换？（本讲的零钱兑换和
   后续模块的分割等和子集,就是"遍历顺序决定对错"的经典反例。）

七道题按这四步整理如下：

| 题目 | `dp[i]` 定义 | 递推关系 |
|---|---|---|
| 爬楼梯 | 爬到第 i 级的方法数 | `dp[i]=dp[i-1]+dp[i-2]` |
| 打家劫舍 | 前 i 间房子能偷到的最大金额 | `dp[i]=max(dp[i-1], dp[i-2]+nums[i])` |
| 最大子数组和 | 以 `nums[i]` 结尾的最大和 | `dp[i]=max(nums[i], dp[i-1]+nums[i])` |
| 乘积最大子数组 | 以 `nums[i]` 结尾的最大/最小乘积 | 三候选取最大/最小 |
| 最长递增子序列 | （优化后）长度为 k+1 的子序列最小结尾 | 二分查找替换/追加 |
| 零钱兑换 | 凑出金额 i 的最少硬币数 | `dp[i]=min(dp[i], dp[i-c]+1)` |
| 单词拆分 | `s[:i]` 能否被完全拆分 | 存在分割点 j 使 `dp[j]` 真且 `s[j:i]` 在词典 |

## 深挖 1：最长递增子序列——从 O(n²) 到 O(n log n) 的关键一跳

**先看 O(n²) 的朴素做法**：`dp[i]` 定义为"以 `nums[i]` 结尾的最长递增子序列长度"。
枚举 `i` 之前所有 `j`，如果 `nums[j] < nums[i]`，说明 `nums[i]` 可以接在"以
`nums[j]` 结尾的序列"后面，`dp[i] = max(dp[i], dp[j]+1)`。这个做法完全正确，但
"每处理一个新数都要回头扫一遍之前所有更小的数"是明显的重复劳动——这正是可以被
优化掉的地方。

**优化的核心 insight**：换一个角度定义状态。不再问"以某个具体元素结尾的最长
序列有多长"，而是问：**"长度恰好为 k 的递增子序列，它的结尾数值最小能是多少？"**
维护一个数组 `tails`，`tails[k]` 就存这个"长度为 k+1 的子序列的最小结尾"。

为什么"最小结尾"这个量值得单独维护：假设有两个长度都是 3 的递增子序列，一个结尾
是 5，另一个结尾是 8。哪一个未来更容易被继续扩展？显然是结尾为 5 的那个——任何
能接在"结尾为 8"后面的数（比如 9），也一定能接在"结尾为 5"后面。所以结尾为 8 的
那个序列已经"没有存在的价值"了，只要记录最小结尾，就不需要保留其他所有长度为 3
的候选序列。

**这个 insight 直接把状态空间从"每个下标一个值"压缩成了"每个长度一个值"**，而
长度的范围是 `1..n`，恰好可以用数组下标表示，且 `tails` 数组本身天然有序（长度
越长，最小结尾一定不小于更短长度的最小结尾——如果一个长度为 5 的子序列的最小
结尾比长度为 3 的还小，那这个长度为 3 的子序列直接删掉最后两个元素就能得到一个
结尾更小的长度为 3 的候选，矛盾）。**`tails` 有序，就可以用二分查找**，而不需要
线性扫描，这才是真正把复杂度从 O(n²) 降到 O(n log n) 的地方。

具体对每个新数 `x`：

```python
i = bisect_left(tails, x)
if i == len(tails):
    tails.append(x)      # x 比现有所有结尾都大，能接在最长序列后面，长度 +1
else:
    tails[i] = x          # x 能让"长度为 i+1"的候选结尾变得更小，替换它
```

**必须强调一个反直觉的事实**：替换发生之后，`tails` 数组本身已经不对应任何一个
真实存在过的子序列了（它可能拼接了来自不同"候选链"在不同时刻的值）。但
`len(tails)` 在任意时刻都精确等于"处理到当前位置为止的最长递增子序列长度"——
这个"长度对、内容不一定对"的性质,正是这个技巧最容易被误用的地方（如果题目要求
"输出具体的 LIS 内容"而不只是长度，就不能直接读 `tails`，需要额外维护"每个位置
的前驱下标"来回溯还原）。

`[7,7,7,7]` 这个样例专门用来检验一个常见的实现错误：如果把 `bisect_left` 误写
成 `bisect_right`，会把"严格递增"错误地放宽成"非严格递增"（允许把相等的数也
接到序列后面），本该是 1 的答案会被错误地算成 4。用 `bisect_left`，遇到相等的
数会被放到"替换现有位置"分支而不是"追加"分支，天然保证了"严格"。

## 深挖 2：零钱兑换——完全背包 vs 01 背包，差在遍历顺序

`coin_change` 的递推关系写出来很直观：

```python
for i in range(1, amount + 1):
    for c in coins:
        if c <= i:
            dp[i] = min(dp[i], dp[i - c] + 1)
```

**这里外层遍历"金额"、内层遍历"硬币种类"，且金额是从小到大正序遍历**——这个
顺序选择不是随便的，而是由题目的一个关键条件决定的：**每种硬币可以使用无限次**
（这就是"完全背包"这个名字的由来：背包容量对应金额，物品对应硬币，每种物品/硬币
不限使用次数）。

对比一下如果题目换成"每种硬币只能用一次"（这就是"01 背包"，本 track 后续模块的
"分割等和子集"就是这个模式）：这种情况下,如果还是正序遍历金额,`dp[i-c]` 有可能
是"用了这一种硬币 c 计算出来的状态"，再把它加到 `dp[i]` 里，就等于**又用了一次
硬币 c**，变成了"这枚硬币被用了两次"，破坏了"每种硬币最多用一次"的约束。01 背包
的标准做法是把**金额维度倒序遍历**，这样计算 `dp[i]` 时，`dp[i-c]` 一定还是"不
包含这一次硬币 c"的旧状态,保证每种硬币在一次外层循环里最多被计入一次。

回到零钱兑换：**正是因为硬币允许重复使用，`dp[i-c]` 里"已经用过硬币 c"这件事完全
不是问题——本来就允许再用一次**。所以遍历顺序可以按最自然的正序来写，不需要
刻意倒序。**一句话总结这条区别：允许重复选 → 正序遍历（状态可以复用同一种物品）；
每种最多选一次 → 倒序遍历（避免同一种物品在同一轮里被"隐式"用了两次）。** 这条
规律在后续 DP 进阶模块的背包类问题里会反复出现，这里先埋下伏笔。

`dp[0]=0` 是递推链的地基（凑出金额 0 天然需要 0 枚硬币）。其余状态先设成一个
明确"不可能"的哨兵值 `amount+1`（因为最优解不可能用超过 `amount` 枚面值为 1 的
硬币），比用 `-1` 或 `0` 当哨兵更安全——`-1`/`0` 都可能和某个中间计算出的合法值
撞车,而 `amount+1` 永远不会是一个合法答案，最后判断 `dp[amount]` 是否还等于这个
哨兵值，就知道有没有解。

## 其余 5 题点睛

- **爬楼梯 `climb_stairs`**：`dp[i]=dp[i-1]+dp[i-2]`——本质是斐波那契数列，因为
  爬到第 i 级的最后一步要么是从 i-1 迈 1 级，要么是从 i-2 迈 2 级，两种情况互不
  重叠又覆盖全部可能。
- **打家劫舍 `rob`**：对每间房子只有"偷"和"不偷"两个选择，偷了当前房就不能偷
  相邻的前一间，`dp[i]=max(dp[i-1], dp[i-2]+nums[i])`。别用"隔一个偷一个"的
  贪心直觉，金额不均匀时会算错。
- **最大子数组和 `max_sub_array`**（Kadane 算法）：`dp[i]=max(nums[i], dp[i-1]+
  nums[i])`——如果前面累积的和是负数，不如从当前元素重新开始。答案是所有 `dp[i]`
  的最大值，不是 `dp[-1]`（最优子数组不一定在末尾结束）。
- **乘积最大子数组 `max_product`**：负负得正会让"最小值"翻转成新的"最大值"，
  所以要同时维护 `cur_max` 和 `cur_min` 两条链，每一步用同一组三个候选值
  `(x, cur_max*x, cur_min*x)` 同时算出新的最大值和最小值。
- **单词拆分 `word_break`**：`dp[i]` 表示 `s[:i]` 能否被完全拆分，枚举分割点 j，
  `dp[j]` 为真且 `s[j:i]` 在词典里则 `dp[i]` 为真。记得把词典转成 `set`，否则
  每次子串查找退化成线性扫描。

### 深挖：LC337 打家劫舍 III —— 为什么树形 DP 的递归要返回一个二元组

**先想明白递归到底要"带回"什么信息。** 在一维打家劫舍（LC198）里，`dp[i]` 只需要一个数：
"前 i 间房子能偷到的最大金额"，因为房子是一条线，`dp[i-1]`、`dp[i-2]` 已经把"偷或不偷第
i-1 间"这两种情况隐式地压缩、揉合进了同一个数字里——反正后面只会用到 `dp[i-1]` 这一个值,
不需要区分"第 i-1 间到底偷没偷"。

**但树形结构不一样：父节点的决策，依赖子节点"偷没偷"这个具体状态。** 如果父节点想偷自己,
两个孩子就**必须**处于"不偷"的状态（相邻限制）；如果父节点不偷，两个孩子可以自由选择偷或
不偷中更优的一种。如果递归只返回"这棵子树的最优解"这一个笼统的数字，父节点做决策时根本
无从得知："这个最优解里，孩子到底偷没偷？"——因而也就无法判断"如果我要偷自己，孩子那边
损失了多少"。

**所以树形 DP 的递归函数必须返回一个二元组** `(skip, take)`：`skip` 是"不偷当前节点时，
以它为根的子树最大金额"，`take` 是"偷当前节点时的最大金额"。这样父节点拿到孩子的
`(skip, take)` 之后，可以精确地做两件事：

```python
take = node.val + left_skip + right_skip          # 偷自己 -> 孩子必须都不偷
skip = max(left_skip, left_take) + max(right_skip, right_take)  # 不偷自己 -> 孩子各自挑更优的
```

这是树形 DP 的通用范式：**只要父节点的决策依赖子节点"处于哪种具体状态"，递归就必须把
"每种状态各自的最优值"都带回来，而不能只带回一个笼统的最优值**。后续如果遇到"树上最大
独立集""树上最小支配集"这类题，都是同一个范式的变形——先想清楚"父节点做决策时需要知道
子节点的哪些信息"，再决定递归返回值的形状。

### 其余 12 题点睛

- **LC213 打家劫舍 II**：环形约束的本质是"第一间和最后一间不能同时偷"，拆成"不偷最后一
  间"（对 `nums[:-1]` 跑线性打家劫舍）和"不偷第一间"（对 `nums[1:]` 跑线性打家劫舍）两个
  互斥子问题，取较大值。**这个"环形拆成两个线性子问题取 max"的技巧可以推广到很多环形 DP
  题**——只要环上的约束只涉及"首尾相邻"这一条边，就可以枚举"这条边不被使用"的两种方式,
  把环切开变成线性问题（本讲的 LC918 环形子数组最大和是另一种处理环形结构的思路：用
  "总和减最小子数组和"而不是"拆成两个线性子问题"，两者适用的场景不同，前者适用于"选择型"
  约束，后者适用于"求和型"目标，值得对比着记）。
- **LC91 解码方法**：`dp[i]` 依赖 `s[i-1]` 单独解码（1~9）和 `s[i-2:i]` 两位解码
  （10~26）两条转移路径的加和。前导 `'0'` 是最大陷阱："06" 不能解码（`0` 既不能单独成字母,
  也不能作为两位数的十位），必须让 `dp[1]` 在 `s[0]=='0'` 时直接为 0。
- **LC279 完全平方数**：把"完全平方数"当作面值不限次数的硬币，结构和零钱兑换（LC322）
  完全同构，外层金额正序、内层完全平方数种类，是完全背包的又一实例。
- **LC343 整数拆分**：`dp[i]` 枚举第一刀切在 j，比较"不再拆"（`j*(i-j)`）和"继续拆"
  （`j*dp[i-j]`）两个候选取较大值——继续拆不一定更优，dp 自动帮你判断该在哪一层停手。
- **LC62/LC63 不同路径 / 不同路径 II**：二维网格 dp 压缩成一维滚动数组，更新顺序必须
  从左到右（和 01 背包"必须倒序"的方向相反，因为这里恰恰需要用到"本行刚算出的新值"）。
  II 版本多了一条规则：障碍物格子直接把 dp 值清零，让"不可达"通过乘法/加法自然传播。
- **LC918 环形子数组的最大和**：答案是"普通最大子数组和"和"总和减最小子数组和"中的较大
  者，但要提防全负数组这个边界——这时最小子数组和会吞掉整个数组（`best_min == total`）,
  说明"跨接缝"的候选其实是空集，必须退化成只返回普通 Kadane 的结果。
- **LC673 最长递增子序列的个数**：在 LIS 朴素 O(n²) dp 的基础上加一条 `count[i]`，
  长度更长时**重置** `count[i]=count[j]`，长度相等时**累加** `count[i]+=count[j]`——
  一个重置一个累加，是这题最容易搞混的两条分支。
  最终答案要过滤出"长度等于全局最大长度"的位置求和，不能把所有长度的方案数都加起来。
- **LC1043 分隔数组以得到最大和**：`dp[i]` 枚举最后一段长度 1 到 k，一边扩展这一段一边
  同步更新这一段的最大值 `cur_max`，避免对每个候选长度重新扫描整段取最大值。
- **LC264 丑数 II**：三指针法，每一步的候选是 `ugly[i2]*2`、`ugly[i3]*3`、`ugly[i5]*5`
  取最小值当作新丑数，命中的指针（可能不止一个）各自独立前进一步——必须用三个独立的
  `if` 而不是 `if/elif`，否则像 6、12 这种同时是多个质因子候选的丑数会被重复生成。
- **LC1027 最长等差数列**：比 LIS 多一个维度，`dp[i][diff]` 按"结尾下标 + 公差"两个
  维度分桶（用字典存，因为公差取值稀疏），转移时 `dp[j].get(diff, 1)` 的默认值必须是
  1（"j 和 i 这两个数本身就是长度为 2 的等差数列"）而不是 0。
- **LC926 将字符串翻转到单调递增**：维护 `zero`（前缀变全 0 的最少翻转）和 `one`（前缀
  已经是合法单调递增串的最少翻转，注意 `one` 不要求全是 1，可能仍是全 0），最终答案取
  两者较小值。

### 深挖：LC887 鸡蛋掉落——从 O(k·n²) 到 O(k·n·log n) 再到 O(k·log n) 的两次跃迁

**先看最直白的暴力**：`dp[k][n]` 表示"k 个鸡蛋、n 层楼，最坏情况下最少试验次数"。枚举
第一次扔的楼层 `x`（1 到 n），扔了之后要么碎（转移到 `dp[k-1][x-1]`，往下找）要么没碎
（转移到 `dp[k][n-x]`，往上找），取最坏情况再加 1 次：

```python
dp[k][n] = 1 + min_x( max(dp[k-1][x-1], dp[k][n-x]) )   # x 从 1 到 n
```

这个方程完全正确，但对每个 `(k, n)` 状态都要枚举 O(n) 个 `x`，状态数本身是 O(k·n)，总
复杂度 **O(k·n²)**——n 到 10⁴ 时，n² 就是 1e8 级别，乘上 k 直接超时。

**第一次优化：注意到 `x` 增大时 `dp[k-1][x-1]` 单调不减、`dp[k][n-x]` 单调不增。** 两条
单调（一条递增一条递减）的曲线，要找它们"交叉点"附近使 `max` 最小的位置，天然适合用
**二分查找**替代线性枚举——对每个 `(k,n)`，二分找到 `dp[k-1][x-1]` 恰好追上
`dp[k][n-x]` 的临界 `x`，把内层的 O(n) 降到 O(log n)，总复杂度变成 **O(k·n·log n)**。
这是很多题解里"二分优化 DP"的标准动作：只要能证明两个候选序列关于枚举变量单调，就可以
用二分替代线性扫描。

**但真正的最优解需要换一个状态定义，靠"数学剪枝"整体绕开对 n 的依赖。** 关键的反问是：
与其问"k 个蛋、n 层楼最少要试几次"，不如反过来问 **"k 个蛋、给定 m 次试验预算，最多能
确定多少层楼？"**——这是原问题的对偶视角。定义 `f(eggs, moves)` 为这个"最大可确定楼层
数"，转移异常简洁：

```python
f(eggs, moves) = f(eggs - 1, moves - 1) + f(eggs, moves - 1) + 1
#                 ↑ 这一层碎了，剩 eggs-1 个蛋、moves-1 次机会能确定的楼层数（当作"下方"）
#                                  ↑ 没碎，剩 eggs 个蛋、moves-1 次机会能确定的楼层数（当作"上方"）
#                                                          ↑ 当前这一层本身
```

这个转移是 **O(1)** 的——不再需要枚举任何东西。只需要让 `moves` 从 1 开始递增，直到
`f(k, moves) >= n` 为止，这个 `moves` 就是答案。因为 `f` 关于 `moves` 是指数级增长的
（每多 1 次机会，能覆盖的楼层数大约翻倍），`moves` 最多到 O(log n) 就能覆盖任意大的
`n`，总复杂度压到 **O(k·log n)**——彻底摆脱了对 n 的多项式依赖，这才是这道 Hard 题"数学
剪枝"的真正含义：不是剪掉搜索树的分支，而是换一个和 n 无关的维度重新定义整个状态空间。

一句话总结这三层递进：**朴素枚举 O(k·n²) → 利用单调性二分优化到 O(k·n·log n) → 换成
"预算换楼层数"的对偶状态、彻底把 n 从状态维度里剔除，O(k·log n)**。三层优化分别对应
"发现子问题重叠该用 DP"、"发现单调性该用二分"、"发现状态定义可以整体翻转"这三种在竞赛
DP 里最常见、也最值钱的优化直觉，缺一不可。

### 其余 10 题点睛

- **LC1289 下降路径最小和 II**：朴素做法对每一行每个 j 都要排除"上一行同列"扫一遍最小值,
  是 O(n³)；关键优化是只维护上一行的最小值 `min1`（及其列下标）和次小值 `min2`——当前列
  撞上 `min1_idx` 时降级用 `min2`，否则放心用 `min1`，把 O(n) 的"排除法找最小值"降到
  O(1)，整体降到 O(n²)。
- **LC132 分割回文串 II**：131 题（回溯列出所有分割方案）的进阶版，只要最少切割次数。
  先用区间 DP 预处理 `is_pal[i][j]`，再用一维 dp 枚举切割点，`dp[i]=0`（当 `s[0:i+1]`
  整体就是回文）这个特判很容易漏掉，误统一写成 `dp[j-1]+1` 会在 `j=0` 时读到错误的负
  索引。
- **LC1553 吃掉 N 个橘子的最少天数**：n 最大 2×10⁹，线性 dp 数组完全不可行，必须靠
  "记忆化搜索 + 每步尽量往 n/2、n/3 跳"把子问题数量压到 O(log n) 级别，这是和上面
  "鸡蛋掉落"完全不同的另一种规避大状态空间的手段——不是换状态定义，而是发现"只有指数级
  少数几个子问题会被真正访问到"。
- **LC1420 生成数组**：三维状态 `dp[i][j][cost]`（长度、当前最大值、搜索代价），转移里
  "产生新最大值"要对所有严格更小的 `prev_j` 求和，暴力实现是 O(n·m²·k)，可以再叠一层
  前缀和优化到 O(n·m·k)——这是"数位 DP 风格"计数题的典型结构：逐位/逐个元素构造，按
  状态分桶计数。
- **LC940 不同的子序列 II**：维护 26 个字母各自"以它结尾的不同子序列数" `ends[c]`，每次
  用当前总数 `total+1` **覆盖**（而不是累加）`ends[ch]`——覆盖正是天然去重的关键，累加
  会让重复字符导致的重复子序列被多算。
- **LC1751 最多可以参加的会议数目 II**：按开始时间排序后，`dfs(i, remain)` 二选一（跳过
  /参加），参加时用 `bisect_right` 二分定位"下一个不冲突的会议"——这里的"二分"不是对
  答案二分，而是对"下一个可选区间"做二分定位，避免 O(n) 线性扫描。
- **LC1937 扣分后的最大得分**：把 `abs(c-c')` 按 `c'<=c` 和 `c'>=c` 拆成两条方向的前后
  缀最值 `left[c]=max(left[c-1]-1, dp[c])` / `right[c]`，把朴素 O(n²) 的"枚举上一行所有
  列"降到 O(n)。
- **LC1531 压缩字符串 II**：记忆化搜索状态 `(i, last_char, last_cnt, k_remain)`，保留
  字符时用**增量** `calc_len(cnt+1)-calc_len(cnt)` 而不是重新计算整个压缩长度，这是
  这题最容易被写复杂的地方。
- **LC1187 使数组严格增**：用字典 `dp[last值]=最少操作数`，每步要么保留原值（要求
  `num>last`），要么二分找 arr2 中"严格大于 last 的最小值"替换——"最小合法替换值"是
  贪心最优的选择，给后续留最大空间。
- **LC1140 石子游戏 II**：博弈 DP，`dfs(i, M)` 返回的是"总量减对手接下来最优能拿到的",
  而不是直接枚举 `X` 算己方贡献——这个"用总量减对方最优解"的间接算法是几乎所有二人零和
  博弈 DP 的通用套路。
