# L18 · 前缀和与差分

> 对应代码：`src/problems/p18_prefix_sum.py`

## 这一类到底在解决什么问题

前缀和的核心思想一句话概括:

> **把"多次查询区间和,每次都要重新把区间内元素加一遍"的重复计算,预处理成"花一次 O(n) 算出前缀和数组,之后每次查询只需要两个前缀和相减,O(1)"。**

典型信号:题目里出现"多次查询某个区间的和/乘积""求满足某个和条件的子数组个数"这类描述,且**同一批数据会被反复查询**时,先想"能不能预处理一张前缀信息表",而不是每次查询都重新遍历一遍区间。

差分数组是前缀和的**逆运算**,解决的是反过来的场景:

> **多次对一个区间内所有元素批量做加法,最后一次性还原出最终结果数组。**

如果直接在原数组上暴力地把区间内每个元素都加一遍,一次操作是 O(区间长度);而差分数组只需要在区间的两个端点各改一个数,O(1) 一次操作,所有操作做完后再对差分数组求一次前缀和,就能"一次性"把所有区间加法的效果还原出来。**前缀和是"以后要多次查询,先做预处理";差分是"以后要多次批量修改,最后再统一求和还原"——两者互为逆运算,一个面向查询、一个面向修改。**

## 深挖例题一:和为 K 的子数组(为什么"前缀和之差"能转化成哈希表计数)

**题目**:给数组 nums 和整数 k,求"和恰好等于 k"的连续子数组的个数(nums 可能含负数)。

**暴力思路**:枚举所有子数组的左右端点,对每个区间重新算一遍和——O(n²)甚至更糟。

**关键转化**:设 `prefix[i]` 表示 `nums[0..i-1]` 的和(`prefix[0]=0`)。子数组 `nums[j..i-1]` 的和,就等于 `prefix[i] - prefix[j]`——这是前缀和最基本的用法。现在题目要求这个区间和等于 k,也就是:

```
prefix[i] - prefix[j] = k   即   prefix[j] = prefix[i] - k
```

扫描到位置 `i` 时,问题变成:**"前面出现过多少次前缀和恰好等于 `prefix[i] - k`"**。这是全题最关键的一步转化——把"任意子数组的和是否等于 k"这个看似需要枚举区间的问题,转化成了"某个具体数值在前缀和历史记录里出现过几次"这个纯粹的计数问题,而"某个数值出现过几次"正是哈希表最擅长回答的问题,O(1) 查询。

**为什么是"哈希表记录前缀和出现的次数"而不是"记录前缀和出现的下标"**:因为题目只要"个数",不要求具体是哪个子数组,所以不需要记下标,只需要记"出现过几次"——这也是为什么用 `dict[前缀和 -> 次数]` 而不是 `dict[前缀和 -> 下标列表]`,更省空间也更直接。

**两个容易翻车的细节**:
1. 哈希表必须预置 `{0: 1}`——表示"前缀和为 0"在还没扫描任何元素之前就已经出现过一次,对应"从数组开头到当前位置"这一段恰好等于 k 的情况(这类子数组要求 `prefix[j]=0` 且 `j=0`,如果不预置,这次计数永远查不到)。
2. **必须先查表统计答案、再把当前前缀和计入表中**——顺序反了,`k=0` 时会把"空子数组"或者"自己减自己"的情况错误地多算一次。

这道题是前缀和技巧里最常被反复考的变体——"区间和等于定值"几乎都能靠"两个前缀和之差 + 哈希表计数"解决,值得牢牢记住这个转化步骤。

## 深挖例题二:航班预订统计(差分和前缀和的互逆关系)

**题目**:n 个航班编号 1 到 n。每条预订记录 `[first, last, seats]` 表示"航班 first 到 last(闭区间)上各预订了 seats 个座位",求每个航班最终的总预订座位数。

**暴力思路**:对每条记录,在结果数组的 `[first-1, last-1]` 区间里逐个位置加 seats。如果区间很长、记录很多,这是 O(区间长度 × 记录数),重复劳动体现在:**同一批位置,可能被很多条记录反复地逐格累加**。

**差分数组的思路**:既然前缀和能把"多次查询区间和"预处理成 O(1) 查询,那么它的逆运算——差分——就应该能把"多次区间批量修改"预处理成 O(1) 修改。具体做法是构造一个和结果数组等长(多开一位)的差分数组 `diff`:

- 对每条记录 `[first, last, seats]`,只需要 `diff[first-1] += seats`(告诉"从这个位置开始,后面的前缀和都要带上这 seats")、以及 `diff[last] -= seats`(告诉"从这个位置开始,不要再带上这 seats 了,抵消掉")。
- 每条记录都是 O(1) 的两次修改,和区间长度完全无关。

**为什么最后要对 diff 求一次前缀和才能还原**:因为 diff 数组本身只是"变化量的记录",`diff[first-1] += seats` 相当于说"这里有个+seats 的台阶",`diff[last] -= seats` 相当于说"这里有个-seats 的台阶"。真正的结果数组是这些台阶累积的效果——而"把一系列增量累积成实际值"正是前缀和在做的事(`result[i] = result[i-1] + diff[i]`)。这就是"差分数组是前缀和的逆运算"这句话的具体含义:**打标记的时候用差分(区间批量加,只改两个端点);还原结果的时候用前缀和(把所有标记累积成最终值)**。

**为什么不能直接在原数组上暴力加**:直接暴力加,一条记录的开销和区间长度成正比,记录一多、区间一长就退化成 O(n·m);差分把每条记录的开销压到 O(1),把"区间有多长"这个因素从单次操作的复杂度里剔除了,代价是要多做一次 O(n) 的前缀和累积去"结算"所有记录——但这个结算只需要做一次,不管有多少条记录。

## 点睛:区域和检索-数组不可变

`NumArray` 是前缀和最基础的应用——数组构造后不再变化,构造时一次性预处理出前缀和数组 `prefix`(`prefix[0]=0` 作哨兵),之后每次 `sum_range(left, right)` 就是 `prefix[right+1] - prefix[left]`,O(1)。这是"预处理换查询速度"最纯粹的例子,后面所有前缀和变体都是在这个基础模板上加条件。

## 点睛:除自身以外数组的乘积

`product_except_self` 是前缀和思想的"乘法版本",而且不能用除法。核心手法是拆成"左前缀积"和"右后缀积"两部分分别处理:第一趟从左到右扫,把每个位置的左前缀积先存进结果数组;第二趟从右到左扫,用一个变量累积右后缀积,乘进结果数组里。两趟都是线性扫描,O(n) 时间、O(1) 额外空间(不算输出数组)。这题提醒我们:前缀和不一定是"求和",只要是"能从左到右/从右到左递推累积的信息"(乘积、最大值、最小值等等)都能套用同样的双向扫描框架。

## 小结

1. 前缀和:预处理 O(n),之后每次区间查询 O(1)——适用于"同一批数据要被多次查询"的场景。
2. 差分:每次区间批量修改 O(1),最后统一做一次前缀和还原——适用于"要对同一批数据做多次批量修改,最后才需要看结果"的场景。两者互为逆运算。
3. "区间和等于定值"的题(如和为 K 的子数组),标准转化是"两个前缀和之差 + 哈希表计数",记住这一步转化能解一大类变体。
4. 前缀和不局限于加法,乘积、按位运算等只要满足"可以从一侧递推累积",都可以套用同样的双向扫描框架(除自身以外数组的乘积)。

### 深挖：LC974 和可被K整除的子数组 —— 为什么"前缀和模k相同"就意味着子数组和能被k整除

这题和 Part I 的"和为 K 的子数组"是同一个母题的取模变体，只是把判断条件从"两个前缀
和之差恰好等于 k"换成了"两个前缀和之差能被 k 整除"，背后的原理是一条同余定理：

设 `prefix[i]`、`prefix[j]`（i<j）是两个前缀和，子数组 `nums[i..j-1]` 的和就是
`prefix[j] - prefix[i]`。这个差要能被 k 整除，写成同余的语言就是：

```
prefix[j] - prefix[i] ≡ 0 (mod k)   即   prefix[j] ≡ prefix[i] (mod k)
```

也就是说，**两个前缀和只要模 k 之后余数相同，它们的差就必然是 k 的整数倍**——这是
"同余定理"最直接的应用：两个数同余，它们的差能被模数整除；反过来，只要一个数的两个
"快照"（这里是前缀和）除以 k 余数相同，中间这一段的"变化量"（子数组和）就一定是 k
的倍数。有了这条定理，问题就从"枚举子数组、逐个判断能否被 k 整除"，转化成了和
"和为 K 的子数组"完全一样的套路：**扫描到位置 j 时，统计前面出现过多少次"和当前前缀
和模 k 同余"的历史前缀和**，这正是一个纯粹的哈希表计数问题——用 `dict[余数 -> 出现
次数]` 边扫边记录，O(1) 查询。

和"和为 K 的子数组"对比着看会更清楚这层"同源"关系：那道题的判据是"两个前缀和之差
== k"，这道题的判据是"两个前缀和之差是 k 的倍数（包括 0）"——前者是"差恰好等于一个
定值"，后者是"差恰好等于某个定值的整数倍"，而"模 k 同余"正是判断"整数倍"关系最直接
的手段。两个容易翻车的细节也完全一致：哈希表要预置 `{0: 1}`（对应"从数组开头开始"
这一段前缀和本身就同余于 0 的情况）、必须先查表统计再计入当前余数（避免把空子数组
错误计入）。

### 其余 6 题点睛

**LC304 二维区域和检索-矩阵不可变**：一维前缀和（`NumArray`）向二维的直接推广，
递推和查询都用"容斥原理"（加两次、减一次重叠部分）实现 O(1) 查询，是"预处理换查询
速度"这个母题在更高维度上的自然延伸。

**LC528 按权重随机选择**：本质是前缀和 + 二分查找——累积和数组把权重转化成一系列
递增的分段边界，随机点落在哪一段用 `bisect_right` 在 O(log n) 内定位。测试时把
"生成随机数"和"根据随机数查表定位"拆成两个方法，只测试后者，是应对"含随机性的接口"
的通用测试手法。

**LC523 连续的子数组和**：和"和可被K整除的子数组"是同一个"前缀和取模"母题的姊妹
应用，区别只是这里只需要"存在性"而不是"计数"，所以哈希表记录的是"每种余数第一次
出现的下标"而不是"出现次数"，并多了一个"子数组长度至少为 2"的额外约束。

**LC1094 拼车**：和 Part I 的"航班预订统计"是同一个"差分数组处理区间批量加"母题，
差异只是这里的区间是左闭右开（上车计入、下车不计入），提醒我们同一个差分模板要
根据题目的区间开闭方式调整"减法标记"打在哪个位置。

**LC1893 检查是否区域内所有整数都被覆盖**：差分数组的另一个变体——不是"求和"而是
"计数覆盖次数"，对每个区间在起点 +1、终点后一位 -1，前缀和还原出"每个点被多少个
区间覆盖"，只需要检查目标区间内每个点是否都 ≥1 次覆盖。

**LC1732 找到最高海拔**：前缀和思想里最轻量的应用——只关心"历史最大值"而不需要
任意历史时刻的具体前缀和时，用一个滚动变量代替整个前缀和数组就够了，是本节里
"什么时候可以不用完整数组、只用滚动变量"这个判断力的最佳练习题。
