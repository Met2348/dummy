# L22 · 高级图论 —— 带权最短路 / 最小生成树 / 二分图判定

> 对应代码：`src/problems/p22_advanced_graph.py`。

## 这批题是 Phase 1 图论的自然延伸

Phase 1 的图论章节（11 · 图 BFS/DFS，17 · 并查集）处理的都是"无权图"或者"只关心
连通性"的问题。这一章把图论往前推一步，处理三类新场景：

- **边有权重**时，BFS 按层扩展（假设每条边权重都是 1）不再适用，要换成
  Dijkstra（边权非负）或 Bellman-Ford（能处理带次数限制、或存在负权边的场景）；
- **要用最少的边把所有点连通**，且要让连通代价最小时，要用最小生成树（Prim 或
  Kruskal）；
- **要判断"能否二染色"**（每条边两端颜色不同）时，要用染色法——这是"连通性"问题
  在"图的结构性质"上的延伸。

## 深挖：网络延迟时间（LC743，Medium）—— Dijkstra 和 BFS 的核心区别

**先回忆 BFS 为什么能求最短路**：在无权图（或者所有边权重都相等）里，BFS 按"层"
扩展，每一层到起点的距离恰好等于层数，第一次到达某个节点时所走的层数，就是从起点
到它的最短路径长度——因为 BFS 保证了"同一层内所有节点距起点的步数相同，不可能
存在一条更短、还没被发现的路径"。

**但边权不同时，这个假设不成立了**：如果边 A→B 权重是 1，边 A→C 权重是 100，
先出队的节点不代表它对应的距离更小——BFS 的"出队顺序 = 距离顺序"这个隐含假设，
只在所有边权重相等时才成立。

**Dijkstra 的核心 insight**：既然出队顺序不能直接代表距离顺序，那就显式地把"距离"
作为排序依据——维护一个"已经确定最短距离"的节点集合，每一步不是简单地按 FIFO
顺序取节点，而是**贪心地从集合外的所有候选节点里，选出当前已知距离最小的那一个**
确定下来。用最小堆维护候选节点（堆顶永远是当前候选里距离最小的），这样每次确定
下来的节点，它的距离一定不会再被后续发现的路径更新（因为堆里剩下的所有候选距离
都不会比它更小，任何经过它们的路径只会让距离更大）。

```python
def network_delay_time(times: list[list[int]], n: int, k: int) -> int:
    graph: dict[int, list[tuple[int, int]]] = defaultdict(list)
    for u, v, w in times:
        graph[u].append((v, w))

    dist: dict[int, int] = {k: 0}
    heap: list[tuple[int, int]] = [(0, k)]
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist.get(u, float("inf")):
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist.get(v, float("inf")):
                dist[v] = nd
                heapq.heappush(heap, (nd, v))

    if len(dist) < n:
        return -1
    return max(dist.values())
```

**为什么堆顶弹出的距离可能是"过期"的**：同一个节点可能因为发现了更短的路径，被
以不同的距离值多次压入堆——这是因为堆不支持"修改已有元素的优先级"，只能
再塞一个更小的候选进去。所以出堆时要检查 `d > dist.get(u, inf)`：如果堆顶的距离
比当前记录的最短距离还大，说明这是一条早就被更新过的过期记录，直接跳过，不会
影响正确性（因为真正的最短距离已经被记录在 `dist` 里了）。

**最终答案是所有最短距离的最大值**，不是把它们相加，也不是走过的边数——因为
"信号传到所有节点"这件事，必须等到*最慢*到达的那个节点收到信号才算完成；如果
`dist` 里记录的节点数少于 `n`，说明有节点从 k 不可达，返回 -1。

## 其余 6 题点睛

- **LC787 K 站中转内最便宜的航班**：不能直接用 Dijkstra——本题引入了"最多经停 k
  站"的次数限制，Dijkstra 的贪心正确性依赖"节点一旦被确定最短距离就不会再变"，
  但这里"更便宜的路线"可能因为经停次数超限而不合法，贪心假设失效。改用
  Bellman-Ford 的"分层松弛"：只做 k+1 轮松弛（对应最多飞 k+1 段航班），每轮必须
  基于上一轮的快照更新，不能原地更新，否则会突破次数限制。
- **LC1584 连接所有点的最小费用**：最小生成树问题，用 Prim（而不是 Kruskal）
  是因为这里任意两点都有一条边（曼哈顿距离），图是稠密图——Prim 每步只需要
  维护"树到每个未入树点的最短距离"，天然利用稠密结构，不需要像 Kruskal 那样先把
  O(n²) 条边全部排序。
- **LC785 判断二分图**：染色法——任选一个未染色节点染色，BFS/DFS 扩展时邻居必须
  染相反的颜色，一旦发现邻居已经染了和自己相同的颜色就说明有矛盾。图可能不连通，
  必须对每个连通分量单独触发一次染色过程，漏掉任何一个分量都可能让矛盾被掩盖。
- **LC1319 连通网络的操作次数**：并查集——n 个节点连通至少需要 n-1 条边，边数
  不够直接返回 -1；否则扫一遍已有连接，让"两个不同分量合并"的边才是有用的边，
  扫完之后"剩余连通分量数 - 1"就是还需要多少次操作才能把所有分量并成一个。
- **LC802 找到最终的安全状态**：反图 + 拓扑排序（Kahn 算法的变体）——出度为 0
  的节点（终端）天然安全，一个节点如果它指向的所有邻居都安全，它自己也安全。
  这是"从终端向外传播安全性"的过程，用反图上的拓扑排序实现：初始把原图里出度为 0
  的节点入队，每次确定一个安全节点就去反图里找"谁指向它"、把这些节点的（原图）
  出度减一，减到 0 就说明它指向的所有节点都已确认安全，自己也随之安全。也可以用
  三色 DFS（白/灰/黑）等价实现。
- **LC1976 到达目的地的方案数**：Dijkstra 求最短路的同时，额外维护 `ways[v]`
  统计"按最短距离到达 v 的方案数"——发现更短路径时覆盖重置 `ways[v]`，发现同样
  短的路径时把方案数累加，"更短覆盖、一样短累加"是本题和普通 Dijkstra 唯一的
  区别，结果记得对 `10**9+7` 取模。
