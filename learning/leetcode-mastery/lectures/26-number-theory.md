# L26 · 数论与数学进阶 —— 从"会用位运算"到"真正理解数论算法"

> 对应代码：`src/problems/p26_number_theory.py`

## 这一类到底在解决什么问题

这是仓库里**第一次系统教数论算法本身**。19 类"位运算与数学"讲的是异或消消乐、
`n & (n-1)`、快速幂这些基础的位技巧和数学工具，但从没真正涉及数论的核心内容——
19 类里的快速幂只是"把 O(n) 次乘法压缩到 O(log n) 次"这个通用倍增思想的一个应用，
没有涉及模运算本身的代数结构。这一类要补的正是这一块：

- **模运算的逆运算**：整数模 m 之后，"除以 a"这个操作还有没有意义？答案是——只有
  当 a 和 m 互质时，才存在一个"模逆元"能让"乘以它"等价于"除以 a"。这是扩展欧几里得
  算法真正的用武之地。
- **多个同余方程的合并**：如果同时知道"x 模 3 余 2""x 模 5 余 3""x 模 7 余 2"，
  能不能一次性合并出 x 模 105 的值？中国剩余定理（CRT）给出构造性的答案。
- **指数级问题的对数级求解**：不仅整数乘法可以用快速幂从 O(n) 压缩到 O(log n)，
  任何满足结合律的运算都可以——矩阵乘法就是最典型的例子，这让**线性递推**（不只是
  斐波那契）也能达到 O(log n)。
- **同余类与循环节**：当问题的状态空间是"模 k 的余数"这种有限集合时，鸽笼原理保证
  了状态一定会循环，循环节本身往往就是问题的答案或者解题的关键突破口。

这一类的 14 道题/练习里，8 道是真实 LeetCode 题（直接对应上面每条主线的实战场景），
6 道是不依赖具体题号的自实现数论算法组件（`ext_gcd`、`mod_inverse`、`crt`、
`pow_mod`、`fib_matrix_pow`、`n_choose_r_mod`）——这些组件是可以直接在其他题目里
复用的"数论工具箱"，也是后面遇到任何数论相关的难题时，第一时间应该想到的基础积木。

## 深挖一：扩展欧几里得算法 + 模逆元 —— 为什么除法在模运算下必须转换成乘法

**先想清楚"除法"在模运算下为什么会失效**。在普通的实数世界里，"除以 a"等价于
"乘以 a 的倒数 `1/a`"——这个倒数永远存在（只要 a 不是 0）。但在"模 m 的整数"这个
世界里，压根没有"分数"的概念：`1/a` 这个记号本身不是一个合法的整数。如果我们直接
在模运算的中间步骤里写 `x / a mod m`，这个表达式在数学上是没有定义的——这不是
"实现起来麻烦"的问题，而是"这个操作根本不存在"的问题。

**模逆元就是"除法"在模运算下的替代品**。我们退一步想：能不能找到一个整数
`a_inv`，使得"乘以 `a_inv`"这个操作，效果和"除以 a"完全一样？也就是要求
`a * a_inv ≡ 1 (mod m)`——只要这样的 `a_inv` 存在，以后凡是遇到"除以 a"，都可以
换成"乘以 `a_inv`"，问题就转化成了模运算天然支持的乘法。这样的 `a_inv` 被称为
**a 模 m 的逆元**。

**扩展欧几里得算法是构造模逆元的钥匙**。普通欧几里得算法（辗转相除法）只回答
"gcd(a, m) 是多少"，把"这个 gcd 究竟是怎么由 a、m 线性组合出来的"这条信息丢弃了。
扩展欧几里得算法在递归回溯的过程中，把这条信息找回来，给出满足
**贝祖等式** `a*x + m*y = gcd(a, m)` 的一组整数系数 `(x, y)`。递推的核心是：

```
ext_gcd(a, 0) = (a, 1, 0)                       # 基准情形：gcd=a，且 a*1+0*0=a
ext_gcd(a, b) = 设 ext_gcd(b, a%b) = (g, x1, y1)
              则 a*y1 + b*(x1 - (a//b)*y1) = g   # 把子问题的系数反推回当前层
              所以 x = y1, y = x1 - (a//b)*y1
```

这个反推公式不是凭空出现的：因为 `a % b = a - (a//b)*b`，把子问题的等式
`b*x1 + (a%b)*y1 = g` 代入这个关系式展开整理，两边按 a、b 重新归并系数，就直接
得到了当前层的 `x, y`——每往上回溯一层，就完成一次"系数换算"。

**贝祖等式在 `gcd(a,m)=1` 时，恰好就是模逆元的定义式**。当 a、m 互质时，贝祖等式
退化成 `a*x + m*y = 1`。两边同时对 m 取模，`m*y` 这一项恒等于 0（它本身就是 m 的
倍数），于是等式变成 `a*x ≡ 1 (mod m)`——这正是"x 是 a 模 m 的逆元"的定义！也就是
说，模逆元根本不需要额外的算法，它是扩展欧几里得算法在"a、m 互质"这个特殊场景下
的直接副产品：调用 `ext_gcd(a, m)` 拿到 `(g, x, y)`，如果 `g == 1`，`x % m`
（取模是为了把可能为负的 x 归一化到 `[0, m)`）就是所求的模逆元；如果 `g != 1`，
说明 a 和 m 不互质，模逆元根本不存在——这不是算法的局限，而是数学事实本身
（比如 `mod_inverse(2, 4)` 无解，因为不存在整数 x 使得 `2x ≡ 1 (mod 4)`：
`2x` 恒为偶数，不可能模 4 余 1）。

**这条"除法转乘法"的主线贯穿了本章节后面的两道题**：中国剩余定理构造解的公式里，
每一项都要用到某个数模某个模数的逆元；组合数取模 `n_choose_r_mod` 计算
`C(n,r) = n!/(r!*(n-r)!)` 时，分母的阶乘同样不能直接"除"，必须先求出阶乘的模
逆元再改写成乘法（这里因为 mod 是质数，还可以用费马小定理 `a^(mod-2) mod mod`
这条更直接的路径求逆元，本质上和扩展欧几里得算法殊途同归，都是在解决"模运算下
如何做除法"这同一个问题）。

## 深挖二：矩阵快速幂求斐波那契 —— 线性递推如何写成矩阵乘法，从 O(n) 降到 O(log n)

**朴素递推的瓶颈在哪**。斐波那契数列 `F(n) = F(n-1) + F(n-2)` 用一个循环从
`F(0), F(1)` 递推到 `F(n)`，是 O(n) 时间——如果 n 达到 10^18 这种级别（在一些
竞赛题或者需要对超大 n 求值的场景中很常见），O(n) 的循环完全跑不完。

**关键洞察：线性递推可以写成矩阵乘法**。把"相邻两项"打包成一个列向量
`[F(n+1), F(n)]^T`，斐波那契的递推关系 `F(n+1) = F(n) + F(n-1)`（以及平凡的
`F(n) = F(n)`）可以整体写成一次矩阵左乘：

```
[F(n+1)]   [1 1] [F(n)  ]
[F(n)  ] = [1 0] [F(n-1)]
```

也就是说，"往前推进一项"这个原本用加法表达的操作，等价于"用常数矩阵
`M = [[1,1],[1,0]]` 左乘当前的状态向量"。既然每推进一项都是乘一次 M，推进 n 项
就是连乘 n 次 M：

```
[F(n+1)]        [F(1)]
[F(n)  ] = M^n * [F(0)]  =  M^n * [1, 0]^T
```

这意味着**斐波那契数列的第 n 项，就藏在矩阵 `M^n` 里**（具体是 `M^n` 的
`[0][1]` 分量）——原来"求数列第 n 项"这个问题，被转化成了"给一个 2x2 矩阵求
n 次幂"这个问题。

**矩阵乘法满足结合律，这是快速幂能够搬过来的充分条件**。整数乘法的快速幂能用
"指数减半、底数平方"这个技巧，本质上依赖的唯一性质是乘法满足结合律
（`(x*x)*x*x = x*(x*x)*x` 这类重新括号不影响结果）——这样才能把 `x^n`
拆成 `(x^(n//2))^2` 而不改变结果。矩阵乘法同样满足结合律（只是不满足交换律，
但快速幂用到的重新括号并不需要交换律），所以完全可以照搬同一套折半框架：

```
M^n = (M^(n//2))^2          当 n 是偶数
M^n = M * (M^(n//2))^2      当 n 是奇数
```

于是"求 `M^n`"也只需要 O(log n) 次矩阵乘法（而不是 O(n) 次），每次矩阵乘法本身
是 O(1)（2x2 矩阵，固定 8 次数乘 4 次加），总时间 O(log n)——这就是矩阵快速幂
比朴素递推快指数级的原因。代码实现上，`fib_matrix_pow` 复用了和整数快速幂
一模一样的迭代结构（`_mat_pow` 里的 `while power > 0` 循环，逻辑和 19 类的
`my_pow`、本章节的 `pow_mod` 几乎逐行对应，只是把"数字乘法"换成了"矩阵乘法"），
这正说明了"折半"是一个**和具体运算无关的通用算法框架**，能不能用它，只取决于
"这个运算是否满足结合律"这一个条件。

**这个技巧的泛化边界远不止斐波那契**：任何能写成
"下一个状态向量 = 常数矩阵 × 当前状态向量"形式的**固定阶数线性递推**——比如
`F(n) = 2*F(n-1) + 3*F(n-2) - F(n-3)` 这种三阶递推，只需要把状态向量扩成三维、
常数矩阵扩成 3x3——都可以用同样的矩阵快速幂从 O(n) 降到 O(log n)。斐波那契只是
这一整类问题里最简单、最常见的代表。

## 其余 12 题点睛

**GCD 基础与逆向欧几里得算法**

- **LC1979 数组的最大公约数**（Easy）：整个章节的热身题——很多人会下意识地以为
  要对数组所有元素求 GCD，但题目只要求最小值和最大值这两个数的 GCD。手写一遍
  欧几里得算法（而不是直接调 `math.gcd`），是为了给扩展欧几里得算法的结构做铺垫：
  两者除了"要不要多维护一组系数"之外，递推结构完全一样。
- **LC780 到达终点**（Hard）：正向从起点出发会指数级分叉，必须**逆向**从终点
  往回推——`(tx, ty)` 的上一步是唯一确定的（`tx>ty` 时一定来自 `(tx-ty,ty)`，
  反之亦然）。用 `tx %= ty` 代替"一次减一个 ty"，正是欧几里得算法"用取模一次性
  跳过很多步减法"这个思想的直接搬用，把复杂度从"坐标数值线性"压到对数级。

**素数筛与循环节**

- **LC204 计数质数**（Medium）：埃拉托斯特尼筛法——不对每个数单独做素性检测，
  而是反过来，找到一个素数就批量标记它的所有倍数为合数。外层循环只需要跑到
  `sqrt(n)`（更大的合数一定能被更小的素因子筛掉），内层标记从 `i*i` 开始
  （更小的倍数已经被更小的素数筛过），这两个优化把复杂度从暴力试除的
  O(n*sqrt(n)) 降到 O(n log log n)。
- **LC1015 可被 K 整除的最小整数**（Medium）：全 1 数可能有几十万位，不能真的
  构造出来。技巧是只维护"当前拼出来的数模 k 的余数"，每次拼一个新的 1 对应
  `r = (r*10+1) % k`。余数只有 k 种可能，鸽笼原理保证：k 步之内出现 0 就有解，
  否则永远进入一个不含 0 的循环节、无解。这是"同余类只有有限种、必然循环"这个
  数论思想的典型应用，和 CRT、模逆元一样，都是把"可能无限增长的大数"投影到
  "模某个数的等价类"里处理。

**快速幂家族——整数快速幂的三种变体应用**

- **LC372 超级次方**（Medium）：指数本身是几千位的数组，不能拼成完整数值。
  关键等式 `a^(10p+q) = (a^p)^10 * a^q` 让我们能从左到右逐位消化这个指数，
  每处理一位只需要两次"小指数"（10 和当前数字）的模幂运算，全程不需要知道
  真实的指数数值是多少。
- **LC1922 统计好数字**（Medium）：先用乘法原理把问题转化成
  `5^(偶数位个数) * 4^(奇数位个数)`，这一步组合计数的洞察做完之后，剩下的纯粹
  是"指数可能大到 10^15，必须用快速幂"——和 LC372 的共同点是都依赖 `pow_mod`，
  区别在于这题的难点在"列出计数公式"，LC372 的难点在"指数本身无法直接表示"。
- **custom `pow_mod`**：本章节所有快速幂相关的题目共用的底层组件，实现结构和
  19 类的 `my_pow` 完全一致，唯一区别是每一步都对 mod 取模，防止中间结果的位数
  随着重复平方而无限膨胀。

**双指针+数学 与 因子分解**

- **LC633 平方数之和**（Medium）：`a` 从 0 递增、`b` 从 `sqrt(c)` 递减，
  双指针相向而行——`s<c` 说明太小只能增大 a，`s>c` 对称减小 b，两个方向不会
  同时错——把枚举范围从 O(c) 压缩到 O(sqrt(c))，先用数学（b 的上界）划定
  边界，再用双指针的单调性收窄。
- **LC829 连续整数求和**（Hard）：把"从 a 开始连续 i 个数之和等于 n"写成等差
  数列求和公式 `n = i*a + i*(i-1)/2`，反解出 `a = (n - i*(i-1)/2) / i`——
  对每个 i 只需要检查这个 a 是否是正整数，枚举范围由不等式 `i*(i-1)/2 < n`
  自然限定在 O(sqrt(n))。这题把"代数变形"和"整除性枚举"这两个数论子技能结合
  在一起，是本章节里综合性最强的一道，压轴收尾。

**custom：CRT 与组合数取模**

- **custom `crt`**：中国剩余定理——给每个方程 `x ≡ r_i (mod m_i)` 构造一个
  "在第 i 个方程里贡献 1、其余方程里贡献 0"的基向量
  `M_i * mod_inverse(M_i, m_i)`（`M_i` 是除 `m_i` 外其余模数的连乘积），按余数
  加权求和就同时满足所有方程——这是线性代数"正交基表示任意向量"的思想在同余
  方程组上的对应物，前提是所有模数两两互质。
- **custom `n_choose_r_mod`**：组合数 `C(n,r) = n!/(r!*(n-r)!)` 取模时不能直接
  做除法，必须把分母的阶乘换成它们的模逆元。预处理阶乘数组之后，只对最大的
  `n!` 做一次费马小定理意义下的快速幂求逆元，其余阶乘的逆元用
  `inv_fact[i-1] = inv_fact[i] * i % mod` 反向递推 O(1) 得到——这是"除法转乘法"
  这条主线在组合数学场景下最实用的落地。

## 小结

1. 模运算不是"普通整数运算加一个取模"这么简单——它是一套独立的代数结构，除法
   在这套结构里默认不存在，必须通过扩展欧几里得算法求模逆元，把"除"转换成"乘"。
2. 中国剩余定理把"合并多个同余方程"这件事，转化成"给每个方程构造一个正交基
   向量再加权求和"，前提是模数两两互质，构造过程本身还是要用到模逆元。
3. 快速幂的"指数减半、底数平方"不是整数乘法的专属技巧——只要一种运算满足
   结合律（矩阵乘法是最典型的例子），就能套用同一套折半框架，把任意固定阶数
   的线性递推从 O(n) 压到 O(log n)。
4. 素数筛、同余类循环节、双指针+数学上界、等差数列反解整除性枚举，是这一类里
   "看信号就能对上技巧"的几个固定套路，和 19 类"位运算与数学"里"看信号选技巧"
   的方法论一脉相承，只是这一次面对的是真正的数论结构而不是位运算。
