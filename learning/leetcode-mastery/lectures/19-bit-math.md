# L19 · 位运算与数学

> 对应代码：`src/problems/p19_bit_math.py`

## 这一类到底在解决什么问题

这一类不像前面的分类那样有一个统一的"母题结构",它更像是**几个固定技巧的合集**,遇到符合特征的题目就套对应的技巧:

**位运算题的通用套路:**
- **异或消消乐**:利用"成对出现的数字异或为 0"这一性质,把"找不成对的那个数"从 O(n) 额外空间的哈希计数,压缩到 O(1) 空间。
- **`n & (n-1)` 消去最低位的 1**:常用来数二进制里 1 的个数,或者判断一个数是不是 2 的幂(是 2 的幂当且仅当只有一个 1 位,即 `n & (n-1) == 0`)。
- **位掩码代替集合**:用一个整数的每一位表示一个布尔状态,常见于状态压缩 DP,这里不展开(留给后面的 DP 专题)。

**数学题的通用套路:**
- 想清楚"能不能把 O(n) 的暴力循环换成 O(log n) 的倍增或者一条封闭公式"。快速幂就是典型代表——把"乘 n 次"变成"乘 O(log n) 次"。

## 深挖例题一:只出现一次的数字(为什么异或的三条性质组合起来正好解出这题)

**题目**:数组里除了一个数字只出现一次,其余每个数字都恰好出现两次,找出只出现一次的那个,要求 O(n) 时间、O(1) 额外空间。

**暴力思路**:哈希表计数每个数字出现的次数,挑出次数为 1 的那个——能做对,但需要 O(n) 额外空间,不满足题目"不使用额外空间"的要求。这个"不能用额外空间"的约束,其实是在提示我们应该用不需要存储的运算来代替计数——异或正是这样一种运算。

**异或的三条关键性质,缺一不可:**
1. **`a ^ a = 0`**——任何数和自己异或得 0。这条性质是"消消乐"的来源:成对出现的数字,只要被异或到一起,就会互相抵消变成 0。
2. **`a ^ 0 = a`**——任何数和 0 异或还是它自己。这条性质保证了"抵消完剩下的东西"不会被 0 污染,最终结果恰好就是那个落单的数字。
3. **交换律 + 结合律**——异或的顺序不影响最终结果,`a ^ b ^ c = c ^ a ^ b`。这条性质是关键中的关键:它意味着**我们不需要精心安排数组的扫描顺序去"凑出"相邻的相同数字**,不管数组里相同的数字隔多远、顺序多乱,只要把所有数字依次异或到一起,相同的数字总能在某种意义上"抵消掉",不受它们在数组中位置的影响。

**三条性质组合起来的效果**:把数组所有元素依次异或,由性质 3,可以任意重排这个异或的顺序,相当于把相同的数字"挪到一起";由性质 1,挪到一起的相同数字两两抵消为 0;由性质 2,抵消完剩下的 0 异或上落单的数字,还是那个数字本身。全程只用一个累加变量从 0 开始异或,天然是 O(1) 额外空间、O(n) 时间——异或的这三条性质,正好一一对应了题目"两两相同、找落单者、不能用额外空间"这三个约束点。

**这个技巧的适用边界**:它依赖"恰好出现两次"这个前提。如果题目换成"其余数字出现三次",单纯异或就不再够用了(三次异或不会抵消为 0),需要换成"按位统计每一位上 1 的个数,对 3 取模"的变体技巧——但核心思想是一致的:利用某种运算,让"重复出现的数字"以某种方式自动抵消或归零。

## 深挖例题二:Pow(x, n)(快速幂的倍增思想,和二分查找是同一类母题)

**题目**:实现 `x` 的 `n` 次幂(n 可正可负)。

**暴力思路**:循环把 x 累乘 n 次,O(n) 次乘法。当 n 很大时(比如接近 2³¹)会直接超时。

**核心洞察——指数减半、底数平方**:
```
x^n = (x^(n//2))^2         当 n 是偶数
x^n = x * (x^(n//2))^2     当 n 是奇数(多出来的一次单独乘)
```
也就是说,要算 `x^n`,不需要真的乘 n 次,只需要先算出 `x^(n//2)`,再把它平方一下就行——而 `x^(n//2)` 又可以用同样的方式,从 `x^(n//4)` 平方得到……每递归一层,指数就减半,总共只需要 O(log n) 层。

**这和二分查找是同一类"折半"母题的不同应用**:二分查找的"折半"是**折半地缩小"答案可能存在的搜索范围"**——每比较一次中点,就能排除一半的候选区间;快速幂的"折半"是**折半地缩小"还需要乘多少次"**——每平方一次底数,就能把"还要乘的次数"减少一半。两者的共同本质是:**面对一个"重复对半"的结构,不需要线性地处理每一个单位,而是通过"倍增"操作,让每一步都能处理掉一半的工作量,从而用 O(log n) 步完成原本需要 O(n) 步的工作**。这类"减半"思想后面在树的高度分析、分治算法里还会反复出现。

**实现要点(迭代版,不需要额外栈空间)**:把指数 `n` 看成二进制展开,从低位到高位处理:维护一个不断自我平方的 `base`(初始是 x),每次看 `n` 的最低位是不是 1——是 1 就把当前的 `base` 乘进结果里,然后无论如何都让 `base` 自我平方、`n` 右移一位,直到 `n` 变成 0。`n` 为负数时,先令 `x = 1/x`、`n = -n`,转化成正数次幂的情形,复用同一套逻辑,只是要小心不要漏掉"连底数也要取倒数"这一步(只把指数变正、不管底数,会漏算倒数)。

## 点睛:位1的个数

`hamming_weight` 用 `n & (n-1)` 消去最低位的 1:`n-1` 会把 n 的最低位 1 变成 0、同时把它右边原本的 0 全变成 1;`n` 和 `n-1` 做与运算,恰好把 n 最低位的那个 1 抵消掉,其余位不变。每执行一次 `n &= n-1`,1 的个数就减少一个,循环到 n 变成 0 为止,循环次数就是答案——这个技巧只跳过 1,不逐位检查 0,当 1 很稀疏时比"移位 32 次逐位检查"更快。这个技巧还有一个常见姊妹应用:判断一个数是不是 2 的幂,只需要判断 `n & (n-1) == 0`(2 的幂只有一个 1 位)。

## 点睛:多数元素

`majority_element` 用 Boyer-Moore 投票算法,把"多数元素记 +1 票、其余元素记 -1 票,总票数一定是正的"这个洞察,转化成"维护一个候选人 + 一个计数器,遇到相同的加票、不同的减票、票数减到 0 就换候选人"的线性扫描。它依赖题目"多数元素一定存在"这个前提,不做验证——如果没有这个保证,还需要再扫一遍数组验证候选人的真实票数。

## 点睛:整数反转

`reverse_integer` 用"不断取最低位、拼到结果末尾"的方式反转数字(`rev = rev*10 + digit`),这一步的思想和前缀和"边扫边累积"是同一种"逐位/逐元素累积"的模式,只是作用对象是十进制数位而不是数组元素。这题的额外要求是**反转后要检查是否超出 32 位有符号整数范围**,超出返回 0——这一步必须在算出完整的反转结果之后再检查(Python 整数不会像其他语言那样自动截断,必须显式比较范围)。

## 小结

1. 位运算题看几个信号:"两两配对、找落单者"→异或;"数 1 的个数/判断 2 的幂"→`n & (n-1)`;"用整数状态表示集合"→位掩码。
2. 数学题的核心思路是"能不能把 O(n) 的重复操作,用倍增/折半的方式压缩到 O(log n)"——快速幂是最典型的例子,和二分查找共享"折半"这个母题。
3. 异或能解决"只出现一次的数字",依赖"恰好出现两次"这个前提,前提变了(比如出现三次)技巧要跟着换。
4. Boyer-Moore 投票算法依赖"多数元素一定存在"的题目保证,是一种不需要额外空间的计数技巧。
