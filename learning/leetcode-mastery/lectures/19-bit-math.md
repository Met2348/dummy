# L19 · 位运算与数学

> 对应代码：`src/problems/p19_bit_math.py`

## 这一类到底在解决什么问题

这一类不像前面的分类那样有一个统一的"母题结构",它更像是**几个固定技巧的合集**,遇到符合特征的题目就套对应的技巧:

**位运算题的通用套路:**
- **异或消消乐**:利用"成对出现的数字异或为 0"这一性质,把"找不成对的那个数"从 O(n) 额外空间的哈希计数,压缩到 O(1) 空间。
- **`n & (n-1)` 消去最低位的 1**:常用来数二进制里 1 的个数,或者判断一个数是不是 2 的幂(是 2 的幂当且仅当只有一个 1 位,即 `n & (n-1) == 0`)。
- **位掩码代替集合**:用一个整数的每一位表示一个布尔状态,常见于状态压缩 DP,这里不展开(留给后面的 DP 专题)。

**数学题的通用套路:**
- 想清楚"能不能把 O(n) 的暴力循环换成 O(log n) 的倍增或者一条封闭公式"。快速幂就是典型代表——把"乘 n 次"变成"乘 O(log n) 次"。

## 深挖例题一:只出现一次的数字(为什么异或的三条性质组合起来正好解出这题)

**题目**:数组里除了一个数字只出现一次,其余每个数字都恰好出现两次,找出只出现一次的那个,要求 O(n) 时间、O(1) 额外空间。

**暴力思路**:哈希表计数每个数字出现的次数,挑出次数为 1 的那个——能做对,但需要 O(n) 额外空间,不满足题目"不使用额外空间"的要求。这个"不能用额外空间"的约束,其实是在提示我们应该用不需要存储的运算来代替计数——异或正是这样一种运算。

**异或的三条关键性质,缺一不可:**
1. **`a ^ a = 0`**——任何数和自己异或得 0。这条性质是"消消乐"的来源:成对出现的数字,只要被异或到一起,就会互相抵消变成 0。
2. **`a ^ 0 = a`**——任何数和 0 异或还是它自己。这条性质保证了"抵消完剩下的东西"不会被 0 污染,最终结果恰好就是那个落单的数字。
3. **交换律 + 结合律**——异或的顺序不影响最终结果,`a ^ b ^ c = c ^ a ^ b`。这条性质是关键中的关键:它意味着**我们不需要精心安排数组的扫描顺序去"凑出"相邻的相同数字**,不管数组里相同的数字隔多远、顺序多乱,只要把所有数字依次异或到一起,相同的数字总能在某种意义上"抵消掉",不受它们在数组中位置的影响。

**三条性质组合起来的效果**:把数组所有元素依次异或,由性质 3,可以任意重排这个异或的顺序,相当于把相同的数字"挪到一起";由性质 1,挪到一起的相同数字两两抵消为 0;由性质 2,抵消完剩下的 0 异或上落单的数字,还是那个数字本身。全程只用一个累加变量从 0 开始异或,天然是 O(1) 额外空间、O(n) 时间——异或的这三条性质,正好一一对应了题目"两两相同、找落单者、不能用额外空间"这三个约束点。

**这个技巧的适用边界**:它依赖"恰好出现两次"这个前提。如果题目换成"其余数字出现三次",单纯异或就不再够用了(三次异或不会抵消为 0),需要换成"按位统计每一位上 1 的个数,对 3 取模"的变体技巧——但核心思想是一致的:利用某种运算,让"重复出现的数字"以某种方式自动抵消或归零。

## 深挖例题二:Pow(x, n)(快速幂的倍增思想,和二分查找是同一类母题)

**题目**:实现 `x` 的 `n` 次幂(n 可正可负)。

**暴力思路**:循环把 x 累乘 n 次,O(n) 次乘法。当 n 很大时(比如接近 2³¹)会直接超时。

**核心洞察——指数减半、底数平方**:
```
x^n = (x^(n//2))^2         当 n 是偶数
x^n = x * (x^(n//2))^2     当 n 是奇数(多出来的一次单独乘)
```
也就是说,要算 `x^n`,不需要真的乘 n 次,只需要先算出 `x^(n//2)`,再把它平方一下就行——而 `x^(n//2)` 又可以用同样的方式,从 `x^(n//4)` 平方得到……每递归一层,指数就减半,总共只需要 O(log n) 层。

**这和二分查找是同一类"折半"母题的不同应用**:二分查找的"折半"是**折半地缩小"答案可能存在的搜索范围"**——每比较一次中点,就能排除一半的候选区间;快速幂的"折半"是**折半地缩小"还需要乘多少次"**——每平方一次底数,就能把"还要乘的次数"减少一半。两者的共同本质是:**面对一个"重复对半"的结构,不需要线性地处理每一个单位,而是通过"倍增"操作,让每一步都能处理掉一半的工作量,从而用 O(log n) 步完成原本需要 O(n) 步的工作**。这类"减半"思想后面在树的高度分析、分治算法里还会反复出现。

**实现要点(迭代版,不需要额外栈空间)**:把指数 `n` 看成二进制展开,从低位到高位处理:维护一个不断自我平方的 `base`(初始是 x),每次看 `n` 的最低位是不是 1——是 1 就把当前的 `base` 乘进结果里,然后无论如何都让 `base` 自我平方、`n` 右移一位,直到 `n` 变成 0。`n` 为负数时,先令 `x = 1/x`、`n = -n`,转化成正数次幂的情形,复用同一套逻辑,只是要小心不要漏掉"连底数也要取倒数"这一步(只把指数变正、不管底数,会漏算倒数)。

## 点睛:位1的个数

`hamming_weight` 用 `n & (n-1)` 消去最低位的 1:`n-1` 会把 n 的最低位 1 变成 0、同时把它右边原本的 0 全变成 1;`n` 和 `n-1` 做与运算,恰好把 n 最低位的那个 1 抵消掉,其余位不变。每执行一次 `n &= n-1`,1 的个数就减少一个,循环到 n 变成 0 为止,循环次数就是答案——这个技巧只跳过 1,不逐位检查 0,当 1 很稀疏时比"移位 32 次逐位检查"更快。这个技巧还有一个常见姊妹应用:判断一个数是不是 2 的幂,只需要判断 `n & (n-1) == 0`(2 的幂只有一个 1 位)。

## 点睛:多数元素

`majority_element` 用 Boyer-Moore 投票算法,把"多数元素记 +1 票、其余元素记 -1 票,总票数一定是正的"这个洞察,转化成"维护一个候选人 + 一个计数器,遇到相同的加票、不同的减票、票数减到 0 就换候选人"的线性扫描。它依赖题目"多数元素一定存在"这个前提,不做验证——如果没有这个保证,还需要再扫一遍数组验证候选人的真实票数。

## 点睛:整数反转

`reverse_integer` 用"不断取最低位、拼到结果末尾"的方式反转数字(`rev = rev*10 + digit`),这一步的思想和前缀和"边扫边累积"是同一种"逐位/逐元素累积"的模式,只是作用对象是十进制数位而不是数组元素。这题的额外要求是**反转后要检查是否超出 32 位有符号整数范围**,超出返回 0——这一步必须在算出完整的反转结果之后再检查(Python 整数不会像其他语言那样自动截断,必须显式比较范围)。

## 小结

1. 位运算题看几个信号:"两两配对、找落单者"→异或;"数 1 的个数/判断 2 的幂"→`n & (n-1)`;"用整数状态表示集合"→位掩码。
2. 数学题的核心思路是"能不能把 O(n) 的重复操作,用倍增/折半的方式压缩到 O(log n)"——快速幂是最典型的例子,和二分查找共享"折半"这个母题。
3. 异或能解决"只出现一次的数字",依赖"恰好出现两次"这个前提,前提变了(比如出现三次)技巧要跟着换。
4. Boyer-Moore 投票算法依赖"多数元素一定存在"的题目保证,是一种不需要额外空间的计数技巧。

### 深挖：LC260 只出现一次的数字III —— "分组降维"如何把 2 个特殊数问题降级成 2 个独立的 1 个特殊数问题

Part I 的"只出现一次的数字"能直接处理"恰好一个数落单、其余成对出现"的情形——整体
异或一遍，成对的数字两两抵消，剩下的就是答案。但这道题有**两个**数字各只出现一次，
如果照搬"整体异或"的做法，得到的只是这两个落单数字的异或值，这两个数字本身仍然
"混在一起"，异或不出单独的答案。

**关键转折点**：整体异或的结果 `xor_all` 虽然不是任何一个答案，但它藏着一条决定性
的信息——**这两个落单的数字，一定在 `xor_all` 为 1 的某一位上互不相同**（因为
`xor_all` 是这两个数字异或的结果，如果它们在所有位上都相同，`xor_all` 就会是 0，
矛盾于"它们是两个不同的数"）。任取 `xor_all` 里一个为 1 的位（代码里用
`xor_all & (-xor_all)` 取最低位的 1），就获得了一个能够**区分**这两个特殊数字的
判据。

用这个判据把整个数组划分成两组——"这一位是 1"和"这一位是 0"——会发生两件事：

1. 两个特殊数字必然被分进**不同**的组（因为它们在这一位上恰好相反）；
2. 原本成对出现的数字，同一对里两个值相同，在这一位上也必然相同，所以**同一对
   永远被分进同一组**，不会被拆散。

这两条性质合起来意味着：**每一组内部，都重新变回了"只有一个数字落单、其余数字都
成对出现"的结构**——也就是说，一个"2 个特殊数混在一起"的复杂问题，被"分组"这一步
操作精确地降级成了两个互相独立的、Part I 已经会解的"1 个特殊数"子问题，只需要对
每一组分别做一次简单异或就能各自求解。

这就是位运算题里"分组降维"的通用范式：**当直接对全体数据做某种运算无法把问题化简
到底时，先从"全体信息的汇总"（这里是整体异或）里提炼出一个能区分不同角色的判据，
再用这个判据把原问题拆分成若干个更小、且已经知道怎么解的独立子问题**。这个思路不
局限于位运算——很多"看似需要同时处理 k 个特殊对象"的题目，都值得先问一句"能不能
找到一个一次性的判据，把 k 个对象的问题拆成 k 个 1 个对象的子问题"。

`n & (n-1)` 这个技巧最常见的两个应用（数 1 的个数 / 判断 2 的幂）里更简单的那一个,
一次位运算就能判断，不需要循环。

### 深挖：LC782 变为棋盘——如何把矩阵变换拆解成两个独立的一维排序问题

**第一步转折——交换行/列不改变"内容"，只改变"顺序"**：这是全题最重要的观察。
既然操作只是"交换整行"或"交换整列"，那么原始 board 里每一行、每一列的**具体
内容**是不可能被改变的，能改变的只是它们的**相对位置**。这意味着：如果原始
board 本身的结构就不满足"能通过重排变成棋盘"，无论怎么交换都无济于事——必须
先做一次结构校验。

**第二步——结构校验：秩 1 关系**：合法棋盘只有两种行内容（一种模式和它的按位
取反）。这意味着对任意的 `(i, j)`，`board[i][j]` 必须能由 `board[i][0]`、
`board[0][j]`、`board[0][0]` 三者唯一确定——即
`board[i][j] xor board[i][0] xor board[0][j] xor board[0][0]` 恒为 0。这一步
是纯位运算：用异或表达"这四个值构成的关系是否自洽"。

**第三步——计数校验：1 的比例**：光结构自洽还不够——每一行、每一列在最终棋盘里
必须是严格的 01 交替，这要求"第一行 1 的个数"（`row_ones`）和"第一列 1 的个数"
（`col_ones`）都必须落在 `[n//2, (n+1)//2]` 这个区间内。这一步是纯计数：如果
1 的比例已经失衡（比如全部行都以 1 开头，第一列全 1），不管结构关系多自洽，也
不可能重排出棋盘——第二个例子 `board=[[1,0],[1,0]]` 恰好就是"结构关系自洽（两行
本来就相同、异或关系恒成立），但计数校验失败（`col_ones=2` 超出了 `n=2` 时的
合法区间 `[1,1]`）"这种情况，返回 -1。

**第四步——两个校验通过之后，问题才真正"降维"**：一旦确认可以重排成棋盘，"把行
排成正确顺序"和"把列排成正确顺序"就变成了两个**完全独立**的子问题，可以分别求
最少交换次数再相加——这是本题最关键的降维：一个 n×n 的矩阵变换问题，被拆解成了
两个长度为 n 的一维序列排序问题。

**第五步——一维子问题：01 序列排成交替序列的最少交换次数**：以"行"这个子问题为例，
只需要看"第一列这一竖列的 01 序列"（它决定了每一行是"原始模式"还是"取反模式"）。
把这个序列变成严格交替的 `0101...` 或 `1010...`，统计"偶数位置是 1 但目标要 0"
的个数和"奇数位置是 0 但目标要 1"的个数——每次交换恰好能同时修正一个这样的"1
错位"和一个"0 错位"，所以最少交换次数就是这两个计数中的一个（可行时两者相等）。
n 为偶数时两种目标模式都可能可行，各自算一遍取较小值；n 为奇数时两种模式各自
要求的 1 的总数不同，只有和实际计数相符的那一种才可行。

**给这道题一个整体的记忆框架**：位运算负责"结构自洽"这一步的校验（异或关系），
组合计数负责"比例是否失衡"这一步的校验，而最终决定答案数值的，是把二维问题拆
成两个一维"01 交替排序"子问题之后的贪心配对——三种技巧（位运算校验、计数校验、
贪心配对）缺一不可，这正是这道题作为"位运算与数学"分类里最难题目的原因：它不
考验单一技巧的熟练度，而是考验"能不能把一个复杂问题正确地拆解成几个已知技巧
分别能解决的子问题"这个更高层次的能力。

**LC393 UTF-8编码验证**：纯粹的"按状态机逐字节校验格式"位运算模拟题——用一个
"还需要读几个续字节"的计数器驱动状态转移，不涉及任何取巧的数学性质，提醒我们
这个分类里也有单纯"老实做"、把格式规则原样翻译成代码的题目。

**LC1178 猜字谜**：位掩码 + 子集枚举的经典组合——把字符串压缩成 26 位掩码后，
借助 `submask = (submask - 1) & rest_mask` 这个子集枚举技巧，把每个谜题的枚举量
从"可能的字母组合数"精确压缩到固定的 `2^6=64`，是本节"用位掩码代替暴力枚举"这
个技巧里最有代表性的一道题。

**LC1611 使数字变为0的最少操作次数**：题目给出的两条操作规则表面上需要复杂的
分类讨论递归，但本质等价于"格雷码的逆变换"——`ans ^= n; n >>= 1` 循环到底，是
本节"发现题目描述的操作规则本质是一个已知的编码变换"这类"降维打击"解法的典型
代表，和 LC782 一样，都属于"真正的突破口不是位运算技巧本身，而是识别出问题背后
更深一层的数学结构"这一类难题。
