# L29 · 计算几何 —— 叉积 / 线段相交 / 凸包 / 矩形与圆的碰撞

> 对应代码：`src/problems/p29_computational_geometry.py`。

## 计算几何题在面试里为什么"少见但致命"

计算几何题在面试里少见，但一旦出现通常是"你没见过就不可能现场想出来"的类型——
不像动态规划或图论那样，题目结构本身会给你不少提示，计算几何的正确性建立在
一套并不显然的几何直觉上（比如"用叉积判断转向"这件事，如果之前没见过，现场
很难独立推导出来）。好消息是：这一类问题的核心只有几个基础几何原语——**叉积
判断三点转向**、**线段相交判断**——一旦掌握，**复杂问题（凸包、矩形碰撞、圆与
矩形重叠）几乎都是这些原语的组合**，而不是需要分别硬记的独立算法。

## 深挖：凸包构造（Andrew 单调链算法）—— 和 LC84 单调栈的同构关系

**问题**：给定平面上一组点，找出包围所有点的最小凸多边形的顶点（凸包）。

**核心想法**：凸包有一个关键性质——如果把凸包顶点按 x 坐标（相同 x 按 y 坐标）
排序，那么"下边界"（下凸包）和"上边界"（上凸包）可以分别独立构造，最后拼在一起
就是完整凸包。构造下凸包的过程是：

```python
lower: list[Point] = []
for p in pts:                                    # pts 已按 (x, y) 排序
    while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
        lower.pop()
    lower.append(p)
```

**这一步在做什么**：每次准备把新点 `p` 接到当前下凸包末尾之前，先看"栈"（这里
用列表模拟）里最后两个点 `lower[-2]`、`lower[-1]` 和即将加入的 `p` 构成的转向——
用叉积 `cross(lower[-2], lower[-1], p)` 判断：如果这个值 `<= 0`，说明从
`lower[-2] -> lower[-1]` 转到 `lower[-2] -> p` 是顺时针（或共线），也就是说
`lower[-1]` 这个点让路径"往内凹"了，不满足凸包应该"一直左转"的性质，必须把它
**弹出**；弹出之后继续检查新的栈顶，直到不再凹陷（或者栈里点数不够比较）为止，
再把 `p` 压入栈。上凸包用完全相同的逻辑对**逆序**遍历的点再做一遍。

**和 LC84（柱状图中最大的矩形）的相通之处**：LC84 的单调栈维护的是"高度单调
递增"的性质——一旦发现新柱子比栈顶矮，就说明栈顶那根柱子的"势力范围"已经确定，
必须弹出并结算；这里的单调链维护的是"凸性"——一旦发现新点让路径变凹，就说明
栈顶那个点不可能是凸包顶点，必须弹出。**两者都是"栈顶不满足某种单调/凸性质就
弹出，向后面的元素让路"这同一个算法模式**，只是判断弹出的条件从"数值大小
比较"换成了"叉积符号判断"。这也是为什么很多人把 Andrew 单调链算法称为"用叉积
做判据的单调栈"。

**一个常见的语义分支**：`<= 0` 弹出意味着"共线的中间点也会被弹出"——这是标准
凸包的语义（只保留真正的转折顶点）。但如果题目要求"保留边界上的所有点"（比如
LC587 Erect the Fence，要求围栅栏的绳子经过边界上的每一棵树，哪怕这棵树只是
恰好落在两个顶点连线中间），只需要把弹出条件从 `<= 0` 改成 `< 0`（只有严格
顺时针才弹出，共线的点保留），复用完全相同的框架就能得到不同的语义——这正是
本文件 `convex_hull` 和 `erect_the_fence` 两个函数唯一的区别。

## 其余项点睛

- **叉积 `cross(o, a, b)`**：`(a-o) x (b-o)`，符号判断"从 o→a 转到 o→b"是
  逆时针（正）、顺时针（负）还是共线（零）。这是所有其他函数的地基。
- **线段相交 `segments_intersect`**：一般位置下，两条线段相交当且仅当各自的
  两个端点分别落在对方所在直线的两侧（四个叉积两两异号）；叉积为零时说明出现
  共线，需要额外用坐标范围检查（`on_segment`）是否真的有重叠，不能笼统地把
  "共线"当作"相交"。
- **812 Largest Triangle Area**：数据规模小（点数 <= 50），直接三重循环枚举
  所有三点组合，用 `|cross(o,a,b)| / 2` 算三角形面积（叉积是平行四边形有向
  面积，三角形是一半），取最大值。
- **939 Minimum Area Rectangle（轴对齐）**：矩形可以看成"两条竖直线段"——把
  点按 x 分组，如果两个不同的 x 下都出现了同一对 `(y1, y2)`，就构成一个矩形，
  面积是 `|Δx| * |y2-y1|`；记录"每对 (y1,y2) 最近一次出现的 x"，逐列扫描更新
  最小面积。
- **963 Minimum Area Rectangle II（可旋转）**：利用"矩形对角线互相平分且长度
  相等"这一充要性质——枚举所有点对，把"中点 + 对角线长度平方"作为分组 key
  （用平方避免开根号的浮点误差），同一组内任取两条对角线即可还原出一个矩形，
  面积用相邻两边长度相乘算出。
- **1401 Circle and Rectangle Overlapping**：把圆心坐标"钳制"（clamp）到矩形
  的 x、y 范围内，得到的点就是矩形上离圆心最近的点；这个点到圆心的距离如果
  不超过半径，两者就重叠——一次钳制 + 一次距离比较，O(1) 解决。
- **587 Erect the Fence**：凸包构造的变体，弹出条件从 `<= 0` 改成 `< 0`，
  让边界上的共线点也保留在结果里（见上文深挖部分）。
