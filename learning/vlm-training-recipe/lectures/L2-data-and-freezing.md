# L2 · 数据与冻结策略: VLM 训得好不好的关键细节

> 30-min lecture · 目标: 搞清 VLM 两阶段各用什么数据、冻结策略怎么定。这些细节决定 VLM 的成败, 也是你 9.4 实验设计要消融的地方。

---

## 0. 配方对了, 细节决定成败

L1 给了两阶段的总路线。但「用什么数据」「冻谁」这些细节, 才真正决定 VLM 训得好不好。本讲讲这两个最关键的细节。

---

## 1. VLM 的三类数据

| 数据类型 | 长什么样 | 用在 | 作用 |
|---|---|---|---|
| **图文对 (image-text pairs)** | 一张图 + 一句 caption | 阶段 1 对齐 | 教 LLM 读视觉, 海量便宜 (网络抓) |
| **交错文档 (interleaved)** | 图和文交替的文档 (如网页) | 阶段 1/预训练 | 教多图 + 上下文中的图文关系 |
| **视觉指令数据 (instruction)** | 图 + 问题 + 答案 | 阶段 2 微调 | 教按指令处理图, 珍贵 (常需构造) |

几个要点:
- **图文对**最便宜 (CLIP 用的那种网络图文对), 但 caption 短、噪声大, 只够做对齐。
- **交错文档**让模型学「一段话里多张图」, 是多图/视频能力的基础。
- **指令数据**最值钱也最难搞 —— 真实的「图+好问题+好答案」少, 所以常用**强 VLM/LLM 合成** (如让 GPT-4V 看图生成问答对)。这接你的 data-curation 专题 + 合成数据思想。

> 数据配比是个真问题 (L4 会再碰): 阶段 2 如果全是视觉指令、没有纯文本指令, LLM 会「忘了怎么纯聊天」(灾难遗忘)。所以常**混入纯文本指令数据**保持语言能力。**数据配比本身就是一个该用 9.4 消融的超参。**

---

## 2. 冻结策略: 冻谁、何时解冻

VLM 有三块: 视觉塔 / 连接器 / LLM。每块「训 or 冻」是关键决定:

```
   阶段 1 (对齐):    视觉塔 ❄冻    连接器 🔥训    LLM ❄冻
   阶段 2 (指令):    视觉塔 ❄冻    连接器 🔥训    LLM 🔥训
   (可选) 阶段 3:    视觉塔 🔥训(部分)  连接器 🔥训   LLM 🔥训
```

逐项交代为什么:
- **连接器: 永远训。** 它是新加的小桥, 必须学。这是 VLM 训练唯一从头学的核心。
- **视觉塔: 默认全程冻。** 它的表示 (M10.1 对比预训练) 已经够好; 训它贵、且可能破坏「懂语言」的对齐。只有数据极充足、追求极致时, 才在最后阶段部分解冻微调。
- **LLM: 阶段 1 冻 (保护语言), 阶段 2 解冻 (学用视觉执行指令)。** 解冻 LLM 是让它「学会用视觉信息」的关键, 但要小心灾难遗忘 (L4)。

> 一条默认原则 (呼应 M10.1-L4): **能冻就冻。** 冻结 = 省算力 + 稳 + 不破坏预训练。VLM 的大部分能力来自冻结的预训练组件, 你只在必要处解冻。**先全冻只训连接器跑通, 再逐步解冻** —— 这是个课程式的解冻策略。

---

## 3. 冻结策略对结果的影响 (该消融)

冻不冻直接影响:
- **可训练参数量**: 全冻只训连接器 = 几万参数; 解冻 LLM = 几亿。差几个数量级 → 算力、过拟合风险、训练时间全不同。
- **效果**: 阶段 2 解冻 LLM 通常比全程冻 LLM 强 (LLM 能真正适配视觉), 但代价是贵 + 可能遗忘。
- **稳定性**: 解冻越多越容易训崩 (模态坍缩 / 遗忘, L4)。

> 这正是一个该用 9.4 实验设计认真消融的地方: 「冻 LLM vs 解冻 LLM」对准确率/语言能力保持的影响。你的 N2 会做这个消融 —— 在 mini-VLM 上对比「冻 LLM 只训连接器」vs「全解冻」, 看准确率和参数量的权衡。**这是 VLM 训练里最经典的一个消融。**

---

## 4. 一个实用的训练顺序 (LLaVA 式, 通往 L3)

```
   1. 拿现成的: 冻结 CLIP/SigLIP 视觉塔 + 冻结 LLM (如 Vicuna/LLaMA)
   2. 阶段 1: 用图文对, 只训连接器 (MLP) → 对齐
   3. 阶段 2: 用视觉指令数据, 训连接器 + LLM (解冻 LLM) → 指令跟随
   4. (混入纯文本指令防遗忘)
   → 一个能问答的 VLM
```

这个顺序最大化复用预训练 (视觉塔 + LLM 都现成), 最小化从头训 (只连接器 + 微调 LLM)。L3 会把 LLaVA 的这个配方逐步拆开。

---

## 5. 本讲小结 + 通往 L3

- VLM 三类数据: **图文对** (对齐, 便宜) / **交错文档** (多图) / **视觉指令** (微调, 珍贵, 常合成)。
- 数据配比要点: 阶段 2 混纯文本指令防语言遗忘 (该消融)。
- **冻结策略**: 连接器永远训; 视觉塔默认全程冻; LLM 阶段1冻、阶段2解冻; 原则是**能冻就冻、逐步解冻**。
- 冻不冻影响参数量 (差几个数量级)、效果、稳定性 —— **该用 9.4 消融**。
