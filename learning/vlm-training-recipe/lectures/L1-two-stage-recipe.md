# L1 · VLM 训练两阶段配方: 对齐预训练 + 指令微调

> 30-min lecture · 目标: 理解为什么 VLM 训练分两阶段, 每阶段训什么、为什么。这是把随机组装的 mini-VLM (M10.2) 真正训成能问答的 VLM 的总路线。

---

## 0. 从「组装」到「训练」

M10.2 你组装了一个 mini-VLM (视觉塔 + 连接器 + LLM), 但它是**随机权重**, 啥也不会。要让它能回答图像问题, 得训练。但 VLM 训练不是「一锅端」, 而是有讲究的**两阶段**:

```
   阶段 1: 对齐预训练 (alignment / feature alignment)
           目标: 让 LLM 学会「读懂」视觉 token
           只训连接器 (视觉塔、LLM 冻结)
        │
   阶段 2: 指令微调 (visual instruction tuning)
           目标: 让 VLM 学会「按指令回答图像问题」
           训连接器 + LLM (视觉塔常仍冻结)
```

---

## 1. 阶段 1: 对齐预训练 (教 LLM 读视觉)

**问题**: 视觉塔输出的视觉 token, 即使投影到 LLM 空间 (M10.2), LLM 也还**不习惯**读它们 —— LLM 一辈子只见过文本 token。

**解法**: 用大量「图 + 简单描述」对, 让模型学「看图说话」(image captioning), **只训连接器**:
- 视觉塔冻结 (它的表示已经够好, M10.1)。
- LLM 冻结 (保护它的语言能力)。
- **只训那个小连接器 (MLP)**, 让它学会把视觉 token「翻译」成 LLM 真正能用的形式。

> 为什么只训连接器: 这一步的任务很窄 (让视觉文本对齐), 不需要动昂贵的视觉塔/LLM。连接器是个小 MLP, 训它便宜、快、不会破坏预训练能力。**这是 LLaVA 高性价比的核心: 大部分能力来自冻结的预训练组件, 只训一座小桥。**

---

## 2. 阶段 2: 指令微调 (教 VLM 按指令答)

阶段 1 后, 模型会「看图说话」, 但不会「按指令做事」(如「图里有几只猫?」「描述左上角」)。**指令微调**用「图 + 指令 + 期望回答」的数据教它:

- 数据: 视觉指令数据 (VQA、图像描述、图像推理、定位……), 格式是「图 + 问题 → 答案」。
- 训什么: **连接器 + LLM** (这次解冻 LLM, 让它学会用视觉信息执行指令); 视觉塔常仍冻结。
- 结果: VLM 从「会描述图」升级到「会按各种指令处理图」。

> 这一步和你已会的 LLM 指令微调 (M1/M4) **几乎一样**, 只是输入多了视觉 token。你的 instruction-tuning 知识直接迁移。区别只在: 数据带图, 且要小心别让 LLM 在微调中遗忘语言能力 (L4 的灾难遗忘)。

---

## 3. 为什么是「两阶段」而不是「一阶段端到端」

新手会问: 为什么不直接拿指令数据一把训完?

- **课程式 (curriculum)**: 先学简单的 (对齐: 看图说话), 再学难的 (指令: 按要求做)。先把「视觉接口」对齐好, 再在好接口上学复杂任务, 比一上来就学复杂任务更稳更省。
- **保护预训练**: 阶段 1 只训连接器, 完全不碰昂贵的视觉塔/LLM, 用极小代价建立对齐。如果一上来就全解冻训, 容易破坏预训练表示、且贵。
- **数据效率**: 阶段 1 用便宜的图文对 (海量), 阶段 2 用珍贵的指令数据 (少而精)。两阶段让你把不同质量的数据用在对的地方。

> 本专题的 mini-VLM (N1) 把这个简化成一个可跑的合成任务: 训练时连接器必须学会把图的类别信息传给 LLM, LLM 才能答对。你会看到 loss 下降、准确率从随机 (4 类 = 0.25) 升到接近 1.0 —— 这就是「视觉→LLM 对齐」在小尺度上真的发生。

## 4. 一张总图 (本专题的骨架)

```
   ┌──── 阶段 1: 对齐预训练 ────┐  ┌──── 阶段 2: 指令微调 ────┐
   │ 数据: 图 + 简单描述         │  │ 数据: 图 + 指令 + 答案     │
   │ 训: 仅连接器                │  │ 训: 连接器 + LLM          │
   │ 冻: 视觉塔 + LLM            │  │ 冻: 视觉塔               │
   │ 学会: 看图说话              │  │ 学会: 按指令处理图        │
   └───────────────────────────┘  └──────────────────────────┘
        (L1 本讲讲为什么两阶段; L2 讲数据+冻结; L3 拆 LLaVA; L4 讲坑)
```

---

## 5. 本讲小结 + 通往 L2

- VLM 训练分**两阶段**: ① **对齐预训练** (只训连接器, 教 LLM 读视觉, 用图文对) → ② **指令微调** (训连接器+LLM, 教按指令答, 用指令数据)。
- 两阶段的理由: **课程式** (先对齐后任务) + **保护预训练** (阶段 1 不碰昂贵组件) + **数据效率** (便宜数据 vs 珍贵数据各用其所)。
- 阶段 2 和你已会的 LLM 指令微调几乎一样, 只多了视觉 token。

> **下一讲 L2「数据与冻结策略」**: 两阶段各用什么数据 (图文对/交错文档/指令数据)? 冻结策略怎么定 (冻谁、何时解冻)? 这些细节决定 VLM 训得好不好, L2 逐个讲。

**动手**: 去 `N1-train-mini-vlm.ipynb`, 用 `mini_vlm` 端到端训一个 mini-VLM, 看 loss 下降、准确率从 0.25 升到接近 1.0 —— 亲眼见证「视觉信息经连接器传给 LLM」。
