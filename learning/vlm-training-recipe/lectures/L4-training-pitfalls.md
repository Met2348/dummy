# L4 · VLM 训练陷阱: 模态坍缩 / 灾难遗忘 / 数据配比

> 25-min lecture · 目标: 认识 VLM 训练的三个常见坑 (配方对了也会崩) 和怎么避。这是把「能训」变成「训得好」的最后一关。

---

## 0. 配方对了, 为什么还崩

L1-L3 给了正确配方。但 VLM 训练比纯文本 LLM 多了「跨模态」这层, 引入了几个独有的失败模式。新手训 VLM 翻车, 大多栽在这三个坑。

---

## 1. 坑一: 模态坍缩 (modality collapse) — 只看文不看图

**症状**: VLM 训完, 回答看起来流畅, 但**根本没在看图** —— 它靠语言先验 (LLM 的常识) 猜答案, 图给不给都一样。

为什么会发生:
- 语言模态太强势: LLM 预训练时见过海量文本, 语言信号「响亮」; 视觉是新加的、信号「微弱」。模型偷懒, 学会**只靠文本**就把 loss 降下去, 忽略视觉。
- 数据问题: 如果指令数据里很多问题「不看图也能答」(语言先验就够), 模型就没动力看图。

怎么避:
- **数据要逼模型看图**: 问题的答案必须依赖图 (如「图里左边的物体是什么颜色」而非「天空通常什么颜色」)。
- **检测**: 做一个对照实验 —— 给模型**空白图/打乱图**, 看准确率是否掉。不掉 = 它没在看图 (模态坍缩)。**这是个该做的诊断实验 (接 9.4)。**

> 这个「给空白图看准确率掉不掉」的诊断, 直接呼应 9.3 的批判式阅读 (一个声称用了视觉的 VLM, 真的用了吗?) 和 10.6 的幻觉评测。**模态坍缩是 VLM 版的「不可证伪」——看起来在用视觉, 其实没有。**

---

## 2. 坑二: 灾难遗忘 (catastrophic forgetting) — 微调后不会纯聊天了

**症状**: 阶段 2 解冻 LLM 微调后, VLM 会看图了, 但**纯文本能力退化** —— 不带图时它变笨了、不会正常对话了。

为什么: 微调 LLM 时, 如果数据全是「图+指令」, LLM 的权重被往「处理视觉任务」拉, 把原来的纯语言能力**覆盖**掉了。这是神经网络的经典灾难遗忘 (你 9.5 也碰过「不可复现」, 这里是「不可保持」)。

怎么避:
- **混入纯文本指令数据** (L2 提过): 阶段 2 的数据里掺一定比例纯文本对话/指令, 提醒 LLM「别忘了怎么纯聊天」。配比是个超参 (该消融)。
- **用 LoRA 而非全量微调** (你 M1 PEFT 的知识): LoRA 只加小量参数, 原 LLM 权重不动, 天然抗遗忘。
- **少解冻**: 解冻越少, 遗忘越少 (但视觉适配也越弱, 权衡)。

---

## 3. 坑三: 数据配比失衡

VLM 训练数据有多个来源 (图文对/交错/各类指令/纯文本), **配比**直接影响结果:
- 对齐数据太少 → 视觉没对齐好, 后面学不动。
- 指令数据太单一 (全是描述、没有推理/定位) → VLM 只会某一类任务。
- 纯文本太少 → 灾难遗忘 (坑二)。
- 某类视觉任务过采样 → 偏科。

> 数据配比是 VLM 工程里最「脏」也最重要的活之一。没有公式, 靠**消融 + 验证集**调 (9.4)。一个实用心法: **先用一个朴素配比跑通, 再针对验证集上最弱的能力补对应数据** —— 迭代式调配比, 别追求一次配好。

---

## 4. 三个坑的统一视角: 跨模态的「偷懒」与「失衡」

```
   模态坍缩  = 模型偷懒, 只用强模态 (文), 忽略弱模态 (图)
   灾难遗忘  = 微调把旧能力 (语言) 挤掉, 偏向新任务 (视觉)
   数据失衡  = 训练信号在不同能力间分配不均
   ────────────────────────────────
   共同根源: 多模态训练是个「平衡」问题 —— 让模型既用好每个模态、又不偏废任一能力
```

> 一句话: **纯文本 LLM 训练是「学一件事」, VLM 训练是「平衡多件事」。** 多出来的复杂度全在「平衡」上 —— 平衡模态 (别坍缩)、平衡新旧能力 (别遗忘)、平衡数据 (别失衡)。理解这点, 三个坑就是同一个问题的三张脸。

---

## 5. 本讲小结 (M10.3 收口) + 通往 M10.4

- **模态坍缩**: 只看文不看图; 数据要逼它看图; 用「空白图准确率掉不掉」诊断 (接 9.3/10.6)。
- **灾难遗忘**: 微调后丢语言能力; 混纯文本数据 / 用 LoRA / 少解冻。
- **数据配比失衡**: 靠消融 + 验证集迭代调; 先朴素跑通再补弱项。
- 统一视角: VLM 训练是**平衡多件事** (模态/新旧能力/数据), 这是它比纯文本难的根源。

> **下一专题 M10.4「视觉 token 化与生成」**: 到此 VLM 会「读」图了。但怎么让它「画」图? 答案是把图变成离散 token (VQ), 然后像生成文本一样自回归生成视觉 token。这通往 any-to-any 模型 (Chameleon)。

**动手**: 在 N1 训好的 mini-VLM 上, 做一个**模态坍缩诊断**: 把测试图换成空白/打乱, 看准确率是否掉到随机 —— 如果不掉, 说明模型在偷懒。亲手验证「VLM 真的在看图吗」。
