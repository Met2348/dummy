# L1 · 视频扩散: 空间 + 时间

> 25-min lecture · 目标: 理解视频扩散如何在「图扩散」上加时间维, 以及时序连贯为什么是核心难点。接你 M10.5 的视频时空 token。

---

## 0. 从图扩散到视频扩散

M13.1-13.3 让你会用扩散生成图。视频 = 一串图 (帧) + 时间。视频扩散 = 在图扩散上**加一个时间维**。但这不是「每帧独立生成」那么简单 —— 视频的灵魂是**帧之间的连贯 (运动)**。

> **视频扩散的核心难题: 时序连贯 (temporal coherence)。** 帧之间要平滑过渡 (运动自然), 不能闪烁、跳变、物体突然变形。一个会画单帧但不会保持连贯的模型, 生成的「视频」是一堆抖动的图, 不是视频。本讲讲怎么让扩散生成连贯的序列。

---

## 1. 朴素做法为什么不行: 逐帧独立扩散

最朴素的想法: 用图扩散逐帧独立生成。问题:

```
   逐帧独立: 每帧用图扩散单独生成
   帧1 (独立) 帧2 (独立) ... → 帧之间没有约束 → 抖动/闪烁/不连贯
```

> 因为每帧独立生成, 模型不知道「上一帧长什么样」, 生成的帧之间没有运动连续性 —— 物体位置乱跳、颜色闪烁。你的 N1 会量化看到: **逐帧模型生成的轨迹「帧间跳变」远大于真实** (抖动)。这就是为什么视频需要**联合建模时间**。

---

## 2. 正确做法: 时空联合建模

视频扩散把**整段视频 (所有帧) 当一个整体**去噪, 让模型在去噪时**看到所有帧**, 学到帧间的运动关系:

```
   时空扩散: 模型同时看 T 帧, 帧之间互相 attend (时空 attention)
   → 学到「运动应该连贯」→ 生成平滑过渡的序列
```

机制 (`video_diffusion` 的 joint 版): 把 T 帧当一个 token 序列, 用 **transformer 让帧之间互相 attend** (时空 attention, 接 M10.5-L1)。去噪时每帧都「看着」其它帧, 于是学会保持连贯。

> 对比你的 N1: **时空模型 (joint) 生成的轨迹连贯 (帧间跳变小), 逐帧模型抖动 (跳变大)。** 同样的扩散框架, 区别只在「模型有没有跨帧交互」。这坐实了: **时序连贯来自联合建模时间, 不是免费的。**

---

## 3. 时空 attention 的两种排法 (接 M10.5)

视频的时空 attention 怎么算? 和 M10.5-L1 一样的权衡:
- **联合时空**: 所有 (帧×空间) token 互相 attend。最全, 但 $O(n^2)$ 在长视频/高分辨率上爆。
- **分解 (factorized)**: 先空间内 attend、再时间上 attend。省算力, 主流。

> 视频的核心工程难题还是 M10.5 说的**时空二次方爆炸**: 帧数 × 每帧 patch 数 = token 巨多, attention 爆。所有视频模型都在和它斗: 分解注意力、时空 patch (M10.5)、latent 视频扩散 (L2) 都是为了压这个。

---

## 4. 视频扩散 = 图扩散 + 时间维 (机制不变)

关键认知: **视频扩散的扩散机制和图扩散完全一样** (前向加噪/反向去噪/CFG, M13.1-13.3), 唯一新增:
- 数据多了时间维 (T 帧)。
- 去噪网络要**跨帧建模** (时空 attention) 以保证连贯。
- token 数暴涨 (帧×空间), 要更狠地压 (latent + 时空 patch)。

> 所以你已经会了视频扩散的 90%! 你的 N1 用的就是 M13.1 的 DDPM 框架, 只是数据从「2D 点」变成「T 帧轨迹」, 去噪网络从「MLP」变成「时空 transformer」。**扩散是统一的, 视频只是更高维 + 时间结构。**

---

## 5. 本讲小结 + 通往 L2

- 视频扩散 = 图扩散 + 时间维; 核心难题是**时序连贯** (帧间平滑)。
- **逐帧独立**生成会抖动; 必须**时空联合建模** (帧互相 attend) 才连贯。
- 时空 attention 两排法 (联合/分解), 都在对抗**时空二次方爆炸** (接 M10.5)。
- 扩散机制不变, 视频只是更高维 + 时间结构; 你已会 90%。
