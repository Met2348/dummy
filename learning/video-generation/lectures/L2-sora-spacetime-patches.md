# L2 · Sora 式时空 patch + latent 视频扩散

> 24-min lecture · 目标: 理解 Sora 的核心设计 (latent 视频扩散 + 时空 patch), 它是 M13.3-L2 (latent) + M10.5 (时空 token) + L1 (时空扩散) 的合体。

---

## 0. 真实视频扩散面对的规模问题

L1 的玩具是 12 帧 × 2D 点。真实视频是 (比如) 16 秒 × 24 帧/秒 × 1080×1920 像素 × 3 通道。直接在像素空间做时空扩散 = token 数天文数字, 算不动。Sora / 主流视频模型怎么办? 两板斧: **latent (压空间) + 时空 patch (统一切块)**。

---

## 1. 第一板斧: latent 视频扩散 (接 M13.3-L2)

和文生图一样 (M13.3-L2), 不在像素空间扩散, 先用一个 **(3D) VAE 把视频压成 latent**:

```
   原视频 (T×H×W×3, 巨大)
      │ 3D VAE 编码 (空间 + 时间都压)
   视频 latent (t×h×w×c, 小得多, 比如空间 1/8、时间 1/4)
      │ 在 latent 空间做时空扩散 (L1 的机制)
   去噪后的 latent
      │ 3D VAE 解码
   生成视频
```

> 关键: **VAE 同时压空间和时间** (3D VAE)。文生图的 VAE 只压空间 (M13.3-L2); 视频的 VAE 还压时间 (相邻帧冗余大, 可压)。压完 token 数降几十倍, 时空扩散才跑得动。**这就是 M13.3-L2 latent diffusion 的视频版。**

---

## 2. 第二板斧: 时空 patch (接 M10.5)

latent 之后, Sora 的招牌设计: 把 latent 视频切成**时空 patch** (spacetime patches), 当 transformer 的 token:

```
   视频 latent (t×h×w)
      │ 切成时空小块: 每块跨 (Δt 帧 × Δh × Δw)
   一串时空 patch token  ◄── 这就是 M10.5-L1 讲的视频时空 token!
      │ DiT (时空 transformer) 去噪 (L1 + M13.3-L1)
   去噪 latent
```

> 这正是你 **M10.5-L1 学的视频时空 token 化**! Sora 把它用作扩散的 token。好处: (1) 统一处理不同分辨率/时长 (token 数随之变, 模型不变); (2) transformer 在时空 patch 上做注意力, 自然建模时空关系。**Sora = 时空 patch (M10.5) + DiT (M13.3-L1) + latent (M13.3-L2) + 时空扩散 (L1)。你已经学过每一块。**

---

## 3. 把 Sora 拆成你学过的部件

```
   Sora 式视频扩散流水线:
   文本 "a cat surfing"
     → 文本编码 (CLIP/T5, M10.1)
     → 噪声 latent 视频 (3D VAE 空间, M13.3-L2 的视频版)
     → 切时空 patch (M10.5-L1)
     → DiT 时空去噪 T 步 (M13.3-L1 + L1):
          cross-attend 文本 (M10.2) + CFG (M13.3-L3)
     → 3D VAE 解码 → 视频
```

> **整张图没有一块是新的!** 每个组件你都在 M10/M13 学过, Sora 只是把它们拼起来 + 海量数据 + 海量算力。这再次坐实课程的核心信念: **前沿系统 = 你已学部件的组合 + 规模。** tiny 和 Sora 只差规模 (同 M10.7 mini-VLM、M13.3 文生图)。

---

## 4. 为什么时空 patch 这么关键 (统一表示)

Sora 论文的核心卖点之一: **用统一的时空 patch 表示**, 让一个模型能吃不同分辨率、不同时长、甚至图片 (1 帧的视频):
- 图片 = 1 帧视频 → 同一套 patch 表示。
- 不同时长/分辨率 → patch 数不同, 但模型 (在 patch 上做注意力) 不变。
- 训练时混合图片 + 视频 + 不同尺寸, 共享一个模型。

> 这是「**patch 是通用接口**」思想 (M10.1 ViT patchify → M10.5 时空 patch) 的极致: 任何视觉信号 (图/视频/任意尺寸) 都切成时空 patch, 喂同一个 transformer。统一表示 = 数据可混合、模型可 scale。

---

## 5. 本讲小结 + 通往 L3

- 真实视频扩散两板斧: **latent (3D VAE 压空间+时间, M13.3-L2 视频版)** + **时空 patch (M10.5 的 token, Sora 招牌)**。
- Sora = 时空 patch + DiT + latent + 时空扩散 + 文本条件; **每块你都学过, 只差规模**。
- 时空 patch = 通用视觉接口: 图/视频/任意尺寸统一表示, 数据可混、模型可 scale。

> **下一讲 L3「时序连贯的真实挑战」**: 时空建模能让短片连贯, 但真实视频还有更难的: 长程一致 (几百帧后物体是否还是同一个)、运动物理合理性、闪烁。L3 讲这些 open 难题 (gap)。

**动手**: 回看 N1 的时空 vs 逐帧对比, 想象把 2D 点换成「latent patch」: 机制完全一样, 只是 token 从「一帧的点」变成「一个时空块」。确认你理解 Sora 和你的 toy 是同一个东西的不同规模。
