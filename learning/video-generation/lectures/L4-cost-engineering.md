# L4 · 视频生成的成本工程 (为什么是少数玩家的游戏)

> 22-min lecture · 目标: 理解视频生成的训练/推理成本结构, 为什么成本驱动架构选择。收口 M13.4, 接你 M8 基础设施。

---

## 0. 视频生成不只是算法难, 更是「贵」

文生图 (SD) 已经能在消费级 GPU 上微调/推理。视频生成完全不同: **训练成本高一两个数量级, 推理也慢得多**。理解成本结构, 才懂为什么视频模型的架构长这样 (全是为省钱)。

---

## 1. 成本从哪来: token 数爆炸

回到 L1/L2 的根: **视频 = 帧数 × 每帧 patch 数**。和图比:
- 一张图: 比如 256 个 patch。
- 一段 16 秒视频: 帧数 × 每帧 patch ≈ 几万 ~ 几十万 token。

attention 是 $O(n^2)$, token 数涨 100 倍 → 计算涨 10000 倍 (最坏)。**这就是成本的源头**: 视频的 token 数天然比图大几个数量级。

> 你的 N1 toy 是 12 帧 × 1 点 = 12 token, 秒级训练。真实视频几十万 token, 这就是为什么需要几百上千张 GPU 训几周。**成本和 token 数直接挂钩, 所以所有省钱招都是在压 token** (latent、时空 patch、分解注意力 —— L2 的设计全是成本驱动的)。

---

## 2. 一个量级感: Open-Sora 的公开数字

社区复现 (Open-Sora 等) 给了量级参考 (粗略, 量级而非精确):
- 训练一个能用的文生视频模型: **几十万美元量级** (~$200k 级别的 GPU 时), 几百张 GPU 训数周。
- 闭源旗舰 (Sora 级): 普遍认为高一两个数量级 (数据 + 算力 + 人力)。
- 推理: 生成几秒视频要数十秒~分钟级 GPU 时间 (图是亚秒级)。

> 量级认知 (别背精确数字): **视频训练 = 百万美元级、推理 = 比图慢一两个数量级**。这就是为什么视频生成是少数有算力的玩家的游戏。和你 M8 基础设施模块呼应: **前沿能力 = 算法 + 算力 + 数据, 缺一不可**。

---

## 3. 成本如何驱动架构 (省钱 = 设计动机)

回看 L2 的每个设计, 全是成本驱动:
| 设计 | 省的是什么 |
|---|---|
| latent 视频扩散 (3D VAE) | 压 token 数 (空间+时间) → 直接砍计算 |
| 时空 patch | 统一表示, 可混合训练 (图+视频共享, 省数据/算力) |
| 分解注意力 (时空分开) | 把 $O((TS)^2)$ 降到 $O(T^2 + S^2)$ |
| 少步采样 (接 M13.2 flow/一致性模型) | 推理步数 50→几步, 砍推理成本 |
| 级联 (低分辨率生成 → 超分) | 主模型在小分辨率跑, 省训练 |

> **架构选择 = 成本优化的结果**, 不是凭空的美学。理解这点你就能读懂任何视频模型的设计: 问「这一招在省哪部分成本」。这和你 M8 (推理引擎、量化、并行) 的思路完全一致 —— **系统设计的核心永远是成本/性能权衡**。

---

## 4. 推理加速: 把 M13.2 的少步采样用起来

视频推理慢的大头是**扩散要多步去噪** × **每步在巨多 token 上跑**。M13.2 学的加速直接用得上:
- **flow matching / rectified flow** (M13.2): 更直的概率路径, 少步采样。
- **一致性模型 / 蒸馏** (M13.2): 几步甚至 1 步生成。
- 这些把视频推理从「几十步」压到「几步」, 是实时视频生成的关键路线。

> 你 M13.2 学的少步采样在视频上**价值放大**: 图省几步是锦上添花, 视频每步都极贵, 少步采样是能不能实时的分水岭。**M13.2 不是孤立的, 它是视频/实时生成的命脉。**

---

## 5. 本讲小结 (M13.4 收口) + 通往 M13.5

- 视频成本源头: **token 数爆炸** (帧×patch), attention $O(n^2)$ 放大。
- 量级认知: 训练百万美元级、推理比图慢一两个数量级; 少数玩家的游戏 (接 M8)。
- **架构 = 成本优化的结果**: latent / 时空 patch / 分解注意力 / 少步采样 / 级联, 每招都在省特定成本。
- M13.2 的少步采样在视频上价值放大, 是实时生成的命脉。

> **M13.4 收口**: 视频 = 图扩散 + 时间维; 核心难题时序连贯; Sora = 你学过的部件 (时空patch+DiT+latent) + 规模; 成本驱动一切架构。
> **下一专题 M13.5「world-models」**: 视频生成再进一步 —— 如果模型能**预测「我做某动作后世界会变成什么样」**, 它就成了世界模型 (可交互的视频生成)。这接 L3 的长程一致 (需要持久状态), 也接 M11 具身 (世界模型给机器人规划)。

**动手**: 给你的 N1 toy 估个「成本」: 把帧数从 12 加到 48、把每帧从 1 点加到 16 点, 测训练时间怎么涨。亲手体会 token 数 → 成本的非线性关系 (这就是真实视频贵的微缩版)。
