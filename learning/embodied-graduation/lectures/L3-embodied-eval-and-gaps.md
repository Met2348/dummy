# L3 · 具身评测 (LIBERO/CALVIN) + 找 gap

> 22-min lecture · 目标: 理解具身评测的思路与坑, 在具身全栈上系统找研究 gap。

---

## 0. VLA 怎么评? 比 NLP 更难

你的 eval 模块知道评估难。具身评测更难: 它要在**物理任务**上测**成功率**, 而不是文本对错。本讲讲具身评测的思路 (LIBERO/CALVIN 等) 和坑, 然后找研究 gap。

---

## 1. 具身评测的基本思路

```
   标准任务套件 (LIBERO / CALVIN / RLBench ...):
     - 一组标准化的操作任务 (抓/放/开/堆...)
     - 固定的初始分布 + 评测协议
   指标: 成功率 (success rate) = 完成任务的 episode 比例
   分层: 看泛化 — 训练任务 / 新物体 / 新布置 / 新指令的成功率
```

> 核心指标是**成功率**, 在**标准化任务套件**上测, 让不同方法可比 (像 NLP 的 benchmark)。LIBERO/CALVIN 提供标准任务 + 协议。你 N1 会建一个 toy 版 mini-benchmark: 多方法 × 多配置的成功率表。

---

## 2. 具身评测的坑 (接 eval 模块 + M9 批判)

- **成功率抓不全**: 任务完成 ≠ 安全/平滑/高效。一个抖动着勉强完成的策略成功率也高。
- **过拟合 benchmark**: 在固定任务套件上刷分, 不代表真泛化 (同 NLP 的 benchmark 过拟合)。
- **sim 评测 ≠ real**: 仿真里成功率高, 真机可能掉 (sim2real gap, M11.6)。
- **初始分布敏感**: 评测的初始分布稍变, 成功率就变 (要固定 + 报告, 接 9.4)。
- **统计噪声**: 成功率要够多 episode 才稳 (你 toy 的 eval 用上百 episode)。

> 一句话: **成功率是必要但不充分的**。真评一个 VLA, 要看**分层泛化** (新物体/布置/指令) + 鲁棒性 + sim2real, 而非单一 benchmark 分数。这和你 eval 模块的「测的是不是你真关心的」一脉相承。

## 3. 在具身全栈上找 gap (M9 框架)

`embodied_capstone.py` 整理了 6 个具身/VLA 研究 gap:
1. **数据高效具身学习** ← M11.4 + M11.5 + 视频

> 注意每个 gap 都标了「连接哪些专题」—— 好 gap 长在交界。对你 (NLP 背景 + IsaacLab 经验): **gap 1 (数据高效, 接你数据直觉)、gap 5 (sim2real, 接你 IsaacLab)** 是低成本切入。

## 4. idea 卡四要素 (接 M9)

每个 gap 结构化成 idea 卡:
1. **问题 (gap)**: 具体没解决什么。
2. **为什么难**: 真障碍 (非没人做)。
3. **最小可验证实验**: toy 上能跑的第一步 (本课每个 gap 都配了 toy 实验)。
4. **连接**: 复用你哪些已学 (降执行成本)。

> 关键还是**最小可验证实验**: 不止说"该研究 X", 而是"我明天能在 toy 上跑的第一步"。你 N2 产出的每张卡都带这一栏 —— 这是从 idea 到行动的桥 (M9 反复强调)。

## 5. 本讲小结 + 通往 L4

- 具身评测 = **标准任务套件 (LIBERO/CALVIN) + 成功率 + 分层泛化**。
- 坑: 成功率抓不全 (安全/平滑) + 过拟合 benchmark + sim≠real + 初始分布敏感 (接 eval + 9.4)。
- 6 个具身 gap, 每个标连接; 对你 gap1(数据)/gap5(sim2real) 最匹配。
- idea 卡四要素, 重点是最小可验证实验。
