# L4 · DDPM 采样 + 步数权衡: 质量 vs 速度

> 25-min lecture · 目标: 把扩散训练和采样完整串一遍, 理解采样步数的核心权衡, 为 M13.2 (flow matching 加速) 铺垫。收口 M13.1。

---

## 0. 完整回顾: 一个扩散模型的生命

```
   训练 (一次):
   for 很多步:
     取真数据 x_0, 随机 t 和噪声 ε
     算带噪 x_t (L2 闭式)
     网络预测噪声, MSE 损失, 更新 (L3)

   生成 (每次):
   x = 纯噪声
   for t = T, T-1, ..., 1:
     x = 用网络预测的噪声去掉一步 + 加小噪声 (L3 采样)
   返回 x (生成样本)
```

你的 N1/N2 把这个完整跑通: 训练 → loss 下降 → 采样 → 看见数据从噪声浮现。

---

## 1. 采样步数 $T$: 扩散的命门

DDPM 生成要**迭代 $T$ 步** (每步过一次网络)。这是扩散相对自回归/GAN 的最大软肋:

```
   T 步采样:
   纯噪声 → [网络] → [网络] → ... → [网络] → 数据
            第1步     第2步           第T步
   每步都要过一次网络 → T 次前向 → 慢
```

- **DDPM 原版**: $T = 1000$ 步。生成一张图要过网络 1000 次 → 慢 (比 GAN 慢几个数量级)。
- **对比**: GAN 一次前向出图; 自回归 N 个 token N 步。扩散的 $T$ 步是它**质量高但慢**的根源。

> 这是扩散的核心工程问题: **质量靠多步细化, 但多步 = 慢**。整个扩散加速研究 (M13.2 flow matching / consistency models) 都在攻「怎么用更少步数 (理想 1 步) 达到同等质量」。理解 DDPM 的 $T$ 步软肋, 才理解后续加速工作的动机。

---

## 2. 步数 vs 质量的权衡

减少采样步数 (跳步) 能加速, 但掉质量:

| 采样步数 | 速度 | 质量 |
|---|---|---|
| 多 (1000) | 慢 | 高 (每步去一点点, 精细) |
| 中 (50, 本专题) | 中 | 尚可 |
| 少 (10) | 快 | 掉 (每步去太多, 糙) |
| 1 步 | 极快 | 朴素 DDPM 做不到 (需特殊方法, M13.2) |

> 你的 N1 用 $T=50$ (玩具数据够了)。真实高分辨率图常用几十到几百步 + 改进的采样器 (DDIM 等)。**步数是扩散部署时最重要的旋钮** (该消融, 9.4): 在质量和延迟间找点。这直接关系 M13.4 视频生成的成本 (帧数 × 步数 → 算力)。

---

## 3. 几个实用的采样改进 (一句话)

- **DDIM**: 一种确定性采样器, 能用更少步数 (如 50 步) 达到接近 1000 步的质量。几乎标配。
- **classifier-free guidance** (M13.3): 控制「跟条件跟得多紧」的旋钮, 提升条件生成质量。
- **flow matching / rectified flow** (M13.2): 把采样路径「拉直」, 大幅减少步数 (理想几步)。
- **consistency models** (M13.2): 训练成「一步直达」, 牺牲一点质量换极速。

> 这些都是 M13 后续专题。本专题 (M13.1) 让你掌握**最基础的 DDPM**; 理解了它的 $T$ 步软肋, 后面所有加速工作的动机就清楚了。

## 5. 本讲小结 (M13.1 收口) + 通往 M13.2

- 扩散完整生命: 训练 (预测噪声) → 采样 ($T$ 步反向去噪)。
- **采样 $T$ 步是扩散命门**: 质量靠多步细化, 但多步 = 慢 (比 GAN 慢几个数量级)。
- 步数 vs 质量权衡: 步数是部署最重要旋钮 (该消融); 改进采样器 (DDIM) 能少步保质。
- 加速是扩散研究主线 (flow matching/consistency, M13.2); 理解 $T$ 步软肋才懂它们的动机。

> **下一专题 M13.2「flow matching」**: DDPM 的 $T$ 步慢是头号痛点。flow matching / rectified flow 把生成路径「拉直」, 大幅减少步数, 是 2024-26 的 SOTA 采样范式 (你的 EE ODE 直觉是优势)。
