# L1 · 生成的两条路: 自回归 vs 扩散

> 25-min lecture · 目标: 在你已会的自回归生成基础上, 建立对扩散生成的直觉对比。理解为什么需要第二条生成范式。

---

## 0. 你已经会一半了

你已经精通**自回归生成** (你的 LLM/推理引擎专题): 一个 token 一个 token 地预测下一个, 串行生成。M10.4 还把它用到了图像 (自回归生成视觉 token)。

但生成数据有**另一条完全不同的路**: **扩散 (diffusion)**。它是 Stable Diffusion、DALL-E 2/3、Sora、以及 M11 机器人 diffusion policy 的核心。不懂它, 你对生成式 AI 的认知有一半空白。本模块补这一半。

---

## 1. 两条路的根本区别

```
   自回归 (你已会)                      扩散 (本模块)
   ─────────────                       ──────────────
   从左到右, 一个个生成 token            从纯噪声, 逐步去噪成数据
   x1 → x2 → x3 → ... (串行, 有顺序)     噪声 → 略清晰 → 更清晰 → 数据 (并行, 无顺序)
   每步: 预测下一个 token                每步: 去掉一点噪声
   离散 token 友好                       连续数据友好 (图像/动作)
```

一句话对比:
- **自回归**: 「写文章」式 —— 从头到尾, 一个词一个词写, 每个词依赖前面。
- **扩散**: 「雕塑」式 —— 从一块噪声 (毛坯) 开始, 一刀刀去掉多余的, 整体逐步显形。

> 关键直觉: 扩散**不按顺序**生成。它对整张图/整个数据**同时**地逐步细化 (所有像素一起从噪声变清晰), 而非从左上到右下。这让它在图像 (空间无天然顺序) 上比自回归更自然、质量常更高。

---

## 2. 扩散的核心思想: 加噪 → 学着去噪

扩散基于一个反直觉但优雅的想法:

```
   前向 (固定, 不学): 给数据逐步加噪声, 直到变成纯高斯噪声
   x_0 (数据) → x_1 → ... → x_T (纯噪声)
        每步加一点点高斯噪声

   反向 (要学的): 训一个网络, 把每一步的噪声去掉
   x_T (纯噪声) → ... → x_1 → x_0 (数据)
        每步去掉一点噪声 = 学出来的

   生成: 从纯噪声 x_T 开始, 反复用网络去噪, 还原出数据
```

逐点交代:
- **前向过程是固定的**: 加噪声不需要学, 是个预定义的、逐步把数据「腐蚀」成噪声的过程 (L2)。
- **反向过程是学的**: 训一个网络, 看着带噪的数据, 预测「该去掉什么噪声」(L3)。
- **生成 = 反向过程**: 从纯噪声开始, 一步步去噪, 最后得到一个看起来像真实数据的样本。

> 为什么这样能 work: 把「生成一个复杂分布」(难) 拆成了「逐步去一点点噪声」(每步简单)。一步从纯噪声直接变数据太难, 但「去掉一点噪声」是个网络能学的简单任务。**把难问题拆成多步简单问题, 是扩散的智慧。**

---

## 3. 一个你能看见的例子 (本专题的演示)

你的 N1 会在 2D 双月分布上看见这个过程:
```
   目标: 生成「双月」形状的点云
   前向: 双月点云 → 逐步加噪 → 变成一团高斯噪声 (形状消失)
   反向: 一团噪声 → 网络逐步去噪 → 双月形状逐渐浮现
   → 你会看见点云从随机噪声「收敛」成双月 (去噪轨迹)
```

> 在 2D 上看「去噪轨迹」是理解扩散最快的方式 —— 你会**亲眼看见**噪声逐步变成数据。一旦看懂 2D 的, 高维图像/视频/动作只是同一个机制在更高维 (M13 后面 + M11)。

---

## 4. 为什么扩散是必须懂的第二范式

- **图像/视频生成的主力**: Stable Diffusion / DALL-E 2-3 / Sora 都是扩散 (M13.4 视频)。
- **机器人控制**: diffusion policy 用扩散生成动作 (M11.3, 多峰动作分布扩散很合适)。
- **世界模型**: 视频扩散当世界模拟器 (M13.5, 接 M11)。
- **和自回归互补/混合**: Transfusion 等前沿混合两者 (M10.4-L3 已埋伏笔)。

> 一句话: **自回归统治了语言, 扩散统治了连续媒体 (图/视频/动作)。** 你做 NLP 已精通前者; 要做多模态/具身, 必须掌握后者。这就是 M13 在你体系里的位置 —— 补上生成范式的另一半, 且直接喂养 M11 具身。

---

## 5. 本讲小结 + 通往 L2

- 生成有两条路: **自回归** (你已会, 一个个生成 token) vs **扩散** (从噪声逐步去噪)。
- 扩散像「雕塑」: 对整个数据**同时**逐步细化, 不按顺序; 连续数据 (图/动作) 上更自然。
- 核心思想: **前向加噪 (固定) → 学着反向去噪 → 生成 = 从噪声去噪**; 把难问题拆成多步简单去噪。
- 扩散是图/视频/机器人/世界模型的主力, 和自回归互补 —— 生成范式的另一半。

> **下一讲 L2「前向加噪过程」**: 先精确理解「加噪」—— 它是个马尔可夫链, 有漂亮的闭式 (能一步算出任意时刻的带噪数据), 这是扩散训练高效的关键。L2 给出数学, 逐项交代。

**动手**: 去 `N1-2d-diffusion.ipynb` 之前, 先在脑子里想象: 一张清晰的图反复加噪声会变成什么 (雪花点)? 反过来, 从雪花点能不能猜回原图? 扩散就是教网络做这个「反猜」。
