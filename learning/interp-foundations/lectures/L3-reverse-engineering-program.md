# L3 · 逆向工程纲领: 把神经网络当程序读

> 22-min lecture · 目标: 把 feature/circuit/superposition 组织成一个研究纲领 —— 系统地逆向工程网络。

---

## 0. 一个类比: 网络是编译后的程序

mech interp 的统一视角:

> **训练好的神经网络 = 一个「编译」过的程序。权重是机器码, 激活是运行时的变量值, 前向传播是执行。逆向工程 = 反编译, 把机器码读回人能懂的算法。**

- 反汇编逆向工程: 机器码 → 汇编 → 伪代码 → 算法。
- mech interp: 权重/激活 → feature → circuit → 算法。

> 这个类比贯穿全模块: 你在做的是**软件逆向工程, 只是对象是神经网络**。你 EE/CS 的「读懂系统」直觉直接迁移。

## 1. 逆向工程的基本循环

```
   ① 选一个具体行为 (如「模型怎么做 in-context 复制」)
   ② 观察: 读激活, 看哪些组件参与 (probing M12.2, logit lens)
   ③ 假设: 提出一个 circuit 假说 (「head X 搬运信息到 head Y」)
   ④ 干预: 用 patching/ablation 因果验证假说 (M12.3) ← 关键
   ⑤ 提炼: 把验证过的组件连成 circuit, 用人话描述算法 (M12.5)
   ⑥ 攻击: 找反例/边界, 证伪或修正 (M9.3 批判)
```

> 核心是 **②观察 → ③假设 → ④干预** 的循环: 观察给线索 (相关), 干预给证据 (因果)。**没有干预 (④), 一切都是相关, 可能误导** (M12.3 的核心)。这是 mech interp 和「看 attention 图猜故事」的本质区别 —— 它要求因果验证。

---

## 2. 从「具体行为」入手 (而非整个模型)

逆向工程不试图一次读懂整个模型 (太大), 而是**挑一个具体、窄、可定义的行为**:
- ✅ 好的切入: 「induction (AB...A→B)」「主谓一致」「IOI (间接宾语识别)」—— 窄、可定义、可测。
- ❌ 不好: 「模型怎么推理」—— 太大太模糊。

> 这和你 M9 的「把大问题切成可验证的小问题」一脉相承。mech interp 的经典成果 (induction heads / IOI circuit) 都是从一个**窄行为**入手, 完整逆向。**先攻一个小而清楚的行为, 是这门学科的方法论。** 你 12.7 capstone 也会选一个窄行为做。

---

## 3. residual stream: 信息流的主干 (接 M12.2)

逆向工程的关键视角: **residual stream (残差流)**。transformer 每层通过残差连接读写一条「信息主干」:

```
   resid → [attn 读取, 计算, 写回 resid] → [mlp 读取, 计算, 写回 resid] → resid → ...
```

- 每个组件 (head/MLP) 从 residual stream **读**信息、**写**信息。
- 信息在这条流上累积, 最后 unembed 成预测。

> 这个视角让逆向工程变可操作: 问「每个组件从 residual 读了什么、写了什么」。你的 `tiny_transformer.py` 缓存了每层的 `resid_post` —— 你能直接看信息怎么在残差流里流动。**residual stream 是逆向工程的「数据总线」**, M12.2 (logit lens) 和 M12.3 (patching) 都在这条流上操作。

## 4. 工具与纲领的对应

把工具映射到逆向工程循环:

## 5. 本讲小结 + 通往 L4

- **网络 = 编译过的程序; mech interp = 反编译** (权重→feature→circuit→算法), 接你逆向工程直觉。
- 基本循环: **观察 (相关) → 假设 → 干预 (因果) → 提炼电路 → 攻击**; 干预是关键 (没干预只是相关)。
- 方法论: 从**窄的具体行为**入手 (induction/IOI), 而非整个模型 (接 M9 切小问题)。
- **residual stream** = 信息主干; 组件从它读写; 是逆向工程的数据总线。
