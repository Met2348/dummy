# Federated Learning Mastery 使用说明

## 为什么这是一个全新专题

联邦学习(Federated Learning)此前在整个面试题库(898点，涵盖 `onsite-mastery` 五个子包和 `foundation-model-mastery`)里完全没有被覆盖过。老手要求 Foundation Model / Federated Learning / Diffusion 三个专题各做100+点、难度"浅到深到社招级别"，这是队列第二个，组织方式沿用 `foundation-model-mastery` 已验证的"难度分层"轴。

## 三层怎么练

1. **Tier 1(浅，`tier1_shallow/`)**：先建立"联邦学习是什么、为什么存在、FedAvg具体怎么跑"这套基础认知，再理解"系统层面怎么实际部署"(client selection/同步异步/straggler)。这一层答不上来说明连联邦学习的地图都没画清楚，不要跳过。
2. **Tier 2(深，`tier2_deep/`)**：机制深水层，三个文件分别对应FL三个最核心的技术挑战——数据异质性(non-IID)、通信瓶颈、隐私安全。这三个挑战贯穿几乎所有FL论文和面试追问，练透这三个文件基本就覆盖了FL算法研究的主干。
3. **Tier 3(社招级别，`tier3_social_hire/`)**：无标准答案的资深判断题，练法是对着 `rubric` 自查覆盖面。第一个文件考"这套系统在生产环境里怎么真实运转"，第二个文件考2026年最前沿的"FL+LLM"交叉判断——这个方向本身还在快速演化，练的不是背答案，是"面对一个新兴、尚未成熟的技术方向，该怎么诚实评估它的成熟度、不夸大也不过度保守"这个更高阶的判断力。

## 一个容易被忽视的陷阱

联邦学习的核心卖点"数据不出域=隐私安全"经常被过度简化。Tier 2 的 `dp_fl_privacy_security.py` 专门讲清楚了"只传梯度不传数据"本身不是绝对安全保证(梯度反演攻击可以从梯度反推出接近原始的训练样本)，这是这个领域里最容易被面试官拿来戳穿"背了口号但没理解本质"的候选人的一个点，练习时要特别留意。
