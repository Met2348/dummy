# L1 · 经费申请全流程 (The Grant Application Lifecycle)

> 30-min lecture · 目标: 把"经费申请"从"写一份proposal然后等结果"这句模糊印象, 拆成一条可执行的全流程: 撰写申请材料 → 预算论证 → 执行阶段 → 结题报告, 并交付一版能扛住财务/合规审查的预算论证草稿。
> 关键区分: **`research-direction-proposal` L3「开题报告写作」教的是研究内容本身的论证**(背景/gap/假设/方法/时间线/风险/贡献, 读者是同行评审, 检验的是科学价值); **本讲教的是钱本身的申请、论证、执行、结算**(budget/budget justification/facilities资源说明等行政附件, 读者常常是项目官员和财务/合规部门, 检验的是预算合理性和执行一致性)。两者是同一份申请材料里并行存在、但被完全不同的人用完全不同的标准审查的两个部分, 缺一不可但不能互相替代。

---

## 0. 为什么这是个真问题

新手写第一份经费申请时, 几乎必然把全部注意力放在研究叙述(narrative)上——毕竟`critical-reading-gap`和`research-direction-proposal`L3已经把"怎么论证一个研究问题值得做"练得很熟。但一份完整的申请材料里, 除了narrative, 还有一整套**行政附件**: budget page(预算表)、budget justification(预算论证)、facilities资源说明、current & pending support(列出你手头正在申请/已获批的其他经费, 避免"double dipping"重复申请同一笔工作量)。这些附件不是可有可无的模板填空——评审小组之外, 还有项目官员(program officer)和机构的财务/合规部门在审查它们, 而且它们的审查标准和narrative完全不同: narrative问"这个问题值得研究吗", 预算附件问"这笔钱花在哪、为什么这么花、将来怎么证明确实这么花了"。

区分两讲的关注对象, 用一张表说清楚:

```
                开题报告写作 (research-direction-proposal L3)      经费申请全流程 (本讲)
关注对象         研究内容本身(背景/gap/假设/方法/时间线/风险/贡献)     经费本身怎么申请/论证/执行/结算
核心问题         "这个研究值得做吗、做得出来吗"                       "这笔钱花在哪、为什么这么花、怎么证明花对了"
文档载体         proposal的研究叙述部分(narrative)                   budget/budget justification/facilities等行政附件
审查的人         同行评审(peer review), 看科学价值                    项目官员+财务/合规部门, 看预算合理性与一致性
时间跨度         写完提交narrative即完成这一环节的任务                 贯穿整个资助周期: 申请→执行→中期报告→结题决算
```

**新手最常踩的坑**: 把narrative写得漂亮, budget justification那一页却随手填一句"申请50万元用于开展本研究的相关支出", 没有把每一项开销对应到具体产出。评审看到的信号不是"这个人不擅长写预算", 而是"这个人没有认真规划过项目怎么执行"——这恰恰是预算论证环节最容易失分、也是本讲要治的问题。

## 1. 经费申请全流程四阶段

```
①撰写申请材料 ──► ②预算论证 ──► ③执行阶段 ──► ④结题报告
(narrative+行政附件)  (每项花费对应产出)  (按科目执行+中期报告)  (科学叙述与财务决算对齐)
```

### ① 撰写申请材料阶段
除了narrative(`research-direction-proposal`L3已详细教过, 本讲不重复), 还需要准备: budget page(直接费用direct costs + 间接费用/管理费indirect costs, 常缩写F&A, 费率由机构和资助方事先谈定, PI个人不能随意更改)、facilities资源说明(你所在机构现有哪些设备/算力支撑这个项目)、current & pending support(列出你名下正在申请或已获批的其他经费及其工作量占比, 资助机构用它核查你是否有足够时间投入新项目, 也防止同一笔工作量被两个资助方重复资助)。

### ② 预算论证阶段
budget justification通常是一到两页的附件, **核心要求: 每一项花费都要对应一个具体产出**, 而不是笼统的"研究经费"。常见科目: 人力(RA/博后的月薪和对应月数)、设备、差旅(会议/合作访问)、耗材、云计算/算力成本。这一阶段也要写清楚indirect cost rate是机构层面预先和资助方谈定的固定比例, 申请人不需要(也不能)自己去改这个数字, 但需要知道它会从总预算里扣除, 影响你能实际花在直接研究活动上的额度。

### ③ 执行阶段
经费到账后按年度/阶段执行。几乎所有资助机构对"预算科目之间挪用"(rebudgeting, 比如把原本给设备的钱挪去发人力工资)都设有容忍度阈值——常见的规则是超过某个比例(具体数字因机构而异, 需查该资助机构当年的政策文件)需要提前书面申请批准, 而不是先斩后奏。年度/中期报告(annual/interim report)通常同时要求科学进展汇报和经费执行情况汇报, 两者需要互相印证。

### ④ 结题报告阶段
final technical report(科学叙述: 做了什么、达到了哪些目标)和financial closeout(财务决算: 钱实际怎么花的)必须相互一致。**这是审计红线**: 如果科学叙述说做了三个大规模实验, 财务记录却显示预算大部分花在了行政差旅上, 这种不一致会直接触发审计问询, 影响PI以后申请同一资助机构经费的信誉记录。

## 2. 一个走查示例: 预算论证从模糊到具体

```
反例(模糊, 无法通过审查):
  "申请50万元用于开展本研究的相关支出, 包括人员、设备和差旅等。"

  问题诊断:
  · 没有分项——审查者无法判断哪部分花费对应哪个具体产出。
  · 没有数量/单价——无法核实合理性(比如GPU-hours到底买了多少小时、单价是多少)。
  · "等" 字暴露了申请人自己也没想清楚具体清单。

改写(具体, 可核查):
  "预算80%(40万元)用于GPU-hours: 按当前云计算单价估算, 覆盖3个大规模训练实验
   (对应研究问题③的三组消融), 每个实验预计需要X GPU-hours; 20%(10万元)用于
   RA工资(6个月, 承担数据预处理和评测脚本维护)及2次会议差旅(对应预期贡献的
   成果发表计划)。"

  为什么这版能通过审查:
  · 每一项花费都标注了对应的具体产出(三组消融/RA的具体工作/会议发表计划)。
  · 有明确的数量和用途拆分, 财务审查能核实合理性。
  · 和narrative里的实验设计、时间线相互印证, 不是两张互不相干的文档。
```

## 3. 常见误区

1. **budget justification写成一句套话** —— 症状: 出现"用于本研究相关支出"这类没有分项的表述。解法: 套第2节的改写模板, 每一项都对应一个具体产出。
2. **不清楚indirect cost/F&A由谁谈定** —— 症状: 申请人自己试图调整间接费用比例, 或完全不知道这笔钱的存在导致预算规划失真。解法: 提前找机构科研处/财务办公室确认当前费率, 把它当作既定约束而非自己能决定的变量。
3. **执行阶段随意挪用预算科目、不书面报备** —— 症状: 觉得"反正总预算没超"就随便在科目间调配。解法: 查清该资助机构当年的rebudgeting阈值政策, 超过阈值提前书面申请。
4. **结题报告的财务数据和科学叙述对不上** —— 症状: 科学叙述夸大完成度, 财务记录却显示大量预算未按计划使用。解法: 执行阶段就同步记录科学进展和经费流向, 而不是等结题前临时拼凑。

## 4. 可参考的官方指南

经费申请全流程的具体格式和规则因资助机构而年年更新, 以官网当年发布的版本为准, 但两份官方指南是理解整体流程绕不开的起点:

- **NSF, *Proposal & Award Policies & Procedures Guide (PAPPG)***——NSF官方发布、定期更新的proposal撰写与执行全流程规范, 涵盖budget、budget justification、current & pending support、rebudgeting阈值等本讲提到的几乎所有行政环节, 是理解"钱怎么申请和管理"最权威的第一手材料。
- **NIH, SF424 (R&R) Application Guide**——NIH官方的申请材料撰写指南, 详细规定了budget page和budget justification的具体格式要求, 和PAPPG分属不同资助机构体系, 但核心逻辑(每项花费对应具体产出)是相通的。

## 5. 本讲小结 + 通往 L2

- 经费申请全流程和研究内容论证(`research-direction-proposal`L3)是同一份申请材料里两个关注对象、审查标准都完全不同的部分, 缺一不可。
- 四阶段: 撰写申请材料(narrative之外的行政附件) → 预算论证(每项花费对应具体产出) → 执行阶段(按科目执行+中期报告, 挪用需书面报备) → 结题报告(科学叙述与财务决算必须一致)。
- 4个常见误区: 预算论证写成套话/不清楚间接费用归属/执行阶段随意挪用/结题报告前后不一致。

> **下一讲 L2「算力资源规划与申请」**: 预算论证里最容易被写得笼统的一项就是算力开销。L2把"算力"单独拎出来, 讲清楚集群quota申请流程、云计算预算怎么估算, 以及组内多人抢卡时怎么谈判争取到资源, 而不是在预算表里写一句"需要更多GPU"就带过。

**动手**: 打开 `templates/funding-plan-worksheet.md` 的「预算论证」小节, 把你手头(哪怕只是设想中)的一个研究项目, 按第2节的改写模板列出至少三项具体开销, 每项配一句"对应哪个实验/里程碑"。
