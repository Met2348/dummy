# AI 智能体时代的记忆：一篇综述（Forms–Functions–Dynamics）

> 本篇对齐 agent-harness 库 v2 标杆范文（`2605.27922-harness-bench`）的密度与诚实度：v1 硬规范（公式前给直觉+符号先定义、指标/判据给定义、数字标 §/Table/Fig 出处、区分宣称 vs 批判）+ 本库 Θ1–Θ5（E/T/C/L/O/V 分层、回扣 `Agent=Model+Harness`、Inspires-Us 打到我们自己 harness、canon/前沿坐标、regime 诚实）。
>
> **一句话定位**：上一篇 Harness-Bench 把 harness 当**被测变量**立了"卡尺"；这一篇把 harness 里最难的那层——**记忆（C 层）**——立了一套"坐标系"。读完它，再看库内任何一个具体记忆方法（MemGPT / Mem0 / A-MEM / HippoRAG…），都能一句话报出它的"三轴坐标"。

## §1　TL;DR（一页讲清这篇在干嘛）

> 主讲提示：开场先说"为什么记忆值得一篇 76 页的综述"，再抛出三轴，最后用一句话点明它和 Harness-Bench 的分工。

**一句话**：智能体（agent）越来越多地要做**长程推理、持续适应、与复杂环境交互**，而支撑这三件事的核心能力就是**记忆**（原文 Abstract）。但"智能体记忆"这个领域已经**严重碎片化**——大家都自称在做"agent memory"，可动机、实现、假设、评测协议各不相同，术语（declarative / episodic / semantic / parametric…）满天飞（§1）。传统的"长期 vs 短期"二分**不足以**刻画当代记忆系统的多样性与动态性（Abstract）。本文给出一套**统一的三轴分类法**，把约 400 个系统装进去：

- **Forms（形态）——记忆"长在哪、以什么形式存"**：分 **token 级**（显式、可读、可编辑的离散单元）/ **参数级**（写进模型权重）/ **潜在级**（藏在 KV cache、隐藏态等内部表示里）三类（§3）。
- **Functions（功能）——智能体"为什么"需要记忆**：超越"长/短期"，细分为 **事实记忆**（agent 知道什么 What does it know）/ **经验记忆**（agent 如何变强 How does it improve）/ **工作记忆**（agent 现在在想什么 What is it thinking about now）三柱（§4）。
- **Dynamics（动态）——记忆"怎么"运转和进化**：**形成（Formation）→ 演化（Evolution）→ 检索（Retrieval）** 的生命周期（§5）。

**3 条带走的结论**：
1. **范式正在从"检索"转向"生成"、从"手工"转向"自治"、从"启发式"转向"RL 驱动"**（§7.1–7.3）。记忆不再是"贴在 LLM 外面的一个检索库"，而要变成"与推理共同进化的、可学习的子系统"。
2. **三轴是正交的、可组合的坐标，而非互斥的桶**。同一个系统（如 MemGPT）会在不同轴上分别落位（token 级形态 / 事实+工作功能 / 增量 summarization 形成）。这正是它比"长短期"更有解释力的地方（见 §3 Why 三连）。
3. **诚实的边界**：本文**不**测性能、**不**给排名（这是 Harness-Bench 干的活），它是一篇**概念框架**论文；它的贡献是"把混乱的术语收敛成一张可检索的地图"+ 汇编 benchmark（Table 8）与开源框架（Table 9）。

**属于 harness 的哪一层（Θ1）**：本篇是 **C（Context/上下文-记忆）层**的**框架综述**。在本库六层解剖（E/T/C/L/O/V）里，记忆是 C 层的本体；但它对 **L（控制循环）**强依赖（检索时机/形成时机都嵌在 ReAct 循环里，§2.2 的三算子 F/E/R "未必每步都调用"）、对 **T（工具）**有交叉（skill-based memory 就是把"工具/技能"当记忆存，§4.2.3）。

**回扣全库论点（Θ2）**：`Agent = Model + Harness`。本文把 **Memory 从 Harness 的一个零件，抬升为 harness 设计的一等公民**——并在 §2.3 用一张 Venn 图（Fig 2）把 agent memory 与 LLM memory / RAG / context engineering 划清边界。它给"换 harness 分数大变"提供了**机制解释的一半**：Harness-Bench 测出 23.8 分摆动里有相当一块来自"状态/续跑"失败（9.3%）与"契约/格式"失败（36.4%）——而这正是**工作记忆**（§4.3）与**记忆形成**（§5.1）在管的事。

**够新够权威（Θ4）**：2026-01 预印本，新加坡国立大学等**十余家机构联合**出品；是**首批用"形态×功能×动态"三轴取代"长/短期"二分**的记忆综述（Abstract 原文：传统分类"have proven insufficient"）。

## §2　问题与动机：为什么"智能体记忆"值得一套新分类法（Why·问题层）

> 主讲提示：这页讲清"领域为什么乱"，给出两条核心动机。这是综述类论文最该讲透的"问题层 why"。

**Why（问题层）——不做这件事会卡住什么？**
原文 §1 给了两条并列动机（论文称之为 ❶❷）：

1. **❶ 既有分类法过时（Limitations of Existing Taxonomies）**：已有几篇记忆综述（如 Zhang et al. 2025s、Wu et al. 2025g）的分类体系，是在 2025 年的"方法学大爆发"**之前**搭的，没能反映当前的广度。例如 2025 年涌现的"从过往经验蒸馏可复用工具"的记忆框架、"记忆增强的测试时扩展"方法，在旧分类里**没有位置**。
2. **❷ 概念碎片化（Conceptual Fragmentation）**：自称研究"agent memory"的论文，在实现、目标、底层假设上**差异巨大**；declarative / episodic / semantic / parametric 等术语泛滥，进一步**模糊了概念清晰度**。

**后果**：领域缺一张**统一地图**，导致同一个词指代不同东西、不同词指代同一东西，新人无法快速定位、综述无法系统比较。

**5 个核心问题（论文 §1 的 Key Questions 框）**：
- ❶ agent memory 到底如何**定义**？它和 LLM memory / RAG / context engineering 什么关系？（→ §2 回答）
- ❷ **Forms**：记忆能取哪些架构/表示形态？（→ §3）
- ❸ **Functions**：为什么需要记忆，它服务哪些角色？（→ §4）
- ❹ **Dynamics**：记忆如何随时间运转、适应、演化？（→ §5）
- ❺ **Frontiers**：推进记忆研究的有前景方向有哪些？（→ §7）

> **读出什么**：和 Harness-Bench 一样，这篇的动机**不是"再造一个更强记忆系统"，而是给领域补一把"游标卡尺"的孪生工具——"坐标系"**。卡尺量"谁强"，坐标系答"它是什么、和谁同类、和谁对立"。这是 A 组（框架综述）作为方法论护城河的意义。

## §3　形式化：把"智能体+记忆"写成一个交互回路（§2 Preliminaries）

> 主讲提示：这页把全篇的数学骨架立起来。每个式子先给直觉，再定义符号。综述里少有公式，这几个是核心，务必讲透——它们决定了"记忆"在 agent loop 里的位置。

**直觉**：一个 agent 就是"在一个会变的环境里，每一步看一眼（observe）、想一下（可能查记忆）、做一个动作（act），动作又改变环境"的循环。记忆 = 这个循环里"跨步、跨任务"保留信息的那个**可演化状态**。

### 3.1 环境与 agent（§2.1）

记号（先定义后用）：
- $\mathcal{I}=\{1,\dots,N\}$：agent 的索引集合；$N=1$ 是单 agent（如 ReAct），$N>1$ 是多 agent（如 planner–executor、辩论）。
- $\mathcal{S}$：环境状态空间；$s_t\in\mathcal{S}$ 是 $t$ 时刻状态。
- $a_t$：$t$ 时刻执行的动作。
- $\mathcal{Q}$：任务规范（task specification，如用户指令、目标、外部约束），**任务内固定**。

环境按一个**受控随机转移**演化（直觉：做了动作 $a_t$，环境从 $s_t$ 概率性地变到 $s_{t+1}$）：

$$ s_{t+1}\sim \Psi(s_{t+1}\mid s_t, a_t) $$

每个 agent $i$ 收到一个**观察**（直觉：agent 看不到全局真状态，只能看到一个被 $O_i$ 过滤后的视图）：

$$ o_t^i = O_i(s_t, h_t^i, \mathcal{Q}) $$

- $h_t^i$：对 agent $i$ 可见的那部分**交互历史**（过往消息、工具输出、推理痕迹、共享工作区状态等）。

**动作空间的异质性**是 LLM-agent 的特色（§2.1）：不止"生成文本"，还包括 **工具调用 / 规划动作 / 环境控制动作（改仓库、改共享 buffer）/ 通信动作**。这一点很重要——它意味着"记忆"要服务的"动作"远比"输出一句话"复杂。

### 3.2 记忆系统：一个会演化的状态 + 三个算子（§2.2）

记号：
- $\mathcal{M}_t\in\mathbb{M}$：$t$ 时刻的**记忆状态**；$\mathbb{M}$ 是所有可行记忆配置的空间。**关键**：原文**不强制** $\mathcal{M}_t$ 的内部结构——它可以是文本 buffer、KV store、向量库、图、或任意混合（§2.2）。短期/长期之分"emerge from usage patterns rather than architectural separation"（从用法涌现，而非架构上分隔）——**这句话是全篇 Functions 轴的种子**。

记忆生命周期由**三个算子**刻画（这是 §5 Dynamics 的骨架）：

**① 形成（Formation）**——直觉：别把整段历史逐字存下来，只把"未来可能有用"的信息抽成候选记忆。

$$ \mathcal{M}_{t+1}^{\text{form}} = F(\mathcal{M}_t, \phi_t) $$

- $\phi_t$：$t$ 时刻产生的**信息性产物**（informational artifacts）：工具输出、推理痕迹、部分计划、自评、环境反馈。

**② 演化（Evolution）**——直觉：新候选记忆要和老记忆"对账"——合并冗余、解决冲突、丢弃低价值、为高效检索重排。

$$ \mathcal{M}_{t+1} = E(\mathcal{M}_{t+1}^{\text{form}}) $$

**③ 检索（Retrieval）**——直觉：选动作时，构造一个"任务感知"的查询，从记忆里取出相关内容，喂给策略。

$$ m_t^i = R(\mathcal{M}_t, o_t^i, \mathcal{Q}) $$

- $m_t^i$：检索出的、可直接被 LLM 策略消费的**记忆信号**（一串文本片段或结构化摘要）。

最终，agent 的策略把"观察 + 记忆信号 + 任务"映射到动作：

$$ a_t = \pi_i(o_t^i, m_t^i, \mathcal{Q}) $$

> **读出什么（这页的核心判断）**：**三算子 F/E/R 不必每步都调用**——这是全篇最重要的"结构性洞察"。原文给了一个判别式：有的系统只在任务开始时检索一次

$$ m_t^i = \begin{cases} R(\mathcal{M}_0, o_0^i, \mathcal{Q}), & t=0 \\ \perp, & t>0 \end{cases} $$

（$\perp$ = 空检索），有的则间歇/持续检索。**"短期 vs 长期"不是两个架构模块，而是 F/E/R 在不同时间频率上被调用所"涌现"出的现象**。这一句直接论证了：为什么本文敢用 Functions（事实/经验/工作）而不是"长/短期"来切——因为后者根本不是架构上的真实分界（详见 §6 的 Why 三连）。

### 3.3 边界划分：agent memory vs LLM memory vs RAG vs Context Engineering（§2.3，Fig 2 Venn 图）

> 主讲提示：这页是"概念清场"。用 Venn 图把四个常被混用的词划清。重点记住每一对的"分界判据"。

| 对比对象 | 重叠（共享什么） | 分界判据（agent memory 独有什么） |
|---|---|---|
| **LLM memory**（§2.3.1） | few-shot prompting、self-reflection、KV 压缩——这些**大部分**其实属于 agent memory | LLM-internal memory（改 KV cache、改架构如 RWKV/Mamba/diffusion-LM、扩 KV-store）目标是**扩模型表征容量**，不支持跨任务持久/环境驱动适应 → **不算** agent memory |
| **RAG**（§2.3.2） | 都构造/组织外部信息库、都有向量索引+语义检索 | RAG 增强的是**静态知识源**、面向**单次推理**；agent memory 是**持续交互中自生成、会演化**的内部记忆库。判据是**任务域**：RAG≈HotpotQA/2WikiMQA 这类多跳 QA；agent memory≈LoCoMo/GAIA/SWE-bench 这类多轮/长程任务 |
| **Context Engineering**（§2.3.3） | 都在用压缩/组织/选择来对付有限上下文窗（working memory 处重叠最大） | Context engineering 做的是**资源分配与接口正确性**（语法有效、调度高效）；agent memory 做的是**认知建模**——管 agent **知道什么、经历过什么、身份如何跨任务保持** |

> **读出什么（Θ2 呼应）**：这张表是本篇对全库的"概念基建"贡献。它告诉我们：当我们说"给 harness 的 C 层加记忆"时，到底是在加 RAG（静态知识）、加 context engineering（窗口调度）、还是加真正的 agent memory（持久演化的认知态）——**这三者在我们自己的 harness 里是不同的组件，不能混为一谈**（见 Inspires-Us a）。

## §4　形态轴（Forms）：记忆"长在哪、以什么形式存"（§3，Fig 1/3/4/5）

> 主讲提示：这是三轴的第一轴，也是最"工程"的一轴。每类先给判据（怎么一眼认出它属于哪类），再给代表系统。

**判据（一句话分三类，§3 开头）**：看**记忆驻留在哪、以什么表示**——
- **token 级**：显式离散单元，**外部可见、可编辑**（不止文本 token，也含视觉 token、音频帧）。
- **参数级**：编码进**模型权重**，前向计算时隐式访问。
- **潜在级**：藏在模型**内部隐藏态/KV cache/latent embedding** 里，既非明文 token 也非专用参数集。

### 4.1 token 级记忆（§3.1）——最常见、最透明，按"拓扑维度"再分三层

**判据（按 inter-unit 拓扑，§3.1）**：单元之间有没有结构、是几维的结构——

| 子类 | 判据 | 代表系统（Table 1 / Fig 3） |
|---|---|---|
| **Flat（1D）** | 无显式拓扑，记忆是序列或"袋"（snippets/trajectories/chunks 的堆叠） | Reflexion、Memento、ExpeL、Voyager、MemGPT、Mem0、MemoryBank、MIRIX、Generative Agents… |
| **Planar（2D）** | 单层结构（图/树/表），单元间有关系但**无跨层** | A-Mem、Memo^g、M3-Agent、D-SMART（图）；MemTree、TME（树） |
| **Hierarchical（3D）** | 多层 + 层间链接，形成"金字塔/分层"卷状记忆 | G-Memory、CAM（金字塔）；HiAgent、HippoRAG、SGMem、Zep、GraphRAG（多层图） |

**Why（设计层）——为什么要按"维度"分 token 记忆，而不是按"内容"分？**
> 朴素做法是按内容分（对话/用户画像/经验…）。→ 会**和 Functions 轴打架**（内容其实是"功能"该管的）。本文改按**拓扑维度（1D/2D/3D）**分，因为拓扑直接决定了"**搜索/更新/推理的效率与能力**"（§3.1 原文）：Flat 好写好读但多跳弱；Planar 支持关系推理但要维护图；Hierarchical 支持跨粒度多跳但"保证所有节点语义有意义 + 设计最优三维布局"很难（§3.1.3 Discussion）。**维度是"能力–成本"的真实坐标，内容不是。**

### 4.2 参数级记忆（§3.2）——写进权重，按"内 vs 外"分

**判据（按存储位置相对核心参数，§3.2）**：
- **Internal（内部）**：直接改原始权重（weights/biases）。代表：KnowledgeEditor、MEND、ROME、MEMIT（知识编辑）、Character-LM、ToolFormer。注入时机可在 **pre-train / mid-train / post-train**（Table 2）。
- **External（外部）**：加**额外参数**（adapter、LoRA、proxy model），不动原权重。代表：K-Adapter、WISE、ELDER、MemLoRA（adapter 类）；MAC、Retroformer（辅助 LM 类）。

**Why（设计层）——为什么要有 External 这个折中？**
> 朴素做法只有两端：要么存成 token（模型理解不深、占窗口）、要么改 internal 权重（更新难、与预训练知识冲突、灾难性遗忘）。→ 两端都有硬伤。External 参数（如 WISE 的双参数记忆 + 路由、ELDER 的多 LoRA + 路由）是"**模块化、可加可删可回滚**"的中间道（§3.2.2），代价是"其影响是**间接的**、要经模型注意力/计算通路中介"，效果取决于外部参数能否和内部参数知识良好接口。

### 4.3 潜在级记忆（§3.3，Fig 4）——藏在隐藏态里，按"来源"分

**判据（按 latent state 的来源，§3.3）**：
- **Generate（生成）**：由独立模型/模块**新造**紧凑连续态（gist token、summary vector）。代表：Gist、AutoCompressor、MemoryLLM、M+、MemGen、CoMEM。
- **Reuse（复用）**：直接沿用先前计算的内部态，最典型是 **KV-cache 复用**。代表：Memorizing Transformers、LONGMEM、FOT。
- **Transform（变换）**：对已有 latent 态做选择/聚合/压缩（≈ KV 压缩）。代表：Scissorhands、SnapKV、PyramidKV、H2O、RazorAttention。

**Why（设计层）——为什么要单独立"潜在级"，而不并入参数级？**
> 很多潜在级方法也依赖"单独训练的模块"，看着像参数级。→ 但本文的**分类锚点是"记忆的形态"而非"学习机制"**：潜在级方法产出的是**显式实例化、可被当独立单元复用的 latent 表示**，而非直接嵌进权重/前向激活（§3.3.1）。判据清晰：**记忆单元能不能被单独拎出来复用**——能（latent vector/KV）就是潜在级，不能（融进 W）就是参数级。

### 4.4 三形态的"适用场景"对照（§3.4，Fig 5）——这是给工程选型的判据表

| 形态 | 特性 | 最适合 | 软肋 |
|---|---|---|---|
| **token 级** | 符号化、可寻址、透明；增删改快；长期稳定、抗灾难性遗忘；即插即用（不改底座） | 多轮 chatbot、个性化、推荐、**高风险域（法/金/医，要可验证溯源）** | 占上下文窗口、检索效率受规模拖累 |
| **参数级** | 隐式、抽象、可泛化；（通常）性能增益更好 | 角色扮演、推理密集、需要"**从根上获得新能力**"的任务 | 更新慢、灾难性遗忘更重 |
| **潜在级** | 隐式、人不可读；效率/灵活折中；天然跨模态融合；token 高效 | 多模态记忆、端侧/边缘部署、低资源/小数据 | 黑箱、难调试/验证、生成过程可能丢信息或漂移 |

> **读出什么（关键工程判断 + Inspires-Us 伏笔）**：**三形态不是"谁取代谁"，而是按"可解释性 vs 效率 vs 能力"权衡的三种选择**。我们自己的 harness（Claude Code）现在的记忆是**纯 token 级**（MEMORY.md 明文 + compaction），落在"透明、可验证、即插即用"这一格——优点是可审计，代价是占窗口、规模一大检索就钝（正好对上这格的软肋）。这张表直接告诉我们"下一步该往哪格借东西"（见 §15 Inspires-Us）。

## §5　功能轴（Functions）：智能体"为什么"需要记忆（§4，Fig 6/7）

> 主讲提示：这是三轴里最有"认知科学味"的一轴，也是本文相对旧综述最大的增量——用三柱替代"长/短期"。每柱配一个"它回答什么问题"的金句。

**三柱（§4 开头的金句，务必板书）**：
- **事实记忆（Factual）**：agent 的**陈述性知识库** → 回答 **"What does the agent know?"**
- **经验记忆（Experiential）**：agent 的**程序性/策略知识** → 回答 **"How does the agent improve?"**
- **工作记忆（Working）**：agent 的**容量受限的动态草稿纸** → 回答 **"What is the agent thinking about now?"**

三者构成一个**认知回路**：encoding（把交互结果固化进长期记忆）→ processing（在工作记忆里推理）→ retrieval（从长期记忆取相关上下文回填工作记忆）（§4 原文）。

### 5.1 事实记忆（§4.1）——按"指向哪个实体"分

认知科学根基：对应**declarative memory**（Riedel & Blokland 2015），含 **episodic**（特定时空的"what/where/when"）+ **semantic**（一般事实/概念）。但原文强调：在 agent 系统里这**不是死板的二分，而是一个处理连续谱**（episodic 原始日志 → 反思/抽取 → semantic 事实库）。

它要保的三性质（§4.1）：**Consistency（行为/自我表述不矛盾）、Coherence（上下文感知、话题连贯）、Adaptability（按用户画像个性化）**。

| 子类 | 判据 | 代表系统 |
|---|---|---|
| **User factual**（§4.1.1） | 关于**用户**的可验证事实：身份、稳定偏好、历史承诺 | MemGPT、MemoryBank、TiM、RMM、COMEDY；防的是 coreference drift（指代漂移）、重复追问、自相矛盾 |
| **Environment factual**（§4.1.2） | 关于**外部世界**的事实：文档/代码库状态、资源、其它 agent 能力 | 知识持久化：HippoRAG、MemTree、LMLM、WISE；共享访问：MetaGPT、G-Memory、Generative Agents、OASIS |

### 5.2 经验记忆（§4.2）——按"抽象层级"分（Fig 7）

认知根基：对应人类**nondeclarative / procedural / habit** 系统（Squire 2004）。这是**continual learning 与 self-evolution 的基础**（"era of experience"，Sutton 2025）。判据是**存的东西被抽象到多高**：

| 子类 | 判据（抽象层级从低到高） | 代表系统 |
|---|---|---|
| **Case-based**（§4.2.1） | 最少加工的**原始事件/轨迹**，高保真、当 in-context 范例直接复用 | Memento（soft Q-learning 选轨迹）、ExpeL、JARVIS-1、Early Experience、DreamGym |
| **Strategy-based**（§4.2.2） | 蒸馏出的**可迁移知识**：insights（规则/启发式）/ workflows（动作序列）/ patterns（认知模板） | Reflexion、Buffer-of-Thoughts、AWM（工作流）、ReasoningBank、ACE、Agent KB |
| **Skill-based**（§4.2.3） | **可执行**的程序性能力：code snippet / function / API / MCP | Voyager（技能库）、Darwin Gödel Machine（自改代码）、Gorilla、ToolLLM、Alita（MCP）、LEGOMem |
| **Hybrid**（§4.2.4） | 混合多层（原始+规则+技能），动态选最合适格式 | ExpeL、G-Memory、Memp、MemEvolve、MemVerse |

**Why（设计层）——为什么要按"抽象层级"分经验，而非按"任务域"？**
> 朴素做法按域分（游戏/网页/代码经验）。→ 无法回答"**同一条经验该存成什么粒度**"这个真问题。本文按抽象层级（case→strategy→skill）分，因为这正是"**保真度 vs 泛化性**"的权衡轴：case 保真但占地、检索贵；strategy 泛化但"只指导规划、不直接执行"；skill 可执行但要保证 callable + 可验证 + 可组合（§4.2 各 Summary）。**层级是经验记忆的"压缩率"坐标。**

### 5.3 工作记忆（§4.3，Table 6）——按"单轮 vs 多轮"分

认知根基：Baddeley 的**容量受限、主动控制**的工作记忆。**关键批判**：原文指出当前 LLM 的上下文窗其实只是个"**被动只读 buffer**"，缺乏主动选择/维持/变换的机制——近期行为证据表明"current models do not exhibit human-like working memory characteristics"（§4.3，Huang et al. 2025a）。所以要把窗口从"被动 buffer"改造成"**可控、可更新、抗干扰的工作区**"。

| 子类 | 判据 | 代表机制/系统 |
|---|---|---|
| **Single-turn**（§4.3.1） | **单次前向内**对付海量输入（长文档/多模态流）→ 输入压缩 + 观察抽象 | 输入压缩：LLMLingua（硬）、Gist/ICAE/AutoCompressor（软）、HyCo2（混合）；观察抽象：Synapse（HTML→state）、VideoAgent |
| **Multi-turn**（§4.3.2） | **跨轮**维持任务态、防历史淹没注意力 → 状态固化/分层折叠/认知规划 | 状态固化：MemAgent、ReSum、MemGen；分层折叠：HiAgent、Context-Folding、AgentFold、DeepAgent；认知规划：PRIME、SayPlan、KARMA |

> **读出什么（Θ3 直接命中我们）**：**工作记忆 = compaction 的学名**。我们 harness 的"上下文压缩 / compaction"，在本文坐标里就是 **Multi-turn 工作记忆**的"状态固化 + 分层折叠"。本文给了一整套比"无脑截断"高级的做法（AgentFold 把 fold 操作做成**可学策略**、IterResearch 用 MDP 式迭代重建工作区防"context suffocation"）——这是我们 compaction 策略的现成升级清单（见 Inspires-Us）。

## §6　动态轴（Dynamics）：记忆"怎么"运转和进化（§5，Fig 8/9/10）

> 主讲提示：这是三轴的"动词轴"——前两轴讲"记忆是什么静态对象"，这轴讲"记忆作为一个活的生命周期怎么转"。对应 §3 的三算子 F/E/R。

**总骨架（Fig 8）**：**Formation（形成）→ Evolution（演化）→ Retrieval（检索）**，三者构成闭环：形成抽取 → 演化整合 → 检索利用 → 推理结果又反馈回形成/演化。三个问题分别是 **"怎么抽（extract）/ 怎么炼（refine）/ 怎么用（utilize）"**。

### 6.1 记忆形成（§5.1，Table 7）——5 种"抽"法

**判据（按压缩粒度 + 编码逻辑，§5.1）**：

| 形成操作 | 在做什么 | 代表机制 |
|---|---|---|
| **语义摘要（Semantic Summarization）** | 把冗长原始数据压成紧凑摘要（保全局语义、滤冗余）；分**增量**（chunk-by-chunk 融合，如 MemGPT/Mem0/Mem1，后者用 RL 优化）与**分区**（按 session/章节切，如 MemoryBank/ReadAgent/DeepSeek-OCR 光学2D压缩） | MemGPT、MemAgent(GRPO)、ReadAgent、LangRepo |
| **知识蒸馏（Knowledge Distillation）** | 更细粒度地抽**可复用知识**（事实/经验），不管存储结构 | 事实：TiM、RMM、MemGuide；经验：AWM、ExpeL、H²R（两层反思）、Memory-R1（RL 训 extractor） |
| **结构化构建（Structured Construction）** | 把无定形数据组织成**显式拓扑**（图/树），增强可解释 + 多跳 | 实体级：KGT、Memo^g、GraphRAG、AriGraph、Zep（三层时序图）；chunk 级：RAPTOR、MemTree、H-MEM、A-MEM |
| **潜在表示（Latent Representation）** | 直接编码进机器原生 latent（向量/KV） | MemoryLLM、M+、MemGen、CoMEM、KARMA |
| **参数内化（Parametric Internalization）** | 经梯度更新固化进权重（知识内化 vs 能力内化） | 知识：MEND/ROME/MEMIT/CoLoR；能力：ToolFormer、Memory Decoder（即插即用参数化 RAG） |

### 6.2 记忆演化（§5.2，Fig 9）——3 种"炼"法

**判据（按目标，§5.2）**：

| 演化机制 | 在做什么 | 代表机制 |
|---|---|---|
| **巩固（Consolidation）** | 合并新旧、反思整合成更高阶 insight；分 local（细粒度合并，RMM top-K）/ cluster（PREMem、EverMemOS、CAM）/ global（MOOM 角色画像、Matrix 迭代优化） | 累积式学习而非孤立 |
| **更新（Updating）** | 解决冲突、纠错补全；分 external memory update（MemGPT/Mem0 硬替换 → Zep 软时序更新 → Mem-α 把更新建成 RL 策略学习）与 model editing（ROME 定位改、MEMORYLLM/M+ latent 自更新） | 关键难题：stability–plasticity 困境 |
| **遗忘（Forgetting）** | 删过时/冗余/低价值；分 time-based（MemGPT 溢出驱逐、MAICC 软遗忘）/ frequency-based（XMem LFU、KARMA Bloom filter、MemOS LRU）/ importance-driven（TiM/MemTool 让 LLM 判重要性） | 防知识过载，但激进剪枝会误删长尾 |

> **读出什么**：**演化这层是"区分死库与活库"的关键**。RAG 是"死库"（写进去就不动）；agent memory 的演化算子 E 让它变"活库"（自我巩固/纠错/遗忘）。这正是 §2.3.2 划清 RAG 与 agent memory 的那条线在动态层的体现。

### 6.3 记忆检索（§5.3，Fig 10）——4 步"用"法

**判据（按检索流水线的阶段，§5.3）**：

| 检索阶段 | 在做什么 | 代表机制 |
|---|---|---|
| **时机与意图（Timing & Intent）** | 何时触发、查哪个库；从"被动/指令驱动"→"自治、自调"。fast-slow 思考（ComoRAG/PRIME 先快答再判是否深检索）；MemGen 把触发做成 latent 可训练决策 | MemGPT、MemTool、AgentRR、MemOS、H-MEM（粗到细路由） |
| **查询构建（Query Construction）** | 弥合"用户原话 vs 记忆索引"的语义鸿沟；**分解**（Visconde/PRIME/Agent KB）vs **改写**（HyDE 生成假设文档、MemoRAG、ToC） | Auto-RAG（自动选 HyDE/Visconde） |
| **检索策略（Retrieval Strategies）** | 实际搜索；**lexical**（TF-IDF/BM25）/ **semantic**（Sentence-BERT/CLIP，默认主力）/ **graph**（AriGraph/HippoRAG PageRank/Zep 时序）/ **generative**（直接生成 doc id）/ **hybrid** | 各给定义式（如 BM25 加 IDF + 长度归一化） |
| **后处理（Post-Retrieval）** | 重排/过滤 + 聚合/压缩；learn-to-memorize 把打分做成 RL；Memento 用 Q-learning 估"该条对正确答案的贡献概率" | ComoRAG/MA-RAG（聚合）、G-Memory（按角色个性化） |

> **读出什么（关键张力）**：§5.3.4 Summary 点破检索的根本矛盾——**自治时机/意图能省算力、压噪声，但制造了一个"沉默失败"漏洞**：当 agent **高估自己的内部知识、该检索却没检索**时，系统会陷入"silent failure"并幻觉。所以要平衡"该给时给、别过度检索引噪"。**这条和 Harness-Bench 的"evidence/grounding 失败 14.6%"是同一个病**。

## §9　前沿与立场（§7）：7 个"回看→展望"+ 3 大范式迁移

> 主讲提示：这是全篇的"风向标"页。不要逐条念，抓住三条主范式迁移线 + 每个子方向的"一句话缺口"。

**三条主范式迁移（贯穿 §7.1–7.3，也是 Abstract 的落点）**：
1. **检索 → 生成（§7.1）**：从"把记忆当静态库去查"转向"按需主动**合成**新记忆表示"。两条路：retrieve-then-generate（ComoRAG/G-Memory/CoMEM，保 grounding）与 direct generation（MemGen/VisMem，绕过显式检索）。未来记忆应：**上下文自适应 / 跨异构信号整合 / 可学习自优化**。
2. **手工 → 自治（§7.2）**：从"人手写规则/阈值/分类"转向"agent 自己管记忆"。未来方向：**把记忆操作（增/删/改/查）做成显式工具调用**，让 agent 像推理一样推理"我现在该做哪个记忆动作"。
3. **启发式 → RL 驱动（§7.3，Fig 11）**：RL-free（MemGPT/Mem0，纯启发式）→ RL-partial（RMM 排序、Memory-R1、MemSearcher）→ fully RL-driven（自设计记忆架构）。未来：**记忆架构由 agent 自己学，最小化人类先验**，agent 对记忆全生命周期有完整控制。

**4 个专题前沿（§7.4–7.7）**：
- **多模态记忆（§7.4）**：缺"真正 omnimodal"的记忆，现有都偏单模态/松耦合。
- **多 agent 共享记忆（§7.5）**：从"隔离本地记忆+消息传递"→"中心化共享（黑板/共享文档）"→ 未来"**角色/信任感知**的 access control + 学习式贡献策略"。
- **世界模型的记忆（§7.6）**：记忆是 world model 维持长程一致性（场景/物体/运动）的基石；从 Data Caching → State Simulation；未来"双系统架构（System-1 SSM 快 / System-2 VLM 慢）+ 主动记忆策略"。
- **可信记忆（§7.7）**：三支柱——**隐私保护 / 可解释 / 抗幻觉**。点名风险：记忆可被间接 prompt 攻击**泄露隐私**（Wang et al. 2025b）；需要 access control + verifiable forgetting + auditable updates（Wu et al. 2025g）。

**1 个认知科学连接（§7.8）**：当代 agent 记忆架构（容量受限上下文窗 + 海量外部向量库）**惊人地复刻了** Atkinson-Shiffrin 多存储模型 + Tulving 的 episodic/semantic/procedural 三分。但有一条**根本分歧**：人类记忆是**建构性的**（每次回忆都按当前认知态**重构**过去），而多数 agent 记忆是**逐字检索不可变 token**——agent 有"如实记录"但缺"记忆扭曲/抽象/动态重塑"的生物能力。未来应引入**离线巩固（sleep-like consolidation）+ 主动遗忘 + 生成式重构**。

> **读出什么（Θ4 时间坐标）**：这 7 个前沿是 2026-01 的"记忆研究路线图"。其中**§7.2（记忆操作即工具调用）+ §7.3（RL 驱动记忆）+ §7.8（生成式重构 / 离线巩固）** 三条，对我们 harness 的 C 层最有直接施工价值（见 Inspires-Us）。

## §10　主要"结论性论断"梳理（区分宣称 vs 边界）

> 主讲提示：综述没有实验数字，但有"论断"。这页把它的核心宣称与诚实边界并列，对应 v1 的"宣称 vs 批判"。

**论文的核心宣称（claims）**：
1. "形态×功能×动态"三轴**比"长/短期"更能刻画**当代记忆（Abstract + §2.2"短长期从用法涌现"的论证）。
2. agent memory **几乎完全包含**传统"LLM memory"，但 LLM-internal memory（改 KV/架构）**不属于** agent memory（§2.3.1）。
3. 领域正经历"检索→生成、手工→自治、启发式→RL"三大迁移（§7）。
4. agent 记忆架构与人类认知模型**结构同构**，但**动态分歧**（建构性 vs 逐字检索）（§7.8）。

**论文诚实承认/隐含的边界（honest limits）**：
- 这是**概念框架**，**不含任何受控实验/排名**——"哪类记忆更好"本文不答（这是 Harness-Bench 的活）。三轴的"更优"是**论证性**的，非实证的。

- 三轴**正交性是设计假设**，并非所有系统都干净落位（如很多 latent 方法"既像生成又像参数")，原文需反复申明分类锚点（§3.3.1）才能消歧。

## §11　局限与批判（论文承认的 + 我的补充）

**论文自陈/可推断的局限**（见 §10）：概念框架无实证、归位示意性、覆盖有遗漏、正交性是假设。

**我的补充批判**：
- **"三轴正交"在工程上是漏的**：Forms 与 Dynamics 高度耦合（"参数内化"既是一种 Form 也是一种 Formation 操作，§5.1.5 与 §3.2 讲的是同一批 ROME/MEMIT）。综述自己也把同一篇论文（如 MemGPT）在 4 张表里反复出现。**正交是叙述方便，不是真实独立**——这会让"按三轴检索"时出现重复定位。
- **缺"该用哪轴"的决策树**：三轴给了"是什么"，但没给"我有一个新任务，该怎么沿三轴选记忆"的**可操作判据流程**。Fig 5 的适用场景表是最接近的，但仍停在定性。
- **评测与框架表是"链接汇编"不是"质量评判"**：Table 8/9 列了 benchmark/框架，但**没有**横向打分（哪个 benchmark 防刷更严、哪个框架 overhead 更低）——这恰恰是 G 组 Harness-Bench 那套"受控协议"能补的。
- **"建构性记忆"被点名却未被解决（§7.8）**：本文敏锐指出"agent 缺生物的记忆扭曲/重构能力"，但这究竟该不该补、补了会不会引入"记忆幻觉"（把没发生的事重构出来），原文留白——这是双刃剑，**可信记忆（§7.7）与建构性记忆（§7.8）天然冲突**，综述未调和。
- **多 agent 共享记忆的"谁污染谁"未量化**：§7.5 提了 write contention / memory clutter，但没有"一个 agent 写错记忆会如何在团队里传播"的机制分析——这正是本库多 agent 编排该警惕的。

## ★ 对我们的启发（Inspires Us）

> 本库独门优势（Θ3）：**我们（Claude Code / 本课 m9.* 的 agent）本身就是一个 harness，而且有真实的记忆文件（MEMORY.md 明文）+ compaction（上下文压缩）**。所以这一节最该做的，是**用本文的三轴给我们自己的记忆机制"归位"，再把"下一步"落到具体组件**。

**先给我们自己的记忆机制做"三轴归位"**（这是本节的地基）：
- **Forms**：我们是**纯 token 级**（MEMORY.md 是明文 markdown，外部可见可编辑），且拓扑是 **Flat（1D）**（按 entry 堆叠，无图无层）。→ 落在 §3.4 表的"透明、可审计、即插即用"格，软肋也精准命中："占上下文窗口 + 规模一大检索就钝"。

➤ **a. 可直接借用的招（method/trick we can reuse）**：

2. **§5.2 的"演化三算子（巩固/更新/遗忘）"**：给 MEMORY.md 补一个**演化层**。最小版：写入新记忆前，先 top-K 检索相似旧条目（RMM 式 local consolidation），让 LLM 判"合并 / 覆盖（更新）/ 新增"，并给每条记忆打 time + access-freq 标签做 importance-driven 遗忘（§5.2.3）。这能直接治 MEMORY.md "越长越乱、旧承诺与新承诺打架"的病。

## §12　版图定位（canon/前沿坐标 + 在本库的位置）

- **时间坐标（Θ4）**：**2026-01 前沿**。它是**首批用"形态×功能×动态"三轴取代"长/短期"二分**的记忆综述（Abstract 明示旧分类 insufficient）。相对本库已有工作的增量：它**不**推进某个具体能力，而是给整个 **C 层（含 D 组所有具体方法）提供"统一坐标系"**——把散落的 MemGPT/Mem0/A-MEM/HippoRAG…一次性归位。Harness-Bench 把 harness 设为**评测变量**（卡尺），本篇把记忆设为**分类对象**（坐标系），二者是"度量"与"地图"的互补。
- **E/T/C/L/O/V 归属（Θ1）**：坐 **C（Context/上下文-记忆）层**，是 C 层的**框架顶层**；强依赖 L（循环决定 F/E/R 调用时机）、交叉 T（skill-based memory = 工具即记忆）。
- **在本库的位置**：**A 组（综述/框架）锚点**。读完它，再回看本库任何一篇记忆相关论文，都能问一句三轴拷问：**"它在哪种 Form？服务哪个 Function？在 Dynamics 的哪一环（形成/演化/检索）动了刀？"** ——这正是本篇作为"坐标系"的最大复用价值。
- **regime 诚实（Θ5）**：不把"记忆越多越好"绝对化。本文自己强调（§3.4 / §5.2.3）——**形态选择分场景**（高风险域要 token 级可验证，端侧要 latent 高效）、**遗忘是必要的**（无界累积反而增噪、拖检索、降推理）、**自治检索有"沉默失败"反作用**（§5.3.4）。所以诚实表述是：**记忆机制的"该加什么、加多少"分 regime**——任务越长程/越需跨会话一致，记忆收益越大；越是单轮纯语言任务，重型记忆几乎无所谓（与 Harness-Bench §12"纯语言任务最不吃 harness"同构）。

## §13　组会讨论问题（留给大家吵）

1. **三轴真的正交吗？**"参数内化"同时是一种 Form（§3.2）和一种 Formation 操作（§5.1.5）——这种耦合会不会让"按三轴检索"退化成"同一篇出现在多个格子"？我们需要的是不是一个**带优先级的决策树**而非三个独立坐标？
2. 本文断言"agent memory 几乎包含 LLM memory，但 KV/架构改动不算"（§2.3.1）。可是 §3.3 的"潜在级 Reuse（KV 复用）"明明被算进了 agent memory——**这条边界是不是自相矛盾？**判据到底是"改不改架构"还是"能不能当独立单元复用"？
3. §7.8 说人类记忆是"建构性的"、agent 缺这能力。但 §7.7 又要"可信/抗幻觉"。**给 agent 加"建构性重构"会不会等于给它发"幻觉许可证"**？这两个前沿如何调和？
4. 我们 harness 的 MEMORY.md 该停在 **Flat(1D)** 还是升 **Planar(2D)/Hierarchical(3D)**？升维的收益（多跳、关系推理）是否抵得过维护图/层的成本（§3.1.3 Discussion 警告的"保证节点语义有意义 + 设计最优布局"很难）？
5. 多 agent 共享记忆（§7.5）里，"一个 agent 写错一条记忆"会怎么在团队里传播？本库的子代理编排该不该给共享记忆配"角色/信任感知的写权限"？
6. 本文是"地图无实证"。如果让你用 Harness-Bench 那套受控协议去**实测**三轴的某个论断（如"token 级在高风险域更优"），你会怎么设计这个实验？

## §14　一页速记（takeaways）

- **命题**：智能体记忆已碎片化，"长/短期"二分不够用；本文立**三轴坐标系**统一之，并汇编 benchmark + 框架。
- **三轴**：
  - **Forms（长在哪）**：token 级（透明可编辑，再分 1D/2D/3D）/ 参数级（写进权重，内 vs 外）/ 潜在级（藏隐藏态，生成/复用/变换）。
  - **Functions（为什么）**：事实（知道什么）/ 经验（如何变强，case→strategy→skill）/ 工作（现在在想什么，单轮压缩 + 多轮折叠）。
  - **Dynamics（怎么转）**：形成（5 种抽法）→ 演化（巩固/更新/遗忘）→ 检索（时机/构建/策略/后处理）。
- **形式化**：$\mathcal{M}_t$ 结构不限；三算子 $F/E/R$ **未必每步调用**——"短/长期"从用法涌现，非架构分隔（全篇最重要的洞察）。
- **边界**：agent memory ⊋ LLM memory；vs RAG 看"静态查询 vs 持续演化"；vs context engineering 看"认知建模 vs 资源调度"（Fig 2）。
- **三大迁移（§7）**：检索→生成、手工→自治、启发式→RL；记忆要变成"与推理共进化的可学习子系统"。
- **认知连接（§7.8）**：与人类记忆结构同构（Atkinson-Shiffrin + Tulving），但缺"建构性重构"——未来引 sleep-like 巩固 + 主动遗忘。
- **诚实**：概念框架、无实证排名；归位示意性；三轴正交是假设；遗忘必要；自治检索有"沉默失败"。
- **对我们（Θ3）**：我们=纯 token-Flat(1D)、事实记忆为主、compaction=多轮工作记忆、**演化层几乎空、检索是全量读入**。下一步：给 MEMORY.md 加**演化层（巩固/更新/遗忘）+ 检索时机探针**，用 Harness-Bench 五类失败标注测 state/continuation 与 contract 两类是否下降；有效再从 1D 升 2D（对标 A-MEM）。
