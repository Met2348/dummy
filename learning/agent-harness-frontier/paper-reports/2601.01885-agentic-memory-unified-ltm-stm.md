# Agentic Memory：为 LLM agent 学习统一的长期/短期记忆管理

> **本篇导读**：这是 agent-harness 库 **D 组（记忆）** 的一篇前沿工作，主打 harness 的 **C（Context / 上下文）层**——
> agent 能读到、记住、忘掉什么，直接决定它在长程任务上的天花板。AgeMem 的核心主张一句话：
> **别再把长期记忆（LTM）和短期记忆（STM）当成两套硬编码的外挂子系统，而是把二者的每一个操作都做成 agent 可调用的工具动作，
> 再用强化学习让 agent 自己学会统一调度。** 本报告严格对齐库内标杆（Harness-Bench）的密度：
> 每个公式前给直觉、先定义符号；指标给定义式；数字标 §/Table/Eq/Fig 出处；区分「论文宣称」与「批判」；
> 强制 `## ★ 对我们的启发`，并把 Θ3 打到我们自己的 harness（MEMORY.md 长期 + 会话短期）上。

## §1　TL;DR（一页讲清这篇在干嘛）

> 主讲提示：开场先讲清「记忆是 agent 的隐形上下文墙」，再点出这篇把记忆从『外挂子系统』升格为『agent 的一等公民动作』。

一句话：**LLM agent 的长程能力被有限上下文窗口卡死；缓解办法是管好记忆，但过去 LTM 和 STM 各自为战、靠启发式规则或外挂专家模型硬编码。AgeMem（Agentic Memory）把 LTM 与 STM 的所有操作都暴露成工具动作**（LTM 用 `Add/Update/Delete`、STM 用 `Retrieve/Summary/Filter`，见 §3.2 Table 1），**让 agent 通过一个统一的工具接口自主决定「何时、存什么、怎么管」记忆；再用「三阶段渐进 RL + 步级 GRPO」把这套统一记忆行为端到端训出来**（§3.3–3.4）。在 5 个长程基准上，AgeMem 在 Qwen2.5-7B / Qwen3-4B 两个底座上平均分分别到 **41.96 / 54.31**（Table 2），相对无记忆基线相对提升 **49.59% / 23.52%**（§4.2）；记忆质量分（MQ）也拿到 **0.533 / 0.605**（Fig 2）。

三条带走的结论：

- **属于 harness 的哪一层（Θ1）**：本篇打的是 **C（Context/上下文）层**——它管的是「agent 在每一步能观察到什么、把什么持久化、把什么从活动上下文里清掉」。它对 **T（Tools）层**有强依赖（记忆操作本身以工具形态实现），对 **L（Loop/控制循环）层**有强耦合（训练靠一个 ReAct 式 think→tool_call→answer 循环，见附录 A.1 系统提示），并且用 **RL** 把记忆策略和任务策略绑成同一个学习问题。
- **回扣全库论点（Θ2）**：本篇是 `Agent = Model + Harness` 在**记忆维度**上的一记实证。同一个底座模型（Qwen2.5-7B），只是把「记忆当外挂启发式」换成「记忆当可学习工具动作」，平均分从 33.43（AgeMem-noRL）抬到 41.96（AgeMem），**RL 训练一项就贡献 +8.53 个百分点**（§4.2）。模型没换，换的是 harness 的记忆层与其训练方式。
- **够新够权威（Θ4）**：2026-04 预印本，阿里巴巴通义团队 + 武大出品。它相对本组基石（MemGPT 把记忆当函数调用、A-Mem 把记忆当动作）推进了两步——**(1) 从「只优化记忆的某一侧」升级到「LTM+STM 联合、端到端优化」；(2) 不再依赖外挂专家 LLM 做记忆控制器，而是把记忆策略直接内化进 agent 自己的 policy**（§2 "Positioning relative to RL-based memory agents"）。

## §2　问题与动机：为什么「统一管理 LTM 与 STM」值得单独做

> 主讲提示：这一页用 Why 三连的「问题层」。要讲清「记忆碎片化」这个盲区，以及三个具体挑战 C1/C2/C3。

**Why（问题层）——不解决会卡住什么？**
在多步推理、长工作流的 agentic 任务里，agent 的表现「从根本上受限于它在任一时刻能注意到的信息量」，作者把这统称为 agent 的 **记忆（memory）**（§1 原文）。记忆分两类：**长期记忆（LTM, long-term memory）**——持久保存用户/任务特定知识；**短期记忆（STM, short-term memory）**——当前输入上下文里的信息。高质量的 LTM 支撑对累积知识的高效检索，有效的 STM 管理则减少冗余、保住关键上下文；二者共同缓解有限上下文窗口的限制，所以**联合管理二者对提升复杂推理下的 agent 表现至关重要**（§1）。

**盲区在哪：LTM 与 STM 被当成「相互独立、松耦合的模块」。** 现有研究几乎都把两者分开处理（§1）：

- **STM 侧**：主流是 RAG（检索增强生成），如 MainRAG、ReSum，通过外部检索或周期性摘要扩展可用上下文。问题是它们**重度依赖预设的时间表（schedule）或启发式规则**，可能让不频繁但关键的细节被漏掉，同时引入无关噪声（§1、§2 STM 段）。
- **LTM 侧**：沿两条独立路线发展——**trigger-based（触发式）** 在预设时刻执行固定记忆操作；**agent-based（智能体式）** 引入一个专门的记忆管理器决定存什么。但后者大多**仍依赖手工规则或外挂专家模型**，牺牲适应性、增加系统复杂度（§2 LTM 段，引 Xiong et al. 2025）。

**后果**：LTM 与 STM 于是被当成**分离、松耦合的模块**（§1）。如 Figure 1 所示，现有架构一般是两种模式：**(a) 静态 STM + 触发式 LTM**，或 **(b) 静态 STM + 智能体式 LTM**。两种模式里，两套记忆系统**各自独立优化、事后临时拼装**，导致记忆构建碎片化、在长程推理任务上次优（§1 末）。

**三个具体挑战（§2 中段，作者明确编号 C1/C2/C3）**：

| 编号 | 挑战 | 具体是什么 | 为什么难 |
|---|---|---|---|
| **C1** | **功能异质性协调** | LTM 决定「存/更新/丢弃什么」，STM 管「检索/摘要/保留活动上下文里什么」——两者目的互补但异质 | 要设计一个**统一机制**去协调二者的相互作用，而不是各调各的 |
| **C2** | **训练范式错配** | LTM 训练常用「交互前就有的会话级信息」；STM 训练常靠「注入干扰项模拟长程」；而标准 RL 假设**连续轨迹 + 稳定奖励** | 记忆操作产生的轨迹**天然碎片化、奖励稀疏且不连续**，端到端优化很难 |
| **C3** | **实际部署约束** | 很多系统靠一个**外挂专家 LLM** 做记忆控制 | 显著抬高推理成本与训练复杂度；如何**不依赖外部专家**把统一记忆管理直接做进 agent，仍是开放问题 |

> **读出什么**：这篇的动机不是「再造一个更强的检索器」，而是「把记忆从 agent 外面的一堆启发式脚手架，搬进 agent 自己的决策里」。三挑战恰好对应它三样东西：C1→统一工具接口（§3.2）；C2→三阶段渐进 RL + 步级 GRPO（§3.3–3.4）；C3→无外挂专家、记忆策略即 agent 自身 policy（§3.1）。

## §3　核心 intention 与三个贡献

**一句话形式化 intention**：把「记忆使用」建模成**一个带延迟监督的、单一可学习的控制问题**——用一个统一 policy 覆盖异质的记忆动作（既含持久的 LTM 操作，也含上下文的 STM 操作），端到端训练，使**存储、检索、过滤、摘要的决策都朝着同一个终局任务奖励对齐**（§2 末 "Positioning" 段原文）。

**三个贡献（§1 末，逐条抄要点）**：

1. **统一的 agentic 记忆框架**：让 LLM agent 能自主决定 **when / what / how** 去管理 LTM 与 STM（把记忆当工具动作）。
2. **三阶段渐进 RL 策略 + 步级 GRPO**：促成「统一记忆管理行为」的有效端到端学习。
3. **全面评测**：跨多个模型、多个长程基准，验证 AgeMem 在复杂 agentic 任务上的鲁棒与有效。

> 主讲提示：把三贡献对回三挑战——贡献 1 解 C1（异质协调）、贡献 2 解 C2（训练范式错配）、无外挂专家这条隐含解 C3。这是一份「问题→方案」严丝合缝的论文，答辩时可用这条主线串全场。

## §4　相关工作定位：它站在谁肩上、跟谁不同

> 主讲提示：这页做一张对比表，把 AgeMem 放进「记忆即动作」这条谱系里。重点是它相对 MemGPT / A-Mem / Memory-R1 的增量。

**Why（设计层预热）——「把记忆当动作」不是 AgeMem 首创，那它新在哪？**
作者自己承认（§2 "Positioning relative to RL-based memory agents"）：近期工作（如 Memory-as-Action、Memory-R1）**已经把记忆操作建模成动作并用 RL 优化**，与本文「表面相似」。但这些方法**通常一次只优化记忆的某一个方面**（storage 或 retrieval 或 summarization 之一），而把其余当固定启发式或单独调的模块。**后果**：早期的存储决策和后期的推理行为**只是松耦合，学习信号并不显式把它们连起来**。AgeMem 的差异就在这一句——**把 LTM 操作 + STM 操作放进同一个动作空间、用同一个终局任务奖励端到端训练**。

| 维度 | trigger-based LTM<br/>(MemoryBank, RAP…) | agent-based LTM<br/>(A-Mem, Mem0, MIRIX…) | 单侧 RL 记忆<br/>(Memory-R1, Memory-as-Action) | **AgeMem（本文）** |
|---|---|---|---|---|
| LTM 决策 | 预设时刻固定执行 | 专用记忆管理器 | 用 RL，但**只覆盖某一侧** | **工具动作，RL 学** |
| STM 决策 | 静态 / RAG | 静态 / RAG | 常固定为启发式 | **工具动作，RL 学** |
| LTM×STM | **分离、事后拼** | **分离** | **松耦合** | **统一动作空间、联合优化** |
| 是否需外挂专家 | 否（但靠规则） | **常需** | 视实现 | **否（策略即 agent 自身）** |
| 优化目标 | 无端到端 | 无端到端 | 部分端到端 | **终局任务奖励端到端** |

作者同时把自身放进两条既有脉络：
- **LTM 谱系**：LangMem（模块化多类型记忆）、A-Mem（Zettelkasten 式链接结构化知识单元）、Mem0（可扩展 extract-update 流水线 + 图变体）、Zep（时序知识图）——这些**擅长组织/检索，但依赖预定义结构或启发式更新规则**；记忆一大，系统复杂度飙升、缺自适应的学习式优先级与遗忘（§2 LTM 段）。
- **RL for LLM 谱系**：GRPO（DeepSeekMath 提出）通过按采样轨迹的相对质量来优化 policy、**免掉显式价值函数**，稳定性好；其变体在复杂推理上表现强。但现有 RL 系统**普遍把记忆当静态或外部组件**，不适配记忆操作带来的不连续、碎片化轨迹（§2 RL 段）——这正是 AgeMem 要把 RL 直接嵌进记忆管理过程的理由。

## §5　方法总览（big picture）：一图看懂 AgeMem

> 主讲提示：先给直觉——把一次训练想成「让 agent 在一个干净沙箱里，先攒长期记忆、再顶着噪声管短期上下文、最后靠两者联合把任务做完，全程录像，用最终成败倒推每一步记忆决策的功过」。先不上数学。

**三块拼图**：
1. **统一工具接口（§3.2）**：6 个记忆工具进入动作空间——LTM 三件套 `Add/Update/Delete`，STM 三件套 `Retrieve/Summary/Filter`。agent 在 ReAct 循环里像调普通工具一样调它们。
2. **三阶段渐进课程（§3.3）**：一条完整轨迹被切成三段——Stage 1 攒 LTM、Stage 2 顶着干扰管 STM、Stage 3 联合推理做题。LTM 跨三阶段持续存在，STM 在阶段间被重置以防信息泄漏。
3. **步级 GRPO（§3.4）**：只有任务做完才有终局奖励（中间步奖励通常为 0），于是把这个终局优势**广播**回轨迹里每一步（含 Stage 1/2 的记忆决策），实现跨异质阶段的长程信用分配。

## §7　方法之一：把记忆做成「工具动作」——统一 RL 表述与工具接口

> 主讲提示：这页是全篇的「地基」。要把「记忆操作 = 动作空间里的一等公民」讲透，再讲清 6 个工具各干什么。公式前先给直觉。

### 7.1 统一 RL 表述（§3.1）

**直觉**：传统做法把记忆管理器搁在 agent 外面，agent 只管说话，一个规则/专家模型在旁边决定存不存。AgeMem 反过来——**把记忆操作塞进 agent 自己的动作空间**，于是「说下一句话」和「把这条信息存进 LTM」在形式上是**同一类决策**，都由 policy $\pi_\theta$ 产出。

**状态**：每步 agent 观察到 $s_t=(C_t,\mathcal{M}_t,\mathcal{T})$——把短期上下文、长期存储、任务规格三者打包。这让 agent 的决策**同时扎根于瞬时上下文与持久知识**（§3.1）。

**动作与轨迹奖励**。先给直觉：一条轨迹的总回报，应当既奖「任务与记忆质量都好」，又罚「冗余存储 / 过度用工具 / 上下文无节制膨胀」。

记号（先定义）：$w_i$ 为各奖励分量的权重；$R_i(\tau)$ 为第 $i$ 个奖励分量（覆盖任务表现与记忆质量）；$P_{\text{penalty}}(\tau)$ 为惩罚项。

$$R(\tau) \;=\; \sum_i w_i \cdot R_i(\tau) \;+\; P_{\text{penalty}}(\tau) \tag{1}$$

读出什么：这是一个**加权和 + 惩罚**的复合奖励；$R_i$ 把「做没做完 + 记忆好不好」计进去，$P_{\text{penalty}}$ 专门压「冗余存、乱用工具、上下文爆」。（分量细节见 §9 的 Eq.7 展开。）

**优化目标**：

$$\theta^{*}\;=\;\arg\max_{\theta}\;\mathbb{E}_{\tau\sim\pi_\theta}[R(\tau)] \tag{2}$$

读出什么：找一组 LLM 参数，使**在自己策略诱导的轨迹分布下、期望复合奖励最大**。关键点——**记忆管理被当成 agent policy 的内在组成，用可学习机制替代手工启发式**（§3.1 原文）。

**经验元组与步级信用**：每步收集 $e_t=(s_t,a_t,r_t,\log\pi_{\theta_{\text{old}}}(a_t\mid s_t))$，其中 $r_t$ 中间步通常为 0、任务完成后才赋值；$\log\pi_{\theta_{\text{old}}}$ 是旧策略下该动作的对数概率。这个表示**支持 GRPO 下的步级信用分配**，让 agent 能把长期奖励**归因到跨阶段的具体记忆决策**（§3.1 末）。

> **Why（设计层）——为什么值得把记忆塞进动作空间，而不是留个外挂管理器？**
> 朴素做法 A：保留一个独立记忆管理器（agent-based LTM）。→ 会撞上 C3——**多一个外挂专家 LLM，推理/训练成本翻倍**，且它的决策与 agent 的推理**并不共享同一个学习信号**，早期存储和后期推理仍脱节。
> 朴素做法 B：给记忆写死一套触发规则（trigger-based）。→ 撞上 C1——**死板**，关键但低频的信息容易被漏，噪声容易被塞进来。
> AgeMem 改成「记忆即动作 + 端到端 RL」：让**同一个 policy、同一个终局奖励**同时管说话与管记忆，早期存储决策因此能被后期任务成败反向监督（这正是 §8 步级 GRPO 的用武之地）。**代价**：动作空间变异质、轨迹变碎片、奖励变稀疏，训练显著变复杂——这份复杂度就是它接下来要用「三阶段课程 + 步级 GRPO」偿还的账。

### 7.2 六个记忆工具（§3.2，Table 1）

**为什么用「工具接口」而非「隐式记忆」（设计层）**：把记忆操作做成**显式工具**，等于把记忆控制从「外部启发式流水线」变成「决策的内在组件」，让 agent 能**按任务结构、历史、上下文自适应地管理记忆**（§3.2 原文）。代价是要设计一套 agent 学得会、又表达力够的工具集。

| 工具 | 目标 | 功能 | 形式化（附录 A.1） |
|---|---|---|---|
| **ADD** | LTM | 把新知识写进 $\mathcal{M}_t$ | $m_{\text{new}}=(c,\text{enc}(c),\text{metadata})$；$\mathcal{M}_{t+1}=\mathcal{M}_t\cup\{m_{\text{new}}\}$（Eq.10–11） |
| **UPDATE** | LTM | 用 `memory_id` 修改已有条目 | $m_i\leftarrow(c',\text{enc}(c'),\text{metadata}')$（Eq.12） |
| **DELETE** | LTM | 删除过时/错误条目 | $\mathcal{M}_{t+1}=\mathcal{M}_t\setminus\{m_i\}$（Eq.13） |
| **RETRIEVE** | STM | 把 $\mathcal{M}_t$ 里 top-$k$ 相关记忆拉进 $C_t$ | $\text{RETRIEVE}(q,k)=\text{TopK}(\mathcal{M}_t,\text{sim}(q,m_i),k)$（Eq.8） |
| **SUMMARY** | STM | 把一段交互历史压成简洁表示，缩上下文、留要点 | $C_t'=C_t\setminus\{u_i|i\in s\}\cup\{\text{Summarize}(\{u_i\}_{i\in s})\}$（Eq.14） |
| **FILTER** | STM | 删掉与某准则语义相似度超阈的上下文消息，抑噪 | $C_t'=\{u_i\in C_t\mid \text{sim}(c,u_i)<\theta_f\}$（Eq.15） |

**RETRIEVE 的相似度定义式**（先给直觉：要「按语义近似」把最相关的旧记忆调进来，用余弦相似度度量 query 与记忆的贴近程度）：

记号：$\text{enc}(q)$、$\text{enc}(m_i)$ 分别是 query 与第 $i$ 条记忆的嵌入向量；$\|\cdot\|$ 为 L2 范数。

$$\text{sim}(q,m_i)\;=\;\frac{\text{enc}(q)^{\top}\text{enc}(m_i)}{\|\text{enc}(q)\|\,\|\text{enc}(m_i)\|} \tag{9}$$

读出什么：取值 $[-1,1]$，越大越相关；$\text{TopK}$ 取最相关的 $k$ 条（实验里 $k$ 典型设 3–5，A.1）。

**SUMMARY 的 span 参数**：可为 `"all"`（摘所有非系统消息）或 `"N"`（摘最近 N 条）；摘要由一个专门系统提示驱动的 LLM 完成，要求「压缩但不丢关键信息/意图/决策/未决问题」（A.1 给了完整 prompt）。**agent 学的是在上下文溢出前就主动摘要**，平衡信息保全与效率。

**FILTER 的阈值**：$\theta_f$ 默认 0.6（A.1）；准则 $c$ 可由 agent 指定（如"要保留什么"的描述）或从当前任务上下文自动导出。该操作在 Stage 2 尤其关键——那里正故意注入干扰项来测 agent 的过滤能力。

> **读出什么**：这 6 个工具把「记忆生命周期」（写入→维护→调入→压缩→抗噪）拆成了 agent 可组合调用的原子动作。注意一个**接口不对称**：LTM 三件套改的是**持久存储 $\mathcal{M}_t$**，STM 三件套改的是**活动上下文 $C_t$**——这与我们自己 harness 里「MEMORY.md（持久）vs 会话上下文（临时）」的二分几乎同构（见 §16 Inspires-Us）。

## §8　方法之二：三阶段渐进 RL 课程（§3.3）

> 主讲提示：这页解释「为什么不能一上来就整段任务硬 RL」。核心 Why——记忆的价值有时间差：先攒的记忆，要到很后面做题才见效。课程就是为「制造并利用这个时间差」而设计。

### 8.1 一条轨迹被切成三段（Eq.3–4）

对每个任务实例 $q\in\mathcal{T}$，agent 生成一条完整轨迹，含 $K$ 条独立 rollout：

$$\tau_k^{(q)}=\big(\tau^{(1)},\tau^{(2)},\tau^{(3)}\big),\quad k=1,\dots,K \tag{3}$$

其中每个子轨迹 $\tau_k^{(i)}$ 对应一个训练阶段；三段拼成完整轨迹（$e_j$ 为第 $j$ 步经验元组）：

$$\tau_k^{(q)}=(e_1,e_2,\dots,e_T),\quad T=T_1+T_2+T_3 \tag{4}$$

**关键设计**：**长期记忆 $\mathcal{M}_t$ 跨三阶段持续存在**，让 Stage 1 早早存下的知识能影响后续决策；但**短期上下文 $C_t$ 在 Stage 1→2 之间被重置**，防止跨阶段信息泄漏（§3.1、§3.3 反复强调）。Stage 2 前的这次重置尤其重要——它**确保 agent 不能靠残留上下文把最终任务糊弄过去，从而逼它真正从 LTM 检索、逼出对记忆操作的有效训练**（§3.1 原文）。

### 8.2 三个阶段各干什么、各练什么

| 阶段 | 场景 | agent 要学什么 | 子轨迹 |
|---|---|---|---|
| **Stage 1：LTM 构建** | 随意对话，agent 被逐步暴露给上下文信息 $I_q$；$C_t$ 自然演化，可按需调 LTM 工具 | **从闲聊里识别值得存的显著信息并写进 $\mathcal{M}_t$**（此时任务 query 还没出现，只能靠通用线索判断"什么以后可能有用"，逼出结构良好、非 query 定制的记忆痕迹） | $\tau_k^{(1)}=\{e_t\}_{t=1}^{T_1}$ |
| **Stage 2：STM 抗干扰** | **重置 $C_t$、保留 $\mathcal{M}_t$**；给 agent 灌 **distractor messages**——像模像样、却故意与目标 query 无关的话 | **学会主动 STM 控制**：用 FILTER 过滤、用 SUMMARY 压缩，把噪声挡掉、把有用信息留住（而不是被干扰牵着走） | $\tau_k^{(2)}=\{e_t\}_{t=T_1+1}^{T_1+T_2}$ |
| **Stage 3：联合推理** | 给出正式 query $q$，需要**准确推理 + 有效记忆检索** | **协调**：从 $\mathcal{M}_t$ 检索相关知识、恰当管理 $C_t$、端到端生成最终答案；评估 agent「统一调度 LTM+STM+解题」的能力 | $\tau_k^{(3)}=\{e_t\}_{t=T_1+T_2+1}^{T} $ |

> **Why（设计层）——为什么非要拆三段、非要在中间重置 STM？**
> 朴素做法：拿整段长任务直接 RL。→ 撞 C2——奖励极稀疏（只有最后成/败一个信号），而记忆决策发生在很早，中间又被记忆操作切得七零八落，**信用分配几乎无从下手**，训练不稳。
> 朴素做法 2：让 STM 上下文一路带到底。→ agent 会**投机取巧**，用 Stage 1 残留在上下文里的信息直接答题，根本不去练"从 LTM 检索"这件正事，记忆操作学不出来。
> AgeMem 的三阶段 + 中间重置，本质是**人为制造「信息暴露」与「任务执行」的时间分离**：先只给暴露（Stage 1）、再施加上下文压力（Stage 2）、最后才给可评分的下游任务（Stage 3）。这样「早期存储/过滤决策到底有没有用」才能**在延迟结果下被评估**（§3.3 "Generalizability" 段原文）。

### 8.3 课程的普适性（§3.3 "Generalizability of the three-stage curriculum"）

作者特意声明：**这套三阶段不绑定 QA 式监督**，它只要求「信息暴露」与「任务执行」在时间上分离，好让记忆决策的用处能在延迟结果下评估——
- **Stage 1** 只需要「任务前的任意上下文」供 agent 选择性存储，可以是环境描述、检索文档、历史对话，**不限于 QA supporting facts**；
- **Stage 2** 的干扰项由 `DISTRACTORGEN`（附录 A.3）**合成生成，不需数据标注**；
- **Stage 3** 提供下游目标，其延迟奖励决定早期存储/过滤是否有用。

> **读出什么**：这段是作者给「可迁移性」上的保险——课程依赖的是**信息时序 + 延迟信用分配**，而非 QA 特有的事实标注。所以理论上换任何「先攒信息、后用信息」的长程任务都能套。（不过实测里三阶段轨迹**仅取自 HotpotQA** 训练集，这条普适性主要是论证性的——见 §14 局限。）

## §9　方法之三：步级 GRPO 与奖励函数设计（§3.4–3.5）

> 主讲提示：这是全篇最该讲透的两页。步级 GRPO 解决「稀疏+不连续奖励下的长程信用分配」；奖励函数把「任务/上下文/记忆」三种质量拧成一个可优化标量。每个公式前给直觉、先定义符号。

### 9.1 步级 GRPO：把「终局优势」广播回每一步

**直觉**：记忆决策的功过要等任务做完才知道，中间步没有可靠的即时奖励。GRPO 的好处是**不需要显式价值函数**——它用「同一任务的一组并行 rollout 里，谁比平均好/差」来当优势信号。AgeMem 的关键改造是：**只在终局步算优势，然后把它原样广播给整条轨迹的每一步**（含 Stage 1/2 的记忆决策），从而给早期决策也发一个一致的学习信号。

**第一步：组归一化的终局优势**（Eq.5）。先定义符号：$G_q$ 是任务 $q$ 的 $K$ 条并行 rollout；$r_T^{(k,q)}=R(\tau_k^{(q)})$ 是第 $k$ 条轨迹的终局奖励；$\mu_{G_q},\sigma_{G_q}$ 是这组内奖励的均值与标准差；$\epsilon$ 防除零。

$$A_T^{(k,q)}\;=\;\frac{r_T^{(k,q)}-\mu_{G_q}}{\sigma_{G_q}+\epsilon} \tag{5}$$

读出什么：某条 rollout 的终局奖励**比同组平均高多少个标准差**，就是它终局步的优势。这是标准 GRPO 的"相对好坏"归一化——**免掉了 critic/价值网络**。

**第二步：广播（broadcast）到所有先前步**（§3.4 原文）：

$$A_t^{(k,q)}\;=\;A_T^{(k,q)},\qquad \forall t \le T$$

读出什么：**同一条轨迹里每一步的优势都等于其终局优势**。这一步是 AgeMem 的灵魂——它把「最终任务成败」当作监督信号，**监督了轨迹里每一个中间记忆决策，包括 Stage 1 的存、Stage 2 的过滤**，实现跨异质阶段的长程信用分配。

**第三步：带 KL 正则的 GRPO 目标**（Eq.6）。先给直觉：在旧策略采的样本上，按优势方向更新新策略，但用重要性比 $\rho_t$ 控制更新幅度，并用 KL 把新策略拽在参考策略附近以稳住训练。

记号：$\rho_t^{(k,q)}=\dfrac{\pi_\theta(a_t\mid s_t)}{\pi_{\theta_{\text{old}}}(a_t\mid s_t)}$ 是重要性比；$D_{\text{KL}}^{(k,q)}$ 是当前 policy $\pi_\theta$ 与固定参考 $\pi_{\text{ref}}$ 的 KL 散度；$\beta$ 是平衡探索与稳定的系数；$\mathcal{E}=\bigcup_{q,k}\{(e_t,A_t)\mid e_t\in\tau_k^{(q)}\}$ 是聚合了优势的全体经验集，$|\mathcal{E}|=B\times K\times \bar{T}$。

$$
J(\theta)=\mathbb{E}_{(e_t,A_t)\sim\mathcal{E}}\big[\rho_t A_t-\beta D_{\text{KL}}[\pi_\theta\|\pi_{\text{ref}}]\big]
=\frac{1}{|\mathcal{E}|}\sum_{q=1}^{B}\sum_{k=1}^{K}\sum_{t=1}^{T_k^{(q)}}\big[\rho_t^{(k,q)}A_t^{(k,q)}-\beta D_{\text{KL}}^{(k,q)}\big] \tag{6}
$$

读出什么：**对所有任务、所有 rollout、所有时间步**求平均后最大化。因为每步的 $A_t$ 都来自终局，**Stage 1 的一次"该不该存"、Stage 2 的一次"该不该过滤"，都会被最终任务奖励拉着朝正确方向更新**——这正是它声称能端到端联合优化 LTM+STM 的机制所在。

> **Why（设计层）——为什么用「广播终局优势」而不是给中间步逐个塑形奖励（reward shaping）？**
> 朴素做法：给每步记忆操作设计一个即时奖励（比如"存了高质量记忆 +0.1"）。→ 需要**大量手工设计的中间奖励**，容易奖到"看起来对但对终局没用"的行为，且和真实任务目标未必对齐（reward hacking 风险）。
> AgeMem 改用"只在终局评分、把优势广播回全程"：**唯一的真理来源是任务成败**，所有中间决策的好坏都由它隐式定义。代价是信号更稀疏、方差更高——但这恰是 GRPO 的组内归一化（Eq.5）与 KL 正则（Eq.6）要压住的东西。

### 9.2 复合奖励函数（§3.5，附录 A.2 给可实现公式）

**总奖励**（Eq.7 / Eq.16）。直觉：一个好 agent 要**同时**做到「任务做对、上下文管得省而全、记忆存得精而准」，任一维垮掉都该扣分。先定义符号：$\mathbf{w}=[w_{\text{task}},w_{\text{context}},w_{\text{memory}}]^\top$ 为可调权重；$\mathbf{R}=[R_{\text{task}},R_{\text{context}},R_{\text{memory}}]^\top$ 为三类奖励；$P_{\text{penalty}}$ 惩罚上下文溢出/超交互轮数等违规。

$$R(\tau)\;=\;\mathbf{w}^\top\mathbf{R}\;+\;P_{\text{penalty}} \tag{7 / 16}$$

**(A) 任务完成奖励 $R_{\text{task}}$**（Eq.17，主学习信号，dominant 分量）。用一个 LLM judge 打分 $S_{\text{judge}}(A_{\text{pred}},A_q)\in[0,1]$；没产出答案则罚 $P_{\text{no-answer}}$（默认 $-1.0$）：

$$R_{\text{task}}=\begin{cases}S_{\text{judge}}(A_{\text{pred}},A_q),&\text{若有答案}\\[2pt]P_{\text{no-answer}},&\text{否则}\end{cases}$$

**(B) 上下文管理奖励 $R_{\text{context}}$**（Eq.18–21）——评 STM 行为。三分量等权（$\alpha_i=1/3$）：

$$R_{\text{context}}=\sum_{i=1}^{3}\alpha_i R_i,\qquad R_i\in\{R_{\text{compression}},R_{\text{preventive}},R_{\text{preservation}}\}$$

- **压缩效率**（Eq.19）：$T_{\text{used}}$ = 生成最终答案时上下文里的 token 数，$T_{\max}$ = 允许预算。
$$R_{\text{compression}}=\max\!\Big(0,\;1-\frac{T_{\text{used}}}{T_{\max}}\Big)$$
读出：越省 token 越高，鼓励经济用上下文。
- **预防性管理**（Eq.20）：$\mathbb{1}[\text{tool invoked before overflow}]$——在触顶前就调压缩类工具则得 1，奖「主动、别等爆了才管」。
- **信息保全**（Eq.21）：$\mathbb{1}_{\text{preserve}}$——从 query 抽出的关键 token/短语 $K_q$（命名实体、时空表达等）在答题时是否仍在场（直接在或经摘要保留），罚「把关键信息压没了」。

> **Why（设计层）**：STM 的两个失败方向是"压得太狠丢信息"和"压得太松上下文爆"。$R_{\text{compression}}$ 拉一头、$R_{\text{preservation}}$ 拉另一头，$R_{\text{preventive}}$ 奖时机——三者把 STM 治理框在"省而全、且主动"的区间里。

**(C) 记忆管理奖励 $R_{\text{memory}}$**（Eq.22–25）——评 LTM 操作。三分量等权（$\beta_j=1/3$）：

$$R_{\text{memory}}=\sum_{j=1}^{3}\beta_j R_j,\qquad R_j\in\{R_{\text{storage}},R_{\text{maintenance}},R_{\text{relevance}}\}$$

- **存储质量**（Eq.23）：$N_{\text{total}}$ 存入条目数，其中 $N_{\text{high\_quality}}$ 被 LLM 依 query $q$ 与期望答案 $A_q$ 判为高质量。
$$R_{\text{storage}}=\frac{N_{\text{high\_quality}}}{\max(1,N_{\text{total}})}$$
读出：高质量条目占比——奖「存有用的」、罚「囤冗余/低质」。
- **维护**（Eq.24）：$\mathbb{1}[\text{update or delete performed}]$——奖做了有意义的更新/删除以抗记忆陈旧。
- **语义相关性**（Eq.25）：$R_{\text{relevance}}=S_{\text{LLM}}(\mathcal{R},q)$，即检索到的记忆集 $\mathcal{R}$ 相对 query $q$ 的 LLM 语义相关分（归一化到 $[0,1]$）。

**(D) 惩罚项 $P_{\text{penalty}}$**（Eq.26）：$N_{\text{rounds}}$ 交互轮数、$N_{\max}$ 上限；$T_{\text{used}}$ 用量、$T_{\max}$ 预算。

$$P_{\text{penalty}}=\sum_{k=1}^{2}P_k\cdot\mathbb{1}[\text{violation}_k],\quad \text{violation}\in\{\mathbb{1}[N_{\text{rounds}}>N_{\max}],\ \mathbb{1}[T_{\text{used}}>T_{\max}]\}$$

默认 $P_{\text{rounds}}=-1$、$P_{\text{overflow}}=-0.5$——违规导致**终局奖励显著下降**，逼 agent 守住安全与效率边界。

> **读出什么（关键机制串联）**：这套奖励的精妙在于——**它给了 §7.1 那个抽象的 $R_i$ 具体内容，又只在终局评分**，于是通过 §9.1 的广播，Stage 1 存的一条记忆好不好（$R_{\text{storage}}$）、Stage 2 有没有守住上下文（$R_{\text{preservation}}$）、Stage 3 检索准不准（$R_{\text{relevance}}$），**全部被折进同一个终局标量、反向监督对应阶段的动作**。这就是「LTM+STM 联合、端到端优化」落到实处的地方。

### 9.3 训练算法（附录 A.3，Algorithm 1–5）

**主循环（Alg 1–2）**：generate-then-optimize。对 batch $\mathcal{B}$ 里每个任务 $q$，用 `DISTRACTORGEN(q)` 造干扰项 $M_{\text{dis}}$；跑 $K$ 条 rollout，每条依次执行 `STAGE1→STAGE2→STAGE3`（Alg 3/4/5），拼成 $\tau_k^{(q)}=\tau^{(1)}\oplus\tau^{(2)}\oplus\tau^{(3)}$，收进统一 buffer $\mathcal{E}$。随后**组归一化优势**（Eq.5）、**广播**到全轨迹每步（Alg 2 第 27–29 行显式写了 `A_t ← A_T // Broadcast`），最后按 Eq.6 做 KL 正则的梯度上升更新 $\theta$，并令 $\theta_{\text{old}}\leftarrow\theta$。

**三个 stage 的 rollout（Alg 3–5）**关键差异：
- **Stage 1**（Alg 3）：从 $I_q$ 采消息 $m_t$，每步先 `RETRIEVE(M, m_t, k) ∪ m_t` 更新 $\mathcal{M}_{\text{ret}}$（作为一种**内省式**检索，让 agent 时刻知道 LTM 现状、便于决定 update/delete），再从 $\pi_\theta$ 采动作、执行记忆工具、记 $e_t^{(1)}$；出答案即 break。
- **Stage 2**（Alg 4）：`C ← ∅` 重置，`M 从 Stage 1 保留`；每步注入 `M_dis[t]` 干扰、采动作、执行记忆工具。
- **Stage 3**（Alg 5）：`C ← C ∪ {q}` present query，`M 从 Stage 2 保留`；协调检索 + STM 管理 + 多步推理，出 $A_{\text{pred}}$。

> **读出什么**：算法把「$\mathcal{M}$ 跨阶段持续、$C$ 阶段间重置」这条设计**在伪代码里坐实**了（Alg 4 第 1 行注释 "M persists from Stage 1"、Alg 5 第 2 行 "C persists from Stage 2 and present query"）。这是它「统一但分阶段训练」的骨架。

## §10　实验设置：数据集 / 指标定义式 / baseline / 超参

> 主讲提示：这页把 setting 写全。重点是五个基准各用什么指标、指标的定义式、以及"只在 HotpotQA 上 RL 训练、直接迁移到其余四个"这条关键设定。

**数据集（§4.1）**——五个 LLM-agent 常用基准，覆盖具身行动、博弈式推理、知识密集问答：

| 数据集 | 类型 | 主指标 |
|---|---|---|
| ALFWorld (Shridhar 2020) | 具身交互 | Success Rate (SR) |
| SciWorld (Wang 2022) | 科学推理/具身 | Success Rate (SR) |
| PDDL (Chang 2024) | 规划 | Progress Rate (PR) |
| BabyAI (Chevalier-Boisvert 2018) | 网格指令跟随 | Success Rate (SR) |
| HotpotQA (Yang 2018) | 多跳知识问答 | LLM-as-a-Judge (J) |

**关键设定**：HotpotQA **同时含问题与 supporting facts**，天然提供 Stage 1 的上下文信息 $I_q$；因此 **AgeMem 只在 HotpotQA 训练集上做 RL 微调，然后直接在全部五个数据集上评测**（§4.1 原文，即"零样本跨域迁移"）。

**评测指标（给定义/含义）**：
- **Success Rate (SR)**：任务成功完成的比例（ALFWorld/SciWorld/BabyAI）。
- **Progress Rate (PR)**：任务进度完成度（PDDL）。
- **LLM-as-a-Judge (J)**：用 LLM 评判答案质量（HotpotQA），对应 $R_{\text{task}}$ 里的 $S_{\text{judge}}$。
- **Memory Quality (MQ)**：**评"存下的长期记忆"与 ground-truth facts 的相关性**——用 HotpotQA 提供的 ground-truth facts，由 LLM 评测器给存储记忆与这些事实的相关度打分（§4.2、Fig 2；prompt 见附录 C.2）。MQ 越高＝存的记忆越贴近真相、越可复用。

**baseline（§4.1）**——四个代表性 agent LTM 系统：**LangMem、A-Mem、Mem0、Mem0$^g$**（Mem0 官方图变体）；外加 **No-Memory**（无记忆下限）与 **AgeMem-noRL**（有工具、未经 RL 微调，用于剥离 RL 贡献）。STM 消融里，STM 工具对比 **RAG** 方法（"-RAG"变体：用 RAG 替代 STM 工具管理）。

**底座 & 实现（§4.1）**：base agent 用 **Qwen2.5-7B-Instruct** 与 **Qwen3-4B-Instruct**；agent 用 **Agentscope** 框架搭，AgeMem 用 **Trinity（Trinity-RFT）** 框架做 RL 微调。评委固定用 LLM judge（HotpotQA 的 MQ 与 J 均由 LLM 评测器给出，prompt 在附录 C.2）。

> **Why（设计层）——为什么敢只在 HotpotQA 上训、却报五个基准的分？**
> 这是把"三阶段课程的普适性主张（§8.3）"押上台面的赌注：如果 AgeMem 学到的真是**通用的记忆调度策略**（而非 HotpotQA 特有的答题捷径），那它就该零样本迁移到具身/规划任务上仍有效。§4.2 的跨域涨分就是这条赌注的兑现证据——**但也正因如此，"训练数据仅 HotpotQA"成了它最该被追问的外推性软肋**（§14）。

1. **AgeMem 拿双底座平均分第一**：Qwen2.5-7B **41.96**、Qwen3-4B **54.31**（§4.2）。相对 No-Memory 的**相对提升 49.59% / 23.52%**；相对最佳 baseline（Mem0/A-Mem）**平均高 4.82 / 8.57 个百分点**（§4.2）。

**STM 有效性（Figure 3）**：在 HotpotQA 上对比"用 STM 工具 vs 用 RAG（-RAG）"的平均 prompt token——AgeMem（用 STM 工具）比 AgeMem-RAG **省 token**：Qwen2.5 上 2117 vs 2186（**−3.1%**）、Qwen3 上 2191 vs 2310（**−5.1%**）。**读出**：学出来的 STM 管理工具确实**压住了上下文膨胀**，在保持任务表现的同时更省 token——即"学出来的上下文管理 > 静态 RAG"。

### 12.1 组件消融（Figure 4，Qwen2.5-7B）

逐件叠加（Base=No-memory；+LT=只 LTM 工具即 AgeMem-noRL-RAG；+LT/RL=RL 训 LTM 工具；+LT/ST/RL=完整 AgeMem）：

|---|---:|---:|---:|---:|---:|

### 12.4 工具用量的定性迁移（Table 3，RL 前后）

| 工具类 | Qwen2.5 noRL→GRPO | Qwen3 noRL→GRPO |
|---|---|---|
| ADD | 0.92 → **1.64** | 2.49 → 2.64 |
| UPDATE | 0.00 → 0.13 | 0.13 → 0.34 |
| DELETE | 0.00 → 0.08 | 0.00 → 0.22 |
| RETRIEVE | 2.31 → **1.95** | 4.62 → **4.35** |

## §13　核心概念抽象：这篇最有迁移价值的三件东西

> 主讲提示：把全篇压成三个可迁移抽象，供组会讨论时反复引用。

1. **「记忆即动作」的接口不对称（LTM↔$\mathcal{M}_t$ / STM↔$C_t$）**：LTM 工具改**持久存储**、STM 工具改**活动上下文**。这是把"记住 vs 注意"这对认知区分，落成两组作用在不同数据结构上的原子动作。
2. **「时序分离 + 延迟信用」的课程范式**：三阶段的本质不是 QA 特有流程，而是**人为制造"信息暴露→上下文压力→任务执行"的时间差，让早期记忆决策的价值可以在延迟结果下被评估**（§8.3）。这套范式可套到任何"先攒后用"的长程任务。
3. **「广播终局优势」的长程信用分配**：不给中间步塑形奖励，只在终局评分再把优势广播回全程（§9.1）。这是"用单一可验证的任务成败，隐式监督一长串异质中间决策"的通用配方——对我们自己的多步 agent 直接可借。

## §14　局限与批判（论文 Limitations + 我的补充）

**论文自陈的局限（诚实，见 Limitations 节）**：
- **工具集是固定的一小组**：`Add/Update/Delete/Retrieve/Summary/Filter` 这套抽象"清晰有效但可扩展到更细粒度的控制"——即目前是**粗粒度、写死的动作集**。
- **评测settings相对受控**：虽在 5 个长程基准上验证并展示零样本跨域迁移，但相较开放式真实部署仍**偏受控**；对**持久的、长期对话 / 真实用户交互**的评测还没做，作者把本研究定位为"受控条件下的跨域迁移基础"。
- **训练数据仅 HotpotQA**：三阶段轨迹**只取自 HotpotQA**；扩到"交互结构更丰富的其它数据源"会进一步拓宽适用性——**这是它自己点名的最大外推性风险**。

**我的补充批判**：
- **奖励里塞了太多 LLM-as-judge**：$R_{\text{task}}$($S_{\text{judge}}$)、$R_{\text{storage}}$(高质量判定)、$R_{\text{relevance}}$($S_{\text{LLM}}$)、MQ、以及 SUMMARY 本身都由 LLM 打分/执行。**用 LLM 当奖励又当摘要器，引入"裁判自身偏好"的系统偏差**，且"谁来 judge the judge"未解——这与本库 Harness-Bench 的"固定评委也是一个 harness 里的模型"是同一隐忧。若 judge 偏好某种冗长风格，$R_{\text{task}}$ 可能奖错方向。
- **"广播终局优势"的方差代价没量化**：把同一个 $A_T$ 发给整条含数十步、跨三阶段的轨迹，会给"其实无关紧要的中间步"也发同号信号（credit misassignment）。论文没报**方差/样本效率**层面的代价，也没和"逐步塑形"做对照消融。
- **6 个工具的"最小充分性"没证**：为什么恰好是这 6 个、少一个会怎样？除了 §12.1 的 LTM/STM/RL 三级消融，**没有逐工具的留一消融**（如去掉 UPDATE 或 DELETE 掉多少分），"这套动作集是不是冗余或有缺"无从判断。
- **零样本迁移的机制未拆**：ALFWorld/BabyAI 这类**具身**任务与 HotpotQA 差异极大，AgeMem 迁移过去仍涨分——但涨分到底来自"通用记忆策略"还是"更省 token/更稳的 ReAct 循环副作用"，**没有归因实验**能排除后者。

## ★ 对我们的启发（Inspires Us）

> 这一节是组会高潮，也是本库相对 auto-research 的独门优势：**我们（Claude Code / 本课 m9.* 的 agent）本身就是一个 harness**——
> 而且我们**恰好有一套长短期记忆二分**：`MEMORY.md`（跨会话持久，≈LTM）+ 当前会话上下文（临时，≈STM）。AgeMem 把这对东西"暴露成工具动作 + RL 统一训练"的路子，几乎是照着我们的结构写的。下面每条都落到具体组件。

➤ **a. 可直接借用的招（method/trick we can reuse）**：
 1. **「记忆即工具动作」的接口不对称**（§7.2）——把"记什么进 MEMORY.md（LTM 的 Add/Update/Delete）"与"当前上下文调入/摘要/过滤什么（STM 的 Retrieve/Summary/Filter）"**做成两组显式工具**，作用在两个不同数据结构上。我们现在 MEMORY.md 的写入基本靠约定和手工，可以直接抄这套六动作接口。
 2. **复合奖励的三分解 + 惩罚**（§9.2）——`R = w·[R_task, R_context, R_memory] + P_penalty`，其中 $R_{\text{storage}}$=高质量记忆占比、$R_{\text{compression}}$=省 token、$R_{\text{preservation}}$=关键实体是否留存。这是一份**现成的"记忆体检打分表"**，即便我们不做 RL，也能拿它离线**评我们每次会话的记忆管理质量**。
 3. **FILTER 阈值 $\theta_f\approx0.6$ 的经验值**（§12.3）——按余弦相似度删上下文噪声，$[0.4,0.8]$ 都稳，可直接作为我们上下文压缩策略的默认起点。

➤ **b. 可迁移到我们课题的思路（transfer）**：把 AgeMem 的**三阶段课程**映射到我们 `learning/auto-research-frontier/m9.*` 的研究 agent 上——`m9.2-research-agent-core` 正是"先读一堆资料（≈Stage 1 攒 LTM）、中间被无关文献干扰（≈Stage 2 抗 distractor）、最后要综合成结论（≈Stage 3 联合推理）"。**迁移时要改的前提**：AgeMem 靠 HotpotQA 的 ground-truth facts 给 MQ 打分，而我们的研究任务**没有 ground-truth**——所以 $R_{\text{relevance}}$/MQ 得换成"引用可核验性"这类**可验证奖励**（呼应 memory `fix-algo-to-match-paper` 的可验证收口精神），否则 LLM-judge 会把我们带进它自己的偏好里。

➤ **c. 它暴露的开放问题 = 我们的机会（open problems → our opportunity）**：论文自认**工具集是固定的一小组、且"广播终局优势"的方差代价没量化**（§14）。机会有二：(1) **给记忆工具集加"留一消融"**——量化去掉 UPDATE/DELETE 分别掉多少分，回答"这套动作集最小充分吗"；(2) 更重要——AgeMem 只在**离线基准**上跑，**没在"持久长期对话/真实用户"上验证**（作者明确留作 future work）。而**我们每天就活在真实的持久会话里**（MEMORY.md 跨会话累积），可下手的第一步：**拿我们自己近 N 次会话当"真实长期对话"数据，离线复现它的三分解奖励**，看我们的 MEMORY.md 写入是否高 $R_{\text{storage}}$、会话上下文是否守住 $R_{\text{preservation}}$。

➤ **d. 与本库其它论文/模块的连接（connect the dots）**：
 - **与同组 MemGPT（把记忆当函数调用的鼻祖）正面接续**——MemGPT 首创"用工具调用在主上下文与外部存储间搬运记忆（分页/中断）",但它的搬运策略是**提示工程 + 启发式（靠 LLM 自己按系统提示决定何时 page in/out）**，没有端到端训练；AgeMem 正是**在 MemGPT 的"记忆即函数调用"骨架上，把'何时/存什么/怎么管'从启发式升级为 RL 学出来的策略**，并进一步把 STM 的 Summary/Filter 也纳入同一动作空间。可以说 **AgeMem = MemGPT 的接口 × GRPO 的训练**。
 - **与本库 Harness-Bench（2605.27922）呼应**——Harness-Bench 的失败症状里"**状态/续跑 9.3%**（多轮任务没保住进度）正是 AgeMem 的 STM 管理要治的病；而 Harness-Bench 强调"harness 优劣在于**有没有把推理忠实搬运到可验证动作**"，AgeMem 的 $R_{\text{preservation}}$（关键实体是否留存）就是这条"execution alignment"在记忆维度的一个可操作代理。
 - **与 auto-research 的 `m9.8`（独立验证收口）** 共享"**谁来 judge the judge**"的隐忧——AgeMem 把 LLM-judge 塞进奖励的每一层，正是 m9.8 红队最爱攻的软肋。

➤ **e. 如果我来做下一步（my next move，第一人称、可执行）**：我会先在**我们自己的 harness 上做一个"记忆即工具"最小原型**——给会话加两个显式工具 `mem_add`（写 MEMORY.md，带质量自评）和 `ctx_filter`（按 $\theta_f=0.6$ 余弦阈删当前上下文噪声）；然后**不急着上 RL**，先用 AgeMem 的三分解奖励**离线评** 10 次真实会话，量出我们当前 MEMORY.md 的 $R_{\text{storage}}$（高质量记忆占比）到底多少。若偏低，再决定是否值得上"广播终局优势"式的训练——毕竟对我们，"先能测、再谈优化"比"直接抄 GRPO"务实得多。

## §15　版图定位（canon/前沿坐标 + 在本库的位置）

- **时间坐标（Θ4）**：**2026 前沿**。它相对本组基石推进了明确一步——**MemGPT（记忆即函数调用，靠提示/启发式）→ A-Mem / Memory-R1（记忆即动作，但只优化某一侧）→ AgeMem（LTM+STM 统一动作空间 + 端到端 RL）**。它把"记忆即动作"这条线从"单侧、启发式或半端到端"收紧到"双侧、全端到端、无外挂专家"。
- **回扣 Agent = Model + Harness（Θ2）**：本篇是该等式在**记忆/上下文维度**的实证——同一底座（Qwen2.5-7B），仅把记忆层从"有工具不会用"训成"会协调用"，平均分 +8.53（§4.2）。**变的不是模型，是 harness 的 C 层及其训练方式。**
- **E/T/C/L/O/V 归属（Θ1）**：坐 **C（Context）层**，强依赖 **T（Tools，记忆以工具实现）** 与 **L（Loop，ReAct 训练循环）**，并把 **RL** 作为把 C 层策略学出来的手段。
- **在本库的位置**：**D 组（记忆）** 的"训练侧"代表作。读完它，再看 D 组任何一篇"记忆结构/检索"的工作，都能追问一句——"**它的记忆操作是写死的启发式，还是像 AgeMem 一样被端到端学出来的？如果换成学出来的，会不会更好？**"这正是 AgeMem 给本组立下的新标尺。
- **regime 诚实（Θ5）**：**别把"记忆即动作 + RL 一定更好"绝对化**。§11 的 PDDL 反例已经说明——**裸工具（noRL）可能比无记忆还差**（8.87 < 10.15），记忆工具的红利**以"学会用"为前提**；且全部增益都在**离线、受控**基准上取得，对真实长程对话尚未验证（§14）。**什么时候记忆层主导**：任务越需要跨多步保状态、越需要从早期信息里挑关键（如多跳 QA、长程规划），记忆层回报越大；纯单步、上下文放得下的任务里，这套记忆脚手架几乎无所谓。

## §17　一页速记（takeaways）

- **命题**：别把 LTM/STM 当两套硬编码外挂；**把二者操作暴露成工具动作，用 RL 让 agent 自管**。属 harness 的 **C（上下文）层**。
- **接口**：LTM 三件套 `Add/Update/Delete`（改持久 $\mathcal{M}_t$）+ STM 三件套 `Retrieve/Summary/Filter`（改活动上下文 $C_t$）——**接口不对称**（Table 1）。
- **课程**：三阶段渐进——① 攒 LTM ② 抗干扰管 STM ③ 联合推理；**$\mathcal{M}$ 跨阶段持续、$C$ 阶段间重置**（防泄漏，逼真检索）。
- **训练**：**步级 GRPO**——只在终局按组归一化算优势（Eq.5），再**广播回每一步**（Eq.6）监督所有中间记忆决策；免 critic。
- **奖励**：`R = w·[R_task, R_context, R_memory] + P_penalty`；上下文侧奖"省而全且主动"（Eq.19–21），记忆侧奖"存精、维护、检索准"（Eq.23–25）。
- **结果**：五基准平均 **41.96 / 54.31**（Qwen2.5-7B / Qwen3-4B），相对无记忆 **+49.59% / +23.52%**；**RL 一项 +8.53 / +8.72 分**；MQ **0.533 / 0.605**；STM 工具比 RAG 省 **3.1%/5.1%** token。
- **反直觉**：RL 后 **RETRIEVE 频次下降**——不是欠训，是"存好了就不用反复检索"的策略质变（Table 3）。
- **诚实**：工具集写死、训练仅 HotpotQA、奖励重度依赖 LLM-judge、仅离线受控评测；**裸工具（noRL）可能比无记忆还差**（PDDL 8.87<10.15）——记忆红利以"学会用"为前提。
- **对我们**：MEMORY.md(≈LTM)+会话上下文(≈STM) 几乎同构；**先把 `mem_add`/`ctx_filter` 做成显式工具、用它的三分解奖励离线体检我们自己的会话记忆，再谈要不要上 RL**。与 **MemGPT**（记忆即函数调用鼻祖）的关系一句话：**AgeMem = MemGPT 的接口 × GRPO 的训练**。
