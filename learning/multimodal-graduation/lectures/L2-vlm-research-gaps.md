## 1. 用 6 类 gap 雷达扫多模态 (9.3 复用)

`mm_capstone.multimodal_gaps` 把 9.3 的 6 类 gap 雷达对准 VLM, 扫出候选题目, 并用优先级公式排序:

$$\text{优先级} = \frac{\text{Importance} \times \text{Tractability}}{\text{Cost}}$$
(重要性 × 可做性 ÷ 成本, 都 1-5 分; 高分 = 重要、可做、还便宜)

扫出的部分 gap (按优先级):

| 优先级 | 类型 | gap | 对你为什么友好 |
|---|---|---|---|
| **20** | 复现 | VLM 评测的 prompt 敏感性有多严重 | 现成 VLM 就能做, 极低成本, 接 9.3/9.4 |
| **15** | 复现 | 哪层视觉特征喂 LLM 最好 (跨任务消融) | 纯消融, 你的实验设计强项 |
| **10** | 方法 | 用 DPO 教 VLM 不幻觉 (抗噪偏好优化) | 你的 dpo-family + 10.6 POPE 直接接上 |
| 8 | 评测 | 视觉 token 压缩 vs 性能帕累托前沿 | 接你 long-context, 工程友好 |
| 5 | 理论 | reasoning VLM 的视觉 CoT 忠实性 | 接 reasoning-r1 + 即将建的 M12 可解释性 |
| 2.5 | 假设 | 跨模态迁移到底有多少 (any-to-any 核心) | 重要但贵, 适合中期 |

> 注意排序的智慧 (9.3-L3): **给博0 的第一个题, 优先选高可做、低成本的** (哪怕重要性中等), 先跑完一个完整研究循环。所以 prompt 敏感性 (20分) / 视觉特征层消融 (15分) 是你最该先动手的 —— 它们**现成 VLM + 你的实验设计就能做, 不烧算力**。

## 2. 三个对你最友好的起手题 (展开)

### 起手题 A: VLM 评测的 prompt 敏感性 (复现类, 优先级 20)
- **gap**: VLM benchmark 分数有多少是真实能力、多少是 prompt 工程? (M10.6-L4 的隐蔽陷阱)
- **最小实验**: 拿一个现成 VLM, 在一个 benchmark 上用 5-10 个 prompt 变体评测, 报分数方差。
- **为什么友好**: 不训模型、用现成 VLM、接你 9.4 (方差) + 9.6 (诚实图)。一周能出第一个信号。
- **so-what**: 「VLM 排行榜的 X% 差距其实是 prompt 噪声」—— 一个让人「哦!」的诚实结论 (复现 gap 富矿)。

### 起手题 B: 哪层视觉特征喂 LLM 最好 (复现类, 优先级 15)
- **gap**: M10.1-L4 提过「最后一层不总最好」, 但没人系统消融过哪层最优。
- **最小实验**: 固定其它, 扫视觉塔的不同层喂 LLM, 看 VLM 下游性能 (你的 mini_vlm 就能做!)。
- **为什么友好**: 纯消融 (你 9.4 强项), 低成本, 你的 10.1+10.3 src 直接用。

### 起手题 C: 用 DPO 教 VLM 不幻觉 (方法类, 优先级 10)
- **gap**: VLM 幻觉是头号障碍 (M10.6-L3), 能不能用偏好优化教它「诚实说没有」?
- **最小实验**: 构造「幻觉回答 vs 诚实回答」偏好对, 用 DPO 微调, 测 POPE 幻觉率变化。
- **为什么友好**: **你的 dpo-family 复现 + 10.6 POPE 直接拼**。这是「工程 ⨯ 研究」交叉, 你独有的双优势。
