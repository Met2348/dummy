# L3 · omni 模型: 一个模型通吃所有模态

> 25-min lecture · 目标: 理解 omni 模型如何把文本/图/视频/音频统一进一个模型, 及其架构与挑战。这是 M10.4 any-to-any 的具体实现。

---

## 0. 把所有模态接上同一条流

到现在, 你能把所有模态 token 化:
- 文本 → BPE token (你已有)
- 图 → VQ token (M10.4)
- 视频 → 时空 token (L1)
- 音频 → mel/语音单元 token (L2)

**omni 模型 (如 Gemini, GPT-4o, Qwen-Omni) 把它们全接进一个统一 transformer。** 这是 M10.4-L4 any-to-any 的工程实现 —— 本讲讲它怎么搭、难在哪。

---

## 1. omni 模型的统一架构

```
   文本 ─┐
   图   ─┤ 各自的 tokenizer/encoder
   视频 ─┼──→ 投影到共享 token 空间 ──→ 一个统一 transformer ──→ 输出 token
   音频 ─┘                              (预测下一 token)           │ 各自 detokenizer
                                                                  → 文/图/音/视频
```

核心和 M10.2 的 early-fusion + M10.4 的 token 化一致:
- **每个模态有自己的编码器/tokenizer** (把原始信号变 token)。
- **所有 token 进同一个序列**, 一个 transformer 统一处理 (预测下一 token)。
- **输出 token 按模态分发**给对应解码器 (文本 detokenize、视觉 VQ 解码、音频声码器……)。

> omni 的「统一」全靠 token 这个**通用货币**: 不管什么模态, 进来都是 token, transformer 一视同仁。难点不在 transformer (它统一处理), 而在**每个模态的好 tokenizer + 模态间的平衡** (M10.4-L4 已点破)。

---

## 2. omni 模型的三大挑战

### 挑战一: 模态间的 token 数极不平衡
- 一句话: 几十个文本 token。
- 一张图: 上百视觉 token。
- 一段视频: 上千时空 token (即使压缩)。
- 一段音频: 几百音频 token。

> 视频/音频的 token **远多于**文本, 容易「淹没」文本信号 (模态不平衡, 回忆 M10.3-L4 模态坍缩)。omni 模型要么狠压视频/音频 token (L1 的时空 patch + 帧采样), 要么在训练里平衡各模态的损失权重。**token 数平衡是 omni 训练的核心难题。**

### 挑战二: 实时性 (尤其语音/视频)
- 语音对话要求**低延迟** (人受不了几秒延迟)。但自回归生成 token 慢 (M10.4-L2 软肋)。
- 视频流要求**在线处理** (边来边处理, 不能等全部到齐)。

> 这把 omni 模型从「离线批处理」推向「实时流式」, 引入一整套工程约束 (L4 详讲, 接你的 harness-engineering)。GPT-4o 的「实时语音对话」就是攻这个。

### 挑战三: 跨模态对齐与训练数据
- 要让模型理解「这段语音说的就是这张图里的东西」, 需要**对齐的多模态数据** (同一事件的多模态记录), 这种数据稀缺。
- 训练目标要覆盖所有模态的所有方向 (文→图、图→文、语音→文、文→语音……), 数据和损失设计极复杂。

---

## 3. omni 模型的价值与赌注

omni 模型押的是 M10.4-L4 说的**跨模态迁移**假设: 一个模态学的帮其它模态, 统一模型涌现出专精模型没有的能力 (如「看着图听着语音回答」的真多模态推理)。

- **如果假设成立**: omni 是通向「真正理解世界」的路 (世界是多模态的, 接 M11 世界模型)。
- **如果不成立**: omni 只是「几个专精模型塞一起」, 不如分开做。

> 这是 2026 frontier 实验室 (Google/OpenAI) 重金押注的方向, 也是个**未定论的大赌注**。对你找 PhD 方向: omni 模型的「模态平衡」「实时生成」「跨模态迁移验证」都是可做、重要、有人在意的 gap。

---

## 4. omni 是 M10 的归宿

```
   M10.1 视觉塔 → M10.2 融合 → M10.3 训练 (读图)
        → M10.4 视觉 token 化 + 生成 (画图)
        → M10.5 视频/音频 token 化 (L1/L2) + omni (本讲)
   ─────────────────────────────────────────────
   归宿: 一个模型, 所有模态 token 化进同一条流, 统一 transformer 通吃
```

M10 从「给 LLM 装上眼睛」(VLM) 一路走到「给 LLM 装上所有感官」(omni)。核心不变: **token 是通用货币, 统一架构是归宿。**

---

## 5. 本讲小结 + 通往 L4

- **omni 模型**: 所有模态 token 化进同一条流, 一个 transformer 通吃 (any-to-any 的实现)。
- 三大挑战: **token 数不平衡** (视频/音频淹没文本) / **实时性** (语音视频低延迟) / **跨模态对齐数据稀缺**。
- 押注**跨模态迁移**假设 —— frontier 重金押、未定论, gap 富矿。
- 是 M10 的归宿: token 通用货币, 统一架构。

> **下一讲 L4「实时多模态约束」**: omni 模型尤其语音/视频要求实时。实时多模态引入一整套工程约束 (流式、低延迟、边界处理), 这接回你的 harness-engineering。L4 收口 M10.5。
