# L1 · 世界模型 (面向控制): 预测下一观测

> 22-min lecture · 目标: 从控制视角理解世界模型 = 学转移动态, 学会后能想象+规划。Module 11.5 开篇, 接 M13.5。

---

## 0. 从「纯模仿」到「带想象」

M11.4 的 BC 是**纯模仿**: 学专家怎么做。但它有分布漂移 (偏出专家就乱), 且必须有**专家 demo**。这一讲换条路: **世界模型** —— 不学「专家怎么做」, 而学「**世界怎么变**」(转移动态)。学会后能在脑内**想象**未来、**规划**动作, 而且**不需要专家**。

> 这正是你 M13.5 学的世界模型, 现在用于机器人控制。**M11.5 的 `world_model.py` 和 M13.5 同源** (同一概念): 学 $P(s_{t+1}\mid s_t, a_t)$。

---

## 1. 世界模型 = 学环境转移

```
   世界模型 f:  (当前状态 s, 动作 a) → 下一状态 s'
   学法:        从 (s, a, s') 三元组监督回归 (预测 Δs 更稳, 同 M13.5-L1)
   数据来源:    可以是**随机探索** (不需要专家!) ← 关键优势
```

> 对比 BC (学 state→action, 需专家), 世界模型学 (state,action)→next_state, **数据可以来自随机乱动** (随机动作 + 看结果)。你 N1 会用纯随机探索的转移学世界模型 —— **零专家 demo**。这是 model-based 最大的数据优势。

---

## 2. 学会世界模型能干什么: 想象 + 规划

```
   想象 (imagine): 用 f 链式预测未来, 不碰真环境 (M13.5-L1)
   规划 (plan):    在脑内试很多动作序列, 用 f 想象结果, 选最好的去执行 (MPC)
```

- **MPC (模型预测控制)**: 每步采样多条动作序列 → 用世界模型想象各自结果 → 选末端最接近目标的 → 执行其首动作 → 下一步重规划。
- 你 N1 的 `plan_action` 就是 MPC (随机射击版): 没学策略, 靠世界模型 + 搜索就能到目标。

> 关键认知: **有了世界模型, 不用学策略也能控制** —— 在脑内搜动作就行 (规划)。这和 BC (直接学策略) 是两条路: 模仿 vs 规划。世界模型把「控制」变成「搜索 + 预测」。

---

## 3. 误差累积: 世界模型的老问题 (接 M13.5)

世界模型想象是自回归的, **误差会累积** (M13.5-N2): 想象越长越不准。这限制了规划视野:

```
   短视野规划 (horizon 小): 想象误差小, 规划可信, 但只看近处
   长视野规划 (horizon 大): 看得远, 但想象误差累积 → 规划基于错误的想象
```

> 所以 MPC 用**短视野 + 每步重规划** (用真观测纠偏) 对抗误差累积 (M13.5-L1/L4)。你 N1 的 MPC horizon 取小 (如 6 步), 每步重规划。**这是世界模型的通用智慧: 别想太远, 走一步看一步再想。**

## 4. M11.5 ↔ M13.5: 同一个世界模型

| | M13.5 (生成侧) | M11.5 (控制侧) |
|---|---|---|
| 状态 | 2D 点 / 图像 | 机器人状态 (toy: agent+goal) |
| 转移 | 学动态 | 学动态 |
| 用途 | 想象/可玩世界 | **规划控制** |
| 误差累积 | 核心难题 | 限制规划视野 |
| 代码 | `world_model.py` | `world_model.py` (同源) |

> M13.5 和 M11.5 是**同一个世界模型概念的两面**: M13.5 偏生成 (想象/可玩), M11.5 偏决策 (规划控制)。这就是为什么它们共享 `world_model.py`。你学一次, 两边都通。

## 5. 本讲小结 + 通往 L2

- 从纯模仿 (BC) 到带想象: **世界模型**学转移动态 (非学专家), **数据可随机探索** (零专家)。
- 学会后能**想象 + 规划 (MPC)**: 不学策略, 靠搜索+预测就能控制。
- **误差累积**限制规划视野 → 短视野 + 重规划 (接 M13.5)。
- M11.5 ↔ M13.5 同源: 同一世界模型, 一面生成一面决策。

> **下一讲 L2「先学想象, 再学行动」**: NVIDIA 的 world-action models 路线 —— 先用海量数据预训练世界模型 (学想象), 再微调成会行动的策略。

**动手**: 去 `N1`, 用**纯随机探索**数据学世界模型, 然后用 MPC 规划到目标 —— **零专家 demo** 解决任务。体会 model-based 的数据优势。
