# L3 · 基于模型的机器人 RL (在想象里 rollout)

> 22-min lecture · 目标: 理解 model-based RL —— 用世界模型当模拟器, 在想象里训策略, 样本效率暴涨。接你 RL 模块。

---

## 0. 真机 RL 的痛 + 世界模型的解

你 RL 模块学的 model-free RL (PPO 等) 要**大量环境交互**。真机器人交互**贵、慢、危险** (会摔坏)。世界模型给出解: **在想象里做 RL** —— 不碰真机, 在世界模型这个「梦」里跑 RL。

```
   model-free RL:  策略 ↔ 真环境 (大量真实交互, 贵)
   model-based RL: 策略 ↔ 世界模型 (在想象里交互, 几乎免费) + 少量真实数据校准世界模型
```

---

## 1. Dreamer 路线: 在「梦」里学策略

代表方法 (Dreamer 系):
```
   ① 从少量真实交互学一个世界模型 (学想象)
   ② 在世界模型里大量 rollout (想象的轨迹), 用 RL 训策略 (学行动)
   ③ 用学到的策略采少量新真实数据 → 改进世界模型 → 回到②
```

> 核心: **把昂贵的「真环境 rollout」换成便宜的「想象 rollout」**。策略在世界模型里练几百万步, 只花极少真实交互。这就是 model-based RL 样本效率高的原因 —— 真实交互省几个数量级。

---

## 2. 样本效率: model-based 的核心优势 (你 N2 量到)

```
   达到同样成功率需要的真实交互:
   model-free (BC/RL): 需大量专家 demo / 真实交互
   model-based:        从少量 (甚至随机) 数据学世界模型 → 想象里练 → 省真实交互
```

> 你 N2 会对比 **model-based (世界模型+规划) vs model-free (BC)** 的样本效率: model-based 能从**随机探索**数据学世界模型 + 规划解任务, 不需要专家; model-free (BC) 必须有专家 demo。**model-based 用便宜数据换好策略** —— 这是它在「数据贵」的机器人上的杀手锏。

---

## 3. 但 model-based 不是免费午餐

诚实地看代价 (接 M9 批判):
- **世界模型不准 → 策略学歪**: 在错误的想象里练, 学到的策略在真实里失败 (model bias)。
- **误差累积** (M13.5/L1): 想象长了不准 → 限制 rollout 长度。
- **复杂环境难学世界模型**: 高维/接触丰富 (灵巧操作) 的动态极难准确建模。

> 所以 model-based 在**动态好学**的任务上香 (你的 toy reach 就很好学 → MPC 0.9 成功率); 在**接触复杂**的任务上, 世界模型不准会拖累。现实常是 **model-based + model-free 混合** (用世界模型加速, 用真实数据兜底)。**没有银弹, 看任务的动态多好学。**

---

## 4. 把 RL 模块和这里连起来

你的 RL 模块 (model-free) 和本专题 (model-based) 的关系:
- **model-free** (PPO/你的 RL 模块): 不学环境, 直接试错。简单通用, 但费交互。
- **model-based** (本专题): 先学世界模型, 在想象里练。省交互, 但靠世界模型准。
- 世界模型 = **RL 的「环境替身」**, 让试错变便宜 (M13.5-L4)。

> 一句话: **model-based RL = 你的 RL + 世界模型加速**。世界模型把「昂贵的真实试错」变成「便宜的想象试错」。对机器人 (真实交互贵), 这个加速是关键。这是你 RL 知识 + M13.5 世界模型的自然合体。

---

## 5. 本讲小结 + 通往 L4

- **model-based RL**: 用世界模型当模拟器, 在想象里 rollout 训策略 (Dreamer), 样本效率暴涨。
- **样本效率优势** (N2): model-based 从少量/随机数据学世界模型+规划解任务, 省真实交互 (model-free 需专家/大量交互)。
- 代价: 世界模型不准→策略歪 (model bias) + 误差累积 + 复杂动态难学; 常混合使用。
- 世界模型 = RL 的环境替身 (你的 RL + 世界模型加速)。
