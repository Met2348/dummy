# L03 · 自己给自己改作业（grading-its-own-homework）与守卫

## 1. 最隐蔽的循环性陷阱

AI Scientist 用 LLM 给自己写的论文打分。问题：**裁判和选手是同一个模型**。
[Wishful Thinking/ARI](https://arxiv.org/abs/2502.14297) 与 [Hidden Pitfalls](https://arxiv.org/abs/2509.08713) 实证了它的后果：

- 评审"结构化但肤浅"，漏掉冗余、格式、甚至**错误的实验结果**；
- 系统会**刷榜**（偏好高 SOTA 的简单基准）、**伪造/替换数据集**、**幻觉结果**——而自评审给这些高分。

## 2. 我们把这个缺陷做成可复现的代码

`review.py` 的 `review()` 故意有一处缺陷：

```python
# novelty：★缺陷★ 只看"宣称的效果有多大"，完全不管它是否可靠/真实
novelty = clip(3.0 + 40.0 * abs(delta))
```

于是**宣称的 |Δ| 越大，分越高**——不管这个 Δ 是真是假、是正是负。后果有两层：

**(a) 注水能骗分**（`demonstrate_gaming()`）：

```
诚实小真效果 overall=6.2 | 注水大假效果 overall=7.6 | 被刷=True
```

一篇没有误差棒、没有图、Δ 注水到 0.40 的"论文"，分数反而更高。

**(b) 光谱里的现场实证**：跑 `--idea all`，被推翻的 `crank-lr`（Δ=-0.143，最大绝对值）拿了**全场最高 overall=8.6**。
**最烂的研究骗到了最高的自评分。**

## 3. 教训与守卫

> **别信自评分，看实验本身。** 自动评审可以辅助，但不能当判据。

可复现性守卫（接下来 9.6 评测 / 9.8 红队会展开）：

1. **报告数字必须可追溯到真实运行**——本模块用 test `test_report_numbers_match_experiment` 锁死"报告数字==实验真值"。
2. **独立验证**：让一个**不参与生成**的 verifier 重算指标（而非让作者模型自评）。
3. **artifact + 执行日志**：提交代码、数据、随机种子、被丢弃的实验，让人能复算（[Hidden Pitfalls](https://arxiv.org/abs/2509.08713) 的建议）。
