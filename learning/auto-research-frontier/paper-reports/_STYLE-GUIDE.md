# 组会汇报文档 · 风格规范与模板（所有 40 篇统一遵循）

> 目标：每篇论文一份 **50 分钟组会汇报级、PPT 风格、约 20 页**的 Markdown 文档，
> 让组里任何人读完就能主讲并展开讨论。**intention / why 比 how 更重要。**

## 0. 硬性要求（逐条满足，缺一不可）

1. **中文撰写**，术语首次出现给中英对照（如「树搜索 (tree search)」）。
2. **每个公式前必须**：① 一句**直觉/为什么要这个式子**（在解决什么、动机是什么）；
   ② **逐个定义式中所有符号**（变量、下标、集合、函数），**定义在公式之前**。
   公式后再补一句**它说明了什么/读出什么结论**。
3. **why > how**：每个方法块先回答「为什么这么设计、不这么会怎样」，再讲怎么做。
4. **setting / metrics / parameters 写全**：数据集、规模、baseline、评测指标（给出**定义式或精确定义**）、
   关键超参数与其取值、算力/成本、随机性控制——能查到的都列。
5. **忠于原文**：所有公式、数字、结论必须来自 PDF；**引用章节/图表/公式编号**（如「见原文 §3.2 / Eq.(4) / Table 2」）。
   **不许编造**数字或公式；原文没有的，写「原文未给出」。区分「论文宣称」与「批判/局限」。
6. **PPT 风格**：多用小标题、要点列表、表格、`mermaid` 流程图（架构/pipeline 用）、关键式子单独成块。
   每个二级标题相当于一张「幻灯片」，配 1–2 句**讲稿提示**（> 引用块，主讲时说什么）。
7. **篇幅**：约 20 页（≈ 6000–9000 中文字 + 公式/表格）。宁可深、不要水。

## 1. 文件位置与命名

- 路径：`learning/auto-research-frontier/paper-reports/<arxiv-id>-<slug>.md`（与 PDF 同名，换 `.md`）。
- 顶部 YAML：`title / arxiv / venue / 机构 / 主题组(A–H) / 一句话`。

## 2. 二十页骨架（固定结构，按需增删子节但保留主干）

```
1.  封面 + TL;DR：标题/作者/会议/一段话「这篇在干什么、为什么重要」+ 3 条带走的结论
2.  问题与动机（why）：领域缺口、为什么现在做、不做会怎样（2 页，核心）
3.  研究问题 / 核心 intention：把问题形式化成一句话 + 假设
4.  相关工作定位：它站在谁肩上、和谁不同（一张对比表）
5.  方法总览（big picture）：架构/pipeline 一图流（mermaid）+ 直觉，先不展开数学
6.  符号与术语表：一张表，定义后文要用的所有记号与术语
7–12. 方法细节（含数学，核心 6–8 页）：逐组件展开；
      **每个公式：直觉→符号定义→公式→读出什么**；给算法伪代码
13. 实验设置：数据集/规模/baseline/**指标定义式**/超参/算力/成本/种子
14. 主要结果：关键表与数字 + 它们意味着什么（别只贴数，要解读）
15. 消融与分析：哪个部件贡献多少、敏感性
16. 局限与批判：原文承认的 + 你/社区的质疑（诚实）
17. 在 auto-research 版图的位置：与本库其它论文的关系（Tool→Analyst→Scientist 阶梯定位）
18. 复现与可用性：代码/数据是否开源、能不能在单卡跑、坑在哪
19. 组会讨论问题：5–8 个能引发讨论的问题
20. 一页速记 takeaways：把全篇压成一页，供汇报当天速览
```

## 3. 公式书写范式（务必照此格式）——一个微例

> 直觉：我们想衡量「一个想法有多新」，但「新」很主观；于是用「与已有文献的最大相似度的反面」来近似——
> 跟所有已知工作都不像，就更可能新。

记号（先定义，后用式）：
- $i$：待评估的研究想法 (idea)；
- $\mathcal{R}=\{r_1,\dots,r_K\}$：检索到的 $K$ 篇相关工作 (retrieved related work)；
- $\operatorname{sim}(i, r)\in[0,1]$：想法 $i$ 与文献 $r$ 的语义相似度（余弦相似度）。

$$ \text{novelty}(i) \;=\; 1 \;-\; \max_{r \in \mathcal{R}} \operatorname{sim}(i, r) $$

读出什么：相似度越高（越像已有工作），novelty 越低；取 $\max$ 是因为「只要和**任何一篇**雷同就不算新」。
> 讲稿提示：强调这是**启发式**，不是真新颖度——埋下「novelty≠feasibility」的批判线。

## 4. 不同类型论文的侧重（自适应）

- **系统/旗舰**（AI Scientist 等）：重 pipeline 形式化、各 agent 角色、搜索/评审目标函数、成本。
- **综述**：重**分类法 (taxonomy)** 与坐标轴、把代表系统填进框架；公式少则聚焦定义与判据。
- **Benchmark/评测**：**指标定义式**是重点（pass@k、Elo、scoring rubric、归一化分数），任务构造、防刷设计。
- **批判**：重**论证结构与证据**，把「宣称 vs 实测」做成对照表；指出方法学漏洞。
- **创意/理论**：重假设、实验设计、统计检验（t 检验/置信区间）的**定义与解读**。
