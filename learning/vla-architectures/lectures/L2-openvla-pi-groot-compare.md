# L2 · OpenVLA / π / GR00T 架构对比

> 22-min lecture · 目标: 用两阶段框架 (backbone + 动作头) 拆三个真实 VLA, 理解各自的设计取舍。

---

## 0. 同骨架, 不同选择

L1 给了通用骨架: backbone + 动作头。三个代表 VLA 都是这个骨架, 区别在**每个阶段选了什么**。拆完你会有一张「怎么设计 VLA」的决策图。

---

## 1. 对比表 (两阶段视角)

| | backbone (感知核) | 动作头 | 动作表示 | 特点 |
|---|---|---|---|---|
| **OpenVLA** | 开源 VLM (Prismatic, ~7B) | 离散 token | 离散 (分桶) | 开源基线, 简单可复现 |
| **π0 / π** | 预训练 VLM | **flow-matching** | 连续, 高频 | 灵巧操作, 50Hz 精细控制 |
| **GR00T** | VLM + 视觉 | 扩散 + 世界模型 | 连续 | 人形, 大数据, 融合世界模型 |

> 主线观察: **backbone 都是 VLM (趋同), 动作头在分化** —— 从离散 token (OpenVLA) 走向 flow/扩散 (π/GR00T)。这正是 L1 说的「创新主要在动作头」。

---

## 2. OpenVLA: 简单可复现的基线

- backbone: 开源 VLM (视觉编码器 + Llama 类 LLM)。
- 动作头: 把动作离散成 token (tokens-as-actions, M11.1-L3), 用 LLM 的输出层预测。
- 训练: Open X-Embodiment 大数据上微调。

> 价值: **像 Llama 之于 LLM** —— 开源、可复现、成为研究基线。缺点: 离散动作精度受限 (L3 的离散化损失), 不够灵巧。你的 mini-VLA 离散头版 ≈ OpenVLA 的玩具内核。

---

## 3. π (Physical Intelligence): flow-matching 动作头

- backbone: 预训练 VLM。
- 动作头: **flow-matching** (你的 M13.2!) 生成连续动作, 高频 (~50Hz), 支持灵巧双手操作。
- 关键: 用 action chunking (一次预测一段动作, M11.3-L3) + 连续生成 → 平滑精细。

> π 的洞察: **精细操作需要连续、多峰、平滑的动作 —— 离散 token 做不到, 扩散/flow 才行** (M11.3-L1)。这把你 M13.2 的 flow matching 直接用作了动作头。π 是「动作头革命」的代表。

---

## 4. GR00T (NVIDIA): 人形 + 世界模型

- backbone: VLM + 强视觉。
- 动作头: 扩散动作头。
- 特色: 融合**世界模型** (M11.5 / M13.5) —— 不只反应式控制, 还能预测/规划; 面向人形机器人 + 海量 (含仿真) 数据。

> GR00T 代表趋势: **VLA + 世界模型融合** —— 从「看到就动」到「想象后果再动」。这把你 M13.5 的世界模型接进了 VLA。是 M11.5 的伏笔。

---

## 5. 设计 VLA 的决策图 (带走)

拆完三个, 提炼 VLA 设计的关键决策:
1. **backbone 选多大的 VLM?** (越大越懂世界, 越贵越慢) — 接 M10 scaling。
2. **动作头用什么?** 离散 token (简单/可复现) vs 连续回归 (精度) vs 扩散/flow (多峰/平滑/灵巧) — 接 M11.3。
3. **要不要世界模型?** 纯反应式 vs 带规划 — 接 M11.5。
4. **冻结策略?** 冻 backbone 省算力 vs 联合微调更强 — 接 M10.3。
5. **数据?** 真机 + 仿真 + 人类视频怎么配 — 接 M11.4/11.6。

> 拿这 5 个决策去读任何新 VLA 论文 (你没见过的), 你都能快速定位它的设计选择和创新点。这就是「会拆系统」的元能力 (同 M13.7-L2)。

---

## 6. 本讲小结 + 通往 L3

- 三个 VLA 同骨架 (backbone+动作头), **backbone 趋同 (都 VLM), 动作头分化** (离散→flow/扩散)。
- OpenVLA: 离散 token 基线 (可复现); π: flow-matching 灵巧操作; GR00T: 扩散+世界模型人形。
- VLA 设计 5 决策: backbone 大小 / 动作头 / 要不要世界模型 / 冻结策略 / 数据配比。

> **下一讲 L3「离散 vs 连续动作」**: 动作头的第一个核心选择。离散 token 和连续动作各自的代价是什么? 你 N2 会实测两者在 toy 上的成功率与平滑度差异。

**动手**: 选一个你感兴趣的真实 VLA, 用本讲的「5 决策」拆它一遍 (backbone? 动作头? 世界模型? 冻结? 数据?)。
