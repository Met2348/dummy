# L3 · 离散动作 token vs 连续动作

> 22-min lecture · 目标: 理解动作头的第一个核心选择 —— 离散 vs 连续, 各自的代价。你 N2 会实测。

---

## 0. 动作头的第一个分叉

L2 说创新主要在动作头。动作头的第一个选择: **动作怎么表示** —— 离散 token 还是连续值? 这个选择决定了精度、表达力、和能不能复用 LLM。

---

## 1. 离散动作 token (RT-1/RT-2/OpenVLA)

把连续动作分桶成离散 token (M11.1-L3):

**优点**:
- **复用 LLM 输出层**: 动作就是 token, 直接用 next-token 预测 + 交叉熵 (零额外机制)。
- **简单稳定**: 分类比回归好训, 输出有界。
- **天然多峰**: 分类分布能表达「可左可右」(softmax 多个峰)。

**代价**:
- **离散化损失**: 动作被量化, 精度受限 (你 N1 量过方向余弦差)。精细操作 (插孔) 不够。
- **维度爆炸**: 高维动作 (多关节) × 每维多桶 = token 序列很长。

> 你的 mini-VLA **离散头** ≈ OpenVLA 内核: 简单、能到目标, 但动作「跳」(只有 8 个方向, N2 会看到平滑度差)。

---

## 2. 连续动作 (回归头)

直接回归连续动作值 (MSE 损失):

**优点**:
- **无离散化损失**: 精度高, 动作平滑 (你 N2 会看到平滑度好很多)。
- **维度友好**: 高维动作直接回归, 不需分桶。

**代价**:
- **难表达多峰**: MSE 回归会学成「平均」—— 若同一状态有两个合理动作 (绕障可左可右), 回归会取中间 (撞上去!)。这是连续回归的致命弱点。
- **较难训**: 回归比分类敏感。

> 关键矛盾: **连续头精度高但怕多峰; 离散头能多峰但精度低**。你 N2 会看到连续头动作平滑 (好), 但在多峰情形下回归「取平均」是它的阿喀琉斯之踵。这正是为什么要扩散动作头 (L3→M11.3): **既连续又多峰**。

---

## 3. 多峰问题: 为什么它如此关键

机器人动作**经常是多峰的**:
- 绕过障碍: 左绕、右绕都对 (两个峰)。
- 抓杯子: 从多个角度抓都行。
- 连续回归对多峰取平均 → 得到一个**两个都不对**的中间动作 (直接撞障碍)。

```
   合理动作分布:   左绕(峰1)        右绕(峰2)
   连续回归(MSE):           ↑ 取平均 = 撞上去 ✗
   离散分类:        能表达两个峰, 但精度粗
   扩散/flow:       连续 + 多峰, 两全 (M11.3)
```

> 这就是 M11.3 整个专题的动机: **多峰 + 连续 + 平滑, 三者要兼得, 只有扩散/flow 动作头能做到**。离散和连续各缺一块, 扩散补齐。理解这个矛盾, 你就懂了为什么 π 用 flow-matching。

---

## 4. 怎么选 (实践指南)

| 场景 | 选择 |
|---|---|
| 快速原型 / 可复现基线 | 离散 token (简单) |
| 单峰精细控制 | 连续回归 (精度) |
| 多峰 + 精细 + 平滑 (灵巧操作) | 扩散/flow (M11.3) |
| 想复用 LLM 全套 | 离散 token |

> 没有万能选择。离散适合入门/复用 LLM; 连续适合单峰精细; 多峰灵巧必须上扩散 (M11.3)。理解三者代价 = 知道什么任务该用哪个头。

---

## 5. 本讲小结 + 通往 L4

- 动作表示第一选择: 离散 token vs 连续。
- **离散**: 复用 LLM + 天然多峰, 但**离散化损失** (精度粗)。
- **连续**: 精度高 + 平滑, 但**怕多峰** (MSE 取平均 → 撞)。
- 多峰问题是关键矛盾 → 推动**扩散/flow 动作头** (既连续又多峰, M11.3)。

> **下一讲 L4「把 M10 VLM backbone 接动作头」**: 动手把抽象落地 —— 你 M10 造的 mini-VLM 怎么接上动作头变成 mini-VLA。收口 11.2 的「装配」。

**动手**: 去 `N2`, 在 toy 上对比离散头 vs 连续头: 看成功率 (都高) 和**动作平滑度** (连续远胜)。体会离散的「跳」和连续的「滑」。
