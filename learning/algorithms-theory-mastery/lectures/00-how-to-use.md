# Algorithms Theory Mastery 使用说明

> **先读 `01-textbook.md`**:那是本科课件式完整教材(七章、连贯教科书叙述),本页只是DeepPoint/ScenarioPoint追问链的使用说明,建议在读完教材之后再来这里练习追问链。

## 这个专题和 leetcode-mastery 有什么不同

仓库里已有的 `leetcode-mastery` 专题练的是544道题的实现能力——给一道题,写出通过测试的代码。这个专题练的是完全不同的一种能力:**证明一个算法为什么是对的**。举个例子,leetcode-mastery会让你实现Dijkstra算法并通过测试用例,而这个专题会让你回答"Dijkstra算法为什么正确,标准证明的反证法具体是怎么论证的,为什么这个证明在有负权边时会失效"。这是面试里经常被问到、但纯刷题练不出来的能力——很多人会写Dijkstra但说不清楚为什么它是对的。

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:建立渐进分析这套"衡量算法好坏"的严格语言(大O不是"约等于",而是有精确的数学定义),以及分治/贪心这两种最基本的算法设计范式。这一层要求理解离散数学专题里学过的证明基础(存在量词、归纳法),如果还没学discrete-math-mastery建议先补一下。
2. **Tier 2(深,`tier2_deep/`)**:三个文件分别对应算法证明里最常考的三块——动态规划(怎么证明状态定义和转移方程覆盖了所有情况)、图算法(Dijkstra/Bellman-Ford/Prim/Kruskal这些经典算法的标准证明套路)、网络流(一个"看起来很难但其实有多项式算法"的经典例子,和计算理论专题里NP难问题形成对比)。这三者都反复用到同一套证明工具:反证法+不变量+归纳法,建议练习时留意这个共性,而不是把每个算法的证明当成孤立的知识点死记。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"给定一个新问题,该用哪种范式设计算法、怎么验证这个选择"(比如怎么系统性验证贪心选择性质是否成立,而不是凭直觉猜);第二个文件考"理论最优是不是工程最优"这个经常被忽视的判断维度——渐进复杂度模型没有捕捉到常数因子、缓存友好性、实现复杂度这些现实因素。

## 一个容易被忽视的陷阱

会写一个算法的代码,和能证明这个算法为什么是对的,是两种不同的能力。很多人能背出Dijkstra/DP的实现模板,但被追问"为什么这样贪心/转移是对的"就说不清楚。这个专题的追问链第二三层反复在考这一点——不是让你重新写一遍算法,而是让你说清楚证明的核心论证结构(反证法假设错误方案存在,构造矛盾)。这也是为什么tier2的每个知识点都要求"标准证明思路"而不是"算法步骤描述",这是这个专题和leetcode-mastery最本质的区别。
