# L4 · 2 周摸清子领域的 SOP + mini-survey 产出

> 30-min lecture · 目标: 把 L1-L3 的方法压成一个 day-by-day 的两周计划, 终点是一份能上白板、给导师讲的 mini-survey。

---

## 0. 为什么要定时两周

摸领域有个隐藏陷阱: **它没有自然终点。** 你永远能再读一篇, 于是很多人「还在看文献」看了两个月, 迟迟不动手 —— 这是高级拖延 (用「我还没准备好」逃避真正的研究)。

> 解药: **给摸领域一个硬时间盒 (time-box): 2 周。** 两周一到, 不管你觉得多不完整, 都要产出一份 mini-survey 并开始动手。摸领域的目的是「装上 GPS 好上路」, 不是「把地图上每条小巷都走一遍」。**地图永远画不完, 但两周足够画出主干道。**

---

## 1. day-by-day SOP (两周)

```
   第 1 周: 建结构 (从锚点到地图)
   ┌─────────────────────────────────────────────────────────────┐
   │ Day 1   找锚点 (L1): 1 篇 survey + 3 奠基作 + 2 benchmark + 3 人 │
   │ Day 2   精读 survey (9.3 三遍读法), 抄它的 taxonomy 当初稿       │
   │ Day 3-4 后向滚雪球 (L2): 读 3 篇奠基作, 补背景, 建公共语言        │
   │ Day 5   前向滚雪球 (L2): 从奠基作爬最新被引, 列出前沿候选         │
   │ → 周末: 用 snowball.py 建引用网, 跑中心度, 校准「谁重要」         │
   ├─────────────────────────────────────────────────────────────┤
   │   第 2 周: 填地图 + 找前线 (从地图到 mini-survey)               │
   │ Day 6-8 按 taxonomy 精读每个流派的代表作 (每流派 1-2 篇)         │
   │ Day 9   排 SOTA 演进线 (L3): 写出「痛点→解法→新痛点」因果链      │
   │ Day 10  标前线 (L3): 争论 / 多篇共有的 limitation / 空白格       │
   │ → 收尾: field_map.py 出地图图; 写 mini-survey; 收 2-3 个 gap 候选 │
   └─────────────────────────────────────────────────────────────┘
```

几个执行要点:
- **第 1 周重结构, 第 2 周重内容。** 先有骨架 (taxonomy + 网), 再往里填肉 (精读代表作)。反过来 (先一篇篇精读) 就会淹死。
- **每天读完立刻起卡** (9.1 的文献卡 + 永久笔记), 别攒到最后。攒着 = 忘光。
- **精读是有预算的**: 两周里你只精读约 8-12 篇 (奠基作 + 每流派代表作), 其余几十篇只扫标题/摘要。把 9.1-L4 的 scan/triage/read 三层漏斗用上。

---

## 2. mini-survey: 两周的交付物

mini-survey 不是论文, 是**一页纸 (+ 几张图)**, 用来给导师汇报、也沉淀进你的第二大脑。用 `templates/mini-survey.md`, 结构:

```
   mini-survey (1 页)
   ├── 一句话定义: 这个子领域研究什么问题
   ├── 流派图 (taxonomy): 2-4 个流派, 各一句话核心思想     ← L3
   ├── 演进线 (timeline): 痛点→解法→新痛点 的因果链        ← L3
   ├── 关键 benchmark: 大家在什么擂台上比                  ← L1
   ├── 前线 / 我看到的洞: 3 个 (接 9.3)                    ← L3 + 9.3
   └── 我的位置: 我的复现/方向落在哪, 我能补哪个洞
```

> mini-survey 的质量检验 (导师视角): 能不能让一个**没读过这些论文的人, 5 分钟看懂这个领域的地形**? 如果你的 mini-survey 是「论文摘要的列表」, 不及格; 如果它是「带流派、带因果演进、带前线」的**结构化地图**, 合格。

## 3. 把整套接回 Module 9

```
   9.2 (本专题) 产出:
   ┌──────────────────────────────────────────┐
   │ · 引用网 + 中心度  → 落进 9.1 第二大脑       │
   │ · 领域地图 (taxonomy+timeline)             │
   │ · mini-survey                             │
   │ · 标出的「前线 / 3 个洞」                   │──┐
   └──────────────────────────────────────────┘  │ 交棒
                                                   ▼
   9.3 critical-reading-gap: 在前线的具体论文上, 用 6 类 gap 雷达
                             把「洞」坐实成 gap 卡 → idea 卡
                                                   │ 交棒
                                                   ▼
   9.4 experiment-design: 把 idea 卡做成严谨的最小验证实验
                          (地图里的 benchmark 就是你的评测台,
                           奠基作就是你的 baseline)
```

> 注意整条链的复利: 你在 9.2 花两周建的地图, 不是一次性的。它**持续服务**到 9.3 (去哪找 gap)、9.4 (baseline 和 benchmark 是谁)、9.7 (related work 怎么写)。**摸领域是一次投入、长期复利的动作** —— 这也是为什么它值得用一套方法认真做, 而不是随缘读读。

---

## 4. 给博0 的三条现实建议

1. **第一次摸领域, 就摸你已有复现对应的那个子领域** (如偏好优化对你的 `dpo-family`)。你已经动过手, 读文献时体会最深, 也最容易发现「我复现时遇到的问题, 论文里有没有人提」。
2. **别追求覆盖完整。** 漏掉几篇不致命, 没有结构才致命。两周到点就交付。
3. **mini-survey 写完, 主动约导师讲一次。** 这既检验你摸得对不对 (导师会立刻指出你漏的大鱼), 也是 9.8 presentation 的第一次实战。

---

## 5. 本讲小结 (9.2 收口)

- 摸领域要**硬时间盒 2 周**, 否则会无限拖延; 地图画不完, 但两周够画主干道。
- day-by-day: 第 1 周建结构 (锚点→滚雪球→网), 第 2 周填内容 (精读代表作→演进线→前线)。
- 交付物是 **mini-survey** (一页结构化地图 + 几张图), 不是摘要列表。
- 9.2 的地图持续复利到 9.3/9.4/9.7; 摸领域是一次投入、长期回报。

**动手**: 跑完 N1+N2 后, 用 `templates/mini-survey.md` 给「偏好优化」写一份真的 mini-survey, 标出 3 个你看到的洞。然后把这 3 个洞带到 9.3, 用 6 类 gap 雷达逐个验证 —— 这就是 9.2→9.3 的活交棒。
