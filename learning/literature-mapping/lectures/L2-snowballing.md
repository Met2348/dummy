# L2 · 滚雪球检索: 后向找奠基, 前向找前沿, 读懂引用网拓扑

> 40-min lecture · 目标: 掌握 snowballing 这套系统检索法, 学会用引用网的**拓扑**和**中心度**自动区分「必读奠基作」与「可跳过的边缘工作」。

---

## 0. 核心隐喻: 雪球

你站在一篇锚点论文上, 像在雪地里推一个雪球: 顺着引用关系滚, 球越滚越大, 一个子领域的骨架就被你裹进来了。

两个滚动方向, 缺一不可:

```
        更早 (背景/奠基)                          更新 (前沿/最新)
   ◄──────────────────  [你的锚点论文]  ──────────────────►
      后向 BACKWARD                            前向 FORWARD
      "这篇引了谁"                              "谁引了这篇"
      = 它站在谁肩上                            = 谁站在它肩上
      → 顺藤摸到奠基作                          → 顺藤摸到最新进展
      → 补齐你的背景知识                        → 看 gap 是否已被填 (9.3)
```

---

## 1. 后向滚雪球 (backward): 顺引用爬向奠基

- **动作**: 打开锚点论文的 **References (参考文献)**, 找出被它郑重引用的几篇 (尤其 intro 和 method 里反复提的)。把这几篇加入你的集合, 再对它们各自重复。
- **何时停**: 当你反复滚到同几篇老论文 (如 PPO、Bradley-Terry), 说明摸到了这个领域的**地基**, 可以停。这几篇就是奠基作。
- **产出**: 一条从「你关心的新论文」回溯到「领域起点」的脉络。读完它, 你就有了这个领域的**公共语言**。

> 为什么后向这么重要: 新手读最新论文时处处卡, 不是因为论文难, 是因为**缺背景**。后向滚雪球是补背景最快的路 —— 它精准地告诉你「要读懂这篇, 你欠哪几篇的债」, 而不是让你去读整个领域。

## 2. 前向滚雪球 (forward): 顺引用爬向前沿

- **动作**: 在 Semantic Scholar / Google Scholar 打开锚点论文的 **"Cited by" (被引)** 列表, 看谁引用了它, 尤其按时间排最新的、按引用数排最高的。
- **何时停**: 当最新的被引论文开始重复主题、或明显偏离你的问题。
- **产出**: 这个方向**最新做到哪了**。这是找 gap 的前线 —— 9.3 就在这里上场。

> 前向滚雪球回答一个你做研究前必须问的问题: **「我想做的这个 idea, 是不是已经有人做了?」** 不查就动手, 最大风险是辛苦三个月发现两年前有篇一模一样的。前向滚雪球 (尤其查奠基作的最新被引) 是这个风险的保险。

## 3. 工具: 三件套

| 工具 | 干什么 | 链接 |
|---|---|---|
| **Semantic Scholar** | 看一篇的 references + citations, 有 API (本专题工具用它) | semanticscholar.org |
| **Connected Papers** | 输入一篇, 自动画出它的引用邻域图 (可视化滚雪球) | connectedpapers.com |
| **Google Scholar** | "Cited by N" 最全, 适合前向滚 | scholar.google.com |

> 本专题 `src/snowball.py` 把滚雪球**自动化**了: 给它一个子领域的引用网, 它替你滚、替你算中心度。但你要先理解手工怎么滚 —— 工具只是把手工动作加速, 不是替代理解。

## 4. 读引用网的拓扑: 谁重要写在网里

把滚来的论文连成一张有向图 (边 = 「a 引了 b」), 它的**拓扑形状**直接告诉你每篇的地位:

```
    Bradley-Terry(1952) ◄──┐
                            │
        PPO(2017) ◄─────────┼──── InstructGPT(2022) ◄──┐
                            │                           │
                            └──── DPO(2023) ◄───────────┼─── KTO/ORPO/SimPO(2024)
                                    ▲                    │
                                    └─ IPO(2023) ◄───────┘
        (箭头 = 引用方向: 晚的指向早的)
```

读这张图的三个拓扑信号:

| 拓扑特征 | 含义 | 你该怎么做 |
|---|---|---|
| **入度高** (很多箭头指向它) | 被大量后续引用 = **奠基作** | 必读, 优先 |
| **出度高 + 年份新** (它指向很多, 没人指它) | 站在很多肩上、还很新 = **前沿/综合性工作** | 看它怎么综合 → 找 gap |
| **孤立点 / 只被自己组引** | 边缘工作 | 可跳过 |

### 用中心度把「重要性」量化

肉眼看拓扑会主观。用 **PageRank** (对, 就是 Google 排网页那个) 把重要性算成一个数:

$$PR(p) = \frac{1-d}{N} + d \sum_{q \to p} \frac{PR(q)}{L(q)}$$

逐项交代每个符号:
- **$p$**: 一篇论文 (网络里的一个节点)。
- **$PR(p)$**: 论文 $p$ 的 PageRank 分 —— 它在这个子领域里的「重要性」。
- **$N$**: 子领域论文总数。
- **$d$**: 阻尼系数 (damping, 通常 0.85), 一个技术性常数, 防止分数全卡在死胡同, 可先不深究。
- **$q \to p$**: 所有引用了 $p$ 的论文 $q$ (注意: 我们在**反向图**上跑, 让「被引多」得高分)。
- **$L(q)$**: $q$ 的出链数 (它引了多少篇), 用来把 $q$ 的分摊平给它引的每一篇。
- **直觉**: **一篇论文重要, 当且仅当「很多重要的论文都引它」。** 这是个递归定义 —— 被奠基作引, 比被边缘作引更值钱。PageRank 把这个递归解出来。

> `snowball.py` 的 `centrality()` 就在反向图上跑 PageRank, 帮你把一个子领域的所有论文按「真实重要性」排序。**这把「我该先读哪几篇」从主观判断变成了一个排序。**

## 5. 一个重要的 caveat: 中心度是「相对采样集」的

注意一个微妙但重要的点 (N1 会亲眼看到):

> 你滚雪球收进来的论文集是有限的, **中心度是相对这个集合算的**。在「偏好优化」这个滚雪球集合里, DPO 的入度最高 (被 KTO/ORPO/SimPO/IPO 都引), 看起来比 PPO 还「中心」。但放到整个 RL 领域, PPO 显然更奠基。

这给你两个实操提醒:
1. **别把工具算出的中心度当绝对真理。** 它是「在你框定的这片领域内」的相对地位, 受你滚雪球的边界影响。
2. **滚雪球的边界要诚实地交代。** 写 mini-survey 时说清「我框定的是偏好优化这一块」, 而不是假装覆盖了整个对齐领域。这也是 9.3 批判式阅读会盯的点 (范围声明是否诚实)。

## 6. 本讲小结 + 通往 L3

- 滚雪球 = **后向** (顺引用爬向奠基, 补背景) + **前向** (顺被引爬向前沿, 查 gap 是否被填), 缺一不可。
- 重要性是**网络属性**: 用拓扑 (入度/出度/年份) + **PageRank 中心度** 把「先读哪篇」从主观变排序。
- 工具: Semantic Scholar / Connected Papers / Google Scholar; 本专题 `snowball.py` 自动化全过程。
- caveat: 中心度**相对采样集**, 别当绝对真理, 滚雪球边界要诚实交代。

> **下一讲 L3「建领域地图」**: 滚出一张网还不是地图。地图要回答「有几个流派、SOTA 怎么演进、前线在哪」。L3 教你把网整理成 **taxonomy (流派分类) + timeline (演进线)**, 画成一张能上白板的图。
