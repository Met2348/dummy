# L3 · 从文献网到领域地图: 建 taxonomy + 排 SOTA 演进线

> 35-min lecture · 目标: 把 L2 滚出来的一张引用网, 整理成一张**能上白板、给导师讲**的领域地图 —— 有流派 (taxonomy)、有演进线 (timeline)、有前线 (frontier)。

---

## 0. 网 ≠ 地图

L2 给了你一张引用网。但网还不能直接用来汇报 —— 它告诉你「谁引谁」, 没告诉你「这个领域被组织成几块」。**地图 = 给网加上一层人为的组织结构。** 这层结构正是导师和审稿人脑子里装的东西。

一张好的领域地图回答三个问题:

```
   ① 横向: 有哪几个流派?        → taxonomy (按方法分类)
   ② 纵向: 怎么一步步演进的?     → timeline (按时间排 SOTA 线)
   ③ 前线: 现在做到哪、在争什么? → frontier (接 9.3 找 gap)
```

---

## 1. taxonomy (分类法): 把论文分成流派

**taxonomy = 一套把领域里的工作分门别类的维度。** 难点是选对**分类维度**。同一批论文, 用不同维度分, 得到不同的地图:

| 分类维度 | 例 (偏好优化领域) | 什么时候用 |
|---|---|---|
| **按方法机制** | 显式 RM (RLHF-PPO) vs 隐式 RM (DPO/IPO/SimPO) | 最常用, 揭示技术路线之争 |
| **按解决的问题** | 对齐质量 / 训练稳定性 / 数据噪声鲁棒性 | 揭示「大家在攻什么难点」 |
| **按任务/场景** | 单轮对齐 / 多轮 / 推理任务 | 揭示适用边界 |
| **按时间阶段** | 第一代 (RLHF) / 第二代 (DPO 家族) / 第三代 (在线化) | 揭示代际演进 |

> 选维度的判据: **哪个维度能让「流派之间的差异」最清楚、让「同流派内的共性」最明显, 就用哪个。** 好的 taxonomy 让每篇论文「一看就知道该放哪一格」, 且格子之间界限分明。如果一篇论文哪一格都能放, 说明你的维度选错了。

实操: 别只用一个维度。常见做法是**主维度 (如按方法) + 在每个流派内标注它解决的问题**。`field_map.py` 的 `taxonomy()` 按方法族 (family) 分组, 就是主维度。

### 一个 taxonomy 长什么样 (偏好优化)

```
   偏好优化 (Preference Optimization)
   │
   ├─ 流派 A: 显式 reward model (重)
   │     RLHF-PPO(InstructGPT), RLAIF
   │     特点: 训独立 RM, 三段式, 强但重且不稳
   │
   ├─ 流派 B: 隐式 reward / 直接偏好 (轻)  ★ 当前主流
   │     DPO → IPO / KTO / ORPO / SimPO
   │     特点: 无独立 RM, 单阶段, 轻但理论假设更强
   │
   └─ 流派 C: 在线 / 迭代化
         Online-DPO, Iterative-DPO
         特点: 在 B 上加在线采样, 攻"离线偏好分布漂移"
```

## 2. timeline (演进线): 把 SOTA 怎么变好讲成一个故事

光分类是静态的。导师真正想听的是**动态的故事**: 「这个领域是怎么一步步走到今天的?」这就是 SOTA 演进线。

排演进线的方法 —— 找出每一步的「驱动力」:

```
   RLHF (2022)          DPO (2023)            SimPO/KTO (2024)       在线化 (2024+)
   能对齐, 但           "RM 太重" →            "还要 reference        "离线偏好会
   三段式重且不稳        把 reward 隐式化,       model" → 去掉它;       漂移" → 加在线
        │               单阶段                 "偏好有噪声" → 鲁棒化     采样
        └──── 驱动力: 每一步都在拆掉前一步的一个痛点 ────►
```

> 关键技巧: **演进线不是「时间顺序的论文列表」, 而是「痛点 → 解法 → 新痛点」的因果链。** 每一篇 SOTA 推进, 都是在回应前一代的某个具体缺陷。能把这条因果链讲清楚, 你就真的「懂」这个领域了 —— 而且这条链的**末端 (最新的新痛点) 就是你的 gap 富矿**。

这也直接喂给 9.7 写作: 一篇论文的 Related Work 节, 本质就是把这条演进线讲给审稿人听, 并指出「我补的是链末端那个洞」。

## 3. frontier (前线): 标出争论和空白

地图的最后一层, 也是对你最值钱的一层: **当前的前线在哪?** 三种前线信号:

- **公开的争论**: 两个流派在争「哪条路对」(如 DPO vs PPO 谁更强, 至今没定论)。争论处 = 还没有定论 = 有你说话的空间。
- **反复出现的 limitation**: 多篇论文的 future work / limitation 节都提到同一个没解决的问题 (如「偏好数据噪声」「多轮场景」)。**多篇共同承认的洞, 是被验证过的真洞。**
- **空白格**: 你的 taxonomy 表里某个格子是空的 (如「在推理任务上的直接偏好优化」还没人系统做)。空格 = 没人做 = 机会 (但要警惕: 空格也可能是「没人做因为做不了/没意义」, 用 9.3 三筛验证)。

> 9.2 和 9.3 在这里精确交棒: **9.2 的地图标出「前线在哪」(宏观), 9.3 在前线的具体论文上「找出 gap」(微观)。** 没有地图, 你不知道去哪找洞; 有了地图, 找洞就有了方向。

## 4. 把地图画出来 (通往 9.6)

文字 taxonomy 之外, 一张图能让导师 5 秒看懂。两种最有用的领域图:

1. **演进时间线图** (`field_map.plot_timeline`): x 轴年份, y 轴流派, 点大小 = 被引数。一眼看出每个流派何时兴起、谁是巨头。
2. **引用网图** (`snowball.plot_net`): 节点按年份铺开, 大小 = 被引数, 颜色 = 流派。一眼看出网络结构。

> 这两张图是你 9.6「科研绘图」的预演。现在先求「能传达结构」, 9.6 再升级到「出版级美观」。**会画领域图的人, 开题报告和 related work 的图就有了。**

## 5. 本讲小结 + 通往 L4

- 网 ≠ 地图; 地图 = 给网加**人为的组织结构** (流派/演进/前线)。
- **taxonomy**: 选对分类维度 (让流派间差异最清晰); 主维度 + 流派内标问题。
- **timeline**: 不是论文列表, 是「痛点→解法→新痛点」的因果链; 链末端是 gap 富矿。
- **frontier**: 争论 / 多篇共同承认的 limitation / 空白格 —— 接 9.3 找 gap。
- 画成演进时间线图 + 引用网图 (9.6 的预演)。
