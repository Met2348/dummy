# L1 · 相关 vs 因果: 为什么必须干预

> 22-min lecture · 目标: 理解读取工具的根本局限 (相关), 为什么 mech interp 必须走向因果干预。Module 12.3 开篇。

---

## 0. 读取的尽头是干预

M12.2 给了读取工具 (探针/logit lens), 但它们只证**相关** (L4 反复强调)。这一讲讲清「为什么相关不够、必须干预」, 然后 L2 给出 mech interp 最核心的干预工具: **activation patching**。

---

## 1. 相关的三个失败模式

读取工具 (看激活/权重) 会在三种情况误导你:

- **旁路编码**: 信息被编码在某层 (探针读得出), 但模型计算时没用它 → 你以为重要, 其实是旁观者。
- **冗余路径**: 多条路径都携带某信息, 你看到一条就以为是「唯一」, 漏了其它。
- **混淆**: A 和行为相关, 但其实是 C 同时导致 A 和行为 (A 不因果)。

> 这些都是「相关 ≠ 因果」的具体形态。读取工具看到的是「什么和行为一起出现」, 不是「什么导致行为」。**要分清, 只有一个办法: 动手改它, 看行为变不变。**

---

## 2. 因果的黄金标准: 干预 (intervention)

因果推断的黄金标准是**干预实验** (randomized intervention): 不是观察 A 和 B 一起出现, 而是**主动改变 A, 看 B 变不变**。

```
   观察 (相关): A 和 B 一起出现           → A 可能因果, 也可能不
   干预 (因果): 我改变 A (其它不变), B 变了 → A 因果影响 B
               我改变 A, B 没变           → A 不因果影响 B
```

> 这是科学方法的核心 (你的实验设计 M9.4 也是这个)。mech interp 把它用到网络内部: **改变一个激活 (干预), 看输出变不变 (因果)**。神经网络有个独特优势: 它是**完全可控的实验对象** —— 你能精确地改任意激活、重跑、观察, 比真实世界的因果实验干净得多。

---

## 3. 在网络里怎么「干预」

网络里干预激活有几种方式 (L2/L3 展开):
- **activation patching**: 把另一个运行 (clean) 的激活贴进当前运行 (L2)。
- **ablation**: 把某激活置零 / 替换成均值 (L3)。
- **方向干预**: 沿某个 feature 方向加减 (调控某概念)。

> 共同点: **主动修改内部激活, 看输出/行为的因果变化**。这把「逆向工程」从「看着猜」(相关) 升级成「做实验」(因果)。你 N1 会亲手做 activation patching, 因果定位「哪个位置携带答案信息」。

---

## 4. 为什么神经网络是因果实验的天堂

mech interp 能做严格因果实验, 因为网络给了真实世界给不了的控制力:
- **完全可观测**: 每个激活都能读 (M12.2)。
- **完全可干预**: 每个激活都能改 (本专题)。
- **完全可重复**: 同输入同输出 (确定性), 干预效果可精确测量。
- **可任意对照**: clean/corrupt 随便造, 单变量对照 (接 M9.4)。

> 这是 mech interp 相对其它科学的奢侈: **它能做真正的因果实验**, 不是观察性研究。你 EE/实验背景对「控制变量 + 干预」的直觉在这里是核心生产力。**别浪费这个优势停在「看激活」(相关), 要用干预拿因果。**

---

## 5. 本讲小结 + 通往 L2

- 读取工具只证**相关**; 三个失败模式: 旁路编码 / 冗余路径 / 混淆。
- 因果黄金标准 = **干预**: 主动改 A 看 B 变不变 (科学方法, 接 M9.4)。
- 网络里干预激活: patching / ablation / 方向干预 (L2/L3)。
- 网络是**因果实验的天堂**: 完全可观测/可干预/可重复/可对照 (你的实验直觉=生产力)。

> **下一讲 L2「activation patching」**: mech interp 最核心工具 —— 把 clean 激活贴进 corrupt 运行, 因果定位哪个组件携带行为所需信息。

**动手**: 去 `N1`, 用 activation patching 因果定位「玩具任务里哪个位置/层负责答案」: 你会看到只有某个位置 patch 能恢复行为 (= 它因果负责)。
