# L5 · 方差与显著性: 戳穿「单种子结论」, 让差距站得住

> 40-min lecture · 目标: 学会用**多种子 + 误差棒 + 置信区间 + 显著性检验 + 效应量** 把「看起来更好」变成「统计上站得住」。这是经验研究诚实性的最后一道、也是新手最常省略的一道关。

---

## 0. ML 头号自欺: 用一次结果当结论

你跑一次 Robust-DPO 得 0.55, 跑一次 DPO 得 0.52, 于是写「Robust-DPO 提升 3 点」。**这句话很可能是错的。** 因为如果你换个种子重跑, DPO 可能是 0.54, Robust 可能是 0.53 —— 差距反了。

> **单种子的一次结果, 是从一个有方差的分布里抽的一个样本。** 拿一个样本当总体均值, 是统计学最基础的错误。ML 论文里它无处不在, 因为训练贵、大家懒得多跑种子。而**认真报方差**, 恰恰是让你的结果区别于「不可复现的灌水」的关键 (回忆 9.3 复现 gap)。

本讲的运行例子在模拟器里埋了 `SEED_STD ≈ 0.018` 的种子噪声 —— 和真实 ML 训练的种子间波动量级相当。你的任务: 在这个噪声下, 判断「13 点差距」是真的, 「3 点差距」可能是假的。

---

## 1. 第一道关: 报均值 ± 方差, 而不是一个数

最低门槛: 每个设置跑 $N$ 个种子, 报 **均值 ± 标准差**。

$$\bar{x} = \frac{1}{N}\sum_{i=1}^{N} x_i \qquad s = \sqrt{\frac{1}{N-1}\sum_{i=1}^{N}(x_i - \bar{x})^2}$$

逐项交代:
- **$x_i$**: 第 $i$ 个种子跑出的 win-rate。
- **$N$**: 种子数。
- **$\bar{x}$**: 样本均值 —— 你真正该报告的「性能」(而非某一次)。
- **$s$**: 样本标准差 (注意分母是 $N-1$ 不是 $N$, 叫 Bessel 校正, 让小样本的方差估计无偏)。它量化「换个种子结果会抖多少」。

报告形式: `Robust-DPO: 0.531 ± 0.021`。**那个 ±0.021 和 0.531 一样重要** —— 它告诉读者你的差距有没有意义。

### 误差棒用 std 还是 SEM?

画图时误差棒有两种, 别搞混:
- **标准差 $s$**: 「单次实验结果的离散程度」。
- **标准误 $\text{SEM} = s/\sqrt{N}$**: 「均值估计的不确定性」。$N$ 越大 SEM 越小。

> 比较两个方法的**均值**时, 用 **SEM** (或置信区间) 更合适, 因为你比的是均值。但务必在图注里**写清楚误差棒是什么** —— 不写, 审稿人无法判断, 这也是 9.3 攻击清单会盯的点。

## 2. 第二道关: 置信区间 (bootstrap)

均值±std 还不够直观地回答「差距可信吗」。更好的是**置信区间 (CI)**: 「真实均值有 95% 把握落在 [lo, hi]」。

ML 指标常常不服从正态分布, 所以别硬套教科书的正态 CI 公式。用 **bootstrap** —— 一个不假设分布、纯靠重采样的办法:

```
   bootstrap 95% CI 的算法 (stats.py 的 bootstrap_ci 就这么做):
   1. 手头有 N 个种子结果 x_1..x_N
   2. 有放回地从中抽 N 个, 算一次均值 → 得到一个 "bootstrap 均值"
   3. 重复第 2 步 10000 次 → 得到 10000 个 bootstrap 均值
   4. 取这 10000 个的 2.5% 和 97.5% 分位数 → 就是 95% CI
```

> 直觉: bootstrap 用「从你的样本里反复重采样」来**模拟「把整个实验重做很多次」**。如果两个方法的 CI **不重叠**, 差距很可能是真的; 重叠很多, 就别声称有差距。这比单看 ±std 更可靠, 且对分布无要求。

---

## 3. 第三道关: 显著性检验 —— 差距是真的吗 (p 值)

把「差距可信吗」正式化: 用**假设检验**算 p 值。回到 L1 的零假设 H0「两法无差异」。

**Welch's t 检验** (不假设两组方差相等, 比标准 t 检验更稳妥) 给你一个 **p 值**:

> **p 值 = 「假如 H0 成立 (两法真没差), 看到当前这么大 (或更大) 差距的概率」。** p 越小, 数据越不像「纯噪声巧合」, 你越有底气拒绝 H0。惯例阈值 p < 0.05。

我们模拟器的结果 (8 种子):
- noise=0.4: Robust vs DPO, **p ≈ 0.0000** → 差距极不像巧合, 拒绝 H0, 这个 13 点是真的。
- noise=0.0: Robust vs DPO, **p ≈ 0.14** → 差距很可能只是种子噪声, **不能**声称有差异。

**这正是交互效应 (L4) 在统计上的体现**: 同一对方法, 高噪声下显著、低噪声下不显著。

### p 值的三个致命误区 (务必记住)

| 误区 | 真相 |
|---|---|
| 「p=0.04 说明效应很大」 | **错**。p 只说「不像巧合」, 不说「效应多大」。大样本下微小差距也能 p<0.05 |
| 「p=0.06 说明没效应」 | **错**。可能只是种子不够 (检验力不足)。「不显著」≠「证明无差异」 |
| 「我跑 20 个对比, 挑 p<0.05 的报」 | **p-hacking / 多重比较**。跑 20 个, 平均会有 1 个纯靠运气 p<0.05。这是造假 |

> 第三条最危险, 因为它常是无意的。如果你做了多个对比 (消融矩阵里有很多格), 要么预先指定**主对比**, 要么做**多重比较校正** (如 Bonferroni: 阈值除以对比数)。**别在矩阵里捞 p<0.05 然后编故事** —— 这就是 9.3 攻击清单 Q5 (cherry-picking) 的统计版。

---

## 4. 第四道关: 效应量 —— 差距有多大 (Cohen's d)

p 值只回答「是不是真的」, 不回答「**有多大、值不值得在意**」。补上**效应量 (effect size)**:

$$d = \frac{\bar{x}_A - \bar{x}_B}{s_{\text{pooled}}}$$

逐项交代:
- **$\bar{x}_A - \bar{x}_B$**: 两组均值差 (原始差距)。
- **$s_{\text{pooled}}$**: 两组合并的标准差 (把差距用「噪声尺度」归一化)。
- **$d$**: Cohen's d —— 「差距相当于几个标准差」。经验阈值: |d|≈0.2 小、0.5 中、0.8 大。

> 为什么需要它: 一个 **p<0.001 但 d=0.05** 的结果是「**统计显著但实践无关**」—— 你跑了 10000 个种子, 0.1 点的差距也会显著, 但没人会在意 0.1 点。反过来 d 很大但 p 不显著, 可能是种子太少。**p 和 d 要一起看**: p 管「真不真」, d 管「大不大」。好结论需要**既真又大** (我们 noise=0.4 的 d≈6, 又真又大)。

---

## 5. 到底要几个种子?

实用经验 (不是铁律):
- **MVE 阶段** (L2): 3 个种子, 够算个粗 std 判断「差距是否淹在噪声里」。
- **正式结果 / 投稿**: 至少 5 个, 理想 8-10 个。波动大的任务 (RL!) 要更多。
- **判断够不够的土办法**: 加种子直到你的**结论 (谁赢/显著性) 不再随加种子翻转**。如果第 4 个种子让结论反过来, 说明远远不够。

> RL / RLHF 类实验种子方差出了名地大 (这正是它复现难、是复现 gap 富矿的原因, 见 9.3)。如果你做对齐/RL, **把多种子当默认, 不是可选**。一篇报了 8 种子 error bar 的对齐论文, 可信度碾压报单种子的。

## 6. 本讲小结 (9.4 收口) + 通往 9.5

- 单种子一次结果当结论 = ML 头号自欺; 它是从有方差分布里抽的一个样本。
- 四道关层层加码: **均值±方差** → **bootstrap 置信区间** → **显著性检验 (p, 真不真)** → **效应量 (d, 大不大)**。
- p 值三误区: 不等于效应大 / 不显著≠无差异 / 别 p-hack 捞结果。
- 种子数: MVE 3 个, 正式 ≥5-10; RL 要更多; 加到结论不再翻转为止。

> **9.4 整条线回顾**: 可证伪假设 (L1) → 最小验证 (L2) → 公平对照 (L3) → 消融矩阵+交互 (L4) → 方差显著性 (L5)。你现在能把一个 idea 卡变成一套**严谨、可信、经得起审稿人追问**的实验。
