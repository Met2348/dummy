# L1 · 可证伪假设: 把一个 idea 变成实验能杀死的陈述

> 35-min lecture · 目标: 学会把模糊的「我觉得 X 可能更好」变成一个**可证伪 (falsifiable)** 的假设 —— 一个实验能明确地杀死它的陈述。这是经验研究的起点, 也是新手最常跳过的一步。

---

## 0. 为什么从「假设」开始, 而不是「跑实验」

新手做研究的典型顺序: 有个 idea → 立刻开始跑一堆实验 → 看哪个数好看 → 反推一个故事。这叫 **HARKing** (Hypothesizing After Results are Known, 看到结果再编假设), 是经验科学的大忌。

为什么大忌: 如果你先跑 20 个设置, 再挑数最好的那个编故事, 你几乎**必然**能找到一个「显著」结果 —— 哪怕底层啥都没有 (L5 会量化这个: 跑得够多, 假阳性是必然的)。你骗过了审稿人, 也骗过了自己。

正确顺序: **先写下一个具体的、可被推翻的假设, 再设计专门检验它的实验。**

> 这背后是 Popper 的**证伪主义 (falsificationism)**: 科学陈述的标志不是「能被证实」, 而是「**能被证伪**」。「Robust-DPO 在某些情况下可能有帮助」不是科学陈述 —— 它怎么都对, 没有任何实验结果能推翻它。「Robust-DPO 在 40% 标签噪声下, win-rate 比 DPO 高至少 5 个点」才是 —— 跑出来差距不到 5 点, 它就死了。**一个不能被任何结果推翻的假设, 是没用的假设。**

---

## 1. 可证伪假设的解剖

一个合格的假设, 必须能填进这个句式:

```
   在 [设定] 下, [自变量] 从 [A] 变到 [B] 会让 [因变量] [方向] [幅度],
   相对 [基线/对照]。
```

逐个交代每个槽位 (用贯穿本专题的 Robust-DPO 例子填):

| 槽位 | 含义 | 例 |
|---|---|---|
| **设定 (setting)** | 实验的边界条件, 限定结论范围 | 「在 7B 模型 + 偏好对齐任务下」 |
| **自变量 (IV)** | 你主动改变的东西 (因) | method: DPO → Robust-DPO |
| **因变量 (DV)** | 你测量的东西 (果) | win-rate (对齐质量) |
| **方向 (direction)** | 升 / 降 | 升高 |
| **幅度 (magnitude)** | 至少多少才算数 (可选但强烈建议) | ≥ 5 个点 |
| **基线 (baseline)** | 跟谁比 | 普通 DPO (同数据同超参) |
| **调节变量 (moderator)** | 效应在什么条件下出现/消失 | **只在高噪声 (40%) 下** |

把它们拼起来, 我们的运行假设是:

> **H1**: 在 7B 偏好对齐任务、标签噪声为 40% 时, 用 Robust-DPO 替代普通 DPO, win-rate 会提高 ≥ 5 个点。在无噪声 (0%) 时, 两者无显著差异。

这个假设的好处:
- **具体到能被杀死**: 跑出来 40% 噪声下差距只有 2 点 → H1 被推翻。
- **带调节变量**: 它不只说「Robust 更好」, 而说「Robust 的好处**取决于**噪声」—— 这是一个**交互效应假设**, 比单调的「A>B」信息量大得多 (L4 详讲)。
- **预先承诺**: 你在跑之前就写下了「≥5 点」和「无噪声时无差异」, 没法事后挪动标准。

## 2. 零假设 (H0) 与备择假设 (H1): 你其实在试图杀死 H0

统计检验的逻辑是反着来的, 这点反直觉但必须理解:

- **零假设 H0 (null)**: 「**什么都没发生**」—— Robust-DPO 和 DPO 的 win-rate 没有差异, 看到的任何差距都只是种子噪声的巧合。
- **备择假设 H1 (alternative)**: 「有差异」—— 就是你真正相信的那个。

研究的实际动作是: **假设 H0 为真, 然后看「数据有多不像 H0」。** 如果数据在 H0 下极其不可能出现 (p 值很小), 你就**拒绝 H0**, 转而支持 H1。

```
   你相信:        H1 (Robust-DPO 真的更好)
   你检验:        H0 (其实没差, 都是噪声)
   实验做的事:    收集证据试图"杀死 H0"
   p 值回答:      "如果 H0 真的成立, 看到这么大差距的概率有多小?"  小 → 杀死 H0
```

> 为什么绕这一圈: 因为你**无法直接证明** H1 (证明「有效应」要排除所有其它解释)。但你**可以反驳** H0 (证明「没效应」这个具体假设和数据矛盾)。这又是证伪主义: 科学进展靠**反驳**, 不靠证实。L5 会把 p 值算出来。

## 3. 新手最常见的四种「坏假设」

| 坏假设 | 病在哪 | 改成 |
|---|---|---|
| 「Robust-DPO 更好」 | 没设定、没幅度、没基线 —— 怎么都对 = 不可证伪 | 「40% 噪声下 win-rate ≥+5 点 vs DPO」 |
| 「我的方法在多数情况下有提升」 | 「多数情况」事后可挪动 = HARKing 温床 | 预先指定**哪些**情况 + 阈值 |
| 「探索 X 对 Y 的影响」 | 这是**研究问题**不是**假设**; 没有方向, 杀不死 | 先变成有方向的预测 |
| 「证明 Robust-DPO 有效」 | 「证明有效」不可达; 科学只能**反驳 H0** | 「在设定 S 下拒绝 H0」 |

> 一个自检: 把你的假设给同学看, 问他「**什么样的实验结果会让这句话变成假的?**」。如果他答不上来 (任何结果都符合), 你的假设不可证伪, 回去重写。**能清楚说出「我会怎么被打脸」, 才算有了一个真假设。**

## 4. 假设 → 整个 9.4 的导航

一个好假设直接决定后面每一步要做什么:

```
   H1: 40%噪声下 Robust-DPO win-rate ≥ DPO +5点; 0%噪声下无差异
        │
        ├─ 自变量 method, noise        → L4 排成消融矩阵 (factorial)
        ├─ 基线 = 同设置的 DPO          → L3 保证对照公平 (控混淆)
        ├─ "最小先验证哪条"             → L2 最小验证实验 (先测 40% 这一格)
        └─ "≥5点 / 无差异 怎么判定"      → L5 方差 + 显著性检验
```

> 假设是实验设计的**宪法**: 后面所有设计决策, 都要回答「这能不能更好地检验我的假设」。没有假设, 你的实验就没有靶子, 跑出一堆数也不知道说明了什么。

---

## 5. 本讲小结 + 通往 L2

- 先写**可证伪假设**, 再设计实验; 反过来 (HARKing) 是自欺。
- 假设要填满: 设定 / 自变量 / 因变量 / 方向 / 幅度 / 基线 / (调节变量)。
- 统计上你检验的是**零假设 H0** (「没差异」), 用数据去**杀死它**; 这是证伪主义。
- 自检: 「什么结果会让这句话变假?」答不上 = 不可证伪, 重写。

> **下一讲 L2「最小验证实验」**: 有了假设, 别急着跑全套消融。先用一个**最小、最快**的实验验证「核心机制到底存不存在」。L2 教你怎么设计 MVE, 一周内拿到第一个 go/no-go 信号, 避免在一个错的方向上烧三个月。

**动手**: 拿你从 9.3 带来的一张 idea 卡, 用 `templates/hypothesis-card.md` 把它写成一个可证伪假设。逼自己填上「幅度」和「什么结果会推翻它」两栏 —— 这两栏最难, 也最值钱。
