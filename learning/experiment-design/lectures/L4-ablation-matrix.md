# L4 · 消融矩阵: 系统拆解「哪个部分、在什么条件下起作用」+ 交互效应

> 40-min lecture · 目标: 学会用**消融 (ablation)** 和**全因子设计 (factorial design)** 系统地拆解你的方法; 理解并检测**交互效应 (interaction)** —— 新手最常忽略、却最有信息量的现象。

---

## 0. 消融是什么, 为什么它是经验论文的脊梁

> **消融实验 (ablation study)**: 系统地**移除或改变**方法的某个组件 / 某个条件, 观察性能怎么变, 从而判断「这个组件 / 条件到底贡献了多少」。

"ablation" 这个词借自神经科学 (切除大脑某区域看功能损失)。在 ML 里它回答审稿人最尖锐的问题: **「你方法里花里胡哨那么多东西, 到底是哪个真正起作用?」** 一篇没有消融的方法论文, 基本会被拒 —— 因为读者无法知道你的提升来自核心创新, 还是来自某个无关的工程 trick。

```
   完整方法 = 创新组件A + 组件B + 一堆工程trick
        │  消融: 一次拿掉一个, 看性能掉多少
        ▼
   去掉A → 掉 8 点   (A 很重要!)
   去掉B → 掉 0.2 点 (B 几乎没用, 也许能删掉简化方法)
   去掉trick → 掉 5 点 (糟糕: 提升主要来自 trick 而非创新, 故事要重写)
```

---

## 1. 消融的两个维度: 组件消融 vs 条件消融

| 类型 | 拿掉/改变什么 | 回答 | 例 |
|---|---|---|---|
| **组件消融** | 方法内部的一个模块 | 「哪个组件贡献了性能?」 | Robust-DPO 去掉鲁棒项 → 退化成 DPO |
| **条件消融 (扫描)** | 一个外部条件 | 「方法在什么条件下还成立?」 | 噪声 0/10/20/30/40% 扫一遍 |

我们的运行例子两者都要:
- **组件消融**: Robust-DPO vs DPO (= Robust-DPO 抽掉鲁棒机制)。证明「是鲁棒机制起效」。
- **条件消融**: noise ∈ {0, 0.2, 0.4} 扫描。证明「鲁棒机制的效果**取决于噪声**」。

这两个维度交叉, 就构成一个**矩阵**。

---

## 2. 全因子设计 (factorial design): 把变量交叉成矩阵

最朴素的消融做法叫 **OFAT (one-factor-at-a-time, 一次只变一个因素)**: 固定其它, 单独扫 method; 再固定其它, 单独扫 noise。问题是 **OFAT 看不到交互效应** (下一节)。

正确做法是**全因子设计**: 把每个因子的每个水平**两两组合**, 排成矩阵, 每格都跑。

```
   factorial: method × noise  (2 × 3 = 6 格, 每格 ×N 种子)

              noise=0.0    noise=0.2    noise=0.4
   DPO          0.62         0.52         0.40        ← DPO 随噪声暴跌
   Robust-DPO   0.62         0.60         0.53        ← Robust-DPO 扛得住
              ─────────    ─────────    ─────────
   两法之差:     ~0           +0.08        +0.13       ← 差距随噪声拉大!
```

逐项交代怎么读这个矩阵:
- **行**: 同一个方法在不同噪声下 → 看「这个方法对噪声多敏感」。DPO 行从 0.62 暴跌到 0.40。
- **列**: 同一个噪声下不同方法 → 看「这个条件下谁更好」。noise=0.4 列, Robust 完胜。
- **最后一行 (差之差)**: 两法差距怎么随噪声变 → **这正是交互效应**, 整个矩阵最有信息量的一行。

> 全因子的代价是格子多 (因子和水平一多就组合爆炸)。所以: **MVE 阶段 (L2) 只跑矩阵的四个角** (2 方法 × 2 极端噪声), 确认交互存在; 确认后才填满中间格子画完整曲线。先角后面, 别一上来填满。

## 3. 交互效应 (interaction): 本讲的核心, 也是新手的盲区

> **交互效应**: 一个因子的效果, **取决于**另一个因子的水平。形式化地: A 对结果的影响, 在 B 的不同水平下不一样。

我们的整个故事就是一个交互效应:

```
   没有交互 (新手默认假设):           有交互 (真相):
   "Robust-DPO 总是比 DPO 好 X 点"     "Robust-DPO 的好处取决于噪声"

   win                                win
   ↑   Robust ────────                ↑           Robust ───
   │   DPO    ────────                │   两法重合 ╱
   │   (两条平行线)                    │          ╱ DPO 暴跌
   └──────────────► noise             └──────────────► noise
       差距恒定 = 无交互                  差距随noise张开 = 有交互
```

为什么交互效应这么重要:
- **它信息量远大于主效应。** 「Robust-DPO 平均高 3 点」(主效应) 是个平淡结论; 「Robust-DPO 在高噪声下高 13 点、低噪声下没差」(交互) 是个**有机制、有故事、有适用边界**的结论 —— 后者才是好论文。
- **它定义了你方法的「适用条件」。** 交互告诉你「我的方法在什么时候值得用」(高噪声场景), 这对实践者极有价值, 也让审稿人信服你**理解**自己的方法, 而不只是刷了个数。
- **OFAT 设计根本看不到它。** 如果你只在 noise=0 下比 method (OFAT), 会得出「两法没差, Robust-DPO 没用」的**错误结论** —— 因为你恰好踩在交互曲线的重合端。**这就是为什么必须 factorial。**

### 怎么量化交互效应

最简单的量化: 看「差之差 (difference-in-differences)」是否非零。

$$\text{Interaction} = \big(\bar{y}_{\text{Robust}, \text{high}} - \bar{y}_{\text{DPO}, \text{high}}\big) - \big(\bar{y}_{\text{Robust}, \text{low}} - \bar{y}_{\text{DPO}, \text{low}}\big)$$

逐项交代:
- **$\bar{y}_{m, n}$**: 方法 $m$ 在噪声 $n$ 下的平均 win-rate。
- **第一个括号**: 高噪声下两法的差距 (≈ +0.13)。
- **第二个括号**: 低噪声下两法的差距 (≈ 0)。
- **两者相减 (Interaction)**: 差距随条件变了多少 (≈ +0.13)。**非零 = 存在交互**; 越大交互越强。
- 严格地说还要检验它是否**统计显著** (L5 的双因素 ANOVA / 对差之差做检验), 但先理解这个量的含义。

## 4. 排消融矩阵的实操清单

设计一个消融矩阵时, 逐项决定:

1. **列出所有因子**: 你假设涉及的自变量 (method, noise) + 你怀疑有影响的 (规模? 数据集?)。
2. **每个因子选水平**: 别贪多。method 2 个、noise 先 2 端后补中间。水平数直接决定矩阵大小。
3. **决定全因子还是部分**: 因子 ≤3 个、水平少 → 全因子。因子多 → 先全因子跑关键 2 个, 其它固定。
4. **标出你最关心的格子 / 对比**: 通常是「交互最强的那两个角」。
5. **每格的种子数**: 留给 L5 (够算方差)。
6. **预先画出「预期的矩阵长什么样」**: 这是 L1 假设的延伸 —— 跑之前就知道「如果假设对, 矩阵该是什么形状」。

> 用 `templates/ablation-matrix.md` 把上面 6 步填一遍。**先在纸上把矩阵设计完整, 再去跑** —— 跑之前矩阵的每一格都该有一个明确的「我为什么要这一格」。没想清就跑, 会得到一堆不知道说明什么的数。

## 5. 本讲小结 + 通往 L5

- **消融**是经验论文的脊梁: 系统移除组件/扫描条件, 判断「哪个部分、在什么条件下起作用」。
- 两个维度: **组件消融** (Robust vs DPO) + **条件消融** (扫噪声); 交叉成**矩阵**。
- **全因子设计** (factorial) 而非 OFAT, 因为只有它能看到**交互效应**。
- **交互效应**是核心: 「方法的好处取决于条件」比「平均高 X 点」信息量大得多, 定义了适用边界; OFAT 会让你错过它甚至得出反结论。
- 量化交互 = 差之差; 排矩阵先列因子/选水平/标关键对比/预画形状。

> **下一讲 L5「方差与显著性」**: 你的矩阵每格现在只有一个数。但「Robust 高 13 点」可能有一半是种子噪声。L5 教你加种子、报 error bar、算 p 值和效应量, 让矩阵里的每个差距都**站得住**, 并戳穿「单种子结论」这个 ML 头号自欺。

**动手**: 去 `N1-design-an-ablation.ipynb`, 用 `experiment.py` 跑完整的 method×noise 全因子矩阵, 亲手算出交互效应的「差之差」, 并对比一下: 如果你只用 OFAT (只在 noise=0 比 method), 会得出什么错误结论。
