# 05 · Transformer 核心架构（面试高频度：全场最高）

如果说八股文里有一个类别是"不会就直接挂"，那就是 Transformer——它是 2026 年几乎所有 AI/算法岗（无论是搜广推、NLP、还是纯研究岗）面试的必考模块，而且是**被追问最深的模块**：面试官往往会顺着一道基础题一路问到具体工程实现（比如从"什么是 KV cache"追到"MQA/GQA 怎么改、FlashAttention 到底加速了什么"）。对 NLP/LLM 方向的 PhD 申请者/研究岗候选人尤其如此——这部分几乎等价于你的专业课水平线，答不出细节会被直接归为"背过论文标题但没有真正读懂"。本讲义覆盖的是比"自注意力复杂度""为什么除以√d"这类入门题更深一层的问题：MHA 的局限与 MQA/GQA 的具体机制、FlashAttention 的原理、位置编码的数学形式，这些正是目前大模型岗位追问链里最容易见真章的地方。

## 追问链深挖 1：MHA 的瓶颈 -> MQA -> GQA，一路问到底

**Q1（基础问题）**：标准多头注意力(MHA)在实际部署里有什么局限？

**A1**：MHA 在训练阶段没什么明显短板，但在自回归推理阶段会暴露瓶颈：每个头都有自己独立的 K、V 投影，解码时要为每个头都缓存一份 KV cache；头数 H 越多，KV cache 占用的显存、以及每一步生成时读取这份 KV cache 所需要的显存带宽开销就越大。而自回归解码每一步本质上是访存密集型(memory-bound)而不是计算密集型的操作——算力早就够用，真正卡脖子的是把 KV cache 从显存搬到计算单元的带宽，这就是 MQA/GQA 被提出来专门解决的问题。

**面试官追问 Q2**：那 MQA 具体是怎么改的？

**A2**：MQA(Multi-Query Attention) 把 K 和 V 的投影从"每个头一份"直接砍成"全部 query 头共享同一份 K 头和 V 头"——query 这一侧还是保留 H 个独立的头各自做注意力计算，但 K、V 的线性投影矩阵全局只有一份，所有头复用同一组 K/V。这样 KV cache 的大小和读取带宽直接从"正比于头数 H"降到"等价于只有 1 个头"，大幅缓解了访存瓶颈。代价是 K/V 的表达能力被压缩了——不同 query 头本来可以各自学一套定制的键值子空间，现在被迫共用同一套，实践中会带来一定的效果下降。

**面试官再追问 Q3**：既然 MQA 效果会掉点，为什么 Llama2/Mistral 这些主流模型最后选的是 GQA 而不是直接用 MQA？GQA 具体怎么做到"折中"？

**A3**：GQA(Grouped-Query Attention) 把 MQA "一刀切到底"的做法变成一个可调节的连续谱：先把 H 个 query 头分成 G 组，同一组内的 query 头共享一份 K/V，不同组各自拥有独立的 K/V 投影。两个极端正好对应我们已经讨论过的两种方案——G=H 时每个 query 头都有自己的 K/V，就是标准 MHA；G=1 时所有 query 头共享同一份 K/V，就是 MQA。工程上选一个中间的 G（比如 32 个 query 头分成 8 组），就能用远小于 MHA 的 KV cache 开销，换回比纯 MQA 更多的键值表达多样性——这正是"用可控的质量损失换取可控的显存带宽收益"，也是它比"MQA 直接砍到 1"更受欢迎的原因。需要强调的是，MQA/GQA 省的核心是 KV cache 的显存带宽而不是注意力本身的计算量(FLOPs)：每个 query 头依然要对共享的 K/V 完整算一遍注意力分数，只是 K/V 这份"原材料"的制造和存储成本被摊薄了。

## 追问链深挖 2：FlashAttention——它到底是不是"近似算法"

**Q1（基础问题）**：FlashAttention 解决的是什么问题？

**A1**：标准注意力实现要显式算出并写回一个 T×T 的完整注意力矩阵到 GPU 的 HBM(显存)，而现代 GPU 的算力(FLOPS)远大于显存带宽——大部分注意力计算其实是访存瓶颈(memory-bound)而不是计算瓶颈，尤其是 softmax 这一步，算得很少却要读写整个 T×T 矩阵。FlashAttention 通过分块(tiling)，把 Q/K/V 切成小块搬到更快的片上 SRAM 里完成计算，全程不把完整的 T×T 矩阵落盘到 HBM，从而把显存读写量从 O(T²) 降到接近线性，同时省显存又提速。

**面试官追问 Q2**：这是不是一种近似算法？会不会牺牲精度换速度？

**A2**：不是。这是它最容易被记错的一点——FlashAttention 是**IO 感知(IO-aware)的精确注意力算法**，计算出来的结果和标准的 softmax(QK^T/√d)V 在数学上完全一致，没有做任何近似或截断。它加速的手段纯粹是重排计算顺序、减少 HBM 读写次数，而不是像线性注意力、稀疏注意力那样通过改变计算本身（丢弃部分注意力权重、用核函数近似 softmax）来降低复杂度。这两类方法（IO 重排 vs 近似算法）经常被面试者混为一谈，是本类别里最该提醒自己的一条。

**面试官再追问 Q3**：既然不落盘完整矩阵、只能看到局部的分块，softmax 需要看到一整行才能归一化，这是怎么解决的？

**A3**：靠"在线 softmax(online softmax)"技巧：FlashAttention 把 Q 按行分块、K/V 按列分块，依次把每一小块搬进 SRAM，在片上算出这一小块的 QK^T、局部的 softmax 统计量（当前看到的最大值和指数和），然后每处理完一个新的 K/V 块，就用 running max 和 running sum 增量地更新之前所有块累积下来的归一化统计量——这套增量更新在数学上被证明和一次性看到整行再做 softmax 完全等价，所以不需要真的一次性看到整行。反向传播时同理，它不保存完整注意力矩阵，只保存每行的 logsumexp 统计量，需要梯度时在 SRAM 里用这些统计量重新计算(recomputation)出对应的注意力权重，用少量额外的 FLOPs 换取大幅减少的显存读写——这也是为什么"重新计算换省显存"在 FlashAttention 里是划算的：GPU 算力远比带宽富余。

## 其余题目速览

完整标准答案见 `src/ai_qa/qbank_transformer.py`（id 前缀 `ai-trf-`）。

- **Transformer 整体结构**：embedding+位置编码 -> N 层(attention + 残差 + LayerNorm + FFN + 残差 + LayerNorm) -> 输出层，三件套反复堆叠。
- **Q/K/V 怎么来**：同一输入分别乘 W_Q/W_K/W_V 三个独立可学习矩阵得到，Q="在找什么"、K="有什么"、V="实际内容"。
- **为什么需要 causal mask**：训练用 teacher forcing 并行算 loss，不加掩码会泄露未来 token，导致训练/推理行为不一致。
- **RoPE 的数学形式**：把 Q/K 的每两维看成一个二维子空间，按位置乘旋转矩阵，点积后只依赖相对位置 m-n。
- **ALiBi 的机制**：不用位置 embedding，直接在注意力分数上加与距离成正比的线性偏置，外推性好。
- **绝对 vs 相对位置编码**：绝对编码学的是"这个位置长什么样"，相对编码学的是"差几步"，后者天然外推。
- **FFN 的作用**：逐位置的两层 MLP，升维再降维，负责非线性特征加工，参数量占比约三分之二。
- **残差连接的作用**：output = x + Sublayer(x)，给梯度提供恒等捷径，支撑深层网络稳定训练。

## 常见误区清单

1. **"FlashAttention 是近似算法"——错。** 它是 IO 感知的精确计算重排，结果和标准 attention 数学等价，这是本类别最容易被面试官当场纠正的误区，务必说清楚"精确"二字。
2. **"MQA/GQA 主要省的是计算量"——不准确。** 它们省的核心是 KV cache 的显存带宽/存储，注意力本身的 FLOPs 并没有显著减少。
3. **把 GQA 和 MQA 混为一谈**——GQA 是可调节的分组方案，G=1 才等价于 MQA，二者不是同一个东西。
4. **"RoPE 是纯粹的相对位置编码"这个说法不严谨**——它是通过对绝对位置做旋转操作、来间接实现相对位置的效果，注意描述的精确性，面试官很可能会在这里追问"到底算绝对还是相对"。
5. **把 causal mask 和 padding mask 当成同一回事**——前者防止看到未来 token，后者屏蔽 padding 占位符，二者可以同时使用但解决的是不同问题。
6. **KV cache 的显存工程细节不属于这一类**——KV cache 具体占多少显存、PagedAttention 怎么管理显存碎片，属于"推理部署与服务化"类别，这里只需要在机制层面理解"为什么 MHA 的 KV cache 会成为瓶颈"。
