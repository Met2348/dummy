# 00 · 八股文通用作答法（先读这篇）

## 八股文和 LeetCode 的本质区别

`leetcode-mastery` 练的是"给一道没见过的题，现场推导出解法"——考的是临场思维。八股文正相反：**面试官问的问题历史上早就问过千百遍，标准答案是公开的、固定的**。你不需要临场想出"什么是 TCP 三次握手"，你只需要把公认的标准答案 + 采分关键词练到条件反射。

这意味着两件事：
1. 八股文不是"理解就够"，是"理解 + 能在 30 秒内脱口而出关键词"——理解和"能被追问三层不露怯"之间还有一道练习的手续，这道手续就是本 track 要练的。
2. 八股文的性价比极高：同样一小时，刷题可能只多会一道变种，背八股文可能直接补上"不会就是不会、答不出来对面立刻扣分"的硬伤。

## 四步作答框架：是什么 -> 为什么 -> 怎么做 -> 追问链

每张 QA 卡片建议这样练：

1. **是什么**：一句话精确定义。不要绕，不要先讲背景故事。
2. **为什么**：这个东西存在是为了解决什么问题、对比它之前的方案差在哪。这一步是**采分点最密集**的地方——面试官通过"为什么"来判断你是理解还是背书。
3. **怎么做**：具体机制/公式/步骤，能落到可以画图或写伪代码的颗粒度。
4. **追问链**：每张卡片自带 `follow_ups`——面试官大概率会追问的第二、三问。**先自己试着口头回答，答不上来再看讲义**，这是本 track 和"抄答案"最大的区别。

## 关键词打分：grade() 怎么用

每张 `QA` 卡片带 `keys`（采分关键词）。练习时把自己的口头回答写下来（或录音转文字），跑：

```python
from ai_qa.qbank_transformer import BANK
from ai_qa.qbank_transformer import grade  # 每个 qbank 模块都有同名 grade()

my_answer = "……你自己的回答……"
score = grade(my_answer, BANK[0])
```

`score` 是命中关键词的比例。**别死抠命中率数字**——它只是个粗糙的自查信号，真正的检验是"面试官听得懂、追问接得住"。命中率低于 0.5 说明你漏掉了对方评分表上明确会打勾的点，值得回去补。

## 后端赛道 vs AI 赛道：怎么分配练习时间

如果你的目标是海外前沿 Lab 的纯研究岗（RS/RE），后端通用八股（操作系统/网络/数据库/JVM）大概率**不会被问**，优先级应该明显低于 `ai_qa/` 和 `interview-prep/` 的真 torch 手撕。如果你会顺手投国内大厂算法岗或者做两手准备，`backend_qa/` 才值得投入——但即便如此，也建议先把 AI 赛道过一遍，因为那是你的核心竞争力所在，后端八股只是"不to被完全卡在门槛外"的保底项。

## 代码验证模块：为什么八股文也要写代码

`ai_coding/` 和 `backend_coding/` 里的模块（如一致性哈希环、LRU/LFU、MoE 路由）不是让你去刷 LeetCode 式的编程题，而是用**能跑的代码把"我以为我懂了"变成"我跑出来验证过了"**。比如"一致性哈希能减少节点增减时的重分布"这句话背下来很容易，但亲手写一遍、断言重分布比例确实约等于 1/N，才会在被追问"为什么"时答得扎实。
