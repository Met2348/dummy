# AI-02 · 正则化与泛化

## 这一类在面试里的分量

正则化/泛化这块是"考理解深度"的重灾区——基础版("Dropout 是什么""L1/L2 区别")几乎所有校招算法岗都会问,但真正拉开差距的是能不能在追问里聊到 double descent、scaling law、隐式正则化这些 2019 年之后才逐渐成为共识的新结论。国内大厂算法岗(推荐/搜索方向尤其明显)喜欢从"过拟合怎么判断"一路问到"你们的模型上线后效果为什么会掉"(本质是在问分布漂移);海外 Lab 的研究向面试更喜欢从 scaling law/double descent 切入,因为这直接关系到你有没有"用统计直觉设计实验"的科研素养,而不只是"调包防止过拟合"。这一类的知识密度是三类里最高的,建议多花时间把每道题的追问链走扎实,而不是贪多。

## 追问链深挖一:过参数化为什么不崩、早停会不会踩坑

> **Q1(基础问题):** 深度网络的参数量经常远超训练样本数,按经典的 bias-variance 权衡,这不是应该严重过拟合吗?为什么实践中效果反而很好?

**A1:** 经典 bias-variance 权衡建立在"模型复杂度是一维旋钮、越复杂方差越大"的假设上,这在参数量小于或接近样本数时基本成立。但当参数量远超样本数、跨过"插值阈值"(模型刚好能把训练集拟合到零误差的临界点)之后,继续增大模型反而会让测试误差重新下降——这就是 double descent。原因是超过插值阈值后有无穷多组参数都能完美拟合训练数据,而 SGD 等优化器的隐式偏好会在这些解里挑一个范数更小、损失面更平的解,这种隐式正则化效果随参数量增大反而更强,抵消了经典理论预期的方差爆炸。

> **面试官追问 Q2:** 那既然过参数化不可怕,是不是意味着"能用大模型就不用管过拟合了"?

**A2:** 不能这么理解。double descent 说的是"跨过插值阈值之后,继续增大模型规模是安全甚至有益的",但插值阈值附近那个"误差先升"的区间依然真实存在——如果模型大小恰好卡在插值阈值附近,反而是过拟合最严重的地方,这段区间不能靠"再加大模型"绕开,只能靠"要么明显更小、要么明显更大"跳出这个陷阱区。而且 double descent 不代表正则化/数据质量不重要,它只是说明经典理论对方差项的估计在过参数化区域不准,不代表可以放弃控制模型复杂度。

> **面试官再追问 Q3:** 我在实践里一直用"验证集 loss 一回升就早停",这套策略在会出现 double descent 的场景下会不会踩坑?

**A3:** 会。epoch-wise double descent 现象表明,固定一个足够大的模型,训练更久时测试误差会先降、在训练集被插值拟合到接近零误差附近升高出现一个峰值,之后如果继续训练反而会再次下降到比第一个低点更好的水平。常规早停一看到验证误差在这个峰值附近回升就会提前停掉,恰好错过"峰值之后其实还能变好"的后半段,拿到一个次优模型。所以在可能撞上 double descent 的场景(大模型、长训练、干净或有一定标签噪声的数据)里,不能简单信"回升就停",要么把训练拉得明显更长再判断,要么换用权重平均等不依赖单点验证 loss 的模型选择方式。

*(完整答案见 `src/ai_qa/qbank_regularization.py` 的 `ai-reg-06` / `ai-reg-08`。)*

## 追问链深挖二:L1/L2 的贝叶斯视角

> **Q1(基础问题):** 从贝叶斯 MAP 估计的角度看,L1 和 L2 正则分别对应权重的什么先验分布?

**A1:** 给权重设一个均值为 0 的高斯先验,负对数先验正比于权重的平方和,MAP 估计就等价于加了 L2 正则;给权重设一个均值为 0 的拉普拉斯先验,负对数先验正比于权重的绝对值之和,MAP 估计就等价于加了 L1 正则。拉普拉斯分布在 0 点附近的概率密度比高斯更"尖",对权重恰好为 0 赋予了更高的先验概率,这是 L1 容易产生稀疏解的贝叶斯根源。

> **面试官追问 Q2:** 那 Elastic Net 把 L1 和 L2 一起用,从贝叶斯角度怎么理解?

**A2:** Elastic Net 的正则项是 α·L1 + (1-α)·L2 的线性组合,对应的先验可以理解成拉普拉斯先验和高斯先验的某种混合(不是简单相乘,但效果上兼顾了两者的性质)。它想同时拿到 L1 的稀疏性(自动做特征选择)和 L2 的"分组"效应——当一组特征高度相关时,纯 L1 倾向于随机只保留其中一个、丢掉其余,而 L2 成分会让相关特征的系数更均匀地被压缩,不会武断地只留一个,这在特征高度共线的场景(比如很多传统机器学习的表格特征)里比单用 L1 更稳健。

> **面试官再追问 Q3:** L1 正则的绝对值函数在 0 点不可导,这在实际训练里怎么处理?

**A3:** 实践中用次梯度(subgradient)约定:在 |θ| 不可导的 θ=0 点,直接取次梯度为 0(或者更严格地说,取次梯度集合 [-1, 1] 里的某个代表值,工程实现里几乎都直接用 0),不影响训练——因为这一点在参数空间里几乎是测度为零的孤立点,浮点数计算中参数精确等于 0 的概率极低,用次梯度顶替普通梯度不会实质影响优化过程。如果需要精确的稀疏解(比如真的要一部分权重严格为 0 做特征选择),经典机器学习里会用坐标下降或 ISTA/FISTA 这类基于软阈值算子(soft-thresholding)的专门算法,而不是直接对不可导点做梯度下降。

*(完整答案见 `src/ai_qa/qbank_regularization.py` 的 `ai-reg-02`。)*

## 其余题目速览

- **Dropout 在卷积层为什么效果打折** —— 空间相关性让信息"渗透",Spatial Dropout/DropBlock 改成整块丢弃。完整答案见 `ai-reg-01`。
- **label smoothing 对校准 vs 蒸馏的矛盾影响** —— 改善校准,但抹掉类间相似度信息("暗知识"),反而损害蒸馏。完整答案见 `ai-reg-03`。
- **early stopping 和 L2 的等价关系** —— 二次损失面下 λ ≈ 1/(ετ),都是"低通滤波器"。完整答案见 `ai-reg-04`。
- **Mixup/CutMix vs 传统数据增强** —— 从 ERM 跳到 VRM,注入"线性过渡"的归纳偏置。完整答案见 `ai-reg-05`。
- **k 折交叉验证在大模型时代为什么少用** —— 训练成本太高 + 数据量本身足够大,固定 hold-out 就够。完整答案见 `ai-reg-07`。
- **Chinchilla 配比为什么被工业界违背** —— 训练损失最优 ≠ 训练+推理总成本最优。完整答案见 `ai-reg-09`。
- **distribution shift 三种类型** —— covariate shift(P(X)变)/label shift(P(Y)变)/concept drift(P(Y|X)变)。完整答案见 `ai-reg-10`。
- **隐式正则化** —— SGD 噪声偏好宽平极小值 + 梯度下降偏好最小范数解。完整答案见 `ai-reg-11`。
- **Dropout 和 L2 同时使用** —— 一个管结构随机化、一个管参数收缩,联合调参要各自调低强度。完整答案见 `ai-reg-12`。

## 易错点 / 常见误区清单

1. **把 double descent 理解成"模型越大越好、不用正则了"**——它只说明插值阈值之后是安全的,插值阈值附近仍然是过拟合最严重的区间。
2. **无脑信"验证集回升就早停"**——在过参数化+长训练场景下可能正好撞上 epoch-wise double descent 的谷底前的峰值,提前停等于错过更好的解。
3. **以为数据增强都是一回事**——传统增强(标签不变、扩充分布覆盖)和 Mixup/CutMix(构造分布外虚拟样本、注入线性归纳偏置)机制完全不同。
4. **label smoothing 无脑加**——它会改善校准,但会损害知识蒸馏效果,如果 pipeline 里有蒸馏这一步要格外小心。
5. **把 Chinchilla 的"计算最优"当成"工业界都应该照做"**——工业界优化的是训练+推理总成本,很多大模型故意用远超 20 tokens/参数的比例训练。
6. **混淆三种 distribution shift**——covariate shift 通常还能用重加权/域适应缓解,concept drift 本质上是任务规律变了,只能靠持续监控和重训练。
