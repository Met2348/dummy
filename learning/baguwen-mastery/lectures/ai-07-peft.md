# 07 · 微调与参数高效方法（PEFT）

## 这类题在面试里的分量

PEFT 是 2026 年几乎所有"训练/微调相关"岗位的必考项——不管你面的是 LLM 应用岗、对齐/RLHF 岗、还是 Applied Research，只要简历上写了"fine-tuned a model"，面试官大概率会追问"怎么 fine-tune 的、为什么选这个方法、显存怎么算的"。对 NLP/LLM 方向的 PhD 候选人尤其危险：你几乎天天在用 LoRA，容易陷入"我知道怎么调 `r` 和 `alpha` 参数，但说不清楚背后在数学上到底发生了什么"的尴尬——这正是八股文要补的那道"知道 -> 讲清楚"的手续。QLoRA 的三个技术点（NF4/双重量化/分页优化器）是最容易被记混的细节，务必分清楚"谁解决谁"。

## 深挖追问链一：LoRA -> QLoRA -> NF4 量化机制

**Q1（面试官）**：介绍一下 LoRA 的核心思想。

**A1**：LoRA 冻结预训练权重 W，只学习一个低秩旁路 ΔW=BA（A 是 in×r，B 是 r×out），前向变成 h=Wx+BAx。依据是"内在维度"假设——下游任务真正需要的有效权重更新集中在一个很低维的子空间里，用满秩矩阵去拟合是浪费。只训 A、B 把可训练参数量从 O(d²) 压到 O(d·r)，且基座完全冻结，训练完可以把 BA 合并回 W，推理零额外延迟。

**面试官追问 Q2**：如果我显存还是不够，LoRA 都带不动怎么办？QLoRA 相比 LoRA 多做了什么？

**A2**：QLoRA 在 LoRA 之上加了三件事，缺一不可：①NF4(4-bit NormalFloat) 量化——把冻结的基座权重压到 4bit 存储，大幅降低静态显存；②双重量化——把 NF4 量化本身产生的量化常数(scale)再量化一次，65B 模型大约再省 3GB；③分页优化器——借助 NVIDIA 统一内存，在梯度检查点导致显存瞬时尖峰时把优化器状态换页到 CPU，防止 OOM。三者合力让 65B 模型能在单张 48GB 卡上微调。注意：LoRA 的 A、B 矩阵本身**不量化**，依然是 bf16/fp16 高精度训练，只有基座权重是 4bit。

**面试官再追问 Q3**：NF4 具体比普通 int4 好在哪？量化损失的精度是靠什么"补回来"的？

**A3**：普通 int4 是等宽量化，16 个格子等距离切分数值范围，格子密度和数据实际分布无关。但神经网络权重训练后近似零均值正态分布，NF4 用"分位数量化"思想，让每个格子包含大致相等数量的权重——也就是零附近格子更密、两侧更稀——这是信息论意义上对正态分布数据最优的 4bit 量化方式。至于精度损失怎么补：前向时基座权重从 4bit 反量化(dequantize)到 bf16 再做矩阵乘法，这本身已经把大部分量化损失控制住了；而 LoRA 的旁路 A、B 全程保持 bf16 高精度训练，相当于给这个有损的基座配了一条"高精度可训练修正项"，任务相关的精细调整主要靠这条旁路承担，进一步稀释了量化带来的误差影响。

## 深挖追问链二：Adapter vs LoRA 结构对比

**Q1**：Adapter（Houlsby/Pfeiffer）和 LoRA 结构上有什么本质区别？

**A1**：Adapter 是串联(serial)结构——在每层子层之后插入一个瓶颈 MLP，前向必须先过完原层再过 adapter；LoRA 是并联(parallel)结构——旁路 BA 和原权重 W 各自独立计算再相加，两条路径互不阻塞。

**面试官追问 Q2**：这个"串联 vs 并联"的区别在部署时有什么实际影响？

**A2**：串联结构无法绕开——推理时必须真的多算一层 adapter 的前向，路径变长、增加延迟，且这个延迟在生成式模型逐 token 解码时会被放大。并联结构的 LoRA 训练完之后可以直接把 BA 合并进 W 得到 W'=W+(alpha/r)BA，模型结构和推理路径与原始模型完全一样，没有推理延迟开销——这是 LoRA 在工业界比 Adapter 更受欢迎的直接原因。

**面试官再追问 Q3**：那如果我要一个基座同时服务好几个客户/任务，各自用不同的 LoRA，还能保持这个"零延迟"优势吗？

**A3**：如果是单任务单部署，合并权重是最优选择。但多任务场景下，通常不合并，而是把每个任务的 A、B 当作几 MB 的小文件按需加载，基座权重只存一份；推理时确实要多做一次旁路矩阵乘法，但这个开销远小于原矩阵乘法（低秩 r 通常远小于 d），工程上是可接受的代价，换来的是不用为每个任务都存一份完整模型。这也是 S-LoRA 等服务框架要解决的核心问题：怎么在一个 batch 里同时高效服务成百上千个不同的 LoRA adapter。

## 其余题目一览

完整答案见 `src/ai_qa/qbank_peft.py`（`ai-peft-02` ~ `ai-peft-12`，除上面深挖的 4 道）：

- **秩 r 怎么选**（`ai-peft-02`）：容量/参数量/过拟合的权衡旋钮，常见 4~64，不同层可以不同 r。
- **alpha/r 缩放系数的作用**（`ai-peft-03`）：让换 r 时旁路的有效更新幅度保持稳定，超参搜索更可控。
- **A 随机、B 置零的初始化**（`ai-peft-04`）：保证起点严格等价原模型，同时保证 B 的梯度不为零、训练能启动。
- **为什么不插在 LayerNorm 之后**（`ai-peft-07`）：LayerNorm 参数量是 O(d)，本来就很小，套一层低秩分解得不偿失。
- **Prompt-tuning / Prefix-tuning vs LoRA**（`ai-peft-08`）：软提示类方法实现更简，但占用上下文长度、优化更不稳定，效果上限通常不如 LoRA。
- **合并权重 vs 保留旁路**（`ai-peft-10`）：单任务部署合并、多任务部署分开存放，是延迟/存储的权衡。
- **全参微调 vs PEFT 的取舍**（`ai-peft-11`）：大数据/大分布偏移/追求上限选全参；小数据/多任务/资源受限选 PEFT。
- **为什么调 <1% 参数也能接近全参效果**（`ai-peft-12`）：内在维度假设——有效更新方向本就集中在低维子空间，模型越大这个优势越明显。

## 易错点 / 常见误区清单

1. **把 QLoRA 的三个技术点混为一谈**——NF4 管"怎么把权重压到 4bit 还尽量不失真"，双重量化管"量化常数本身也要再省一点显存"，分页优化器管的是"训练时显存瞬时尖峰"，三者解决的是三个不同层面的问题，不是同一件事的三个说法。
2. **以为 QLoRA 训练时全程都在 4bit 下计算**——错，4bit 只是基座权重的**存储**格式，实际矩阵乘法前会反量化到 bf16 计算；LoRA 的 A、B 全程是 bf16 高精度，不参与量化。
3. **搞反 A、B 的初始化**——是 A 随机、B 置零，不是反过来；说反了会导致"为什么能保证起点等价"这个逻辑直接讲不通。
4. **把 alpha 和 r 当成两个独立无关的超参**——起作用的是 alpha/r 这个比值，脱离 r 单独讨论 alpha 的绝对值没有意义。
5. **认为 LoRA 一定要插满所有权重矩阵**——实践中经常只插 Q/V（甚至只插 Q/K/V/O 里的一部分），插得越多可训练参数越多但不代表效果线性变好，需要结合任务和显存预算取舍。
6. **忽略 Adapter 的推理延迟代价**——只强调"Adapter 参数少"而不提"串联结构一定会增加推理路径长度"，这是面试官最容易抓住的漏洞。
