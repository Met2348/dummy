# L2 · 会议 talk + slides 设计: 结构、幻灯片、时间、临场

> 35-min lecture · 目标: 把 L1 的原则落实成可操作的 talk —— 标准结构 + 一图一点的 slide 设计 + 时间预算 + 临场演讲技巧。

---

## 0. 一个 talk 的标准结构

会议 talk (通常 8-15 分钟) 有一个久经考验的骨架, 和论文叙事同源但更稀疏:

```
   motivation        approach          result            takeaway
   (~20%)            (~30%)            (~30%)            (~20%)
   为什么这问题重要   核心 idea (非细节) 关键结果 + 那张图   一句话 + future
   + gap (9.2/9.3)   + Figure1 (9.6)   + 误差棒 (诚实)     让人记住
        │                 │                 │                 │
        └──── 一条线: 钩住听众 → 给方法 → 给证据 → 留 takeaway ──┘
```

`talk_planner.plan(total_min, takeaway)` 按这个结构帮你把时间**预算**到每部分, 并估算合理的 slide 数。两条新手必踩的坑, 它会自检:
- **motivation 拖太长** (>30%): 新手最爱在背景上磨蹭, 听众却急着想听「你到底做了什么」。**钩子要短、要快。**
- **结果讲太碎**: 把论文所有表都搬上来。talk 只讲**那一个支撑 takeaway 的主结果** + (也许) 一个关键消融。

## 1. slide 设计: 一图一点, 少字

slide 不是你的演讲稿, 是**给听众看的视觉辅助**。三条铁律:

### ① 一张 slide 一个点 (one slide, one point)

每张 slide 只传达一个意思, 标题就写那个意思 (不是「实验结果」这种名词, 而是「Robust-DPO 在高噪声下胜出」这种**论断**)。听众扫一眼标题就知道这页要说什么。

### ② 少字, 别堆文字

> **slide 上的字是给听众读的, 不是给你念的。** 听众一旦在读你密密麻麻的文字, 就没在听你说话 —— 两个通道打架。规则: 关键词 + 图, 不是整句整段。**如果你的 slide 拿掉你的讲解还能完全读懂, 那它太满了** (它该是辅助, 不是讲稿)。

经验阈值: 一张 slide 不超过几个要点, 每点几个词。能用图就别用字。

### ③ 视觉化 > 文字化

把 9.6 学的出版级图搬上来: 交互效应图、Figure 1 示意图。**一张好图胜过一段话**, 尤其在不能倒回的听觉媒介里。slide 上的图甚至可以比论文里更大、更简化 (去掉次要曲线, 只留支撑当前 point 的)。

## 2. 时间控制: 排练到秒

超时是 talk 的大忌 (尤其会议, 主持人会打断你, 你的 takeaway 还没讲就被赶下台)。

- **用 `talk_planner` 排时间预算**, 每部分卡死分钟数。
- **真排练**: 对着计时器讲 2-3 遍。新手几乎总是**讲得比排练慢** (现场紧张会拖)，所以排练时要留 buffer (排到 11 分钟, 现场往往正好 12)。
- **设检查点**: 心里记「讲到方法时该是第 4 分钟」, 现场对表, 超了就跳过细节直奔结果。
- **takeaway 永远留足时间**: 哪怕前面超时, 也要砍中间、保 takeaway。**没讲到 takeaway 的 talk 等于白讲。**

## 3. 临场演讲技巧

内容之外, 表达本身也影响效果。几条对博0 最实用的:

| 技巧 | 怎么做 |
|---|---|
| **节奏** | 慢一点。紧张会让你越讲越快, 听众跟不上。**关键句后停顿一下**, 给听众消化。 |
| **眼神** | 看听众 (扫视全场), 不要盯着 slide 或地板念。眼神接触建立连接。 |
| **别背稿** | 记结构和 takeaway, 不背逐字稿 (背稿一卡壳就全乱)。熟练到能用自己的话讲。 |
| **紧张** | 正常的。深呼吸; 开头先讲最熟的 (motivation), 顺起来就稳了。手抖就别拿激光笔抖给大家看。 |
| **过渡** | 每部分之间给一句过渡 (「讲完了为什么, 现在看我们怎么做」), 帮听众跟上结构。 |

> 对博0 (尤其英语非母语) 的现实建议: **第一次会议 talk 前, 至少完整排练 5 遍**, 其中 1-2 遍讲给同学听并收反馈。**讲清楚 > 讲流利 > 口音完美。** 口音不重要, 结构清晰、卡住时间、讲到 takeaway 才重要。

## 4. 一个常见反模式: 「方法细节淹没症」

新手 talk 最常见的翻车: 在「方法」部分陷入所有技术细节 (每个公式、每个超参), 结果时间耗光, 结果和 takeaway 草草带过。

> 治法: **talk 的方法部分只讲「核心 idea + 为什么它能 work 的直觉」, 不讲实现细节。** 细节留给论文 (听众有兴趣自己去读) 和 Q&A (有人问再展开)。一张 Figure 1 + 一句直觉, 远胜五张公式 slide。记住: **听众要的是"你做了什么聪明的事", 不是"你怎么实现的每一行"。**

---

## 5. 本讲小结 + 通往 L3

- talk 标准结构: **motivation(20%) → approach(30%) → result(30%) → takeaway(20%)**; `talk_planner` 帮你排时间预算 + 抓 motivation 过长。
- slide 三铁律: **一张一个点** (标题是论断) / **少字** (辅助非讲稿) / **视觉化** (搬 9.6 的图)。
- 时间控制: 排练到秒、留 buffer、设检查点、**takeaway 永远保住**。
- 临场: 慢节奏 + 停顿、看听众、别背逐字稿、管理紧张、给过渡句。
- 反模式「方法细节淹没」: 只讲核心 idea + 直觉, 细节留论文/Q&A。

> **下一讲 L3「poster + 电梯演讲」**: talk 是单向广播; poster 和 pitch 是双向、即兴的。L3 教你 poster 的视觉动线设计, 以及把研究压成 10秒/1分钟/5分钟三档、针对不同听众调整的电梯演讲阶梯。

**动手**: 去 `N1-talk-storyboard.ipynb`, 用 `talk_planner` 给你的研究排一个 12 分钟 talk 分镜, 填上每部分打算用几张 slide, 看自检会不会提示你 slide 过密 / motivation 过长。
