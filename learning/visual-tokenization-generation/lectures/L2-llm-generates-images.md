# L2 · LLM 生成图像: 像生成文本一样画图

> 25-min lecture · 目标: 理解「自回归生成视觉 token」如何让 LLM 画图 —— 和生成文本完全相同的机制。

---

## 0. 一旦图变成 token, 生成就「免费」了

L1 把图变成了离散 token 序列。现在惊人的事发生了: **生成图像, 和生成文本是同一回事。**

```
   生成文本: 预测下一个词 token, 一个个生成, 直到结束
   生成图像: 预测下一个视觉 token, 一个个生成, 最后用码本解码成图
            ↑ 完全相同的「自回归生成 token」机制
```

你已会的所有文本生成知识 (自回归、采样、temperature、top-k/top-p, 你的推理引擎专题) **全部直接迁移**到图像生成。这就是离散化的回报。

---

## 1. 自回归图像生成的流程

```
   1. (可选) 给一个条件: 文本「a red square」→ 编码成文本 token
   2. 自回归生成视觉 token: 模型一个个预测 v1, v2, ..., vN
        p(v_i | v_1...v_{i-1}, 文本条件)   ← 和预测下一个词一模一样
   3. 把生成的 token 序列 [v1...vN] 用码本解码 → 重建图
```

机制就是预测下一个 token:
$$p(\text{图}) = \prod_{i=1}^{N} p(v_i \mid v_1, \dots, v_{i-1}, \text{条件})$$
逐项: $v_i$ 是第 $i$ 个视觉 token (码本编号); 模型按从左到右、从上到下的顺序, 基于已生成的 token (和文本条件) 预测下一个。生成完整串 $[v_1...v_N]$ 后, 查码本解码成图。**这就是一个在视觉 token 上训练的自回归 transformer**, 和 GPT 生成文本结构相同。

> 你的 N2 会真做这个: 训一个 tiny transformer 在视觉 token 序列上预测下一个 token, 然后采样生成一串新 token、解码成图。亲眼看「LLM 画图」= 自回归 + 解码。

---

## 2. 采样: 和文本生成同款旋钮

既然是自回归生成 token, 文本生成的采样技巧全适用:
- **temperature**: 控制随机性 (低=确定/保守, 高=多样/冒险)。
- **top-k / top-p**: 截断低概率 token, 平衡质量和多样性。
- **贪心 vs 采样**: 贪心 (总取最高概率) 生成单调; 采样生成多样。

> 图像生成里, 这些旋钮控制「生成图的多样性 vs 保真度」, 和文本里控制「回答的多样性 vs 确定性」是同一回事。你的推理引擎专题知识在这里零成本复用。

---

## 3. 自回归 vs 扩散: 两条生成图像的路

值得指出: 自回归 token 生成不是生成图像的唯一路。另一条是**扩散** (M13 整个模块):

| | 自回归 (本讲) | 扩散 (M13) |
|---|---|---|
| 机制 | 一个个生成离散视觉 token | 从噪声逐步去噪成图 |
| 和文本的关系 | 同机制 (统一 token) | 不同范式 |
| 优势 | 和 LLM 统一、可图文混合 | 图像质量常更高 |
| 代表 | DALL-E 1, Parti, Chameleon | Stable Diffusion, DALL-E 2/3 |

> 现在的前沿 (如 Transfusion) 甚至**混合**两者: 文本用自回归、图像用扩散, 在一个模型里。M13 会深入扩散; 本讲先让你掌握「自回归 token 生成」这条和 LLM 统一的路。两条路你都要懂, 才看得懂前沿的混合架构。

---

## 4. 自回归图像生成的软肋

诚实地说, 纯自回归 token 生成有代价:
- **慢**: 一个 token 一个 token 生成, 一张图几百上千 token, 串行慢 (和文本长生成同款问题, 接你投机解码专题)。
- **量化损失**: 受限于码本 (L1), 细节有上限。
- **顺序偏置**: 从左上到右下的固定顺序, 不如扩散的「全局同时细化」自然。

这些软肋正是扩散 (M13) 在图像质量上常胜出的原因, 也是混合架构 (Transfusion) 想两全的动机。

---

## 5. 本讲小结 + 通往 L3

- 图变成离散 token 后, **生成图 = 自回归生成视觉 token + 码本解码**, 和生成文本同机制。
- 文本生成的全套 (自回归、temperature、top-k/p、投机解码) **直接复用**。
- 自回归 vs 扩散是两条生成路; 前沿 (Transfusion) 混合两者。
- 自回归软肋: 慢、量化损失、顺序偏置 —— 扩散 (M13) 在质量上常胜。

> **下一讲 L3「理解+生成一体」**: 既然图能当 token 读、也能当 token 生成, 那一个模型就能**同时读图和画图** —— 这就是 Chameleon 式的统一多模态模型。L3 讲这个「一个模型通吃理解和生成」的架构。
