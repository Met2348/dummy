# L1 · 离散视觉 token: 把图变成「视觉的词」

> 30-min lecture · 目标: 理解 VQ-VAE/VQGAN 如何把连续图像量化成离散 token (码本), 以及为什么「离散化」是让 LLM 画图的关键一步。

---

## 0. 从「读图」到「画图」的鸿沟

M10.3 的 VLM 会读图 (图当输入)。但要让模型**画图** (输出图), 有个根本障碍:

> LLM 生成的是**离散 token** (从词表里一个个挑)。但图像是**连续**的像素值。LLM 怎么「生成」连续的东西? 答案: **先把图离散化成 token**, 这样生成图就变成「自回归生成视觉 token」, 和生成文本机制完全一样。

这个「把图变成离散 token」的工具就是 **VQ (vector quantization, 向量量化)**, 由 VQ-VAE / VQGAN 实现。它是 any-to-any 多模态模型 (Chameleon) 的基础。

---

## 1. VQ 的核心: 一个视觉「码本 (codebook)」

```
   图 → 切 patch → 每个 patch 找码本里最近的条目 → 用那个条目的「编号」代替它
                   码本 = K 个「视觉原型」(常见的 patch 模式)
   结果: 图变成一串编号 (离散 token), 像文本变成一串词 id
```

类比文本:
- 文本词表: ~5 万个词, 每个文本 token 是一个词 id。
- 视觉码本: K 个视觉原型 (如 1024 个), 每个视觉 token 是一个原型 id。

机制 (`vq_tokenizer` 实现):
1. **学码本**: 用 k-means (或可学习的 VQ 层) 在大量 patch 上聚类, 得到 K 个「代表性 patch 模式」当码本。
2. **量化 (编码)**: 每个 patch 找码本里最近的条目, 记下它的编号 → 一个离散 token。
3. **反量化 (解码)**: 拿编号查码本, 取回那个原型 patch → 重建图。

$$\text{token}_i = \arg\min_{k} \|p_i - c_k\|^2$$
逐项: $p_i$ 是第 $i$ 个 patch; $c_k$ 是码本第 $k$ 个条目; $\text{token}_i$ 是离它最近的条目编号 (这就是第 $i$ 个视觉 token)。

---

## 2. 码本大小的权衡 (一眼可见)

码本越大, 重建越精细, 但 token 空间越大:

| 码本大小 K | 重建质量 | token 信息密度 |
|---|---|---|
| 小 (如 2) | 粗糙 (只有 2 种 patch) | 每 token 信息少, 但词表小 |
| 大 (如 1024) | 精细 (足够多原型) | 每 token 信息多, 但词表大 |

> 这是 VQ 的核心权衡: **码本大小 = 视觉词表大小**。太小重建糊 (信息丢失); 太大则视觉「词表」巨大、难学难生成。真实 VQGAN 用几千~上万的码本。你的 N1 会画出「码本大小 → 重建 MSE」曲线, 亲眼看 MSE 随码本增大而下降。

---

## 3. VQ-VAE vs VQGAN (一句话区别)

- **VQ-VAE**: 编码器把图压到隐空间 → 量化 → 解码器重建。用重建损失训练。重建偏「平均」、糊。
- **VQGAN**: VQ-VAE + **对抗损失 (GAN)** + 感知损失。重建更锐利、更真实。是现在主流的图像 tokenizer (生成图质量好)。

> 对你的意义: 知道 VQGAN = VQ-VAE + GAN/感知损失让重建更好即可。本专题用最简的 k-means VQ 演示核心 (离散化), 不纠缠 GAN 细节。关键是理解「图 → 离散 token」这个转换, 而非具体训练技巧。

---

## 4. 离散化的代价与意义

**代价**: 量化必然有信息损失 (连续 → 离散)。重建不可能完美 (除非码本无限大)。这是 VQ 的固有损失。

**意义**: 一旦图变成离散 token, **整个文本生成的机器全部可复用**:
- 生成图 = 自回归生成视觉 token (L2)。
- 图文混合 = 视觉 token 和文本 token 拼在一个序列 (M10.2 的 early-fusion)。
- 理解 + 生成统一在「预测下一个 token」(L3)。

> 这就是为什么 VQ 如此关键: 它把「图像」翻译成「LLM 的母语 (离散 token)」, 让一个模型既能读又能写、既能处理文又能处理图。**离散化是多模态统一的钥匙。**

---

## 5. 本讲小结 + 通往 L2

- 要让 LLM 画图, 得先把图**离散化**成 token (LLM 只会生成离散 token)。
- **VQ**: 学一个**码本** (K 个视觉原型), 每个 patch 量化成最近条目的编号 = 一个视觉 token。
- **码本大小**权衡重建质量 vs 词表大小 (太小糊、太大难学)。
- VQGAN = VQ-VAE + GAN/感知损失, 重建更锐利。
- 离散化有信息损失, 但换来「复用全部文本生成机器」—— 多模态统一的钥匙。

> **下一讲 L2「LLM 生成图像」**: 图变成离散 token 序列后, 怎么生成新图? 答案出奇简单 —— **像生成文本一样, 自回归地一个个预测视觉 token**, 最后解码成图。L2 让你看见这个过程。
