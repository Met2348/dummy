# L06 · ML 系统设计(研究向格式)

## 你有素材,缺的是"格式"

你的 M5(serving)、M6(eval)、M8(infra)、M7(agent)选型树是**一流原材料**。面试挂在没有**固定框架**——想到哪说到哪。用下面的骨架,任何题都能 45 分钟走完。

## 万能骨架(7 步)

1. **澄清需求**:用户是谁?规模(QPS/数据量)?延迟/成本约束?离线还是在线?成功长什么样?
2. **成功指标**:业务指标(留存/成本)+ ML 指标(AUC/perplexity/胜率)+ 护栏(延迟 p99、安全)。
3. **数据**:来源、标注、规模、偏斜、隐私、冷启动。
4. **模型 / 方法**:baseline → 迭代。为什么这个方法(引你学过的选型树)。
5. **训练 / pipeline**:特征、损失、正则、分布式(引 M8)、复现。
6. **serving / 扩展**:批处理、KV cache、量化、缓存、蒸馏(引 M5)。
7. **评测 / 监控 / failure mode**:离线评测(引 M6)、在线 A/B、漂移检测、回滚。收尾谈 tradeoff。

## 研究向题目(比纯工程更可能问你)

- "设计一个评测 coding agent 的 pipeline"(引 `agent-code-eval` + τ-bench)。
- "设计一个 LLM-as-judge 系统,并防 reward hacking"(**你 PhD 方向,主场**)。
- "设计一个可复现的 RLHF 训练系统"(引 M4 + M8)。
- "设计一个检测模型幻觉的离线评测"(引 M6)。

## 关键动作:出声澄清 + 画边界

面试官最看重**你主动澄清需求、显式声明假设、谈 tradeoff**。不要闷头给"标准答案"。每一步先问一句"我假设是在线低延迟场景,对吗?"。

## 练法

选 3–4 道上面的题,**掐表 45 分钟、对着白板/纸出声讲**。录音回放,看哪一步含糊。研究向的两道(judge 评测、RLHF pipeline)是你的强项,先把它们打磨成"招牌答卷"。
