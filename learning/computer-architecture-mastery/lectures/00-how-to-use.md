# Computer Architecture Mastery 使用说明

> **先读 `01-textbook.md`**:那是本科课件式完整教材(七章、连贯教科书叙述),本页只是DeepPoint/ScenarioPoint追问链的使用说明,建议在读完教材之后再来这里练习追问链。

## 为什么要做这个专题,以及和GPU架构专题的边界

老师要求补齐科班CS本科理论必修课空白,这是第四个。仓库里已有的 `gpu-architecture`/`cuda-essentials`/`kernel-engineering` 是GPU专用架构(面向ML训练推理场景),这个专题是**通用CPU体系结构**——指令集、流水线、乱序执行、缓存一致性,是理解"CPU为什么这样设计"的基础知识,两者是互补而非重复。如果你已经学过GPU架构,会发现很多概念是相通的(比如cache局部性、流水线并行),但通用CPU体系结构的很多机制(乱序执行/分支预测/精确异常)是GPU架构里没有对应物的,因为GPU走的是完全不同的大规模数据并行路线而不是单线程指令级并行路线。

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:建立"CPU怎么执行一条指令"的基础模型——指令集架构定义了软硬件的接口契约,数据通路是执行指令的物理机制,存储层次解释了为什么"内存"不是一个单一均匀的概念。这一层几乎不需要预备知识。
2. **Tier 2(深,`tier2_deep/`)**:三个文件分别对应现代CPU设计里"怎么让单核跑得更快"的三个层面——流水线(指令级别的重叠执行)、乱序执行(打破程序顺序限制挖掘更多并行性)、多核缓存一致性(多个核心协同工作时如何维持数据一致)。这三者是历史演进关系:先有流水线,再有乱序执行挖掘流水线之外的并行性,再有多核和随之而来的一致性问题,建议按这个顺序理解演进逻辑而不是把三者当成孤立知识点。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"怎么定位一个性能问题的硬件层面根因"(CPU-bound/memory-bound/cache miss/分支预测失败),第二个文件考"怎么做硬件感知的软件优化,以及什么时候不值得做"——这两个文件的核心训练目标是让你能把"程序跑得慢"这个现象往下追一层,追到具体的硬件机制,而不是停留在"要不要加个缓存"这种表面判断。

## 一个容易被忽视的陷阱

计算机体系结构的知识很容易被当成"背概念"(MESI四个状态/流水线五个阶段),但真正的工程价值在于**建立"程序行为和硬件机制之间的因果链条"**——为什么按行遍历二维数组比按列快、为什么多线程加了核心数性能反而下降(false sharing)、为什么优化在一台机器上有效换一个CPU型号就失效了。这个专题的tier3判断题反复在训练这条因果链条的构建能力,而不是让你复述某个协议的定义。
