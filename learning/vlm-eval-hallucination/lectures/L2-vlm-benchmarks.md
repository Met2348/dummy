# L2 · VLM benchmark: MMMU / MME / POPE 各测什么

> 25-min lecture · 目标: 认识主流 VLM benchmark, 理解它们各测哪个维度、怎么用, 尤其 POPE 怎么量化幻觉。

---

## 0. 一张 benchmark 地图

VLM 的多维能力 (L1) 对应一批不同的 benchmark, 各测一面:

| benchmark | 测什么维度 | 形式 |
|---|---|---|
| **MMMU** | 大学级多学科**推理** (图+知识) | 多选题, 跨学科 |
| **MME** | **多维感知 + 认知** (14 个子任务) | 是非题 |
| **POPE** | **视觉幻觉** (物体存在性) | 是非题 |
| **MathVista** | 视觉**数学推理** | 开放/多选 |
| **TextVQA / OCRBench** | **OCR / 读图中文字** | 开放问答 |
| **RefCOCO** | **定位 (grounding)** | 框 |

> 没有一个 benchmark 测全 VLM —— 你要**组合**多个覆盖不同维度 (感知/推理/OCR/定位/幻觉), 得到能力剖面。这正是 L1 说的「多维剖面 > 单一总分」, 也是 M6 多维评测在 VLM 的延续。

---

## 1. MMMU: 大学级多学科推理

- **测什么**: 给一张专业图 (化学结构/电路/医学影像/图表) + 一个大学级问题, 要 VLM 看图 + 调用领域知识推理。
- **难在**: 不是「图里有什么」(感知), 而是「基于图能推出什么」(推理 + 知识)。是 VLM 的「高考」。
- **用法**: 衡量 VLM 的高阶视觉推理上限, 是旗舰模型的常报指标。

> 对你的意义: MMMU 这类「图 + 专业推理」最难, 也最能区分强弱模型。它和你的 reasoning-r1 (文本推理) 是「视觉版」—— 视觉推理是个研究热点 (接 M11 reasoning VLA / M10.7 Capstone 找 gap)。

---

## 2. MME: 多维感知与认知

- **测什么**: 14 个子任务 (存在性、计数、位置、颜色、OCR、常识推理……), 每个是是非题。
- **形式**: 每张图配「Yes/No」问题, 算各维度准确率。
- **价值**: 给一个**细粒度的能力剖面** (这个模型计数强但 OCR 弱), 而非单一总分。

> MME 的是非题形式简单、易自动评分, 是快速诊断 VLM 各维度的好工具。它的「逐维度报告」正是 9.6 的「一图一信息」: 别把 14 维压成一个数。

---

## 3. POPE: 量化视觉幻觉 (本专题重点)

POPE (Polling-based Object Probing Evaluation) 专测**视觉幻觉**, 方法巧妙:

```
   对每张图, 问一堆「图里有 X 吗?」是非题:
   - 一半 X 真的在图里 (正样本)
   - 一半 X 不在图里 (负样本, 但选「常一起出现的物体」当诱饵)
   看模型对「不在的 X」是否大量说「有」→ 量化幻觉
```

关键指标 **yes-rate (说「有」的比例)**:
- 真相是正负各半 → 理想 yes-rate ≈ 0.5。
- 如果模型 yes-rate 远 > 0.5 → 它对很多「不在的物体」也说有 = **yes-bias = 幻觉**。

> POPE 的精妙在「负样本选诱饵物体」(图里没有但语境常有的, 如厨房图问「有没有刀」)。这专门勾出「靠语言先验猜」的幻觉 —— 模型如果不真看图、靠常识答, 就会在这些诱饵上翻车。`vlm_eval.run_pope` 实现了这套方法; 你的 N1 会看到「幻觉率↑ → yes-rate 偏离 0.5 → 准确率↓」。

## 4. 怎么用这些 benchmark (实操心法)

- **组合覆盖**: 选 3-5 个覆盖不同维度 (如 MMMU 推理 + MME 感知 + POPE 幻觉 + OCRBench)。
- **报剖面不报总分**: 给每维度单独的数 + error bar (9.4-L5 / 9.6)。
- **警惕污染**: 新模型可能见过老 benchmark 的图 (9.3 Q4)。关注新出的、或私有 benchmark。
- **配定性样例**: 量化指标外, 给几个真实问答样例 (但标明是不是 cherry-picked, 9.6-L4)。

> 给博0: VLM benchmark 演进极快 (每几个月有新的)。别死记某个 benchmark, 记住**它们各测哪个维度**这个框架 —— 框架不过时, 具体 benchmark 会换。

---

## 5. 本讲小结 + 通往 L3

- VLM 能力多维, 对应多个 benchmark: **MMMU** (专业推理) / **MME** (多维感知, 14 子任务) / **POPE** (幻觉) / OCR / 定位 等。
- **组合覆盖** + **报剖面不报总分** + 警惕污染 (M6 + 9.6 延续)。
- **POPE** 用「问有没有 X (正负各半 + 诱饵)」+ **yes-rate** 量化视觉幻觉: yes-rate >> 0.5 = 幻觉。

> **下一讲 L3「视觉幻觉机理与探测」**: POPE 能测幻觉, 但幻觉**为什么**发生? 怎么探测和缓解? L3 深入幻觉的机理 (语言先验过强) 和检测方法, 接你的模态坍缩诊断。

**动手**: 去 `N1-pope-hallucination.ipynb`, 用 `vlm_eval` 对不同幻觉率的模拟 VLM 跑 POPE, 看 yes-rate 怎么随幻觉率偏离 0.5, 亲手用 yes-bias 量化幻觉。
