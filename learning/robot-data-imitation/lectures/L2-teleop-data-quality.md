# L2 · 遥操作数据采集 + 数据质量

> 22-min lecture · 目标: 理解机器人 demo 从哪来 (人类遥操作), 及数据质量怎么决定 BC 的上限。

---

## 0. BC 吃的数据从哪来

L1 说 BC 从专家 demo 学。机器人的「专家 demo」不像文本能从网上爬 —— 它要**人去采**, 主要靠**遥操作 (teleoperation)**: 人通过设备 (VR 手柄/主从臂/手套) 控制机器人完成任务, 录下 (观测, 动作) 序列。

> 这是机器人数据的**根本特殊性**: 贵、慢、要人。一条 demo 可能要人操作几十秒到几分钟。这就是为什么数据是机器人的最大瓶颈 (M11.1-L4)。

---

## 1. 遥操作采集流程

```
   人戴 VR / 操作主臂 → 实时控制机器人 → 完成任务 (抓/放/开)
        ↓ 同步录制
   (相机图, 本体感觉, 人给的动作) 序列 = 一条 demo
```

要点:
- **动作来自人**: 录的 action 是人遥操时机器人执行的动作 (= 专家动作)。
- **多视角 + 本体**: 通常录多个相机 + 关节状态 (本体感觉, M11.3-L4)。
- **任务多样**: 采不同任务/物体/布置, 为了泛化 (scaling, L3)。

> 你的 toy 用**合成专家** (`toy_env.expert_action` 朝目标走) 代替人类遥操作 —— 省去采集, 但 (state, action) 对的角色完全一样。真实里这一步是最贵的人力环节。

---

## 2. 数据质量决定 BC 上限

BC 是「学专家」, 所以**专家有多好、数据有多干净, 决定 BC 的天花板**:

- **一致性**: 同样情况人每次操作差不多吗? 不一致 = 多峰/噪声 (需扩散动作头 M11.3, 或会被回归平均掉)。
- **覆盖度**: demo 覆盖了多少状态/任务? 覆盖不到 → 分布漂移 (L1)。
- **质量**: 有没有失败/犹豫/抖动的 demo? 脏 demo 会教坏策略。
- **平滑度**: 人操作的动作平滑吗? 抖的 demo 学出抖的策略。

> 一句话: **garbage in, garbage out**, 机器人版。BC 不会超过它的 demo。这就是为什么数据采集要严控质量 (筛选/标注/重采)。和你 Module「数据治理」(data-curation) 一脉相承 —— 数据质量是模型质量的上游。

---

## 3. 数据的两难: 量 vs 质

采数据要权衡:
- **多而糙** (快速众包采集): 量大但质量参差, 可能含坏 demo。
- **少而精** (专家精心采集): 质量高但量小、贵。

> 现实折中: 大量采集 + 质量筛选 (自动过滤失败/异常 demo) + 关键任务精采。这正是你 data-curation 模块的思路搬到机器人: **scaling 不只是量, 是「有效数据」的量** (干净 + 多样 + 覆盖)。L3 会看数据量怎么换成功率。

---

## 4. 降低采集成本的方向 (gap)

数据贵是核心瓶颈, 降本是热门方向 (potential research):
- **更便宜的遥操作**: 低成本设备 (如 手机/手套), 让更多人能采。
- **仿真生成数据**: 在仿真里自动生成 demo (M11.6 sim2real)。
- **人类视频当数据**: 不用遥操作, 直接从人做事的视频学 (L4 egocentric)。
- **自主数据飞轮**: 部署的机器人自己采数据, 回流改进 (像推荐系统)。

> 每条都是降低「数据瓶颈」的尝试。对你 (找研究方向): **数据高效的具身学习** (少 demo 泛化、仿真/视频替代真机) 是缺口大、价值高的方向, 且和你 NLP 的「数据高效」直觉相通。

---

## 5. 本讲小结 + 通往 L3

- 机器人 demo 主要靠**人类遥操作**采集 (贵/慢/要人 = 根本瓶颈)。
- **数据质量决定 BC 上限**: 一致性/覆盖度/质量/平滑度 (garbage in garbage out)。
- 量 vs 质两难; scaling 是「有效数据」(干净+多样+覆盖) 的量 (接 data-curation)。
- 降本方向 (gap): 便宜遥操作 / 仿真生成 / 人类视频 / 数据飞轮。
