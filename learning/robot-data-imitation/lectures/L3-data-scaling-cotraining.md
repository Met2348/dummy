# L3 · 数据 scaling 教训 + 与 VLM 数据 co-train

> 22-min lecture · 目标: 把 LLM 的 scaling 教训搬到机器人, 理解数据量→成功率, 及和 VLM 数据联合训练防遗忘。

---

## 0. 把 LLM 的 scaling 福音搬过来

你懂 LLM scaling (数据/模型/算力越多越好)。机器人正在重演这一幕, 但**数据规模还在很早期** (远小于 LLM)。这一讲讲机器人的数据 scaling, 以及一个关键技巧: **和 VLM 数据 co-train**。

---

## 1. 数据量 → 成功率 (你 N2 会量到)

最直接的 scaling: **更多 demo → 更高成功率**。

```
   少 demo (覆盖少状态) → 分布漂移严重 → 成功率低
   多 demo (覆盖多状态) → 漂移缓解      → 成功率高, 渐饱和
```

> 你 N2 会画出这条**数据 scaling 曲线** (demo 数量 vs 成功率): 从少到多, 成功率上升然后渐饱和。这和 LLM 的 scaling 曲线同形。**机器人的成功率主要被数据量+多样性卡住** —— 这是当前最大杠杆 (M11.1-L4)。

> 注意 (接 9.4 实验设计): scaling 曲线要在**固定其它变量** (模型/训练步/评估集) 下画, 才是干净的因果对比。

---

## 2. 数据多样性 > 单纯数据量

scaling 不只是「更多」, 更是「更广」:
- 100 条同一任务的 demo < 100 条覆盖多任务/物体/布置的 demo。
- 多样性带来**泛化**和**正迁移** (M11.1-L4: 每条 demo 改进所有机器人)。

> 这和 LLM 一样: 数据多样性决定泛化。机器人 scaling 的关键不是「同一任务采一万次」, 而是「覆盖足够多样的场景」。你 N2 的 toy 里, 每条 demo 是随机起点+目标, 天然多样 —— 所以少量 demo 就能泛化到新起点/目标。

---

## 3. 与 VLM 数据 co-train: 防遗忘 + 保泛化

一个关键技巧 (RT-2 的做法): 训 VLA 时**不只用机器人数据, 还混入原始 VLM 数据** (图文) 一起训:

```
   只用机器人数据微调 → VLM 可能"遗忘"它的视觉/语言常识 → 失去泛化
   机器人数据 + VLM 数据 co-train → 保住 VLM 知识 + 学会控制 → 泛化保留
```

> 为什么? 机器人数据相对少, 纯用它微调会让强大的 VLM backbone **灾难性遗忘** (M10.3 的遗忘问题)。混入 VLM 数据当「锚」, 保住预训练知识。这就是 RT-2 涌现泛化的关键之一: **co-train 让机器人会动的同时, 不忘怎么理解世界**。

> 这和你 M10.3 的「冻结/联合微调防遗忘」、Module「数据治理」的「数据配比」完全一脉相承。VLA 训练 = BC + 数据配比 (机器人数据 : VLM 数据)。

---

## 4. 机器人 scaling 的现状 (诚实)

诚实地看:
- 机器人数据规模 (~百万级轨迹) **远小于** LLM (万亿 token)。
- scaling 定律刚开始显现, 还没到 LLM 那种「越大越好」的成熟阶段。
- 数据是瓶颈 → 仿真 (M11.6)、人类视频 (L4)、数据高效方法都在补这个缺口。

> 判断 (接 M9 研究品味): 机器人正处在「数据 scaling 的早期」—— 这意味着**数据是当前最大杠杆也是最大机会**。谁解决了「数据高效/便宜地 scale」, 谁就推动了领域。这是个缺口大、价值高的方向。

---

## 5. 本讲小结 + 通往 L4

- **数据量 → 成功率**: scaling 曲线 (N2 量到), 上升渐饱和, 同 LLM。
- **多样性 > 单纯量**: 覆盖多样场景才泛化/正迁移 (M11.1-L4)。
- **与 VLM 数据 co-train**: 防灾难性遗忘 (M10.3), 保住 VLM 知识 → 涌现泛化 (RT-2 关键)。
- 现状: 机器人数据规模远小于 LLM, 数据是最大瓶颈也是最大机会。
