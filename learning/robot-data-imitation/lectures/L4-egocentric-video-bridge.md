# L4 · egocentric video 当桥: 人类视频 → 物理智能

> 20-min lecture · 目标: 理解人类第一人称视频怎么当机器人数据的桥, 绕过昂贵遥操作。收口 11.4。

---

## 0. 终极的数据来源: 人类视频

L2 说遥操作贵。有没有更便宜、更海量的机器人数据? 有 —— **人类做事的视频**, 尤其是 **egocentric (第一人称) 视频** (人戴相机做家务/操作)。互联网 + 专门数据集 (Ego4D 等) 有海量这种视频。如果能从中学到「怎么操作物体」, 就绕过了遥操作瓶颈。

---

## 1. 为什么人类视频是诱人的桥

```
   遥操作 demo:  贵, 慢, 要人专门操作机器人      → 百万级
   人类视频:     海量, 便宜 (人本来就在做事 + 网上有) → 十亿级
```

人类视频里藏着丰富的**物理操作先验**:
- 物体怎么被抓/推/倒 (操作物理)。
- 手怎么接近物体 (运动学先验)。
- 任务怎么分步完成 (任务结构)。

> 如果能把这些先验迁到机器人, 就用**便宜海量**的人类视频补上**昂贵稀缺**的机器人数据。这是 2024-2026 具身的热门方向 (也是 GR00T 等的数据策略之一)。

---

## 2. 难点: 人和机器人不一样 (embodiment gap)

人类视频不能直接当机器人 demo, 因为:
- **没有动作标签**: 视频只有画面, 没有「关节指令」(机器人要的动作)。
- **形态不同 (embodiment gap)**: 人的手 ≠ 机器人夹爪, 人的视角/比例 ≠ 机器人。
- **视角/相机不同**: 第一人称人眼 ≠ 机器人相机。

> 核心难题: **怎么把「人做事的视频」翻译成「机器人能学的 (观测, 动作)」**。这是 open 问题。

---

## 3. 怎么搭这座桥 (几条路线)

- **学动作的潜在表示**: 从视频自学「潜在动作」(像 M13.5-L3 的 Genie), 不需显式动作标签。
- **预训练表示 + 少量真机微调**: 用海量人类视频**预训练感知/动力学表示**, 再用少量机器人 demo 微调动作 (迁移学习, 同 M10 预训练→微调)。
- **手部姿态当代理动作**: 从视频估计人手轨迹, 当成机器人末端动作的代理。
- **video → world model**: 人类视频训世界模型 (M13.5), 给机器人当模拟器/先验。

> 共同思想: **用人类视频学「物理/操作的通用先验」(便宜海量), 用少量真机数据学「具体形态的动作」(昂贵稀缺)**。这是「预训练 + 微调」范式 (你 M10 的核心) 搬到具身: 视频预训练 + 真机微调。

---

## 4. 串起 11.4 的数据故事

```
   BC (L1):          从专家 demo 监督学 (基础), 但分布漂移
   遥操作 (L2):      demo 主来源, 贵/慢, 质量决定上限
   scaling (L3):     更多更广数据 → 更高成功率 + co-train 防遗忘
   人类视频 (L4):    便宜海量的桥, 补真机数据缺口 (embodiment gap 是难点)
   ─────────────────────────────────────────
   = 机器人数据的全景: 贵但精 (真机) + 便宜但糙 (视频/仿真) 的组合
```

> 大图景: 机器人数据 = 「昂贵精准的真机 demo」+ 「便宜海量的视频/仿真」的组合拳。前者教具体动作, 后者教通用先验。**数据策略 = 怎么配比这两类**, 是 VLA 工程的核心决策之一 (接 M11.6 仿真)。

---

## 5. 本讲小结 (11.4 收口) + 通往 11.5

- **人类 egocentric 视频** = 便宜海量的数据桥, 补真机数据缺口。
- 难点 **embodiment gap**: 没动作标签 + 形态/视角不同。
- 搭桥路线: 潜在动作 (Genie 式) / 视频预训练+真机微调 / 手部姿态代理 / video→世界模型。
- 11.4 数据全景: 真机 (贵精) + 视频/仿真 (便宜糙) 的组合; 数据策略 = 配比。

> **M11.4 收口**: BC 是训练基础但有分布漂移; 数据靠遥操作 (贵) + scaling + co-train; 人类视频是降本的桥。
