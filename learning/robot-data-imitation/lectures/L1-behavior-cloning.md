# L1 · 行为克隆 (behavior cloning) 基础

> 24-min lecture · 目标: 理解 BC = 从专家 demo 监督学策略, 及它的经典毛病 (分布漂移/复合误差)。Module 11.4 开篇。

---

## 0. 动作头怎么训? 最基础的答案: 模仿

M11.2/11.3 造了动作头 (离散/连续/扩散)。怎么训它? 最基础、也是机器人基础模型训练的大头: **行为克隆 (behavior cloning, BC)** —— 从专家 demo 监督学。

```
   专家 demo: 一堆 (观测, 专家动作) 对
   BC:        监督学一个策略 π, 让 π(观测) ≈ 专家动作
   = 纯监督学习 (回归/分类), 你最熟的范式
```

> **BC = 把控制变成监督学习**: 给状态预测专家会做的动作。你 11.1 的 next-action、11.2 的 mini-VLA 训练、11.3 的动作头训练, 本质都是 BC。这一讲讲透 BC 的原理和坑。

---

## 1. BC 就是监督学习

```
   收集: 专家演示 N 条轨迹 → 拆成 (state, action) 对
   训练: min Σ loss(π(state), expert_action)
         连续动作 → MSE; 离散动作 token → 交叉熵
   部署: 用 π rollout
```

> 没有新东西: BC = 监督回归/分类。你 `bc_train.py` 就是 `state→action` 的回归 MLP。**机器人控制最基础的训练 = 你最熟的监督学习。** 大规模 VLA (OpenVLA 等) 的主训练目标就是 BC (在海量 demo 上)。

---

## 2. BC 的经典毛病: 分布漂移 (distribution shift)

BC 有个著名的坑 —— **复合误差 (compounding error)**:

```
   训练: 只见过专家走过的状态 (专家很完美, 不会走到"坏"状态)
   测试: π 有小误差 → 走到专家没去过的状态 → π 在这些状态没学过 → 更大误差 → 更偏...
         (误差像滚雪球, compounding)
```

> 这和 M13.5 世界模型的误差累积是**同一类问题** (自回归式的误差滚雪球)。BC 只在「专家轨迹分布」上训练, 一旦偏出去就没救。你 N1 会看到: demo 太少时, BC 在专家没覆盖的状态乱走 (分布漂移)。

---

## 3. 怎么缓解分布漂移

经典解法 (知道名字即可):
- **更多更广的数据**: 覆盖更多状态 (含「从错误中恢复」的 demo)。最直接, 也是 scaling 的动力 (L3)。
- **DAgger**: 让专家标注 π 实际走到的状态 (把「坏状态」也教了)。
- **加噪声 demo**: 专家演示时故意加扰动, 教 π 从偏差恢复。

> 主线: **让训练分布覆盖测试时会遇到的状态**。最暴力也最有效的是**数据 scaling** (更多更广 demo, L3)。这就是为什么机器人界拼命采数据 (M11.1-L4 的「每条 demo 改进所有机器人」)。

---

## 4. BC vs RL (要不要环境交互)

| | BC (模仿) | RL (试错) |
|---|---|---|
| 数据 | 专家 demo (离线) | 环境交互 (在线) |
| 信号 | 「专家怎么做」 | 奖励 (好坏) |
| 难点 | 分布漂移 | 探索/奖励设计/样本效率 |
| 机器人现状 | **主力** (安全/可控) | 辅助/仿真里用 (M11.5/11.6) |

> 机器人现在**以 BC 为主** (真机 RL 太贵太危险)。RL 多在仿真里用 (M11.6) 或配世界模型 (M11.5)。BC 简单、安全、可 scale —— 是机器人基础模型的训练基石。

---

## 5. 本讲小结 + 通往 L2

- **BC = 从专家 demo 监督学策略** (state→action 回归/分类); 机器人训练的大头。
- 经典毛病 **分布漂移/复合误差**: 偏出专家分布就滚雪球 (同 M13.5 误差累积)。
- 缓解: 更多更广数据 (scaling, 主力) / DAgger / 加噪 demo。
- 机器人以 BC 为主 (安全可 scale), RL 辅助 (仿真/世界模型)。

> **下一讲 L2「遥操作数据采集 + 质量」**: BC 吃的数据从哪来? 人类遥操作机器人采集。数据质量怎么决定 BC 上限。

**动手**: 去 `N1`, 在 toy 上做 BC (从合成专家 demo 学), 看它能走到目标; 再看 demo 太少时的分布漂移 (走到没覆盖的状态乱走)。
