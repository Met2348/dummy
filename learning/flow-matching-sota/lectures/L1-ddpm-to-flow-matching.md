# L1 · 从 DDPM 到 flow matching: 连续视角与速度场

> 30-min lecture · 目标: 理解 flow matching 如何换一个视角 (学速度场而非去噪) 攻克 DDPM 的 T 步慢, 建立连续时间 (ODE) 的直觉。你 EE 的微分方程/向量场背景是优势。

---

## 0. DDPM 的痛点回顾

M13.1-L4: DDPM 生成要迭代 $T$ 步反向去噪, 慢 (原版 1000 步)。整个采样加速研究都在攻这个。**flow matching 是当前最优雅的答案** —— 它换个视角看「从噪声到数据」, 让路径可以「拉直」, 用极少步走完。

---

## 1. 换视角: 不学去噪, 学「速度场」

DDPM 学的是「每步去掉多少噪声」(离散步)。flow matching 学的是一个连续的**速度场 (velocity field)**:

> **想象「噪声分布」和「数据分布」之间有一条河, 每个点上有一个箭头 (速度), 指示「该往哪个方向、多快地流向数据」。学会这个速度场, 生成就是: 从噪声出发, 顺着箭头流, 流到数据。**

```
   DDPM (离散去噪):              flow matching (连续流):
   噪声 →去噪→ ... →去噪→ 数据    噪声 ──沿速度场积分──→ 数据
   T 个离散步                     一条连续 ODE 轨迹 (步数可调)
```

机制 (`flow_matching.build_velocity_field`): 一个网络 $v_\theta(x, t)$, 输入当前点 $x$ 和时间 $t$, 输出速度向量 (该往哪走)。

---

## 2. 训练: 匹配「噪声→数据」的直线速度

flow matching 训练简单到惊人 (conditional flow matching):

```
   1. 取一个数据点 x1, 配一个随机噪声 x0
   2. 在它们之间走直线: x_t = (1-t)·x0 + t·x1,  t ∈ [0,1]
   3. 这条直线的速度恒为 (x1 - x0)
   4. 训练 v_θ(x_t, t) 去匹配这个速度
```

训练目标:
$$\mathcal{L} = \mathbb{E}_{x_0, x_1, t}\left[\ \big\|\ v_\theta(x_t, t) - (x_1 - x_0)\ \big\|^2\ \right]$$

逐项交代:
- **$x_0$**: 随机噪声 (路径起点)。**$x_1$**: 真数据 (路径终点)。
- **$x_t = (1-t)x_0 + t x_1$**: 噪声和数据之间的**直线插值** ($t=0$ 是噪声, $t=1$ 是数据)。
- **$x_1 - x_0$**: 这条直线的速度 (恒定, 因为是直线)。
- **$v_\theta$**: 网络预测的速度。损失 = 预测速度 vs 直线速度的 MSE。
- **整体**: 又是一个简单的 MSE 回归 (和 DDPM 一样简洁), 但学的是「速度」而非「噪声」。

> 关键洞察: 每个数据点配一个随机噪声、走直线、学这条直线的速度。**单看一对 (x0, x1) 是直线; 但很多对叠加起来, 网络学到的是一个把整个噪声分布「流」成数据分布的速度场。** 这就是 `train_flow_matching` 干的。

---

## 3. 生成: 沿速度场积分 ODE

训好速度场后, 生成 = 从噪声出发, 沿速度场走 (ODE Euler 积分):

```
   x = 噪声
   for i in range(steps):
       v = v_θ(x, t)          # 当前点的速度
       x = x + v · dt          # 沿速度走一步 (Euler)
       t += dt
   返回 x (生成样本)
```

逐点 (`flow_matching.sample`):
- **$dt = 1/\text{steps}$**: 步长。steps 可以**很小** (这是卖点)。
- **$x \leftarrow x + v \cdot dt$**: Euler 积分一步 (沿速度走)。
- **steps 越多积分越准, 但 flow matching 的路径近直, 少步也够。**

> 你 EE 的 ODE 数值积分直觉直接用上: 这就是用 Euler 法解一个 ODE $\frac{dx}{dt} = v_\theta(x,t)$。步数 = 积分精度。DDPM 的随机去噪 (SDE) 路径弯曲、要多步; flow matching 的路径可拉直 (L2), 少步就够。

---

## 4. 连续视角: SDE vs ODE

更深的统一 (你 EE 会喜欢): 扩散和 flow matching 都可看成连续时间过程:
- **DDPM ≈ 随机微分方程 (SDE)**: 每步有随机噪声, 路径是随机游走, 弯曲, 要多步。
- **flow matching ≈ 常微分方程 (ODE)**: 确定性速度场, 路径可设计成直, 少步。

> 这个「SDE vs ODE」视角 (Song et al. 的统一框架) 是扩散理论的高峰。对你的意义: ① flow matching 的少步优势来自「确定性 + 可拉直路径」; ② 这通向 rectified flow (L2) 的「把路径拉得更直」。你的随机过程/微分方程数学在这里是真正的理解力。

---

## 5. 本讲小结 + 通往 L2

- flow matching 换视角: 不学**去噪** (DDPM), 学**速度场** $v_\theta(x,t)$ —— 每点指示「往哪流向数据」。
- **训练**: 数据配随机噪声走直线, 学这条直线的速度 (简单 MSE)。
- **生成**: 从噪声沿速度场 ODE 积分 (Euler), 步数可小。
- **SDE vs ODE**: DDPM 是随机 (弯曲, 多步), flow matching 是确定 (可拉直, 少步)。

> **下一讲 L2「rectified flow」**: flow matching 的路径已比 DDPM 直, 但还能更直。rectified flow 通过「重流 (reflow)」把路径拉成几乎直线, 让 1-2 步生成成为可能。L2 讲这个「拉直」技巧。
