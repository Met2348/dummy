# L4 · 采样器全景: 一张地图理清所有方法

> 22-min lecture · 目标: 把 DDPM/DDIM/flow matching/rectified flow/consistency 放在一张统一地图上, 学会按需选采样方法。收口 M13.2。

---

## 0. 别被一堆名字淹没

扩散采样方法名字一大堆 (DDPM/DDIM/DPM-Solver/flow matching/rectified flow/consistency/...)。本讲给你一张地图, 让你理解它们的**统一视角**和**怎么选**, 不被名字淹没。

---

## 1. 统一视角: 都在「从噪声走到数据」

所有方法做同一件事: **从噪声分布走到数据分布**。区别只在两个维度:

```
   维度 1: 路径是随机 (SDE) 还是确定 (ODE)?
   维度 2: 路径有多直 (决定要多少步)?
```

| 方法 | 路径类型 | 直度 | 典型步数 |
|---|---|---|---|
| **DDPM** | 随机 (SDE) | 弯 | ~1000 |
| **DDIM** | 确定 (ODE) | 弯 | ~10-50 |
| **DPM-Solver** | 确定 + 高阶积分 | 弯但积分准 | ~10-20 |
| **flow matching** | 确定 (ODE) | 较直 | ~10-50 |
| **rectified flow** | 确定 + reflow 拉直 | 很直 | ~1-4 |
| **consistency** | 直接学一步映射 | (绕过路径) | ~1-4 |

> 一句话统一: **要少步, 要么把路径变直 (rectified flow), 要么用更准的积分器 (DPM-Solver), 要么绕过路径直接学映射 (consistency)。** 所有方法都是这三招的变体。理解了「路径弯曲 = 步数代价」这个核心 (L2), 整张地图就清晰了。

---

## 2. 怎么选 (决策树)

```
   你的约束是什么?
   │
   ├─ 要极致质量, 不在乎慢 (离线生成图/视频)?
   │     → DDPM 多步 / DDIM 中步  [稳, 质量天花板高]
   │
   ├─ 要快 + 还要不错质量 (交互式生成)?
   │     → DPM-Solver / flow matching 少步  [折中]
   │
   └─ 要极速, 接近实时 (实时 omni / 机器人控制)?
         → rectified flow / consistency 1-4 步  [极快, 质量够用]
```

逐层逻辑 (呼应 9.4 的「有约束的选择」):
- **离线高质量**: 步数不是问题, 用稳的多步方法。
- **交互式**: 平衡速度和质量, 用 DPM-Solver 或 flow matching 几十步。
- **实时**: 速度优先, 用 1-4 步方法 (接 M10.5 实时 / M11 机器人)。

---

## 3. 训练范式也在统一: flow matching 成新主流

值得指出一个趋势: **2024-26 的大模型 (SD3、Flux、部分视频模型) 训练直接用 flow matching 范式**, 而非 DDPM。原因:
- flow matching 训练目标更简单稳定 (速度匹配)。
- 天然支持 rectified flow 拉直 → 少步采样。
- 连续视角更优雅, 易扩展 (多模态、条件)。

> 对你的意义: 如果现在入门扩散, **flow matching 是更现代的起点** (虽然 DDPM 是理解地基, M13.1)。理解 DDPM (机制) + flow matching (现代实践) 两者, 你就跟上了 2026 的扩散主流。

## 4. M13.2 收口: 速度是落地的命门

```
   M13.1 DDPM: 扩散机制 (但 T 步慢)
   M13.2: 攻速度
     L1 flow matching (学速度场, ODE 视角)
     L2 rectified flow (拉直路径, 逼近 1 步)
     L3 consistency (直接学一步映射)
     L4 采样器全景 (统一视角 + 怎么选)
   ───────────────────────────────────
   速度从「上千步」压到「1-4 步」, 让实时生成成为可能
```

> 速度是扩散从「能生成」到「能落地」的关键。实时 omni (M10.5)、机器人实时控制 (M11.3)、交互式创作, 全靠这些少步方法。**会扩散的机制 (M13.1) + 会让它快 (M13.2) = 能用扩散做实际系统。**

## 5. 本讲小结 (M13.2 收口) + 通往 M13.3

- 所有采样方法统一在「从噪声走到数据」, 区别在**路径随机/确定 + 直度** (决定步数)。
- 少步三招: **拉直路径** (rectified flow) / **高阶积分** (DPM-Solver) / **绕过路径学映射** (consistency)。
- 选法 (有约束): 离线高质量→多步; 交互→折中; 实时→1-4 步 (接 M10.5/M11)。
- **flow matching 成 2024-26 训练新主流** (SD3/Flux); DDPM 是理解地基, flow matching 是现代实践。
