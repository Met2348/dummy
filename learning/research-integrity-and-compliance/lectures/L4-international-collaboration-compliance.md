# L4 · 国际合作合规 (International Collaboration Compliance)

> 30-min lecture · 目标: 搞清出口管制(export control)的基础概念在什么情况下会套用到再正常不过的学术研究上, 以及跨境数据传输限制会怎么约束一次国际合作项目的数据共享方式。
> 关键区分: **`research-funding-and-resources`(9.14)L5「供应商/API合规」管的是你和第三方厂商之间的合同性合规(数据处理协议DPA、API使用条款)——本质是"你和一个具体商业伙伴之间怎么约定"; 本讲管的是国家层面强制施加的合规义务(出口管制法规、跨境数据传输法律)——本质是"不管你和合作方怎么约定, 政府规定你不能做什么"。** 前者可以通过谈判改条款, 后者是硬性法律边界, 不可协商。

---

## 0. 为什么这是个真问题

NLP/LLM方向的国际合作极其普遍: 和海外实验室联合发论文、招募国际学生/访问学者、把训练数据存在境外云服务上、和跨国公司的海外团队共享代码。多数博士生对这些日常操作完全没有"这可能触发法律合规义务"的意识——直到某天导师或机构合规办公室提醒"这个合作方在受限清单上""这批数据不能直接传到境外服务器"时才第一次意识到, 学术研究并不因为"以发表为目的"就自动豁免所有国家层面的管制规则。

```
                    没有本讲的默认心态                          有本讲之后
国际合作             "都是搞学术的, 互相帮忙很正常"                合作前先核实对方机构/个人是否在受限清单上, 是否
                                                               涉及需要额外许可的技术类别
境内的国际学生/访客   "他/她只是我实验室的一员, 跟合规没关系"        意识到"deemed export"概念: 向本国境内的外国籍
                                                               人员传授受控技术, 本身就可能被视为一种"出口"
跨境数据传输          "数据加密传输就没有合规问题了"                意识到加密解决的是安全问题, 不解决法律上"是否
                                                               允许把这类数据传到境外"这一独立义务
```

## 1. 出口管制基础概念

出口管制原本是国家安全领域的概念, 但学术研究在两种情况下会被卷入这套规则:

```
是否落入"基础研究豁免"(fundamental research exclusion)?
      │
      ├─ 是: 研究以公开发表为目的, 未接受资助方对发表/访问施加的限制性条款
      │      → 通常豁免出口管制, 这也是为什么绝大多数纯学术研究不受影响
      │
      └─ 否: 一旦出现以下任一情形, 豁免可能失效
             · 资助合同里包含"发表前需审查/可能限制发表"的条款
             · 项目限制了特定国籍人员的访问权限(这本身有时反而触发管制而非规避)
             · 研究内容属于明确受控的技术类别(如特定军民两用技术)
```

美国的出口管制主要由两套并行法规构成: **《出口管理条例》(Export Administration Regulations, EAR, 由商务部工业与安全局BIS执行)**面向民用及军民两用技术, **《国际武器交易条例》(International Traffic in Arms Regulations, ITAR, 由国务院国防贸易管制局DDTC执行)**面向国防相关物项和技术, 两套体系管辖范围不同, 判断一项研究该对照哪一套(或都不涉及), 通常需要机构出口管制合规办公室介入而非研究者自行判断。

一个格外容易被忽视的概念是**"视同出口"(deemed export)**: 把受控技术信息传授给身处本国境内的外国籍人员(比如你实验室里的国际博士后), 在法律上可能被视同"出口"到该人员的国籍所属国——这意味着"合作是境内进行的"不等于"这件事和出口管制无关"。此外, 合作前核实对方机构/个人是否出现在美国商务部工业与安全局维护的**受限实体清单(Entity List)**等类似名单上, 是国际合作立项阶段就该完成的尽职调查步骤, 而不是等合作出问题才去查。

## 2. 跨境数据传输限制

除了"技术"本身的出口管制, 数据跨境流动本身也受越来越严格的法律约束, 而且不同法域规则并不统一:

- **欧盟GDPR第五章(第44-50条)**: 个人数据向欧盟以外地区传输, 原则上需要满足"充分性认定(adequacy decision, 欧盟委员会认定目的地国家法律保护水平足够)"或采用"标准合同条款(Standard Contractual Clauses, SCCs)"等法定传输机制之一, 不能简单地"把数据拷到境外服务器"就算合规。
- **中国《个人信息保护法》(PIPL, 2021年施行)**: 对涉及重要数据或达到一定规模的个人信息跨境提供设有安全评估要求, 是理解"数据出境不是默认自由, 需要走评估程序"这一原则在中国法律体系下的对应规定。
- **一般原则**: 越来越多法域采用"数据本地化(data localization)"或"传输前评估"的立法思路, 这意味着国际合作项目在设计数据共享方案时, 不能想当然地假设"我们互相信任, 数据传过去就行", 而需要先核实数据来源地和目的地各自的法律要求。

> 这里和`research-funding-and-resources`L5提到的GDPR第28条(处理者需签署数据处理协议DPA)是同一部法律里的不同条款: 第28条管的是"你和处理数据的供应商之间的合同关系", 本讲说的第44-50条管的是"数据本身能不能跨越国境"——两条规则经常同时适用于同一个国际合作项目, 但解决的是不同问题, 不能用一份文件同时打发两边的审查。

## 3. 一个完整走查示例

```
场景: 你所在的实验室计划和一所海外大学联合研究一种新的多语言对齐训练方法, 对方提供
      一批本地语言的人类反馈标注数据, 你方提供算力和训练代码。

①合作前尽职调查   核实对方机构和主要合作者是否出现在相关受限清单上; 确认这项多语言对齐
                  方法本身不涉及需要出口许可的受控技术类别(多数NLP训练方法属于此列,
                  但涉及特定敏感应用场景时需专门核实)。
②基础研究豁免核实  确认合作协议中不包含"资助方可审查/限制发表"这类条款, 以保住基础研究豁免。
③跨境数据传输方案  标注数据若含被试的地理位置/人口统计等个人信息, 核实数据来源地(对方国家)
                  的跨境传输规定, 视需要采用标准合同条款或走安全评估流程, 而非直接FTP传输了事。
④境内协作人员      如果对方派驻访问学者到你方实验室参与该项目, 核实该访问学者接触的技术
                  内容是否涉及"视同出口"层面的限制。
```

## 4. 常见误区

1. **"都是发表为目的的学术研究, 不受出口管制"** —— 只有在真正满足基础研究豁免条件时才成立, 一旦资助合同附带发表限制或访问限制条款, 豁免可能随之失效。
2. **"实验室里的国际学生/博士后跟出口管制没关系, 因为合作是在境内做的"** —— "视同出口"概念恰恰说明, 向境内的外国籍人员传授受控技术信息本身可能构成需要审查的行为。
3. **"数据加密传输就解决了合规问题"** —— 加密解决的是数据安全(防止被截获), 不解决"是否被法律允许传输到该目的地"这一独立的合规义务, 两者不能互相替代。
4. **"我们互相信任, 口头说好数据怎么用就够了"** —— 跨境合作的责任划分和数据使用范围应当参照`research-funding-and-resources`L4提到的书面合作协议原则, 尤其在涉及出口管制/数据跨境规则时, 书面记录是证明"我们尽了合理核查义务"的关键材料。

---

## 5. 本讲小结 + 通往 L5

- 出口管制原本管的是国家安全相关技术, 但"基础研究豁免"是否成立取决于合作协议条款, 一旦被资助方施加发表/访问限制, 豁免可能失效; "视同出口"意味着境内的外国籍协作者本身也可能触发合规审查。
- 跨境数据传输受GDPR第44-50条、中国PIPL等法律约束, 和"数据处理协议(DPA)"是同一份国际合作里两个不同层面的合规义务, 不能互相替代。
- 国际合作立项阶段就该做受限清单尽职调查, 而不是等合作出问题才去查。

> **下一讲 L5「安全与负责任披露」**: 本讲谈的是国家/法律层面强制施加的合规边界, L5要谈一个方向不同但同样"事前没规划、事后就麻烦"的场景——当你的研究(尤其是AI安全/red-teaming类研究)发现了一个真实存在的漏洞或危险能力, 在决定怎么、什么时候、公开到什么程度这件事上, 有没有一套负责任的流程可以遵循。

**动手**: 打开 `templates/compliance-checklist.md` 的第4小节(`export_control`), 对你现在(或设想中)的一次国际合作, 写下合作对象所在地区是否核实过受限清单、涉及的数据跨境传输是否有具体的合规安排, 而不是一句"应该没问题"。
