# L3 · domain randomization 弥合 gap

> 22-min lecture · 目标: 理解 domain randomization (DR) 怎么弥合 sim2real gap, 及它为什么有效。你 N1 实测。

---

## 0. 与其追求完美仿真, 不如随机化

L1 说 sim2real gap 来自 sim≠real。一个想法是「把仿真做得无限真实」—— 但做不到 (永远有未建模因素)。**domain randomization (DR)** 换个思路:

> **与其让 sim 精确等于某一个 real, 不如训练时随机化 sim 参数, 让策略见过一大片可能的 real。** 真实只要落在这片随机化范围里, 策略就鲁棒。

这是 sim2real 最实用、最主流的解法。

---

## 1. DR 的机制: 随机化训练分布

```
   不用 DR: 在一个固定 sim 配置训练 → 只对那个配置鲁棒 → real 偏了就掉点
   用 DR:   每个训练 episode 随机化参数 (摩擦/光照/质量/噪声/...) → 见过一大片 → real 落在范围内就稳
```

随机化什么 (真实 IsaacLab 里):
- **物理**: 摩擦系数、质量、电机强度、关节阻尼。
- **视觉**: 光照、纹理、相机位置/角度。
- **传感器**: 噪声、延迟、畸变。
- **场景**: 物体位置、干扰物。

> 核心: **把「真实的不确定性」在训练时主动注入**。策略被迫学到对这些变化鲁棒的行为, 而不是死记某个 sim 配置。这就是为什么 DR 能弥合 gap: **它把 real 的多样性提前在 sim 里见了。**

---

## 2. 为什么 DR 有效: 覆盖 = 泛化 (接 M11.4)

DR 的本质和你 M11.4 的分布漂移、机器学习的泛化是**同一回事**:

```
   模型只在"见过的分布"上可靠 (M11.4)
   real 是 sim 没见过的分布 → gap (L1)
   DR = 主动把训练分布(sim)扩到覆盖 real → real 不再是"没见过" → 泛化
```

> 你 N1 会坐实这点: 在**窄**配置 (sim) 训的策略, 到**全**配置 (real) 掉点 (覆盖不足); 用 DR (训练随机化到**全**配置) → real 泛化。**DR = 用覆盖换泛化**, 和「数据多样性→泛化」(M11.4-L3) 是同一智慧。

> 你 N1 的数字会清楚显示: 无 DR 在真实全区域成功率掉 (如 0.7), DR 拉回 (如 0.99)。**gap 被覆盖弥合。**

---

## 3. DR 的代价与权衡 (诚实)

DR 不是免费 (接 M9 批判):
- **更难训**: 随机化范围大 → 任务变难 (要对一大片鲁棒) → 训练更慢/可能学不到极致性能。
- **保守性**: 为鲁棒可能牺牲单一配置的最优 (鲁棒 vs 最优权衡)。
- **范围要对**: 随机化范围要**覆盖但不过宽** —— 太窄漏掉 real (没弥合 gap), 太宽任务太难 (学不动)。这个范围是关键超参 (该调, 接 9.4)。

> 实践智慧: **随机化范围要"刚好覆盖真实的不确定性"**。太窄=gap 没补上, 太宽=训练崩。怎么定范围是 DR 的核心工程 (常配合真机少量数据校准范围)。没有银弹, 范围要按真实的不确定性来定。

---

## 4. DR 之外的补充 (组合拳)

DR 常和其它手段组合:
- **系统辨识 + DR**: 先测真实参数大致范围, 再以此为中心随机化 (范围更准)。
- **真机微调**: DR 预训练 + 少量真机微调 (M11.4 预训练+微调)。
- **自适应 DR**: 训练中动态调随机化范围 (从易到难)。
- **更真渲染** (sim fidelity): 减小需要随机化的差异。

> 主线仍是 DR, 其它是让 DR 更准/更省。一句话: **sim2real = DR (覆盖不确定性) + 少量真机 (校准/微调)**。这是当前最实用的范式。

---

## 5. 本讲小结 + 通往 L4

- **DR = 训练时随机化 sim 参数** (物理/视觉/传感器/场景), 让策略见过一大片可能的 real。
- 为什么有效: **覆盖=泛化** (同 M11.4); 把 real 的多样性提前在 sim 见了 → real 不再"没见过"。
- N1 实测: 无 DR 真实掉点 (覆盖不足), DR 拉回 (覆盖弥合 gap)。
- 代价: 更难训 + 鲁棒vs最优 + **范围要刚好覆盖** (关键超参); 常配真机校准/微调。
