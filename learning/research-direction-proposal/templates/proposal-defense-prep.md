# 开题答辩准备清单

> 配套 L4、`src/proposal_audit.py` 的 `defense_focus()`。开题答辩关心"值不值得做/做不做得完",和毕业答辩(关心"做得对不对")性质不同。

## 1. 预判质询维度(结合L2可行性框架)
- [ ] 资源:算力/数据是否够?
- [ ] 时间线:是否过于乐观?
- [ ] 方法风险:主方案不work怎么办?
- [ ] 范围:方向是选窄了还是选宽了?

## 2. 口头陈述结构(把开题报告压缩成陈述)
- [ ] 一句话动机(为什么现在做这个问题)
- [ ] 一张图讲清研究缺口
- [ ] 具体研究问题(可证伪的那句话)
- [ ] 时间线概览(不需要逐月讲,给阶段性里程碑)
- [ ] 主动亮出1-2个最大风险 + 应对预案(不要等被问)

## 3. 应对"边界性质询"的心理预案
- 如果评审说方向选窄了/选宽了 → 先复述对方的关切,再回应(不要立刻改口显得没有主见,也不要完全不听显得固执)
- 如果评审建议改方向 → 记录具体建议,会后重新走一遍`direction-scorecard.md`,不要当场仓促承诺或拒绝

## 4. 用 `proposal_audit.py` 自动预判追问点
```python
from proposal_audit import defense_focus
for q in defense_focus(my_proposal_dict):
    print(q)
```
