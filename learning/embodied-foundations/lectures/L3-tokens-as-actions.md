# L3 · tokens-as-actions: 控制 = 预测下一个 token

> 22-min lecture · 目标: 理解「把动作当 token」这个范式, 它让 VLM 能直接拿来控制机器人 (复用你 LLM 的全套)。

---

## 0. 一个让人会心一笑的想法

你的 LLM 会做一件事: **预测下一个 token**。RT-2 的核心洞察是:

> **如果把机器人动作也变成 token, 那么「控制机器人」就 = 「预测下一个 token」** —— 你 LLM 的全套 (架构/训练/采样/scaling) 直接可用。

这就是 **tokens-as-actions** 范式。它是 VLA 能复用 VLM 的根本原因。

---

## 1. 怎么把动作变成 token

连续动作 (如关节角 [0.3, -0.7, 1.2, ...]) 怎么变 token? **离散化 (discretization)**:

```
   连续动作维度 → 分桶 (如每维分 256 桶) → 每个桶 = 一个 token
   [0.3, -0.7] → [token_182, token_44]
```

- RT-2: 把每个动作维度量化成 256 个离散值, 复用 (或新增) 词表里的 token。
- 一个动作 = 几个动作 token 的序列 (每维一个)。

> 你的 `action_serialize.py` 是最小版: 把 2D 连续动作离散成 9 个方向 token。你 N1 会看到「离散化损失」—— 离散化必然损失精度 (方向被量化), 这是 tokens-as-actions 的固有代价 (M11.2-L3 离散vs连续会深入)。

---

## 2. 于是控制变成 next-token 预测

序列化之后, 一切都熟悉了:

```
   输入序列: [图像 token...] [指令 token...] [当前状态] → 预测: [动作 token1][动作 token2]...
                                                            ↑ 就是 next-token 预测!
```

- 训练: 和 LLM 一样的交叉熵 (预测正确的动作 token)。这就是**模仿学习**的 token 版 (M11.4)。
- 生成: 和 LLM 一样自回归吐出动作 token, 再解码回连续动作执行。

> 你 N2 会做这个: 一个 tiny transformer 在「状态序列 → 下一动作 token」上训练, 就是**具身版的 next-token 预测**。机制和你的 LLM 一模一样, 只是 token 的含义从「词」变成「动作」。

---

## 3. 为什么这个范式如此强大

tokens-as-actions 让机器人**白嫖** LLM/VLM 的全部基础设施:
- **架构**: 直接用 transformer/VLM, 不用设计新结构。
- **预训练知识**: VLM 的视觉/语言/常识全部可迁移 (RT-2 的涌现泛化)。
- **训练机器**: 同一套 loss/优化器/scaling 经验 (你的 M1-M8 全用得上)。
- **多模态统一**: 图/文/动作都是 token, 一个序列搞定 (接 M10.5 统一 token 化)。
- **语言可指挥**: 指令是 token, 自然融入。

> 这就是为什么 2023 后机器人界全面拥抱 VLA: **不用重新发明轮子, 直接站在 LLM/VLM 的肩上**。你作为 LLM 人, 这个范式对你几乎零门槛。

---

## 4. 但 tokens-as-actions 不是终点 (离散化的代价)

把动作离散成 token 有代价, 这推动了 M11.3 的演化:
- **精度损失**: 离散化丢精度 (你 N1 的离散化损失), 精细操作 (插孔/倒水) 不够。
- **多峰分布**: 同一状态可能有多个合理动作 (绕障碍可左可右), 离散 argmax 难表达 (M11.3-L1)。
- **平滑性**: 逐 token 独立预测的动作可能不平滑 (抖)。

> 所以前沿动作头从「离散 token」转向**扩散/flow 动作头** (M11.3, 接你 M13): 直接生成连续、多峰、平滑的动作。tokens-as-actions 是入门范式和很多 VLA 的基础, 但不是精细控制的终点。

---

## 5. 本讲小结 + 通往 L4

- **tokens-as-actions**: 把动作离散成 token → 控制 = next-token 预测 → 复用 LLM/VLM 全套。
- 离散化: 连续动作分桶成 token; 一个动作 = 几个动作 token (有离散化损失)。
- 强大之处: 白嫖 VLM 架构/预训练知识/训练机器/多模态统一/语言可指挥。
- 代价: 精度损失/多峰难表达/不平滑 → 推动 M11.3 扩散动作头。

> **下一讲 L4「数据 scaling 的魔法」**: tokens-as-actions 让所有机器人数据能放一起训。L4 讲「每条 demo 改进所有机器人」的数据复用逻辑 —— 这是机器人基础模型最反直觉也最强的地方。

**动手**: 去 `N2`, 训一个 tiny transformer 做「状态 → 下一动作 token」预测 (具身版 next-token), 看它学会专家行为。这就是 RT-2 的玩具内核。
