# L1 · 什么是具身 AI + 为什么用基础模型

> 24-min lecture · 目标: 理解具身 AI = 把感知-推理-行动闭环交给学习模型, 以及为什么 2024-2026 转向「机器人基础模型」。Module 11 开篇, 接你 LLM/VLM 全栈。

---

## 0. 从「会说」到「会做」

你的 LLM (会说) 和 VLM (会看会说) 都在**信息空间**里。具身 AI (Embodied AI) 多一步: **在物理世界里行动** —— 控制一个身体 (机器人) 去改变世界。

```
   LLM:  文本 → 文本
   VLM:  图+文 → 文本                    (你的 M10)
   具身: 图+文+本体感觉 → 动作 → 改变世界  (M11, 闭环)
```

> 具身 AI 的核心多了两样: **动作输出** (不只是说, 还要动) 和 **闭环** (动了之后世界变了, 要持续感知-决策)。这把你的「生成/预测」能力接到了物理世界。

---

## 1. 具身的核心问题: robot learning

机器人要学的是一个**策略 (policy)**: 给定观测, 输出动作。

```
   π(action | observation)
   observation: 相机图 + 本体感觉 (关节角/位置) + 指令
   action: 关节力矩 / 末端位移 / 离散动作
```

- 传统机器人: 每个任务**手工编程** (逐任务硬编码运动规划)。脆、不泛化、换任务重写。
- 具身 AI: **学**一个策略, 最好一个模型干很多任务、泛化到新物体/新场景。

> 你的 toy 环境 (`toy_env.py`) 就是这个的最小形态: 状态 (agent+goal) → 动作 (2D 速度) → 转移。整个 M11 都在这个闭环上做文章: 怎么学好 π。

---

## 2. 为什么转向「机器人基础模型」(2024-2026 大势)

机器人界正在重演 NLP 的「基础模型革命」:
- **旧范式**: 每个任务/每个机器人单独训一个小策略。数据不共享, 不泛化。
- **新范式**: 训一个**大的、预训练的机器人基础模型** (VLA, vision-language-action), 在海量多任务/多机器人数据上学, 然后泛化/微调到新任务。

为什么现在? 三个条件成熟 (和 LLM 一样):
1. **架构**: VLM (你的 M10) 提供了强大的「感知+推理」核, 只需接个动作头 → VLA (M11.2)。
2. **数据**: 大规模机器人数据集 (Open X-Embodiment 等) + egocentric 人类视频 (M11.4)。
3. **算力 + 范式**: tokens-as-actions (L3) 让机器人控制复用 LLM 的全套训练机器。

> 一句话: **机器人正在变成「VLM + 动作头 + 海量数据」**, 走 LLM 走过的路。你已经懂 VLM 和 scaling, 所以你懂具身基础模型的 80%。M11 教你补上「动作」那 20%。

---

## 3. 为什么「基础模型」比「逐任务硬编码」强

| | 逐任务硬编码 | 机器人基础模型 (VLA) |
|---|---|---|
| 泛化 | 几乎不 | 跨任务/物体/场景 |
| 数据利用 | 每任务独立 | 所有数据共享 (L4) |
| 新任务 | 重写代码 | 少量 demo 微调 / 零样本 |
| 语言指令 | 难 | 天然 (继承 VLM) |

> 关键优势是 **数据复用 + 语言可指挥**: 一个 VLA 吃所有机器人的数据一起学 (L4 的「每条 demo 改进所有机器人」), 还能听自然语言指令 (继承 VLM)。这是逐任务硬编码永远做不到的。

## 5. 本讲小结 + 通往 L2

- 具身 AI = 学一个策略 π(动作|观测), 在物理世界闭环行动 (多了动作输出 + 闭环)。
- 2024-2026 转向**机器人基础模型 (VLA)**: 重演 NLP 的基础模型革命 (架构 VLM + 数据 + tokens-as-actions)。
- 基础模型胜逐任务硬编码: **数据复用 + 语言可指挥 + 泛化**。
- 你已懂 VLM + scaling = 懂具身基础模型 80%, M11 补「动作」那 20%。
