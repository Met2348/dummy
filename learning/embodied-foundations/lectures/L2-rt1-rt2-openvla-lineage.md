# L2 · RT-1 → RT-2 → OpenVLA: 机器人控制的「LLM 化」

> 22-min lecture · 目标: 沿谱系理解机器人控制怎么一步步变成「VLM + 动作 token」, 看清 VLA 的来路。

---

## 0. 一条清晰的演化线

机器人基础模型不是凭空出现的, 有一条清晰的「越来越像 LLM」的演化线:

```
   RT-1 (2022)    专用 transformer, 图+指令 → 离散动作 token
   RT-2 (2023)    直接用预训练 VLM, 把动作当 token 接到词表 → 涌现泛化
   OpenVLA (2024) 开源 VLA: 开源 VLM backbone + 动作头, 人人可复现
   π / GR00T(2024+) 更强动作头 (扩散/flow), 更大数据, 走向产品
```

> 主线: **从「专用机器人网络」走到「通用 VLM + 动作接口」**。每一代都更彻底地复用 LLM/VLM 的成果。理解这条线 = 理解 VLA 为什么长这样。

---

## 1. RT-1 (2022): 把控制 transformer 化

- 输入: 相机图像序列 + 语言指令。
- 输出: **离散化的动作 token** (把连续动作分桶成离散值)。
- 架构: 专门训练的 transformer (还不是预训练 VLM)。

> RT-1 的贡献: 证明「transformer + 动作离散成 token」能学多任务机器人控制。**第一次把控制问题变成 token 预测**。但它还从零训练, 没复用 VLM 的世界知识。

---

## 2. RT-2 (2023): 关键飞跃 — 直接用 VLM

- 核心想法: **不另训网络, 直接拿预训练 VLM (PaLI-X/PaLM-E), 把机器人动作当成额外的 token 接到它的词表后面**。
- 训练: 在「机器人数据 + 原始 VLM 数据」上联合微调 (co-fine-tune)。
- 结果: **涌现泛化** —— 因为复用了 VLM 的视觉/语言/常识, 机器人能泛化到训练没见过的物体/指令 (如「把草莓放到正确的碗里」需要知道草莓是什么)。

> RT-2 是范式飞跃: **机器人控制 = VLM 的下游任务**。VLM 学到的世界知识 (草莓/碗/空间关系) 直接迁移到控制。这坐实了 L1 的论点: 懂 VLM 就懂 VLA 大半。这就是 **tokens-as-actions** (L3) 的威力。

---

## 3. OpenVLA (2024): 开源 + 标准化

- 用开源 VLM backbone (如 Prismatic/Llama+视觉编码器) + 动作头。
- 在 Open X-Embodiment (大规模多机器人数据) 上训练。
- 开源权重/代码 → 成为社区基线 (像 Llama 之于 LLM)。

> OpenVLA 的意义: 把 RT-2 的闭源飞跃**民主化**, 人人可复现/微调。你的 M11.2 mini-VLA 就是它的玩具版 —— 同样的「VLM backbone + 动作头」结构, 只是规模小。

---

## 4. π 系列 / GR00T (2024+): 更强动作头 + 走向产品

- **π (Physical Intelligence)**: 用 **flow-matching 动作头** (接你 M13.2!), 高频精细控制, 真机灵巧操作。
- **GR00T (NVIDIA)**: 人形机器人基础模型, 大数据 + 世界模型 (接 M11.5)。

> 演化的最新方向: backbone 还是 VLM, 但**动作头越来越精致** (从离散 token → 扩散/flow, M11.3), 并开始融合**世界模型** (M11.5)。这正好把你 M13 (扩散/世界模型) 接了进来。

---

## 5. 谱系的两条主线 (带走)

把整条线压成两句:
1. **感知核越来越复用 VLM**: RT-1 自训 → RT-2 用 VLM → OpenVLA 开源 VLM。你的 M10 = 这个核。
2. **动作头越来越精致**: 离散 token (RT-1/2) → 扩散/flow (π, M11.3)。你的 M13 = 这些动作头的地基。

> 所以 **VLA = M10 (VLM 感知核) + M13 (扩散动作头/世界模型) + 机器人数据**。M11 的全部内容, 就是把你已学的两块接到物理世界。你不是在学全新东西, 是在组装。

---

## 6. 本讲小结 + 通往 L3

- 演化线 RT-1→RT-2→OpenVLA→π/GR00T: 从「专用控制网络」到「VLM + 精致动作头」。
- **RT-2 是飞跃**: 直接用 VLM + 动作当 token → 涌现泛化 (复用 VLM 世界知识)。
- VLA = M10 (VLM 核) + M13 (扩散动作头/世界模型) + 机器人数据。
- 两主线: 感知核复用 VLM; 动作头从离散 token 走向扩散/flow。

> **下一讲 L3「tokens-as-actions 范式」**: 深入 RT-2 的核心机制 —— 把动作当 token, 控制就变成 next-token 预测。这是「为什么 VLM 能直接拿来控制机器人」的关键。

**动手**: 想一想你的 M10 mini-VLM 要怎么改才能控制机器人? (答案: 接一个动作头, L3/M11.2)。带着这个问题进 L3。
