# L4 · 数据 scaling: 「每条 demo 改进所有机器人」

> 20-min lecture · 目标: 理解机器人基础模型最反直觉的优势 —— 数据复用/正迁移。收口 M11.1。

---

## 0. 机器人界的「数据稀缺」诅咒

机器人数据**极贵**: 真机遥操作采集慢、需人、会摔坏 (不像文本能从网上爬)。这曾是机器人学习的根本瓶颈。tokens-as-actions (L3) 带来了破局之道: **让所有机器人数据共享一个模型**。

---

## 1. 关键逻辑: 一个模型吃所有数据

旧范式 vs 新范式:
```
   旧: 机器人A数据→训模型A;  机器人B数据→训模型B  (数据隔离, 各学各的)
   新: 机器人A+B+C...所有数据 → 训一个 VLA  (数据汇聚, 共享学习)
```

> 因为 tokens-as-actions 把不同机器人的观测/动作都统一成 token, **不同机器人、不同任务的数据可以喂进同一个模型一起训**。这就是 Open X-Embodiment 的思路: 把全世界的机器人数据集汇到一起。

---

## 2. 「每条 demo 改进所有机器人」的正迁移

最反直觉也最强的点: **正迁移 (positive transfer)**。
- 教机器人 A 开抽屉的 demo, 也会让机器人 B 开柜门变好 (共享「拉」的概念)。
- 厨房任务的数据, 帮到办公室任务 (共享物体/空间常识)。
- 甚至**人类视频** (egocentric) 也能帮 (M11.4): 人怎么操作物体的先验。

> 这和 LLM 的「多任务正迁移」是**同一个故事**: 一个大模型在多样数据上学, 任务之间互相帮助, 涌现出泛化。**机器人基础模型把 LLM 的 scaling 福音搬到了物理世界。** 每加一条 demo, 受益的不是一个任务, 是整个模型。

---

## 3. scaling 定律也来了 (数据/模型/任务)

像 LLM 一样, 机器人基础模型也观察到 scaling 趋势:
- **更多数据** (更多机器人/任务/demo) → 更好/更泛化的策略。
- **更大模型** (更强 VLM backbone) → 更强。
- **更多任务多样性** → 更好的零样本/少样本泛化。

> 你 M11.4-N2 会画一条玩具版的「数据量 vs 成功率」曲线 (接你 M9.4 实验设计), 亲眼看 scaling。机器人 scaling 还很早期 (数据量远小于 LLM), 这正是机会: **数据是当前最大瓶颈也是最大杠杆**。

---

## 4. 这对你 (和招聘) 意味着什么

- **方向价值**: 「机器人基础模型 / VLA」是 2026 公认的高需高薪前沿 (Physical Intelligence/Figure/1X/NVIDIA 等在抢人)。
- **你的优势**: 你懂 LLM scaling + VLM (M10) + 扩散 (M13)。VLA = 这些 + 机器人数据。你的地基迁移成本极低。
- **入口**: 数据是瓶颈 → 数据高效学习 (少 demo 泛化)、sim2real (M11.6, 你碰过 IsaacLab)、与人类视频 co-train 都是热门切入点。

> 一句话: 机器人基础模型正处在「LLM 的 GPT-2 时刻」—— 范式确立、数据是瓶颈、scaling 刚开始。对懂 LLM 的你, 这是一个地基复用、红利期、缺人的方向。

## 5. 本讲小结 (M11.1 收口) + 通往 11.2

- 机器人数据贵 → tokens-as-actions 让**所有数据共享一个模型** (Open X-Embodiment)。
- **正迁移**: 每条 demo 改进所有机器人/任务 (同 LLM 多任务正迁移, scaling 福音搬到物理世界)。
- scaling 定律 (数据/模型/任务) 也成立; 数据是当前最大瓶颈也是最大杠杆。
- 对你: VLA = LLM scaling + VLM + 扩散 + 机器人数据, 地基复用、红利期、缺人。
