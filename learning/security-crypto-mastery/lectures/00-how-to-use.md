# Security & Crypto Mastery 使用说明

> **先读 `01-textbook.md`**:那是本科课件式完整教材(七章、连贯教科书叙述),本页只是DeepPoint/ScenarioPoint追问链的使用说明,建议在读完教材之后再来这里练习追问链。

## 为什么要做这个专题,以及和LLM安全专题的边界

老师要求补齐科班CS本科理论必修课空白,这是第六个也是最后一个。仓库里已有的 `safety-defense`/`red-team-jailbreak` 是LLM安全(大模型越狱/对齐攻防),这个专题是**通用信息安全和密码学基础**——两者是完全不同的领域,通用信息安全关注的是传统软件系统(Web应用/网络/操作系统)的攻防,LLM安全关注的是大模型本身的对齐失效和滥用风险,只是恰好都叫"安全",但知识体系几乎不重叠。

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:建立两块基础——密码学(加密/哈希/签名这些"密码学原语"分别解决什么问题)、认证与访问控制("你是谁"和"你能做什么"这两个经常被混淆的概念)。这一层几乎不需要预备知识,但要注意密码学部分容易陷入"背算法名字"的误区,真正重要的是理解每个原语解决什么威胁模型下的什么问题。
2. **Tier 2(深,`tier2_deep/`)**:三个文件分别对应"攻"(常见漏洞:理解攻击原理才能写出防御代码)、"防"(网络安全协议:TLS/PKI/VPN这些保护数据传输的机制)、"管"(应用安全工程:怎么把安全实践系统性地融入整个软件开发流程,而不是上线前临时补救)。这三者是完整的安全工程闭环。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"安全事件发生后该怎么应急处置"(遏制/取证/沟通的顺序权衡),第二个文件考"该怎么给系统选安全架构"(零信任vs边界防御、该用什么加密方案)。这两个文件的核心训练目标是让你理解安全决策从来不是"越安全越好"的单目标优化,而是安全性/可用性/性能/合规/成本这几个维度的真实工程权衡。

## 一个容易被忽视的陷阱

"未定义行为不是bug的借口"和"这里是内网所以不需要加密"是这个专题里最容易被戳破的两种想当然假设。前者提醒我们缓冲区溢出这类漏洞的根源往往是对内存安全语言特性的误解;后者提醒我们零信任架构的核心洞察正是"内网可信"这个传统边界防御模型的假设在现代混合办公/云原生场景下已经不再成立。tier3的安全架构选型判断文件反复在训练"识别隐含威胁模型假设是否仍然站得住脚"这条判断线,这也是CS科班理论六件套走到最后一个专题时,回望前五个专题(离散数学的证明严谨性/计算理论的能力边界/算法理论的正确性证明/计算机体系结构的机制理解/编译原理的工程落地)共同指向的一个主题:**任何工程决策背后都有一组隐含假设,资深判断力就是能不能把这些假设找出来并检验它们是否仍然成立**。
