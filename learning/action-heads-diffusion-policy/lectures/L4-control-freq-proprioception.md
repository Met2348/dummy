# L4 · 控制频率 / 本体感觉 / 实时约束

> 20-min lecture · 目标: 理解机器人控制的硬约束 (高频/本体感觉/实时), 及它们怎么决定动作头设计。收口 11.3。

---

## 0. 机器人不是聊天机器人

LLM 推理慢一点没关系 (用户等几秒)。机器人**不行**: 它在物理世界实时闭环, 控制慢了就摔、就撞。这一讲讲这些**硬约束**, 以及它们怎么反过来决定动作头怎么设计 (成本驱动架构, 同 M13.4-L4)。

---

## 1. 控制频率: 为什么 30-100Hz

机器人要以固定频率输出动作:
- **30Hz** (~33ms/步): 一般操作任务的下限。
- **50-100Hz**: 灵巧操作 / 动态平衡 (人形) 需要。
- 低于这个 → 动作不连续, 控制不稳 (像低帧率游戏)。

> 这是个**硬实时约束**: 每 10-33ms 必须出一个动作。这直接卡死了动作头: **朴素扩散 50 步去噪太慢, 出不来** → 必须 flow-matching 少步 (M13.2/L3) 或 action chunking (一次出一段摊薄, L3)。**控制频率决定了动作头必须快。**

---

## 2. 本体感觉 (proprioception): 机器人的「内感」

观测不只有相机图, 还有**本体感觉** —— 机器人对自己身体的感知:
- 关节角度/速度、末端位置、力/力矩、夹爪状态。
- 它告诉机器人「我现在的姿态/接触状态」, 是精细控制必需的。

```
   观测 = 相机图 (外部世界, VLM 处理) + 本体感觉 (自身状态)
        → backbone 融合两者 → 动作头
```

> 本体感觉是机器人比 VLM 多的输入。它通常和视觉特征**拼接/融合**后喂动作头 (类似 M10.2 的多模态融合, 但模态是「本体状态」)。你的 toy 里 state 就是简化的本体感觉 (agent 位置)。真实里还有关节力矩等。**本体感觉让控制能精细 (知道自己在哪、碰到没)。**

---

## 3. 实时约束怎么决定动作头设计

把硬约束串成一条设计逻辑 (成本驱动, 同 M13.4-L4):

| 约束 | 逼出的设计 |
|---|---|
| 高频 (30-100Hz) | flow-matching 少步 (M13.2) / chunking 摊薄 |
| 闭环纠错 | chunk 不能太大 (L3 反应性权衡) |
| 延迟容忍 | action chunking (预测一段留缓冲) |
| 精细操作 | 连续动作 (扩散/flow, 非离散) + 本体感觉 |
| 安全 | 动作限幅 / 平滑约束 |

> **动作头的设计 = 实时约束的优化结果**。π 选 flow-matching + chunking 不是审美, 是被 50Hz 高频 + 延迟 + 多峰逼出来的。理解约束, 你就懂任何 VLA 动作头为什么那样设计 (拿这表去拆)。

## 4. 把 11.3 串起来 (动作头全景)

```
   多峰 (L1)        → 需要扩散/flow (非回归非离散)
   机制 (L2)        → 条件 DDPM = M13.1 + state
   快 (L3)          → flow-matching 少步 (M13.2)
   稳 (L3)          → action chunking (平滑/抗抖, 但权衡反应性)
   硬约束 (L4)      → 高频/本体感觉/实时决定一切设计
   ───────────────────────────────────────
   = π 式 SOTA 动作头 (VLM + flow + chunking + 高频 + 本体感觉)
```

> 这就是机器人动作头的完整图景。每一块来自你学过的: M13.1 (扩散)、M13.2 (flow/少步)、M13.3 (条件)、M10.2 (融合)、本讲 (实时约束)。**会拆 = 会造。**

## 5. 本讲小结 (11.3 收口) + 通往 11.4

- 机器人是**实时闭环**, 控制频率 30-100Hz 是硬约束 → 逼出 flow-matching 少步 + chunking。
- **本体感觉**: 机器人对自身的感知 (关节/力/接触), 和视觉融合后喂动作头, 精细控制必需。
- 实时约束**决定动作头设计** (成本驱动, 同 M13.4): 拿约束表去拆任何 VLA 动作头。
