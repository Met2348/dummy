# L1 · 为什么动作要用扩散/flow (多峰 + 平滑)

> 24-min lecture · 目标: 理解机器人动作的多峰本质, 以及为什么扩散/flow 动作头能同时解决多峰+连续+平滑。Module 11.3 开篇, 接 M11.2-L3 的矛盾。

---

## 0. 接上 M11.2 留的矛盾

M11.2-L3 留了个矛盾:
- **离散 token 头**: 能表达多峰, 但离散化损失 (精度粗、动作跳)。
- **连续回归头**: 精度高、平滑, 但**怕多峰** (MSE 取平均)。

两个都缺一块。这一讲讲为什么**扩散/flow 动作头**能补齐: 既连续平滑、又能表达多峰。这是 π 系列的核心, 也是把你 M13 直接接到机器人的地方。

---

## 1. 机器人动作天生多峰

「多峰」不是边缘情况, 是机器人动作的常态:

```
   绕过障碍:  左绕 ✓  右绕 ✓        → 两个峰
   抓杯子:    多个角度都能抓          → 多个峰
   避让行人:  让左 ✓  让右 ✓        → 两个峰
```

> 同一个观测下, 经常有**多个同样合理的动作**。这就是多峰动作分布 $p(a|s)$ 有多个峰。你 N1 的玩具 (绕障碍, 上绕/下绕都行) 就是最小的多峰任务。

---

## 2. 连续回归为什么死在多峰上

连续回归 (MSE) 学的是**条件均值** $\mathbb{E}[a|s]$。对多峰分布, 均值落在**峰之间的谷**:

```
   合理动作:   上绕(峰1: a=+0.6)         下绕(峰2: a=-0.6)
   MSE 均值:                  0  ← 取平均 = 既不上也不下 = 直冲障碍 ✗
```

> 这是连续回归的**数学必然**: MSE 的最优解就是均值, 而多峰的均值在谷里 (一个两个峰都不是的烂动作)。你 N1 会看到: 回归在障碍前的动作 ≈ 直行 (y≈0), 正对障碍。**不是回归没学好, 是它本质只能给均值。**

---

## 3. 扩散动作头为什么能行

扩散 (你的 M13.1) 学的不是均值, 是**整个分布** —— 它能表达任意复杂的多峰分布, 并且**采样**出具体的一个动作 (而非均值):

```
   扩散动作头: 学 p(a|s) 的完整形状 (含两个峰)
   采样:       这次采到上绕(峰1), 下次可能采到下绕(峰2)  ← 表达了多峰!
   且动作连续平滑 (不像离散 token 量化)
```

> 关键: **扩散建模分布、靠采样产出动作** —— 所以它能给「上绕」或「下绕」这样的**具体的、连续的、属于某个峰的**动作, 而不是均值。这同时拿下了「连续平滑」(像回归) 和「多峰」(像离散分类) —— 两全。你 N1 会看到扩散动作头在障碍前采样出**双峰** (有的上、有的下), 而回归塌成中间一个点。

---

## 4. 这就是把 M13 接到机器人

扩散动作头 = 你 M13.1 的条件扩散, 只是:
- **数据**: 从「2D 点」变成「动作」(state→action)。
- **条件**: 条件于状态 (M13.3 的条件机制)。
- **机制**: 完全是 DDPM (前向加噪/反向去噪预测噪声), 一模一样。

> 你 N1 的 `diffusion_policy.py` 就是 M13.1 的 `diffusion.py` 加状态条件 (同源)。**你已经会扩散动作头的全部机制, 只是用途从「生成图」变成「生成动作」。** 这是 M13 → M11 最直接的桥: π 用 flow-matching 动作头 = 你 M13.2 的 flow matching 用于动作。

---

## 5. 本讲小结 + 通往 L2

- 机器人动作**天生多峰** (绕障/抓取/避让常有多个合理动作)。
- **连续回归死在多峰**: MSE 给均值, 多峰均值落谷里 (烂动作) —— 数学必然。
- **扩散动作头**建模完整分布 + 采样产出动作 → 既连续平滑、又多峰 (两全)。
- 扩散动作头 = M13.1 DDPM + 状态条件 (同源); 你已会全部机制。

> **下一讲 L2「diffusion policy 原理」**: 把扩散动作头的机制讲透 —— 怎么训 (预测噪声)、怎么采样动作、和 M13.1 的对应关系。

**动手**: 去 `N1`, 训一个扩散动作头和一个回归头, 在障碍前采样动作: 看扩散是**双峰** (上/下), 回归塌成中间一点 (直冲障碍)。亲眼见证多峰矛盾的解法。
