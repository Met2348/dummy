# L2 · diffusion policy 原理 (接 M13 扩散)

> 22-min lecture · 目标: 把扩散动作头的机制讲透 (训练=预测噪声, 采样=反向去噪), 看清它和 M13.1 的一一对应。

---

## 0. 它就是你的 M13.1, 换了用途

diffusion policy 的机制 = M13.1 的 DDPM, 只是把「生成 2D 点」换成「生成动作 (条件于状态)」。本讲把对应关系讲清, 你会发现没有新东西要学。

---

## 1. 训练: 预测噪声 (条件于状态)

和 M13.1 一样, 但条件于状态:

```
   取一对专家 (state, action)
   随机噪声水平 t, 给 action 加噪: a_t = √ā_t · a + √(1-ā_t) · ε
   去噪网络 (条件于 state): 预测加进去的噪声 ε̂ = model(a_t, state, t)
   损失: MSE(ε̂, ε)
```

> 唯一的新东西: 去噪网络多吃一个 **state** 当条件 (M13.3 的条件注入)。其余和 M13.1 字字对应: 加噪公式一样、预测噪声一样、MSE 一样。你 `diffusion_policy.py` 的 `train_diffusion_policy` 就是 `diffusion.py` 的 `train_diffusion` + state 输入。

---

## 2. 采样: 反向去噪生成动作

给一个状态, 从纯噪声反向去噪出一个动作:

```
   a_T ~ 噪声
   for t = T..1:
       ε̂ = model(a_t, state, t)        # 条件于当前状态
       a_{t-1} = 去噪一步 (用 ε̂)         # 和 M13.1 的反向公式一样
   返回 a_0 = 一个动作
```

> 因为每次从噪声开始, **不同的噪声 → 不同的动作** —— 若 p(a|s) 多峰, 这次采到上绕、下次采到下绕。这就是扩散表达多峰的机制 (L1)。和 M13.1 采样 2D 点的反向过程一模一样, 只是条件于 state、产出的是动作。

---

## 3. 一一对应表 (M13.1 ↔ diffusion policy)

| M13.1 (生成 2D 点) | diffusion policy (生成动作) |
|---|---|
| 数据 = 双月 2D 点 | 数据 = 专家动作 (条件于状态) |
| 去噪器 model(x_t, t) | 去噪器 model(a_t, **state**, t) |
| 训练: 预测噪声 (MSE) | 训练: 预测噪声 (MSE), 多吃 state |
| 采样: 反向去噪出点 | 采样: 反向去噪出动作 |
| 多峰: 双月两簇 | 多峰: 上绕/下绕两簇 |

> 看这张表: **diffusion policy = 条件版 M13.1**。你 M13 学的一切 (前向加噪/反向去噪/少步采样 M13.2/条件 M13.3) 全部直接迁移。这是「会拆系统」的胜利: diffusion policy 不是新东西, 是 M13.1 + state 条件。

---

## 4. 推理成本与少步采样 (接 M13.2)

扩散动作头有个现实问题: **采样要多步去噪**, 而机器人要**高频控制** (30-100Hz, L4)。每个动作都跑 50 步去噪太慢。解法正是你 M13.2:
- **flow-matching 动作头** (π 用的): 更直的路径, 少步采样 → 高频可行 (L3)。
- **一致性/蒸馏**: 几步甚至 1 步出动作。

> 又一次, M13.2 的少步采样在机器人上**价值放大**: 图少几步是锦上添花, 机器人控制频率是硬约束, 少步采样是能不能实时的分水岭 (同 M13.4-L4 视频)。这就是为什么 π 用 flow-matching 而非朴素扩散。

---

## 5. 本讲小结 + 通往 L3

- diffusion policy = **条件版 M13.1**: 训练预测噪声 (多吃 state)、采样反向去噪出动作。
- 不同噪声 → 不同动作 → 表达多峰 (L1 的机制)。
- 和 M13.1 字字对应; 你 M13 学的全部直接迁移。
- 推理要多步去噪 vs 机器人高频控制 → 需 M13.2 少步采样 (flow-matching, L3)。

> **下一讲 L3「flow-matching 动作头 + action chunking」**: π 的两个关键工程 —— 用 flow-matching (M13.2) 少步出动作, 用 action chunking 一次预测一段动作 (平滑/抗抖动)。
