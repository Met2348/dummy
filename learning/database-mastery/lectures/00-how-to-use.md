# Database Mastery 使用说明

## 三层怎么练

1. **Tier 1(浅,`tier1_shallow/`)**:先建立关系模型的基础词汇(关系/元组/属性)和SQL的核心操作,再理解事务和并发控制"要解决什么问题"(ACID/隔离级别/脏读不可重复读幻读)。这一层的核心是把"为什么需要这些概念"想清楚——比如三种隔离级别异常现象(脏读/不可重复读/幻读)之间的区别经常被面试追问到第二三层,不能只会背名词。
2. **Tier 2(深,`tier2_deep/`)**:机制深水层,三个文件分别对应"数据库怎么变快"(索引与查询优化)、"数据库怎么保证并发正确性"(存储引擎与并发控制,MVCC/redo-undo log/两阶段锁)、"数据库怎么变大"(分布式数据库,分库分表/CAP/共识算法/分布式事务)。这三条线索有内在联系:单机优化到极限之后才会走向分布式,分布式又引入了新的一致性问题需要新的并发控制思路(如Paxos/Raft),建议练习时留意这条从"单机优化"到"分布式权衡"的演进逻辑。
3. **Tier 3(社招级别,`tier3_social_hire/`)**:无标准答案的资深判断题。第一个文件考"选型和容量规划怎么判断"(SQL vs NoSQL、分片时机、读写分离一致性),第二个文件考"生产故障怎么排查"(慢查询/死锁/主从延迟/误删数据应急)。这两个文件的价值在于把tier1/tier2的机制知识串成"真实工作场景里怎么用",`explain` 字段是建立判断框架,不是给出唯一正确答案。

## 一个容易被忽视的陷阱

数据库面试追问链经常在"是什么"层面很容易答对,但第二三层追问"为什么"或"边界条件"时容易露怯——比如"为什么MySQL InnoDB默认隔离级别是REPEATABLE READ而Oracle默认是READ COMMITTED""为什么幻读在InnoDB的REPEATABLE READ级别下大部分情况已被间隙锁规避但还留有例外""CAP定理里的一致性和ACID里的一致性是不是同一个概念"——这类"看似简单但细究会翻车"的辨析题是这个专题的核心训练目标,`pitfall`/`trap` 字段反复指向的就是这类陷阱。
