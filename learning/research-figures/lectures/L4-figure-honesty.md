# L4 · 图的诚实性 + 整合到论文

> 30-min lecture · 目标: 识别并避免误导性可视化 (9.3 攻击清单的可视化版), 学会写 self-contained 的图注, 让图与正文呼应 —— 让你的图既有力又诚实。

---

## 0. 图可以清晰, 也可以骗人

L1-L3 教你把图画**清晰**。但清晰和**诚实**是两回事 —— 一张技术上精美的图, 可以系统性地误导读者。而审稿人 (用 9.3 的攻击清单) 会盯着这些。**无意的误导也是误导, 砸的是你的可信度。**

> 你做 9.3 时学会了攻击别人的图 (「这个 y 轴怎么从 0.5 开始?」)。这一讲是反过来: **保证你自己的图经得起那套攻击。** 你希望别人怎么诚实地对你, 就怎么对读者。

---

## 1. 头号骗术: 截断 y 轴 (truncated axis)

最常见、最有效的误导: y 轴不从 0 开始 (或从一个高值开始), 把微小差距在视觉上放大成「巨大提升」。

```
   诚实 (y 从合理基准)            骗人 (y 轴截断)
   win                          win
   1.0┤                         0.54┤        ▇
      │   ▇   ▇                      │   ▇    ▇
   0.5┤   ▇   ▇                  0.52┤   ▇    ▇
      │   ▇   ▇                      │   ▇    ▇
   0.0┴───────────              0.50┴───────────
      0.52 vs 0.53                  同样的数, 看起来差一倍!
      (几乎没差)                     (其实只差 0.01)
```

> 规则: **柱状图的 y 轴几乎总应从 0 开始** (柱的长度直接编码数值, 截断就扭曲了比例)。折线图为看趋势可以不从 0, 但**必须明确标注轴范围**, 且不能用截断去夸大一个本质微小的差距。判断标准: 「如果读者只瞥一眼柱子高度的比例, 会不会被误导?」会 → 不诚实。

这正是 9.4-L5 的延续: 一个 0.01 的差距, 如果统计上不显著 (p 大), 你连画都不该把它画得「看起来很大」。**图的视觉冲击应该匹配效应的真实大小。**

---

## 2. 其它常见误导 (逐个避开)

| 骗术 | 怎么误导 | 怎么诚实 |
|---|---|---|
| **没有误差棒** | 把有方差的均值画成确定的值 | 永远带误差棒 + 注明是什么 (9.4-L5) |
| **挑有利的区间** | 只画方法领先的那段 x 轴 | 画完整范围, 包括它不占优的地方 |
| **双 y 轴障眼** | 用两个不同刻度的 y 轴制造虚假相关 | 慎用双轴; 或明确标注、分开画 |
| **彩虹色图造结构** | jet 色图的亮度不单调, 凭空造出/掩盖模式 | 用 viridis (感知均匀, L1) |
| **3D 扭曲比例** | 3D 透视让近的柱显得更大 | 永远用 2D |
| **cherry-pick 样例** | 只展示效果好的那个例子 | 标明是 cherry-picked, 或给随机样例 |

> 这张表和 9.3 的攻击清单一一对应。**你在 9.3 学会用它攻击别人, 在这里用它自查。** 投稿前拿这张表过一遍你的每张图。

## 3. 图注 (caption): 让图自成一体

图注不是图标题的复述。一条好图注让图**脱离正文也能被读懂** (self-contained), 因为审稿人/读者经常**只看图和图注**就跳过正文。

一条完整图注的结构:

```
   图 N: [一句话说这张图展示什么/得出什么结论].
   [必要的设定: 模型/数据/指标]. [误差棒是什么: 如 阴影为 8 种子的 SEM].
   [怎么读: 如 两线张开表示交互效应]. [(可选) 关键观察的指引].
```

例 (我们的交互效应图):

> **图 3: Robust-DPO 的优势随偏好标签噪声增大而扩大。** 在 7B 偏好对齐任务上测 win-rate; 误差棒为 8 个随机种子的 SEM。两条线在无噪声时几乎重合, 在 40% 噪声下张开至 13 个点的差距, 表明 method 与 noise 之间存在显著交互效应 (p<0.001)。

逐项交代为什么这样写:
- **首句是结论, 不是描述。** 「win-rate vs noise」是描述; 「优势随噪声扩大」是结论。读者要的是结论。
- **写清设定和误差棒** —— 否则不可信 (9.4-L5 / L2)。
- **给读图指引** —— 「两线张开表示交互」帮读者一眼抓到重点。

## 4. 图与正文的呼应

图不是孤立的, 要和正文形成**一个论证**:

- **每张图对应一个 claim**: 正文里有一句「Robust-DPO 在高噪声下显著更优 (图 3)」, 图 3 就是这句话的证据。**一图一 claim**, 呼应 L1 的一图一信息。
- **正文引用图、解读图**: 别把图扔那不管。正文要「指着图说话」: 「如图 3 所示, 差距从 0 扩大到 13 点」。
- **图的顺序 = 论证的顺序**: Figure 1 (方法) → 主结果图 → 消融图 → 分析图, 跟着你的故事线 (9.7 写作)。

> 检验: 把你所有图的**首句图注**抽出来连读, 应该构成你论文的**论证骨架** (方法→主结果→为什么→边界)。如果连起来不成故事, 说明你的图没有服务于一条清晰的论证线。

## 5. 整个 9.6 收口: 一张图的完整生命

```
   9.4 实验设计 → 数据 (交互效应)
        │
   9.5 留痕 → jsonl (可复现的结果)
        │
   L1 判断力: 一图一信息 (优势随噪声扩大), 选折线 (趋势)
   L2 出版级: set_pub_style + 误差棒 + 矢量 PDF
   L3 (若需) 方法图: Figure 1 讲 Robust-DPO 怎么运作
   L4 诚实: y 轴不截断, 图注写清, 对应一个 claim
        │
   一张能进顶会、又经得起 9.3 攻击的图
```

注意整条链复用同一份数据 (9.4 的 Robust-DPO 噪声鲁棒性): **从设计→留痕→出图, 是一个研究结果的完整生命周期。** 这就是 Module 9 「按生命周期编排」的兑现。

## 6. 本讲小结 + 通往 9.7

- 图能清晰也能骗人; 无意的误导也砸可信度。用 9.3 攻击清单**自查**。
- 头号骗术**截断 y 轴**: 柱状图 y 轴几乎总从 0; 视觉冲击要匹配效应真实大小。
- 其它: 没误差棒 / 挑区间 / 双轴 / 彩虹色 / 3D / cherry-pick —— 逐个避开。
- **图注 self-contained**: 首句是结论、写清设定和误差棒、给读图指引。
- **一图一 claim**, 图与正文呼应; 首句图注连读 = 论证骨架。
