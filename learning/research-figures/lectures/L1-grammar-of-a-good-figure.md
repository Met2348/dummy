# L1 · 一张好图的语法: 目的、data-ink、图类型、色彩

> 35-min lecture · 目标: 建立「什么是一张好科研图」的判断力 —— 一图一信息、最大化 data-ink、选对图类型、用对颜色。先有审美与原则, 再谈 matplotlib 操作 (L2)。

---

## 0. 为什么图这么重要 (尤其对你的论文命运)

审稿人读你的论文, 顺序往往是: 标题 → 摘要 → **图** → (有兴趣才) 正文。

> **图是你论文的「橱窗」。** 一个累、忙、要审 6 篇的审稿人, 经常是看完你的 Figure 1 (方法图) 和主结果图, 就形成了「接受 / 拒绝」的初步判断, 再回头找正文验证。一张糊、乱、读不懂的图, 会让一个好工作被低估; 一张清晰有力的图, 能让审稿人「秒懂你的贡献」。**图不是论文的装饰, 是论文的论点本身。**

这一讲不教画图工具, 教**判断力**: 看到一张图, 你能说出它好在哪、烂在哪、怎么改。没有这个判断力, 工具再熟也只是把烂图画得更精致。

---

## 1. 第一原则: 一图一信息 (one figure, one message)

每张图都该能用**一句话**说清它要让读者「看到」什么。画图前先问:

> **「这张图我想让读者得出的那**一个**结论是什么?」**

我们 9.4 的交互效应图, 那一句话是: 「**Robust-DPO 的优势随噪声增大** (两条线张开)」。整张图的每个元素都该服务于让这句话**一眼成立**; 不服务的, 删掉。

反模式: 一张图塞 5 条信息 (顺便给你看准确率、速度、参数量、还标了 10 个数据点的名字)。读者不知道该看哪, 等于什么都没看到。**信息多 = 信息少。** 宁可拆成 3 张各讲一件事的图。

## 2. data-ink ratio: 删掉所有不传递信息的墨水

统计图形大师 Edward Tufte 的核心概念:

$$\text{data-ink ratio} = \frac{\text{用于呈现数据的墨水}}{\text{图上全部的墨水}}$$

逐项交代:
- **分子 (data-ink)**: 真正编码数据的部分 —— 柱子的高度、点的位置、线的走向。
- **分母 (total ink)**: 图上一切墨水, 包括背景网格、3D 阴影、花哨边框、冗余图例……
- **目标: 让这个比值尽量接近 1** —— 即**删掉一切不传递信息的墨水**。

那些该删的「**chartjunk (图表垃圾)**」:
- 3D 立体柱状图 (3D 扭曲比例, 纯属误导)
- 浓重的背景网格线 / 背景色块
- 多余的边框 (上、右 spine, L2 会去掉)
- 给每个柱子标花纹、加渐变、加阴影
- 和数据无关的装饰图标

```
   chartjunk 满满 (data-ink 低)        干净 (data-ink 高)
   ┌────────────────────┐             win
   │▓▓▓ 3D 渐变柱 + 网格 │              ↑  ▇    ▇
   │▓▓▓▓ + 阴影 + 边框   │   ──删→      │  ▇  ▇  ▇
   │░░░░░░░░░░░░░░░░░░░░│              └──────────►
   └────────────────────┘                两组柱, 一目了然
```

> 一句话: **好图是"减"出来的, 不是"加"出来的。** 默认 matplotlib/Excel 图通常 data-ink 偏低 (网格、边框多), 你的工作是做减法。L2 的 `set_pub_style` 帮你默认去掉一批 chartjunk。

## 3. 选对图类型: 数据关系决定图

图类型不是随便选的, 由你要展示的**数据关系**决定:

| 你要展示 | 用 | 例 (我们的数据) |
|---|---|---|
| 随连续变量的**趋势** | 折线图 line | win-rate 随 noise 变化 (看交互, 趋势!) |
| 离散类别的**比较** | 柱状图 bar | 几个方法在固定 noise 下比高低 |
| 两变量的**相关** | 散点图 scatter | 模型大小 vs 性能 |
| **分布 / 方差** | 箱线/小提琴/带误差棒 | 多种子的 win-rate 分布 (9.4-L5!) |
| **矩阵 / 二维强度** | 热力图 heatmap | 消融矩阵 (method×noise) |
| **占比** | 堆叠柱 (慎用饼图) | 各来源贡献 |

几条选型铁律:
- **趋势用线, 不用柱。** 我们的交互效应是「随 noise 的趋势」→ 必须用折线 (两条线张开才能看出交互); 用柱状图反而看不清趋势。
- **少用饼图。** 人眼比长度 (柱) 比角度 (饼) 准得多。超过 3 类的占比, 用柱不用饼。
- **永远配上不确定性。** 任何含均值的图 (柱/线), 都要带误差棒 (9.4-L5 / 9.5)。没有误差棒的均值图, 在审稿人眼里是「不可信」的。

## 4. 色彩: 色盲安全 + 感知均匀

颜色是最容易出错的一环。三条规则:

1. **色盲安全 (colorblind-safe)**: 约 8% 的男性有红绿色觉障碍。**别用红配绿区分两类** (他们看是一样的)。用经过验证的调色板, 如 **Okabe-Ito** (本专题 `plotstyle.OKABE_ITO`, 8 色, 学术界推荐) 或 matplotlib 的 `tab10`。
2. **顺序数据用感知均匀色图**: 表示「从小到大」(如热力图) 时, 用 `viridis` / `cividis` (亮度单调递增, 色盲也能读), **别用 `jet`/彩虹** (彩虹色图亮度不单调, 会凭空制造/掩盖结构, 是经典误导)。
3. **颜色要有意义, 不是装饰**: 同一个方法在所有图里用同一个颜色 (读者建立映射); 别为了好看乱配色。

> 一个快速自检: 把你的图**转成灰度**打印, 还能区分各系列吗? 能 → 色盲也能读, 且省了彩印钱。不能 → 改用「颜色 + 形状/线型」双重编码 (如 DPO 实线圆点、Robust-DPO 虚线方块)。

## 5. 本讲小结 + 通往 L2

- 图是论文的**橱窗和论点**, 审稿人常看图定判断; 判断力比工具更重要。
- **一图一信息**: 画前先想「这张图要让读者看到的那一句话」, 每个元素服务它。
- **data-ink ratio**: 删掉一切 chartjunk (3D/网格/边框/阴影), 好图是减出来的。
- **选对图类型**: 趋势用线、比较用柱、分布带误差棒、少用饼/彩虹色图。
- **色彩**: 色盲安全 (Okabe-Ito)、顺序数据用 viridis、颜色有意义; 灰度自检。

> **下一讲 L2「matplotlib 出版级实操」**: 有了判断力, 现在把它落实成代码。L2 教你用 rcParams / 字号 / 矢量格式 / 误差棒, 把「能看」的默认图升级到「能印进顶会」的水准, 并把这套固化成可复用的 `plotstyle`。

**动手**: 找一张你之前某个复现专题里画的图 (或论文里见过的烂图), 用本讲四个维度 (一图一信息 / data-ink / 图类型 / 色彩) 给它挑刺, 写下「至少 3 处能改」。
