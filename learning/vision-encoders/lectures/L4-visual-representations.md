# L4 · 视觉表示的性质: 接进 LLM 前的接口决策

> 30-min lecture · 目标: 回答把视觉塔接进 LLM 前的几个工程决策 —— 用 [CLS] 还是 patch token? 哪层表示? 冻结还是微调? 这些直接通往 M10.2 的融合架构。

---

## 0. 视觉塔之后, LLM 之前

L1-L3 给了你一个懂语言/懂细节的视觉塔 (ViT + CLIP/SigLIP/DINOv2)。它输出一串视觉 token。但在把这串 token 喂给 LLM 之前, 有几个看似琐碎、实则影响全局的「接口」决策。本讲讲这些, 它们是 M10.2 融合架构的前提。

---

## 1. 决策一: 用 [CLS] 还是所有 patch token?

视觉塔输出 $N+1$ 个 token: 1 个 [CLS] (图级摘要) + $N$ 个 patch token (位置级细节)。喂 LLM 时用哪些?

| 选择 | 给 LLM 的信息 | 代价 | 适用 |
|---|---|---|---|
| **只用 [CLS]** | 1 个 token, 全图摘要 | 极省, 但丢空间细节 | 图级任务 (分类式问答) |
| **所有 patch token** | $N$ 个 token, 保留空间 | LLM 上下文变长 $N$ 倍 | 需要定位/细节 (VQA、OCR、grounding) |
| **压缩后的 patch token** | $M < N$ 个 (resampler 压) | 折中 | 主流 VLM (如 Flamingo 的 perceiver) |

> 核心权衡: **视觉 token 数 = LLM 要吃的额外上下文长度。** 一张高分辨率图的 patch token 可能上千, 直接撑爆 LLM 上下文 (接你的 long-context 专题)。所以「保留多少视觉细节 vs 上下文成本」是 VLM 的核心工程张力。M10.2 的 Q-Former/resampler 就是为压缩 token 而生。

---

## 2. 决策二: 用哪一层的表示?

ViT 有多层, 哪层的输出最适合喂 LLM?

- **最后一层**: 最「高层语义」, 但对比训练 (CLIP) 的最后一层可能过度专门化于「对齐 caption」, 丢了细节。
- **中间层**: 经验上常更通用、细节和语义平衡更好。很多 VLM 实际用**倒数第二层**或某个中间层。
- 这是个**经验超参**, 要消融 (接你 9.4 实验设计: 这正是一个该做的消融 —— 哪层喂 LLM 效果最好)。

> 一个反直觉点: 「最后一层」不总是最好。对比学习把最后一层优化成「能配对 caption」, 这个目标和「给 LLM 提供丰富视觉信息」不完全一致。**训练目标决定了表示的形状** —— 这个洞察呼应 9.3 的批判式阅读 (一个方法的目标决定它的盲区)。

---

## 3. 决策三: 冻结视觉塔还是一起微调?

造 VLM 时, 视觉塔的参数动不动?

```
   冻结 (frozen)                     微调 (fine-tune)
   ─────────────                     ──────────────
   视觉塔参数不动, 只训连接器+LLM      视觉塔也参与训练
   ✓ 省算力、稳、不毁预训练表示         ✓ 视觉表示能适配下游
   ✗ 视觉表示固定, 不能适配             ✗ 贵、可能灾难遗忘、毁掉对齐
   → LLaVA 阶段1 用这个                → 后期/数据足时部分解冻
```

主流配方 (M10.3 详讲) 是**分阶段**: 先冻结视觉塔、只训连接器 (让 LLM 学会「读」视觉 token); 再视情况解冻视觉塔微调。这是个「先对齐、后精调」的课程式策略。

> 给造 VLM 的人的默认: **先全冻视觉塔, 只训那个小连接器。** 这是 LLaVA 证明过的高性价比起点 —— 一个 ViT + 一个 MLP 连接器 + 冻结的 LLM, 就能造出能用的 VLM。能不动的就别动, 是 VLM 训练省钱省稳的第一原则。

---

## 4. 把三个决策串起来: 一个 VLM 视觉接口的样子

```
   图 → SigLIP ViT (冻结) → 取倒数第二层的所有 patch token
       → (可选) resampler 压缩到 M 个 token
       → 连接器 (M10.2) 投影到 LLM 的 embedding 空间
       → 当作「视觉 token」插进 LLM 的输入序列
```

这串决策 (哪些 token / 哪层 / 冻不冻) + 连接器设计 (M10.2) + 训练配方 (M10.3), 共同定义了一个 VLM。本专题 (M10.1) 管好了视觉塔本身和这些接口决策; 下一专题接手「视觉怎么真正接进 LLM」。

---

## 5. 本讲小结 (M10.1 收口) + 通往 M10.2

- **[CLS] vs patch token**: 视觉 token 数 = LLM 额外上下文; 细节 vs 成本的核心张力; 主流用压缩后的 patch token。
- **哪层**: 最后一层不总最好 (对比目标过度专门化); 常用中间/倒数第二层; 该消融 (9.4)。
- **冻结 vs 微调**: 默认先全冻视觉塔只训连接器 (LLaVA 配方, 省稳); 后期视情况解冻。
- 三决策 + 连接器 + 配方 = 一个 VLM。

> **下一专题 M10.2「VL 融合架构」**: 视觉塔和接口都备好了, 核心问题来了 —— **视觉 token 到底怎么接进 LLM?** cross-attention (Flamingo) vs 投影 adapter (LLaVA) vs early-fusion (Chameleon)? 这是造 VLM 最关键的架构决定, M10.2 给你一张决策树。

**动手**: 回到 N1, 在 mini-ViT 输出上分别取「只 [CLS]」和「所有 patch token」, 对比两者的形状和信息量, 体会决策一的权衡。
