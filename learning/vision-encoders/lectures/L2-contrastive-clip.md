# L2 · 对比学习与 CLIP: 让视觉「懂语言」

> 35-min lecture · 目标: 理解对比学习如何把图和文拉进同一个语义空间, 掌握 CLIP 的 InfoNCE 损失。这是 VLM 视觉塔「懂语言」的来源。

---

## 0. 一个关键问题: ViT 的视觉 token 凭什么「懂语言」?

L1 的 ViT 给了视觉 token, 但如果只用 ImageNet 分类训练它, 它学到的是「猫 vs 狗」这种**封闭标签**, 不懂开放的自然语言。VLM 要的是一个**懂语言的视觉塔** —— 给它一张图, 它的表示能和任意文本描述对齐。

> CLIP (2021) 的答案: **不用人工标签, 用互联网上海量的「图 + 它的 caption」对**, 训练目标是「让配对的图和文在向量空间里靠近, 不配对的远离」。这叫**对比学习 (contrastive learning)**。训完, 视觉塔的表示天然和语言对齐 —— 这就是为什么 CLIP/SigLIP 是 VLM 的标配视觉塔。

---

## 1. 对比学习的核心图景

```
   一个 batch: B 张图 + B 段配对文字
   图编码器 (ViT) → B 个图嵌入       文编码器 (transformer) → B 个文嵌入
        │                                    │
        └────────── 算 B×B 相似度矩阵 S ──────┘
                    S[i,j] = sim(图i, 文j)
   目标: 对角线 S[i,i] (配对的) 最大, 非对角 (不配对的) 最小
```

直觉: 把「图 i」和「文 i」(它的描述) 当**正样本对**, 把「图 i」和 batch 里其它 B−1 段文当**负样本**。训练就是**把正样本拉近、负样本推远**。一个 batch 里有 B 个正对、B(B−1) 个负对 —— **负样本是免费的** (其它样本的配对), 这是对比学习高效的关键。

`contrastive.similarity_matrix` 算的就是这个矩阵; 训好后对角线应该最亮。

---

## 2. InfoNCE 损失 (CLIP 的核心)

CLIP 用的损失叫 **InfoNCE** (噪声对比估计的一种)。先看图→文方向: 对第 $i$ 张图, 在所有文里找「对的那段文 $i$」, 这是一个 B 选 1 的分类:

$$\mathcal{L}_{\text{img}\to\text{txt}} = -\frac{1}{B}\sum_{i=1}^{B} \log \frac{\exp(S_{ii}/\tau)}{\sum_{j=1}^{B}\exp(S_{ij}/\tau)}$$

逐项交代每个符号:
- **$S_{ij} = \text{sim}(\text{图}_i, \text{文}_j)$**: 图 $i$ 和文 $j$ 的余弦相似度 (先 L2 归一化再点积)。
- **$S_{ii}$**: 配对的 (图 $i$ 和它的描述文 $i$), 我们要它最大。
- **$\tau$ (temperature, 温度)**: 一个缩放系数。$\tau$ 越小, softmax 越「尖锐」, 越严厉地放大正负样本差距 (CLIP 用 ~0.07)。
- **分子**: 配对对的相似度 (指数化)。**分母**: 图 $i$ 和**所有**文的相似度之和 (含配对 + 所有负样本)。
- **整体**: 这是个 softmax 交叉熵 —— 让「配对文」在「所有候选文」里的概率最大。

CLIP 是**对称**的: 同样对文→图方向做一遍 ($\mathcal{L}_{\text{txt}\to\text{img}}$, 对每段文找对的图), 两个方向取平均:
$$\mathcal{L}_{\text{CLIP}} = \tfrac{1}{2}(\mathcal{L}_{\text{img}\to\text{txt}} + \mathcal{L}_{\text{txt}\to\text{img}})$$

> `contrastive.info_nce_loss` 实现了这个对称损失。你在 N2 会看到: 配对噪声越大 (图文越不对齐), 损失越高 —— 损失在直接度量「图文对齐得多好」。

---

## 3. 为什么 batch 越大越好 (对比学习的脾气)

InfoNCE 的分母是「batch 内所有负样本」。**batch 越大, 负样本越多, 任务越难, 学到的表示越好。** 这是 CLIP 要用超大 batch (上万) 训练的原因, 也是它工程上贵的地方。

> 这个「batch size 即负样本数」的特性, 引出了 L3 的 SigLIP: 它换一种损失, 摆脱对大 batch 的依赖。理解了 InfoNCE 的这个软肋, 才理解 SigLIP 为什么被提出。

---

## 4. 训完之后: zero-shot 的魔法

CLIP 训完, 视觉塔和文本塔共享一个语义空间, 于是能**zero-shot 分类**: 给一张图, 把候选类名 (「a photo of a cat」「a photo of a dog」) 编码成文嵌入, 看图嵌入和谁最近 —— 不需要为新任务重新训练。这展示了「图文对齐」的威力: 视觉表示被语言「锚定」了。

> 对 VLM 的意义: 一个 CLIP/SigLIP 视觉塔, 拿来当 VLM 的「眼睛」, 它输出的视觉 token 天然「会说话」(和语言空间对齐), LLM 接上就容易理解。这就是 LLaVA 等 VLM 直接用冻结的 CLIP 视觉塔的原因 (M10.2/M10.3)。

---

## 5. 本讲小结 + 通往 L3

- 对比学习: 让**配对的图文靠近、不配对的远离**, 用海量「图-caption」对、无需人工标签。
- **InfoNCE**: 对称的 softmax 交叉熵, 把「找配对」变成「batch 内 B 选 1 分类」; $\tau$ 控制尖锐度; 负样本是 batch 内其它样本 (免费)。
- batch 越大负样本越多、表示越好 → CLIP 工程上贵。
- 训完视觉表示被语言锚定, 能 zero-shot; 这就是 VLM 视觉塔「懂语言」的来源。

> **下一讲 L3「SigLIP 与 DINOv2」**: InfoNCE 依赖大 batch 是个软肋。SigLIP 用 sigmoid 损失摆脱它; DINOv2 则走另一条路 —— 完全不用文本的**自监督**视觉学习。两者都是 2026 VLM 常用的视觉塔。
