# L10 · W4A8（激活也量化）

## 1 · 动机
W4A16 weight 已 int4，但 activation 还是 fp16 → matmul kernel 必须把 int4 dequant 成 fp16 再算 → kernel 复杂。
**W4A8**: activation 也 int8 → 算 int4 × int8 matmul → kernel 更简单 + 更快。

## 2 · 难点
activation 比 weight 更难量化：
- weight 静态（一次性 PTQ 即可）
- activation 动态（per-prompt 不同）

## 3 · 解
- per-token / per-batch 动态 scale (online quantization)
- 或者用 SmoothQuant 把 outlier 转移到 weight 后再量 activation

## 4 · 精度
| 方案 | MMLU |
|------|------|
| W4A16 (AWQ) | 44.9 |
| W4A8 (SmoothQuant + AWQ) | 43.5 |
| W4A8 (Marlin kernel) | 44.0 |

精度损失 1-2pp，换得 30% 速度。

## 5 · 速度
| kernel | tok/s (decode, 7B) |
|--------|-------------------|
| fp16 | 130 |
| W4A16 (Marlin) | 220 |
| **W4A8 (Marlin)** | **280** |

## 6 · 使用场景
- 极致 throughput 服务
- batch 大的离线场景
- agent 推理（high QPS）

## 7 · 部署
TRT-LLM 内置；vLLM 0.7+ 支持。
