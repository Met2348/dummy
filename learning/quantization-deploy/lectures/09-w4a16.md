# L09 · W4A16（bitsandbytes / autoawq 实战）

## 1 · 定义
- **W**eight 量化为 int4（每个 element 4 bit）
- **A**ctivation 保留 fp16

工业最常见配置。

## 2 · 显存
- 7B 模型: weight 14 GB → **3.5 GB**
- 加上 KV cache + activation → 总 ~8 GB
- 5090 24GB 跑 33B-W4A16 没问题

## 3 · 速度
- weight load 是 decode 瓶颈 → 减 4x weight 读 → decode 加速 1.6-2x
- activation 还是 fp16 → matmul 用 dequant_into_fp16 + fp16 matmul
- 或用 W4·A16 fused kernel（vllm marlin kernel）

## 4 · 两个软件栈
| lib | 量化算法 | 部署 |
|-----|---------|------|
| **bitsandbytes** | nf4 (NormalFloat 4) | 训练时 LoRA 常用 |
| **autoawq** | AWQ | 生产推理 |
| **auto-gptq** | GPTQ | 生产推理 |

vLLM 都支持。

## 5 · 端到端流程
```python
# 1. 量化（用 AWQ）
from awq import AutoAWQForCausalLM
model = AutoAWQForCausalLM.from_pretrained("Qwen/Qwen2.5-7B")
model.quantize(tokenizer, quant_config={"w_bit": 4, "q_group_size": 128})
model.save_quantized("./qwen-awq-w4")

# 2. 用 vLLM 部署
from vllm import LLM
llm = LLM("./qwen-awq-w4", quantization="awq")
out = llm.generate("Hello")
```

## 6 · 精度
| 方案 | MMLU |

| GPTQ-W4 | 44.5 |
| **AWQ-W4** | **44.9** |
| bnb-nf4 | 43.5 |
