# L3 · gap 分类学 (A Taxonomy of Research Gaps)

> 40-min lecture · 目标: 从「碰运气发现一个 gap」升级到「系统地扫出所有 gap」。
> 给你一张 **6 类 gap × 嗅探信号** 的雷达图, 以后读任何论文, 都按这 6 类逐项扫一遍。

---

## 0. 什么是「gap」, 为什么它是 PhD 的核心货币

> **gap (研究缝隙)**: 当前知识 / 方法 / 证据里, 一个**真实存在、尚未被填上、且值得填**的空洞。

一句话点破 PhD 的本质:

> **本科和硕士课程是「消费已有的 gap 的答案」; PhD 是「自己找到一个 gap, 并生产出它的答案」。** 你 48 个工程专题练的是前者 (复现别人填过的洞), Module 9 要练的是后者。

找 gap 难, 是因为新手有两种失败模式:
1. **看不见洞** — 默认「已发表 = 已完美」, 读完只觉得「好厉害」, 看不出任何缺。(L2 治这个)
2. **看见的全是假洞** — 「为什么不用更大模型」这种洞要么没价值, 要么已被做过。(L3 + L4 三筛 治这个)

L3 的任务是把第 1 个问题彻底解决: **给你一张分类雷达, 保证你不会漏扫任何一类洞。**

```
                       一篇论文 / 一个子领域
                              │
        ┌──────────┬─────────┼─────────┬──────────┬──────────┐
     方法gap     评测gap    假设gap    泛化gap   复现gap    理论gap
   (怎么做更好) (怎么测才对)(前提对吗) (换场景还行吗)(能重来吗)(为什么work)
```

## 1. 六类 gap 逐个讲

下面每一类给: **定义 · 嗅探信号 (读论文时哪句话/哪张表会触发它) · 真实 NLP 例子 · 翻成题目的方向**。

---

### ① 方法 gap (Method gap) — 「能不能做得更好/更省」
- **定义**: 现有方法在效果、效率、简洁性上有明确改进空间。
- **嗅探信号**:
  - 「we leave ... to future work」(作者亲口承认的洞)
  - 方法很有效但很笨重 (要 3 阶段训练 / 要额外的 reward model)
  - 两个方法各有所长, 没人把它们合起来
- **NLP 例子**: RLHF 经典三段式 (SFT→RM→PPO) 又重又不稳 → DPO 把它压成「无需 reward model 的单阶段」。**DPO 本身就是填了 RLHF 的一个方法 gap。**
- **翻题方向**: 简化 / 加速 / 融合 / 替换某个笨重组件。

### ② 评测 gap (Evaluation gap) — 「现在的测法对吗」
- **定义**: 现有 benchmark 或指标无法真实衡量我们关心的能力。
- **嗅探信号**:
  - 大家都在刷的指标, 其实和真实目标脱节 (刷 BLEU ≠ 翻译真的好)
  - benchmark 老旧 / 可能被预训练背过 (回忆 L2 的 Q4 泄露)
  - 没有衡量某个重要维度的测法 (如「推理过程是否真的合理」而非只看最终答案)
- **NLP 例子**: GSM8K 被普遍怀疑泄露 → 催生 GSM-1k、过程奖励 (PRM) 评测、以及「只看答案对不对」到「看每一步推理对不对」的转变。**你的 `process-reward` 专题正是评测 gap 的产物。**
- **翻题方向**: 造更干净/更难/更贴近真实目标的 benchmark; 设计新指标。

### ③ 假设 gap (Assumption gap) — 「它默默依赖的前提对吗」
- **定义**: 方法成立依赖某个**没明说**或**未必总成立**的前提, 一旦前提破裂, 方法就崩。
- **嗅探信号**:
  - 方法只在「数据充足 / 标注干净 / 单语言 / 在线可访问 reward」等理想条件下验证
  - 论文里出现「we assume ...」却没讨论假设破裂会怎样
- **NLP 例子**: DPO 假设「偏好数据是一致、无噪声的」。现实偏好数据充满标注分歧和噪声 → 一批后续工作 (如各种 robust-DPO) 专门攻击「偏好有噪声时怎么办」。**这就是在假设 gap 上做文章。**
- **翻题方向**: 找到隐藏假设 → 构造它破裂的场景 → 提出在该场景下仍 work 的方法。**这是最容易出 strong 论文的一类, 因为它直击根基。**

### ④ 泛化 gap (Generalization gap) — 「换个场景还成立吗」
- **定义**: 结论只在论文测的那个窄设定成立, 换数据集/规模/语言/领域就未知或失效。
- **嗅探信号**:
  - 只在 1 个数据集 / 1 个模型规模 / 1 种语言验证 (回忆 L2 的 Q6)
  - 「scales to larger models」只是口号, 没有 scaling 曲线
- **NLP 例子**: 一个在 7B 上 work 的对齐技巧, 到 70B 是否还 work? 很多技巧有「规模脆性」—— 小模型上的提升在大模型上消失。系统性地研究「X 随规模如何变化」常常本身就是一篇论文。
- **翻题方向**: 跨规模 / 跨语言 / 跨域 / 跨任务的系统性泛化研究。

### ⑤ 复现 gap (Reproducibility gap) — 「别人能重来吗」
- **定义**: 论文的结果无法被独立复现, 或复现后结论站不住。
- **嗅探信号**:
  - 没代码 / 超参不全 / 单种子 (回忆 L2 的 Q3、Q9)
  - 社区里已有「我复现不出来」的讨论
- **NLP 例子**: 很多 RL/RLHF 论文以「难复现、对超参极度敏感」著称。一篇扎实的**复现 + 公平再评估 + 公开方差**的报告 (哪怕结论是「其实没那么强」) 是真实贡献, 适合 workshop / 也常被高引。
- **翻题方向**: 严肃复现 + 公开代码 + 报告真实方差 + 给出「什么时候它真 work / 什么时候不」。
- **对你尤其友好**: 你已经有 48 个复现! 复现 gap 是你**现在就能做**的研究类型, 几乎不需要新基础设施。

### ⑥ 理论 gap (Theory gap) — 「为什么它 work, 我们其实不懂」
- **定义**: 一个经验上有效的方法, 缺乏解释「为什么有效」的理论, 或现有理论与现象矛盾。
- **嗅探信号**:
  - 「surprisingly」「we observe that ... (but don't know why)」
  - 一个 trick 大家都在用, 但没人说清机理
- **NLP 例子**: 为什么 RLHF 之后模型会「变谄媚 (sycophancy)」? 为什么 CoT 能提升推理、它到底改变了什么? 这些「现象清楚、机理不清」的地方是理论 gap。
- **翻题方向**: 提出可验证的机理假设 + 设计实验证实/证伪 (注意: 这类对数学/分析能力要求高, 你 EE 背景的数学功底在这里是优势)。

## 2. 一张速查雷达 (读论文时贴在手边)

| gap 类型 | 一句话 | 最强嗅探信号 | 难度 | 对你(博0)友好度 |
|---|---|---|---|---|
| ① 方法 | 做得更好/更省 | "leave to future work" | 中 | ★★★ |
| ② 评测 | 测法不对 | 旧 benchmark / 指标脱节 | 中 | ★★★ |
| ③ 假设 | 隐藏前提崩了 | "we assume" 后不讨论 | 中高 | ★★ (出彩) |
| ④ 泛化 | 换场景失效 | 只在 1 设定验证 | 中 | ★★★ |
| ⑤ 复现 | 重来不了 | 单种子/无代码 | 低 | ★★★★ (你现成有料) |
| ⑥ 理论 | 不懂为啥 work | "surprisingly" | 高 | ★★ (数学优势) |

## 3. gap 不是平等的: 给它打分 (通往 idea 三筛)

找到一堆 gap 后, 不能都做。引入一个**优先级分数**, L4 会接着用:

$$\text{Priority} = \frac{\text{Importance} \times \text{Tractability}}{\text{Cost}}$$

逐项交代每一个量 (都用 1–5 打分, 凭判断):
- **Importance (重要性)**: 填上这个洞, 对领域/对人有多大价值? 多少人会在意? (1=没人care, 5=领域级问题)
- **Tractability (可做性)**: 以你现在的能力/资源, 你真能做出来吗? (1=遥不可及, 5=下周就能出初步结果)
- **Cost (成本)**: 需要多少算力/时间/数据/人? (1=几乎免费, 5=要几百卡几个月)
- **Priority (优先级)**: 分子是「值得且能做」, 分母是「要花多少」。**分数高 = 高杠杆题目: 重要、可做、还便宜。**

> 给博0 的现实建议: 你的第一个题目, 应该刻意选 **Tractability 高、Cost 低** 的 (哪怕 Importance 只是中等)。先完成一个完整研究循环 (找洞→做→写→投), 比一上来就挑领域级难题更重要。**第一篇论文的最大价值是「证明你能跑完全程」。** 这正是 ⑤复现 gap 对你友好的原因。

## 4. 本讲小结 + 通往 L4

- gap 有 6 类, 用雷达逐类扫, 保证不漏洞。
- 每类有专属嗅探信号 (大多和 L2 的攻击清单一一对应)。
- gap 用 Priority = Importance×Tractability/Cost 排序; 博0 优先选高可做、低成本的。

> **下一讲 L4「从 gap 到 idea」**: 找到 gap 只是「这里有个洞」。把洞变成一个**具体、可执行、可证伪的研究 idea** (有假设、有最小实验), 是另一道工序。L4 给你 5 种把 gap 翻成 idea 的手法 + idea 三筛。

**动手**: 在 `N2-find-gaps-in-own-work.ipynb` 里, 拿你自己的 `reasoning-r1` 复现, 按这 6 类逐项扫, 至少落地 3 张 gap 卡并打分。
