# L2 · 攻击式阅读 (Adversarial Reading)

> 40-min lecture · 目标: 戴上「想拒掉这篇论文的审稿人」的眼镜, 把 L1 Pass 3 里那张模糊的「存疑清单」, 升级成一张精确的 **10 问攻击清单**。
> 心法一句话: **凡是你能挑出的破绽, 审稿人一定能挑出; 凡是审稿人能挑出的, 就是你未来论文的攻击点, 也是别人论文的 gap。**

---

## 0. 为什么要「带着恶意」读

学习式阅读默认作者是对的, 你的任务是理解。
攻击式阅读默认作者**想说服你**, 你的任务是**抵抗被说服, 直到证据逼你投降**。

这不是愤世嫉俗, 而是科学的默认姿态。复习一个你 EE 背景熟悉的概念:

> **零假设思维**: 默认「这个方法并不比 baseline 更好」(H₀), 论文的全部实验是在试图让你拒绝 H₀。你的工作是检查: 它给的证据, 真的强到能拒绝 H₀ 吗? 还是只是噪声 / 不公平对比 / 选择性呈现造出的假象?

```
作者的目标:  让你相信  "我的方法 M > baseline B"
你的目标:    找出      "这个 '>' 是真的, 还是被制造出来的?"

           ┌─────────────────────────────────────┐
           │  '>' 可能是假的, 常见 7 种制造方式:    │
           │  1. B 没调好 (弱基线)                  │
           │  2. M 多用了数据/算力 (不公平)         │
           │  3. 只报有利的 seed/数据集 (cherry-pick)│
           │  4. 测试集泄露进了训练 (data leakage)  │
           │  5. 差距在噪声范围内 (无显著性)         │
           │  6. 指标选得偏袒 M (metric gaming)     │
           │  7. 结论超出实验范围 (overclaiming)    │
           └─────────────────────────────────────┘
```

这 7 种「制造 '>' 的方式」就是攻击清单的来源。

## 1. 十问攻击清单 (交付前逐条问)

> 用法: Pass 2/3 时, 对照下表逐条审。每命中一条, 记进解剖卡的「❓可攻击点」, 它就是一个潜在 gap。

### Q1. baseline 公平吗?
- **弱基线陷阱**: 它对比的是 5 年前的方法, 还是当前 SOTA? 它有没有「认真调过」baseline 的超参, 还是用了 baseline 论文里的默认值而把自己的调到飞起?
- 嗅探信号: baseline 的数字明显低于该 baseline 原论文报告的数字 → 它故意没调好基线。
- NLP 例子: 一个新 RLHF 方法只对比 vanilla PPO, 却不对比同期更强的 DPO/GRPO。

### Q2. 算力/数据是不是偷偷多给了自己?
- M 用了更大的模型 / 更多训练 token / 更多标注数据, 却宣称是「方法」带来的提升?
- **iso-FLOP / iso-data 原则**: 公平对比应在「相同算力」或「相同数据量」下进行。
- 嗅探信号: 方法表里 M 和 B 的参数量/数据量那一列对不齐, 且正文不提。

### Q3. 差距有统计显著性吗? 还是落在噪声里?
- 一个常见骗局: M=82.3, B=81.9, 宣称「超越」。但如果换 3 个随机种子, 方差就有 ±0.8, 这个 0.4 的差距毫无意义。
- **看 error bar / 标准差 / 置信区间 / 种子数。** 没报方差的「提升」要高度警惕。
- 一个粗略判据 (两组均值差是否可信):

  $$\text{可信度} \approx \frac{\bar{x}_M - \bar{x}_B}{\sqrt{\sigma_M^2/n_M + \sigma_B^2/n_B}}$$

  逐项交代:
  - $\bar{x}_M, \bar{x}_B$: 方法 M 和基线 B 在多次实验下的**平均**指标。
  - $\sigma_M^2, \sigma_B^2$: 各自的**方差** (反映随种子/数据波动有多大)。
  - $n_M, n_B$: 各自跑了**多少次** (多少个种子)。
  - 整个分数: 类似 t 统计量。**分子是「差多少」, 分母是「噪声有多大」。** 分数越大, 差距越可能真实。粗略地, 这个值 < 2 就别太当真。
  - 直觉: 如果只跑了 1 个种子 ($n=1$), 你根本算不出分母 → **单种子结果不可信**, 这是最该攻击的点之一。

### Q4. 测试集干净吗? (data leakage / 数据泄露)
- **数据泄露**: 测试集的内容 (或高度相似内容) 出现在了训练数据里, 导致分数虚高。
- LLM 时代尤其致命: 预训练语料是全网爬的, 很可能已经包含了 GSM8K/MMLU 的题目。
- 嗅探信号: 在「模型可能背过的」老 benchmark 上暴涨, 在「新发布、不可能背过的」benchmark 上平平。
- NLP 例子: 一个数学推理模型在 GSM8K 上 95%, 但在结构相同、2024 年才发布的 GSM-1k 上只有 70% → 强烈的泄露/过拟合信号。

### Q5. 指标选得偏心吗? (metric gaming)
- 它报的指标, 是对它有利的那个吗? 该领域公认的主指标它报了吗, 还是悄悄换成了一个偏袒自己的?
- 嗅探信号: 主表用 BLEU (容易刷), 却不报人工评估或更严格的指标; 或只报 pass@k 里 k 很大的那个。

### Q6. 实验覆盖够吗? 还是只在一个有利设定下成立?
- 只在 1 个数据集 / 1 个模型规模 / 1 种语言上验证, 就宣称普适?
- 嗅探信号: 「Due to space, we report on X」却恰好 X 是最有利的。

### Q7. 消融做全了吗? 真正起作用的是哪一块?
- 方法有 A+B+C 三个组件, 它做消融了吗? 会不会其实只有 A 有用, B/C 是凑数的?
- 嗅探信号: 没有消融表, 或消融只去掉最不重要的组件。**没有消融 = 你不知道它为什么 work。**

### Q8. 结论有没有超出证据? (overclaiming)
- 标题/摘要说「general / robust / reasoning」, 但实验只在一个窄设定做的。
- **桥接检查**: 它结论里每一个形容词, 能在实验里找到对应证据吗?

### Q9. 复现性如何?
- 有没有放代码? 超参全不全? 关键实现细节藏在附录还是干脆没有?
- 嗅探信号: 「code will be released」(经典空头支票) / 关键 trick 一笔带过。

### Q10. 它没测什么? (最重要的一问 → 直通找问题)
- 一个有趣的方法, 作者**故意或无意没做的那个最显然的实验**, 往往就是你的论文。
- 问法: 「如果我是 reviewer, 我会要求补哪个实验?」那个实验, 你现在就可以去做。

## 2. 把「攻击」翻译成「机会」

攻击式阅读不是为了显得聪明, 是为了**把别人的弱点变成你的题目**。每个攻击点都可以翻面:

```
攻击点 (它的弱点)                    →   机会 (你的题目)
─────────────────────────────────    ─────────────────────────────
Q1 它只比了弱基线                     →   在强基线下重测, 看结论还成立吗
Q3 它只跑了 1 个种子                  →   多种子复现, 报告真实方差 (可发 workshop)
Q4 它可能数据泄露                     →   在 decontaminated / 新 benchmark 上重评
Q6 它只在英文上验证                   →   跨语言/跨域泛化研究
Q7 它没做消融                        →   补消融, 找出真正起作用的组件
Q10 它没测 X                         →   去测 X
```

> 这张「攻击 → 机会」转换表, 是 L2 通往 L3/L4 的桥。**批判不是终点, 找到题目才是。**

---

## 3. 红线: 攻击 ≠ 抬杠, 也 ≠ 学术不端

- **合法攻击 (我们做的)**: 基于证据指出「这个主张的支撑不足」。目标是逼近真相。
- **抬杠 (别做)**: 「我就是不喜欢」「为什么不用我喜欢的方法」这类无证据的挑剔。审稿时这叫低质量 review。
- 反过来, 当**你**是作者时, 你 `how_to_write_a_paper` 技能里讲的「审稿人视角检查清单」和这张攻击清单是**同一张表的两面**: 写作时你提前堵住这 10 个洞, 投稿时审稿人就无从下手。

> 这里有个深刻的对称: **会攻击, 才会防守。** 你越擅长攻击别人的论文, 你自己的论文就越无懈可击。这就是为什么 9.9 `research-life` 专题会专门练「给别人审稿」—— 审稿是最好的写作训练。

## 4. 本讲小结 + 通往 L3

- 默认姿态: 抵抗被说服, 直到证据逼你投降 (零假设思维)。
- 10 问攻击清单 = 7 种「制造假 '>'」方式的展开。
- 每个攻击点都能翻面成一个题目 (攻击→机会表)。
- 攻击是为了找题, 不是为了抬杠; 会攻击才会防守。

> **下一讲 L3「gap 分类学」**: 你现在能挑出一篇论文的毛病了。但「毛病」是零散的。L3 把所有研究缝隙归成 **6 大类**, 每类配一组「嗅探信号」, 让你从「碰运气发现一个 gap」升级到「系统地扫出所有 gap」。

**动手**: 在 `N1-dissect-a-paper.ipynb` 里, 用这 10 问逐条审你选的论文, 至少落地 3 个可攻击点。
