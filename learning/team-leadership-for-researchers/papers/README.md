# papers/ — team-leadership-for-researchers 参考源

本专题教**带教新手、向下反馈、组会主持、冲突处理、团队健康诊断**这五件事(L1-L5), papers/ 收团队管理与组织行为学类的经典参考。

## 关于"工程管理书籍迁移到科研团队场景"的说明

下面引用的多本书(Fournier/Scott)是面向软件工程团队管理者写的, **不是专门为科研团队写的**——本专题引用它们, 是因为"怎么带一个新人""怎么给建设性反馈""怎么主持团队会议"这几件事的底层原则(任务分级、具体可执行的反馈、结构化的会议设计), 在工程团队和科研团队之间高度可迁移, 但书中的具体场景(如KPI考核、绩效评审)不能照搬到科研团队, 引用时请注意这一点。

## 指导低年级学生/实习生 (L1)
- **Camille Fournier, 《The Manager's Path: A Guide for Tech Leaders Navigating Growth and Change》**(O'Reilly, 2017): 工程管理书籍, 概念迁移到科研团队场景。书中关于"怎么带一个新人上手"和"怎么做建设性 code review" 的章节, 是本讲第1-2节任务难度阶梯和新手向code review框架的重要参考来源。

## 向下管理与建设性反馈 (L2)
- **Kim Scott, 《Radical Candor: Be a Kick-Ass Boss Without Losing Your Humanity》**(St. Martin's Press, 2017): 工程/职场管理书籍, 概念迁移到科研团队场景。书中"关心个人(care personally)+ 直接挑战(challenge directly)"的四象限框架, 是本讲"balance不是表扬三明治式公式"这一论点的直接理论来源。
- **Camille Fournier, 《The Manager's Path》**: 同上, 书中关于反馈时机和具体化建议的章节。

## 高效组会与Brainstorm会主持 (L3)
- **Google re:Work 公开的 Project Aristotle 团队效能研究**: Google 多年追踪内部团队后发现"发言机会是否大致均等"(equal conversational turn-taking)是团队效能的重要预测因子之一, 这一发现最初来自 Woolley 等人关于集体智能的研究。是本讲"让安静的人发言"背后的实证支撑。
- **Camille Fournier, 《The Manager's Path》**: 书中关于主持团队会议、设计议程的章节。
- 也见本仓库 `critical-reading-gap` L4「从gap到idea」的五种idea生成手法(组合/迁移/极限/反向/未测), 本讲第3节直接把它们复用为brainstorm会的结构化引导轮次。

## 团队协作冲突处理 (L4)
- **Patterson, Grenny, McMillan & Switzler, 《Crucial Conversations: Tools for Talking When Stakes Are High》**(McGraw-Hill): 高风险沟通场合的经典参考, "把事实和故事分开陈述"的原则直接适用于authorship纠纷的处理流程。
- **Patrick Lencioni, 《The Five Dysfunctions of a Team》**(Jossey-Bass, 2002): "害怕冲突"被列为团队功能失调的核心一环, 是本讲"回避不等于和谐"这一论点的理论对照。
- 也见 `research-life` L3「署名与学术伦理」——那一讲讲的是预防(项目早期谈清楚署名预期), 本讲讲的是应对(分歧已经发生之后怎么公开化处理), 两讲互补而非重复。

## 团队健康度诊断 (L5)
- **Google re:Work 公开的 Project Aristotle 团队效能研究**: 核心发现——心理安全感是团队效能最强的预测因子, 强于团队成员个体能力。本讲第2节直接建立在这项研究之上。
- **Amy Edmondson, 《The Fearless Organization: Creating Psychological Safety in the Workplace》**(Wiley, 2018): 心理安全感概念的系统阐述, 明确区分"心理安全感"和"随和/低标准", 是本讲澄清常见误区的直接理论来源。
- 也见 `research-life` L4「可持续科研」的burnout自查——那一讲是个体层面的自我诊断, 本讲是团队结构层面的诊断, 两者粒度不同, 互补而非重复。
