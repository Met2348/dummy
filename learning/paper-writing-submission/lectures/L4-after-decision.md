# L4 · 录用 / 拒稿之后: camera-ready、改投、arxiv、学术诚信

> 25-min lecture · 目标: 走完投稿循环的收尾与重启 —— 录用了怎么做 camera-ready, 被拒了 (大概率) 怎么高效改投, arxiv 怎么用, 学术诚信红线在哪。

---

## 0. 决定来了, 两条路

通知日, 两种结局:
- **录用 (accept)**: 恭喜! 但还有 camera-ready、版权、presentation (9.8) 要做。
- **拒稿 (reject)**: **大概率是这个** (顶会接收率 20-25%)。不是终点, 是改投的起点。

这一讲讲两条路各怎么走, 以及贯穿始终的学术诚信。

---

## 1. 录用之后: camera-ready

录用不等于完事。**camera-ready (终稿)** 阶段要做:
- **改进**: 把 rebuttal 里承诺的修改全部落实 (审稿人/AC 会对照检查)。
- **去匿名**: 加回作者、单位、致谢 (双盲解除)。
- **加页**: 很多会给录用论文 +1 页放 rebuttal 补的内容。
- **版权**: 签 copyright form。
- **代码/数据**: 兑现可复现承诺 (9.5), 放出非匿名的正式仓库。
- **注册 + presentation**: 准备 talk / poster (交棒 9.8)。

> camera-ready 是兑现承诺的时刻。**rebuttal 里答应改的一定要改** —— 不仅是诚信, AC 真的会查, 食言可能影响后续 (甚至极端情况撤稿)。

---

## 2. 拒稿之后: 把拒稿变成燃料

拒稿信会附上审稿意见和 meta-review。**这是一份免费的、详细的改进清单**, 价值极高。高效改投流程:

```
   拒稿 → 冷静期(几天) → 系统读所有意见(连 rebuttal 一起) → 提炼"真正的问题"
        → 决定: 大改 or 小改 → 改 → 选下一个会 → 重投
```

几个要点:
- **区分「真问题」和「审稿噪声」**: 多个审稿人共同指出的, 是真问题, 必须解决 (回忆 9.2: 多篇共有的 limitation 是被验证的真洞)。只有一个人提的怪意见, 可酌情。
- **大改还是小改**: 如果核心贡献被质疑 → 可能要补关键实验/重构叙事 (大改); 如果只是表述/小实验 → 小改快速改投。
- **选下一个会**: 看日历 (L2), 选最近的合适 deadline; 别因为「再等等改完美」拖半年 —— **改投的速度也是产出**。
- **诚实对待新审稿人**: 改投后是新一轮审稿, 通常不要求你提之前被拒 (除非同系列), 但**别隐瞒重大变化**。

> 心态: **几乎所有著名论文都被拒过, 很多还不止一次。** 拒稿是科研的常态机制, 不是对你的判决。一个高产研究者的区别不在「不被拒」, 而在「被拒后快速、理性地改投」。**把情绪从拒稿里抽出来, 把意见当数据用。**

## 3. arxiv: 预印本策略

arxiv 是预印本服务器, 投稿循环里要不要用、何时用, 有讲究:

| 何时挂 arxiv | 考量 |
|---|---|
| **投稿前/同时挂** | 抢占先机 (idea 防被抢)、早传播、早拿引用; **但**破坏双盲 (审稿人可能搜到你) |
| **录用后挂** | 安全 (不破坏匿名); 但可能被人抢先发类似工作 |
| **被拒后挂** | 让工作先见光、拿反馈, 同时改投 |

> 双盲与 arxiv 的张力: 多数会**允许** arxiv 预印本, 但要求审稿人「不主动去搜」。现实是抢先占位 (priority) 常常压过匿名顾虑, 尤其热门方向。**给博0 的建议: 和导师商量** —— 这与方向竞争激烈程度、组里习惯有关, 没有统一答案。挂之前确认目标会的政策 (有的会有 anonymity period 禁止临近 deadline 新挂 arxiv)。

## 4. 学术诚信: 不可碰的红线

投稿循环里, 几条红线**碰了可能毁掉学术生涯** (撤稿、禁投、声誉尽毁):

| 红线 | 说明 |
|---|---|
| **伪造 / 篡改数据** | 编数字、改结果。绝对禁止 (9.4/9.5 教你诚实做实验正是为此) |
| **一稿多投 (dual submission)** | 同一工作同时投多个会。被查到双方都拒 + 记录在案 |
| **抄袭 (plagiarism)** | 抄文字/idea 不引用。包括抄自己已发表的 (self-plagiarism) |
| **不当署名** | 把没贡献的人挂上 / 漏掉有贡献的人 (9.9 详谈) |
| **数据泄露当卖点** | 测试集进了训练 (9.3 攻击清单 Q4) |
| **暗改 benchmark** | 在评测上做手脚 |

> 回到技能包反复强调、9.3/9.5 也强调的那条线: **合法策展** (用真实数字讲聚焦故事) vs **学术不端** (无中生有)。前者是手艺, 后者是犯罪。两者的分界是: **你的每个数字都是真跑出来的吗?** 是 → 你只是选择性呈现 (合法); 否 → 不端。**博0 务必从第一篇就把这条线刻进骨子里** —— 学术圈很小, 一次不端跟一辈子。

## 5. 整个 9.7 收口 + 投稿循环全景

```
   研究做完 (9.1-9.6)
        │
   L1 装配成论文 (+ how_to_write_a_paper 写细则)
        │
   L2 选会 / 卡 deadline / 匿名 / 提交
        │
   审稿 (~6-8 周)
        │
   L3 读意见 → 写 rebuttal (补实验是最强武器)
        │
   ┌─ accept ─→ L4 camera-ready → 9.8 presentation
   └─ reject ─→ L4 把意见当数据 → 改 → 改投 (大概率走这条, 别灰心)
        │
   贯穿: 学术诚信红线
```

> 一句话送你: **第一篇论文的目标不是中顶会, 是「跑完整个循环」** (写→投→评审→rebuttal→改投)。跑完一遍, 你就从「会复现的工程师」真正变成了「会做研究的人」。这也是整个 Module 9 的目的地。

## 6. 本讲小结 + 通往 9.8

- **录用**: camera-ready 要兑现 rebuttal 承诺、去匿名、放正式代码、准备 presentation。
- **拒稿** (大概率): 把审稿意见当**免费改进清单**; 区分真问题 (多人共指) vs 噪声; 快速理性改投。
- **arxiv**: 投稿前挂抢先机但破匿名, 录用后挂最安全; 和导师商量、查会议政策。
- **学术诚信红线**: 伪造/一稿多投/抄袭/不当署名/数据泄露 —— 碰了毁生涯。合法策展 ≠ 不端, 分界是「数字是否真跑出来」。
- 第一篇的目标是**跑完整个循环**, 不是中顶会。

> **下一专题 9.8「research-presentation」**: 论文录用 (或哪怕没录) 你都要把工作**讲给人听** —— 会议 talk、poster、组会、答辩、电梯演讲。9.8 教你把研究讲清楚、讲精彩。

**动手**: 模拟一次拒稿复盘 —— 拿 N2 你写过 rebuttal 的那组意见, 假设这次被拒了, 用 L4 的流程写出: ① 哪些是真问题 (多人共指) ② 大改还是小改 ③ 改投哪个会 (查日历)。
