# L5 · 负结果的处理与发表策略: file drawer problem 与诚实报告失败分支

> 30-min lecture · 目标: L1-L4 全程默认了一件事——你的假设最终被数据支持了。但真实科研里, **负结果 (negative result, 假设没被支持) 比正结果多得多**。本讲教你三件事: ① 认识 file drawer problem 这个系统性偏差、② 判断一个负结果什么时候值得单独写成论文 (而不是默默扔掉)、③ 在你的正面工作里怎么诚实报告失败的分支, 而不是把它们藏起来。

---

## 0. 一个你迟早会遇到的处境

回忆 L1 的 Robust-DPO 例子: 你验证了「置信度加权能让 DPO 在高噪声偏好下更鲁棒」, 这是一个正结果, 论文顺利写出来了。但在拿到这个正结果之前, 你大概率已经试过至少两三个**没 work** 的变体——比如「给偏好对加课程学习 (curriculum learning), 先学容易样本再学难样本」, 你满心以为这会进一步提升鲁棒性, 结果多跑几个种子后发现: 加了课程学习和不加, 结果在统计上没有显著差异。

这时候你有三个选择: **① 假装没做过这个实验, 论文里绝口不提** ② **把它写成一段"我们还试了 X, 没有效果"塞进论文某个角落** ③ **认真评估这个负结果本身是否值得单独写成一篇论文**。选 ① 是本讲要教你避免的坏习惯; 选 ② 和 ③ 之间怎么权衡, 是本讲的核心。

---

## 1. file drawer problem: 负结果去哪了

**file drawer problem (文件抽屉问题)** 由心理学家 Rosenthal 在 1979 年提出 (Rosenthal, "The File Drawer Problem and Tolerance for Null Results", *Psychological Bulletin*, 1979): 期刊上发表的几乎全是「显著」「有效」的正结果, 而大量做出「不显著」「无效」结果的研究, 从没被写出来、投出去, 就静静地躺在研究者的文件抽屉里, 没人知道它们存在过。

这不是某个人偷懒, 而是一套**系统性的激励结构**共同造成的:
- **审稿人偏好**: 「我们试了 X, 没用」天然读起来不如「我们提出 X, 显著提升」有卖点, 更容易被拒。
- **作者自我审查**: 你自己也会觉得「没做出来的东西, 谁要看」, 干脆不投, 甚至懒得写。
- **领域激励**: 顶会看重的是 leaderboard 数字, 「阴性」发现在传统评价体系里不算「贡献」。

**这套偏差对 NLP/LLM 领域的具体危害**, 比很多人以为的更实际:
- **重复造轮子的时间浪费**: 如果「课程学习对噪声偏好没帮助」这件事只有你知道, 下一个博0 会重新花两周去验证同一件事, 全领域重复浪费算力和时间。
- **meta-analysis / 综述失真**: 9.2 mini-survey 式的领域地图, 如果只看得到发表出来的正结果, 会系统性高估某类技巧的有效性——因为所有说「这招没用」的研究都没被看见。
- **把调参变成玄学而非科学**: insights-workshop.github.io (「Insights from Negative Results in NLP」workshop, 与 EMNLP 同期举办) 在其征稿说明里明确指出: NLP 领域对 benchmark 提升的近乎唯一关注, 隐性地打压了假设驱动的研究, 把模型调优变成了「手艺」而非「科学」——这正是 file drawer problem 在这个领域的具体后果。

> 一句话记住: **file drawer problem 不是"负结果不重要", 是"负结果的价值被系统性地埋没了"。** 本讲后两节教你怎么在恰当的场合把它挖出来。

## 2. 三个层级: 不是所有负结果都值得同样对待

不是每个「没 work」都该扔进文件抽屉, 也不是每个都该单独写成论文。用三个层级分类, 决定处理方式:

```
层级①「实现问题」            层级②「窄假设的负结果」          层级③「颠覆广泛假设的负结果」
──────────────────          ──────────────────────         ─────────────────────────────
课程学习加进去 bug了,        课程学习在"这一个具体设定下"      "课程学习对噪声鲁棒性没有普适帮助",
根本没跑对                   没有效果 (你自己的一次尝试)        而这是领域内被广泛默认成立的假设
                                                             (多篇论文隐含假设它有效)
→ 不是负结果, 是bug,         → 值得诚实记录, 但通常不单独       → 值得考虑单独写成一篇论文,
  debug 到跑对再评估            成文 (见第4节: 放进正面工作)      投 Insights workshop 或作为主论文
                                                                的一节 systematic negative finding
```

- **层级①「实现问题」**: 你的负结果其实是 bug——数据没对齐、超参抄错、评测脚本有洞。**这根本不是负结果, 是没做完的实验。** 混淆这一层是新手最常见的错误: 把「我还没调对」当成「这个方法真的没用」, 过早放弃一个其实可行的方向, 或者反过来, 把一个真实的 bug 当负结果写进论文, 被审稿人 (或更糟, 被读者复现时) 揭穿。回忆 9.5 experiment-ops-repro 强调的**留痕**——固定 seed、记录 git sha、多种子——正是帮你排除「是不是我做错了」这个可能性, 才能放心进入层级②③的判断。
- **层级②「窄假设的负结果」**: 你认真做对了实验 (排除了层级①), 但结论只是「在我这一个具体设定下没有效果」——样本量小、只测了一个数据集、只跑了 2-3 个种子。这类负结果**真实、诚实, 但通常不足以支撑一篇独立论文** (审稿人会问「你怎么知道换个数据集/规模它不 work」)。正确去处: 诚实记录进你正在写的正面工作 (见第 4 节), 而不是单独包装成一篇论文硬投。
- **层级③「颠覆广泛假设的负结果」**: 这才是**值得单独发表**的负结果。判据是: **① 严谨性** (多种子/多数据集/系统性消融, 排除「你只是做错了」这个最常见的审稿质疑) **② 广泛性** (你反驳的不是你自己随手的猜测, 而是领域里多篇论文隐含或明确依赖的一个假设) **③ 有实际后果** (如果这个假设错了, 会改变别人怎么做研究/调参)。

层级③的负结果论文, 是`critical-reading-gap` `L4-idea-generation.md` 讲过的 **手法④「反向 (Reverse/Contrarian)」** 的一个具体子类: 反向手法问「大家都认为 P 越多越好, 真的吗」, 而层级③负结果就是**系统地把这个反问跑出数据、证明"真的不是"**。判断一个负结果够不够格进层级③, 其实就是回去套用 idea 三筛 (`critical-reading-gap` L4): 筛 1 (真的没人证伪过这个假设吗) + 筛 3 (证伪它有人在意吗, 会不会让人「哦!」)。

## 3. 层级③负结果的发表去处与写法

顶会主赛道 (ACL/EMNLP/NeurIPS 主会) 对「纯负结果」论文接受度仍然偏低, 因为审稿人下意识用「有没有 new method」的尺子衡量。给博0 两条更现实的路:

- **专门的负结果 venue**: 「Insights from Negative Results in NLP」workshop (insights-workshop.github.io, 与 EMNLP 同期举办, 已连续多届) **明确欢迎**这类投稿——征稿范围包括: 之前发表结果的 (不)稳定性 (换随机种子/硬件复现不出来)、对已提出模型做消融发现某组件贡献与原论文所称不符、新数据集/探测任务证明先前方法不能泛化到别的领域或语言现象。这些恰好对应层级③的判据。**如果你的负结果符合这些模式之一, 这是最合适、接受门槛也更合理的去处。**
- **作为主论文的一节「系统性负发现」**: 如果你的层级③负结果和你正在做的正面工作强相关 (比如你在验证 Robust-DPO 时, 顺手系统性推翻了"课程学习对噪声鲁棒性有帮助"这个假设), 可以把它写成主论文里独立成节的「Systematic Negative Finding」, 而不是硬凑成单独一篇——审稿人会认可"这是我们意外收获的一个有价值的副产品", 前提是你像层级③要求的那样做扎实 (多数据集/多种子/系统消融)。

无论投哪条路, 层级③负结果论文的叙事结构和正结果论文一样需要 how_to_write_a_paper 的「叙事先行」——只是**核心 claim 从「X 有效」换成「大家以为 X 有效, 但系统性证据表明并不」**, 证据链 (`src/paper_assembler.py` 的 `narrative_audit`) 一样要闭合: 每个「X 无效」的子论断, 都要有对应的消融/多种子实验撑住, 不能只凭「我们试了一次没 work」就下结论——**这正是层级③和层级②的本质区别: 层级③需要跟做正结果同等量级的严谨性来证明"无", 而不是随手一次尝试。**

## 4. 层级②怎么办: 在正面工作里诚实报告失败分支

对大多数博0 更常遇到的是层级②——你的主论文有一个正结果 (Robust-DPO work 了), 同时你诚实地做过几个没 work 的分支尝试 (课程学习没帮助、换一种加权函数效果打平)。**这些不该被藏起来**, 原因有三:

1. **学术诚信** (呼应 L1 第 4 节、L4 第 4 节的红线): 藏起来不等于造假, 但如果你的方法描述暗示"我们尝试的每一步都顺利导向最终方案", 而实际上你走了不少弯路才找到有效配置, 这是一种**误导性的叙事包装**——和"合法策展"的区别在于, 策展是省略偏题的真实支线, 而完全隐瞒会让读者误判方法的鲁棒性和你调参的工作量。
2. **对后来者的价值**: 你的「课程学习没帮助」这一条信息, 对下一个做同类工作的人价值可能不亚于你的正结果本身——它替别人省下了重新验证一遍的时间, 这正是第 1 节讲的 file drawer problem 想解决的问题。
3. **对审稿人的说服力**: 一份「我们系统尝试了 A/B/C/D 四种设计, 只有 D work, 原因是……」的论文, 比一份「我们提出 D, 效果很好」的论文**更让审稿人信任你的方法论严谨性**——前者证明你不是运气好蒙对了一个配置, 而是系统探索后找到的最优解。

**具体写法** (不需要单独发一篇, 放进现有论文即可):
- **位置**: 通常放进「实现细节/消融」附录一个小节, 标题可以直接叫「Negative Results / What We Tried That Didn't Work」, 或放进 Limitations 节 (回忆 L2 第 3 节: Limitations 是多数会强制要求的一节, 天然是安放诚实负面信息的地方)。
- **篇幅**: 几句话到半页足够——不需要像层级③那样展开完整实验, 目的是**留痕**, 不是重新论证。
- **格式模板**: 「我们还尝试了 [具体方法], 动机是 [为什么觉得可能有效]; 在 [具体设定] 下, 结果为 [具体数字/无显著差异]; 我们推测原因可能是 [简短假说, 不需要证实]。」——**给出具体数字比只说"没有效果"更有价值** (哪怕只是一次尝试的数字, 只要如实标注"仅一次尝试, 未做多种子验证", 就不违反诚实原则)。
- **和 narrative_audit 的关系**: `paper_assembler.py` 的 `narrative_audit` 目前检查的是"正面 claim 有没有证据"; 你可以把"失败分支"当成一种特殊的 claim——「我们尝试了 X, 未能带来提升」同样应该有它自己的最小证据 (哪怕只是一行数字), 而不是空口一句"我们也试过其他方法"。

## 5. 常见误区

| 误区 | 真相 |
|---|---|
| 「负结果不算贡献, 写出来也没人看」 | file drawer problem 恰恰证明: 领域缺的正是这些信息; layer③负结果 (颠覆广泛假设) 常常比一个平庸的正结果引用更多、影响更持久 |
| 「把所有失败分支都详细写进论文才叫诚实」 | 过度堆砌层级①/②的失败细节会稀释主线、消耗审稿人耐心——诚实 ≠ 不做取舍; 该做的是**留痕但克制** (第4节的模板), 大量层级①的调试性失败不需要写进论文 |
| 「负结果论文比正结果论文好发, 因为竞争少」 | 恰恰相反——审稿人对负结果论文的严谨性要求更高 (必须排除"你只是做错了"), 层级③门槛不比正结果低, 只是**评价标准不同** (新颖性来自"证伪"而非"提出") |
| 「藏起没 work 的分支只是策展, 不算学术不端」 | 要看藏起来的方式: 省略偏题支线 (合法) vs 让读者误以为方法一路顺利、从而低估调参代价或误判鲁棒性 (误导性叙事), 界限在于是否影响读者对方法可靠性的判断 |

## 7. 本讲小结 + 通往 L6

- **file drawer problem**: 负结果系统性地不被发表, 造成重复劳动、领域认知失真; NLP 领域因为唯 benchmark 论的激励结构而尤其严重。
- **三层级**: 层级①(实现问题, 先 debug) / 层级②(窄假设负结果, 诚实记录进正面工作即可) / 层级③(颠覆广泛假设, 值得单独发表)。
- 层级③要严谨 (多种子/多设定排除"做错了") + 广泛 (反驳的是领域共识而非自己的猜测); 去处包括 Insights workshop 或主论文里独立的 systematic negative finding 一节。
- 层级②不该被藏起来: 用「尝试了什么+具体结果+简短推测」的克制模板放进 Limitations 或消融附录, 既诚实又给后来者省时间。

> **下一讲 L6「多篇论文组合发表策略」**: 你现在会诚实处理单篇论文里的正负结果了。但一个博士生手里往往同时有好几个课题的产出——L6 教你怎么把一个大项目切成几篇可独立发表的论文, 而不是憋成一篇"大而全", 也不是切得太碎变成灌水。

**动手**: 回顾你自己 (或想象中) 做过的一次实验尝试, 把它按第 2 节的三层级分类: 它是层级① (还没调对)、层级②(诚实的窄负结果) 还是层级③候选 (可能颠覆了某个广泛假设)? 如果是层级②, 用第 4 节的模板写一段「负结果记录」; 如果怀疑是层级③, 用 idea 三筛 (筛1/筛3) 检验它是否真的值得单独展开。
