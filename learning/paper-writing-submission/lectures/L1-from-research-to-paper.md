# L1 · 从研究到论文: 把 9.1-9.6 的产物装配成一篇论文

> 30-min lecture · 目标: 把你做完前几个专题攒下的零散产物 (领域地图/gap/假设/实验/图), 用一条叙事装配成论文骨架。**本讲不重复写作方法论** —— 那在你已有的 `how_to_write_a_paper` 技能包里; 本讲讲「装配 + 叙事链自检」。

---

## 0. 你其实已经有了一篇论文的大半

走完 9.1-9.6, 你手里已经有:

| 产物 | 来自 | 进论文哪节 |
|---|---|---|
| mini-survey / 领域地图 | 9.2 | 相关工作 + 引言动机 |
| gap 卡 | 9.3 | 引言 (我们补的洞) |
| 可证伪假设卡 | 9.4-L1 | 引言贡献 + 方法 |
| 消融矩阵 + 交互效应 | 9.4-L4 | 实验 + 消融 |
| 多种子 + 显著性 | 9.4-L5 / 9.5 | 实验 (可信结果) |
| 出版级图 + Figure 1 | 9.6 | 全篇 |

> **论文不是从零写, 是把这些产物用一条叙事串成闭合证据链。** 这正是 `how_to_write_a_paper` 的核心: **叙事先行** —— 先定要讲什么故事, 再让所有材料服务它。本讲帮你把材料**对齐**到论文骨架, 并在动笔前**自检证据链通不通**。

## 1. 两层分工: 本专题 vs how_to_write_a_paper 技能包

你已经有一个成熟的写作技能包, 别重复造。两层各管什么:

```
   ┌─────────────────────────────────────────────────────────┐
   │ how_to_write_a_paper 技能包 (写作"细则层")                  │
   │   · 叙事先行方法论 (灵魂)                                   │
   │   · 逐部分写作细则 (标题/摘要/引言/方法/实验/消融/局限...)    │
   │   · 图表/公式/清晰性                                       │
   │   · 审稿人视角检查清单                                      │
   └─────────────────────────────────────────────────────────┘
                          ▲ 写每一节时打开它
   ┌─────────────────────────────────────────────────────────┐
   │ 本专题 9.7 (投稿"流程层" + 装配)                            │
   │   · L1 把研究产物装配成骨架 + 叙事链自检 (本讲)              │
   │   · L2 投稿全流程 (venue/时间线/格式/匿名/OpenReview)        │
   │   · L3 同行评审 + rebuttal (你以前完全没有的下半场)          │
   │   · L4 录用/拒稿之后 (camera-ready/改投/arxiv 策略)          │
   └─────────────────────────────────────────────────────────┘
```

> 一句话: **写「怎么写好一句话/一节」→ 查技能包; 写「怎么把研究装成论文、怎么投出去、怎么应对评审」→ 看本专题。** 两者配合, 覆盖从「研究做完」到「论文录用」的全程。

## 2. 装配: 把产物映射到骨架

`src/paper_assembler.py` 把你的产物字典映射到标准顶会骨架 (对应技能包的 `sections.md`):

```
   你的产物                →  论文骨架的节
   narrative(核心叙事)     →  标题 / 摘要 / 引言
   mini_survey(9.2)        →  相关工作 / 引言
   gap(9.3)                →  引言 / 相关工作
   hypothesis(9.4)         →  方法 / 引言贡献
   results(9.4/9.5)        →  实验 / 摘要
   ablation(9.4-L4)        →  消融
   figures(9.6)            →  方法 / 实验
   limitations(9.5)        →  局限
```

`assemble_skeleton(artifacts)` 生成一个带占位的骨架, 每节标出「哪些素材可用 / 哪节还空着」。**这把"对着空白页发呆"变成"往一个有结构的骨架里填已有素材"。**

## 3. 叙事链自检: 动笔前先确认证据链闭合

装配只是把材料归位。更关键的是 `narrative_audit`: **检查你论文的每个核心 claim 有没有 evidence 支撑。**

回忆技能包的最高原则: **论文是一条闭合的证据链, 任何 claim 都要有证据, 任何无证据的 claim 都是审稿人的攻击点** (现在很多是 AI 审稿, 破绽必被抓)。自检做两件事:

1. **节覆盖**: 哪些节还没素材 (如忘了写 limitations)?
2. **claim→evidence**: 每个主张能不能指向一个具体证据 (实验/图/分析)?

例 (工具的输出):
```
   ✅ 「Robust-DPO 在高噪声下显著更优」← results (有 9.4 实验支撑)
   ✅ 「提升来自鲁棒机制而非 trick」  ← ablation (有消融支撑)
   ❌ 「在所有规模上都成立」          ← (无证据!)   ← 危险: 删掉它或补实验
```

> 第三条是典型陷阱: 一个**没有实验支撑的过度宣称**。技能包教你「笃定地写」, 但**笃定 ≠ 无证据地吹**。正确做法: 要么补一个跨规模实验 (回到 9.4), 要么把这个 claim 删掉/弱化 (它属于 future work)。**在动笔前用 audit 抓出这些, 比被审稿人抓出代价小得多。**

## 4. 叙事先行的一个再强调 (易被博0误解)

技能包有一条容易被理工科新手误读的原则, 这里再交代清楚:

> **「严禁主动展示负面内容 / 一个实验不支持不代表叙事崩塌」** —— 这指的是**策展 (curation)**: 从真实、复杂、矛盾的结果里, 挑出最清晰的一条线深刻呈现, 把偏离主线的真实支线留给 follow-up。**这不是造假。**

务必分清 (技能包的红线, 也是 9.3 / 9.5 反复强调的):
- **合法策展**: 用**真实数字**讲**聚焦干净**的故事; 略去的是真实但偏题的支线。
- **学术不端**: 伪造数据、数据泄露、没跑 baseline 就宣称超越、藏 cherry-pick。**绝不碰。**

> 给你的具体落地: 9.4/9.5 让你**诚实地把实验做对、把方差报全** (对自己/对证据); 9.7/技能包让你**聚焦地把故事讲清** (对读者)。两者不矛盾: 先有干净可信的证据 (9.4/9.5), 才有底气聚焦地策展 (9.7)。无证据的策展是空中楼阁, 会在 rebuttal (L3) 被戳穿。

## 5. 本讲小结 + 通往 L2

- 走完 9.1-9.6, 你已有一篇论文的大半素材; 论文是用**叙事**把它们串成闭合证据链。
- 两层分工: **写作细则 → `how_to_write_a_paper` 技能包**; **装配 + 投稿流程 → 本专题**。
- `paper_assembler` 把产物**映射**到骨架; `narrative_audit` 在动笔前查**节覆盖 + claim→evidence**。
- 无证据的 claim 是头号攻击点; 用 audit 提前抓出 (删掉或补实验)。
- 分清**合法策展** (真实数字讲聚焦故事) 和**学术不端** (伪造/藏 cherry-pick)。

> **下一讲 L2「投稿全流程」**: 骨架装好、论文写完, 接下来是你以前完全没接触的部分 —— 选哪个会、怎么卡 deadline、双盲怎么匿名、OpenReview 怎么提交。L2 给你投稿的全流程地图。

**动手**: 去 `N1-paper-skeleton-assembler.ipynb`, 把你某个复现专题 (如 reasoning-r1 + 一个 gap) 的产物喂给 `paper_assembler`, 生成骨架并跑叙事链审计, 看你的「论文」现在缺哪节、有没有无证据的 claim。
