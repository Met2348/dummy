# L3 · 交互式/可玩世界模型 (加动作条件)

> 22-min lecture · 目标: 理解怎么把被动视频模型变成「可按键控制」的交互世界 (Vid2World/AVID/Genie 路线)。接 L2, 通 L4 具身。

---

## 0. 从「被动续帧」到「可玩」

L2 的视频模型是**被动**的: 它续帧, 但你没法干预。要让它成为机器人/游戏能用的世界模型, 得加一件事: **动作条件** —— 让生成的下一帧取决于「你按了什么键 / 机器人做了什么动作」。

```
   被动视频模型:   过去帧           → 下一帧 (你只能看)
   交互世界模型:   过去帧 + 动作 a   → 下一帧 (你能控制!)
```

> 这就把 L1 的「$s,a\to s'$」搬到了图像空间: 状态=帧, 转移=动作条件的续帧。**加上动作条件, 视频模型就从「会做梦」变成「可交互的世界」**。

---

## 1. 动作怎么注入 (你已会的条件机制)

把动作接进视频生成, 用的全是你学过的条件注入手段 (M13.3-L3/L4):
- **动作当条件 token**: 像类别/文本条件一样, 把动作编码成 token 注入去噪 (M13.3-L4)。
- **每帧对齐动作**: 第 t 帧的生成 cross-attend 第 t 步动作 (M10.2 cross-attn)。
- **CFG 式控制**: 放大动作的影响 (M13.3-L3 的 guidance 思想)。

> 又一次: **没有新机制**。交互世界模型 = 视频扩散 (M13.4) + 动作条件 (M13.3 的条件注入)。你 N1 的玩具世界模型已经是这个结构的最小版: `(state, action) → Δstate`, 只是状态是 2D 点而非图像。

---

## 2. 三条代表路线 (批判式扫描, 接 M9)

- **Genie (DeepMind 2024)**: 从无标注视频**自学**潜在动作 (latent actions), 生成可玩的 2D 世界。亮点: 不需要动作标签, 从视频里无监督发现「可控的动作维度」。
- **Vid2World**: 把预训练视频扩散模型**改造**成动作条件的世界模型 (复用 M13.4 的底座 + 加动作)。亮点: 站在视频大模型肩上。
- **AVID / 动作条件扩散**: 在扩散续帧上显式加动作通道, 做交互预测。

> 共同主题: **复用强大的视频生成底座 (M13.4) + 想办法接上动作**。区别在动作从哪来 (标注 vs 自学) 和怎么接。读这些论文用 M9.3 的批判框架: 问「动作信号哪来的」「长程一致怎么保证」「物理对不对」。

---

## 3. 可玩世界的硬难题 (gap 富矿)

让世界「可玩且一致」非常难, 全是 open 问题:
- **长程一致** (接 M13.4-L3): 走出房间再回来, 房间还在吗? 需要**持久状态/记忆**, 而扩散续帧默认没有。
- **动作-后果对齐**: 按「右」真的向右、力度对不对 —— 学准动作语义不易。
- **可控性 vs 真实性**: 加太强动作条件 → 画面崩 (过约束); 太弱 → 不听控制。
- **实时性**: 可玩要求实时生成 (接 M13.4-L4 少步采样/M13.2)。

> 这些是世界模型最热的研究前沿 (2026)。每一个都是潜在的 idea: 给扩散世界模型加显式记忆、用动作的物理先验正则、把 M13.2 少步采样用于实时交互…… 你的 N2 会量化「长程一致」这个最难的尺度。

---

## 4. 为什么这是通往具身的桥 (通 L4/M11)

可玩世界模型 = 机器人的「想象空间」:
- 机器人在世界模型里**预演**: 「我这样伸手, 杯子会不会倒」, 选好动作再真做 (省真实试错)。
- 这就是 **model-based 具身智能**: 世界模型 + 规划 = 不靠海量真机数据也能学 (接 M11 sim2real)。

> L4 会把这条桥走完: 世界模型怎么和具身 (M11) 的 world-action models 接上。**世界模型是「生成」(M13) 和「具身」(M11) 的共享地基** —— 这也是为什么 `world_model.py` 和 M11 同源。

---

## 5. 本讲小结 + 通往 L4

- 交互世界模型 = 被动视频模型 + **动作条件** (用 M13.3 的条件注入, 无新机制)。
- 代表路线: Genie (自学潜在动作) / Vid2World (改造视频大模型) / AVID (动作条件扩散), 共性是「复用视频底座 + 接动作」。
- 硬难题 (gap): 长程一致 (需记忆)、动作-后果对齐、可控性vs真实性、实时性。
- 它是通往具身的桥: 世界模型 = 机器人的想象空间 (model-based 具身)。

> **下一讲 L4「世界模型 × 具身」**: 把桥走完 —— 世界模型怎么给机器人当模拟器/规划器, 怎么和 M11 的 world-action models 对接, 以及 sim2real 的世界模型路线。

**动手**: 在 N1/N2 里, 你的玩具世界模型已经是「动作条件」的。想象把 2D 点换成图像帧: 机制完全一样, 只是状态维度爆炸。确认你理解 Genie/Vid2World 和你的 toy 是同一结构的不同规模。
