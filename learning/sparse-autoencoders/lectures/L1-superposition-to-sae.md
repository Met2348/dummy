# L1 · superposition → 为什么需要 SAE

> 22-min lecture · 目标: 理解 superposition 为什么让「读神经元」失败, 以及 SAE 如何解叠加。Module 12.4 开篇。

---

## 0. 回到头号难题

M12.1-L2 讲了 superposition: 网络把多于神经元数的 feature 叠加压缩 → 神经元多义。你 12.1-N1 在真 gpt2 上亲眼看到了多义神经元。这一讲讲 mech interp 对它的主力解法: **稀疏自编码器 (SAE)** —— Anthropic 称之为「显微镜」。

---

## 1. 为什么不能直接读神经元 (回顾 + 深化)

```
   理想: 一个神经元 = 一个概念 (能直接读)
   现实 (superposition): 概念数 >> 神经元数
        → 网络把多个概念叠加到同一组神经元 (利用概念稀疏: 大多不同时出现)
        → 单个神经元多义 (混了多个概念), 读它读不出干净概念
```

> 几何直觉: feature 是激活空间 (d_model 维) 里的方向。要装的 feature 数 > d_model → 方向必须**非正交地挤在一起** (叠加)。读单个神经元 (一个坐标轴) 会同时碰到多个非正交 feature → 多义。**问题不是「概念没编码」, 而是「编码被叠加压缩了, 沿坐标轴读会串台」。**

---

## 2. 解叠加的思路: 找回「干净的方向」

既然 feature 是叠加在一起的方向, 解叠加 = **把这些方向重新分离出来**:

```
   叠加的激活 x (d_model 维, 方向挤在一起)
     → 分解成「许多稀疏激活的特征」(每个特征 = 一个干净方向)
     → 每次只有少数特征激活 (稀疏), 每个特征对应一个概念 (单义)
```

> 关键洞察: **如果概念本身是稀疏的 (任一时刻只有少数概念出现), 那么用一个"过完备 + 稀疏"的字典就能把叠加解开。** 这正是 SAE 干的事 (L2): 学一个比 d_model 宽得多的特征字典, 强制稀疏, 让每个特征单义。

---

## 3. 为什么是「过完备 + 稀疏」

两个要素缺一不可:
- **过完备 (n_features >> d_model)**: 特征数要足够多, 才装得下所有概念 (一个概念一个特征, 不用挤)。
- **稀疏 (L1 惩罚)**: 强制每次只少数特征激活 —— 这逼模型把概念分到**不同**特征上 (而非又叠加回去)。

```
   只过完备不稀疏: 模型可能又把概念叠加进多个特征 (没解开)
   只稀疏不过完备: 特征不够装, 还得叠加
   过完备 + 稀疏:  够多特征 + 逼着分开 → 单义特征
```

> 这是 SAE 设计的灵魂 (L2 逐项推): **用过完备装下概念, 用稀疏逼着分开**。结果是一个比原始神经元多得多、但每个更单义的特征集。你 N1 会看到 SAE 特征比原始神经元**单义得多** (纯度对比)。

---

## 4. SAE = 字典学习的神经版

SAE 本质是经典的**稀疏字典学习 (sparse dictionary learning)** 用在神经网络激活上:
- 字典 = SAE 的解码器权重 (每列是一个特征方向)。
- 编码 = 把激活表示成少数字典原子的稀疏组合。
- 这是信号处理的老思想 (压缩感知/稀疏编码), 你 EE 背景很熟。

> 所以 SAE 不是新发明, 是**把稀疏编码 (你的信号处理) 用到 mech interp**。Anthropic 的贡献是证明它在大模型激活上有效 (找出「金门大桥特征」等单义特征, L3)。你的数学背景在这里直接是优势。

---

## 5. 本讲小结 + 通往 L2

- superposition: 概念叠加压缩 → 神经元多义 → 沿坐标轴读会串台 (问题是编码方式)。
- 解叠加 = 把叠加的方向重新分离; 前提是概念本身稀疏。
- **SAE = 过完备 + 稀疏字典**: 过完备装下概念, 稀疏逼着分开 → 单义特征。
- SAE = 稀疏字典学习的神经版 (接你信号处理/EE 直觉)。

> **下一讲 L2「SAE 原理」**: 把 SAE 的结构和损失逐项讲清 (encode/decode + 重建损失 + L1 稀疏), 你会亲手训一个。

**动手**: 去 `N1`, 在玩具 transformer 激活上训一个 mini-SAE, 看它把激活分解成稀疏特征, 且特征比原始神经元更单义 (纯度对比)。
