# L2 · SAE 原理: 结构与损失逐项

> 24-min lecture · 目标: 把 SAE 的结构和损失函数逐项讲清, 你能亲手训一个。

---

## 0. SAE 就是一个特殊的自编码器

SAE = autoencoder (压缩-重建), 但有两个特殊设计: **隐层更宽 (过完备)** + **稀疏惩罚**。L1 讲了为什么; 这一讲讲具体怎么搭、怎么训。

---

## 1. 结构: encode → decode

```
   输入: 激活 x (d_in 维, 如 d_model)
   encode: f = ReLU(W_enc · (x − b_dec) + b_enc)      特征码 (n_features 维, 远 > d_in)
   decode: x̂ = W_dec · f + b_dec                       重建回 d_in 维
```

逐项:
- **f (特征码)**: n_features 维 (过完备, 如 d_in 的 4-8 倍)。**ReLU** 保证特征非负 (特征「激活」或「不激活」, 像神经元放电)。
- **W_dec (字典)**: 每一**列**是一个特征方向 (字典原子)。重建 = 这些方向的稀疏加权和。
- **b_dec (中心)**: 先减再加, 把数据中心化 (稳定训练)。

> 你 `sae.py` 的 `build_sae` 就是这个结构。**特征码 f 是核心产物**: 它把叠加的 x 分解成「哪些干净特征激活了多少」。读 f 比读 x (原始神经元) 单义得多。

---

## 2. 损失: 重建 + 稀疏 (逐项)

```
   loss = ||x − x̂||²        ← 重建损失: 特征要能还原原激活 (没丢信息)
        + λ · ||f||₁         ← 稀疏损失 (L1): 逼大多数特征 = 0 (每次只少数激活)
```

两项的拉锯:
- **重建项**: 逼 SAE 保留 x 的所有信息 (否则特征没意义)。
- **稀疏项 (L1)**: 逼特征码稀疏。**L1 范数 (绝对值和) 是关键** —— 它在 0 处有尖角, 倾向把小值压到**恰好 0** (而非 L2 的「都变小」)。结果: 每个样本只有少数特征非零。
- **λ (稀疏系数)**: 旋钮。大 → 更稀疏更单义但重建差; 小 → 重建好但特征可能又叠加。

> 这就是 SAE 的全部数学。**重建保证「没丢信息」, L1 保证「分得开」**。你 N1 会调 λ 看「稀疏度 vs 重建质量」的权衡 (接 9.4)。L1 选择不是随意 —— 它是「稀疏」的数学实现 (你 EE 的压缩感知里 L1 诱导稀疏是核心定理)。

---

## 3. 训练与读取

```
   训练: 收集大量激活 (跑模型过语料) → 在激活上训 SAE (上面的 loss)
   读取: 把任意激活 x 过 encode → 看哪些特征激活 (f 的非零项)
        → 每个特征是什么? 找它的「最大激活样本」(L3, 哪些输入最点燃它)
```

> 关键: SAE 在**激活上**训练 (不是在原任务上)。它是一个「事后」装在某层的显微镜, 不改原模型。你 N1 在玩具激活上训, N2 找每个特征对应什么概念 (最大激活样本)。

---

## 4. 实践要点 (会踩的坑)

- **死特征 (dead features)**: 有些特征从不激活 (L1 太强或初始化差)。要监控、重置。
- **特征分裂 (feature splitting)**: 一个概念被拆到多个特征 (n_features 太大)。是特征还是噪声要判断。
- **重建-稀疏权衡**: λ 要调到「够稀疏又重建好」的甜点 (接 9.4 消融)。
- **评估难**: 「特征真单义吗」没有完美指标 (L4 的争议)。

> 这些是训真实 SAE 的工程现实。你 N1 的 mini-SAE 会遇到缩小版 (稀疏度、死特征)。**SAE 好训但「评估它好不好」很难** —— 这是 L4 和这个领域的 open 问题。

---

## 5. 本讲小结 + 通往 L3

- SAE 结构: **encode (ReLU, 过完备 f) → decode (字典 W_dec 重建)**; f 是核心产物。
- 损失: **重建 (保信息) + L1 稀疏 (逼分开)**; L1 诱导稀疏 (你的压缩感知)。
- λ 是旋钮: 稀疏 vs 重建权衡 (接 9.4)。
- 坑: 死特征 / 特征分裂 / 评估难。
