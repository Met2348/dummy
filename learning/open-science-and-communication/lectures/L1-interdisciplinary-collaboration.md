# L1 · 跨学科合作方法论: 默认假设/术语/评审标准的落差, 与共享词汇表的搭建 (Interdisciplinary Collaboration Methodology)

> 30-min lecture · 目标: 搞清楚"跨学科合作为什么总是比预想中难"这件事背后真正的三个病灶(默认假设落差/术语落差/评审标准落差), 并学会一套具体可执行的方法——共建一张双向术语对照表——把"合作卡壳"变成"合作可以真的推进"。
> 关键区分: **本仓库`research-team-operations` L5「跨专业背景团队协作」教的是研究者和工程师/PM/设计师这类**功能角色**之间的协作(比如工业界研究团队里, 研究科学家和产品经理谁对哪类决定拍板), 关注的是同一家公司/团队内部不同职能的边界划分; 本讲教的是研究者和**另一个学术学科**的研究者之间的合作(比如你是NLP方向, 合作者是认知科学家/社会学家/生物学家), 关注的是两套完全不同的知识生产传统——默认假设不同、术语不同、连"什么样的证据才算数"这个评审标准本身都不同。前者是同一个项目里"分工"的问题, 后者是两个学科"世界观"本身不兼容的问题, 难度和解法都不是一回事。**

## 0. 一次真实会容易发生的对话事故

想象这样一个场景: 你是NLP方向的博士生, 手头的研究涉及"大语言模型在处理长文本时的注意力分布是否和人类阅读时的注意力分布类似", 于是你联系了心理学系一位研究人类注意力机制的教授, 约了第一次跨学科讨论会。会议开始十分钟, 你已经隐约感觉到哪里不对劲——你说"我们模型的attention权重在这几层明显更集中", 对方教授愣了一下问"你说的attention, 是指被试主观报告的注意力分配, 还是某种行为学指标(比如眼动追踪的注视时长)?"。你这才意识到, 你说的"attention"(注意力机制, attention mechanism)是Transformer架构里一个具体的数学对象——一组通过softmax归一化、决定每个token该"看"多重要其他token的权重矩阵; 而对方教授说的"attention"(注意力)是心理学里一个研究了上百年的认知构念(construct)——描述人类认知资源如何在多个刺激之间分配、可以被眼动追踪或反应时实验间接测量, 但这个构念本身有非常具体、经过几十年争论才逐渐稳定下来的操作化定义, 和你模型里那组权重矩阵除了共享一个英文单词之外, 未必有任何直接对应关系。**这不是谁比谁更懂科学的问题, 是两个学科各自独立发展出来的同一个词, 从一开始就指向不同的东西。**

这次对话事故不是运气不好, 而是**跨学科合作的默认发生形态**——两边都带着"这个词大家都懂"的假设走进会议室, 结果发现连最基础的共同语言都需要重新协商。本讲要讲清楚的, 就是这类落差具体分几种、每种怎么被识别、以及怎么用一套具体的方法(不是"多沟通"这种正确但没用的建议)真正把落差补上。

先定义几个术语, 免得后面讲起来空对空:
- **默认假设(default assumption)**: 一个学科内部所有人都默认成立、通常不会被明说、也很少被质疑的前提。比如ML研究者默认"更多数据+更多算力通常能带来更好的结果"; 认知科学研究者默认"一个现象必须用精心控制混淆变量的小样本实验才算被严谨验证过", 两边都觉得自己的假设"显然成立", 但两条假设之间未必兼容。
- **术语落差(terminological gap)**: 同一个词在不同学科里指向不同(有时完全无关)的概念, 上面的"attention"就是最典型的例子。
- **评审标准落差(evaluation-standard gap)**: 不同学科判断"这个结论站得住脚吗"用的是完全不同的证据标准——ML会议看重benchmark上的数字提升(哪怕只解释"提升了多少", 不解释"为什么提升"); 心理学/认知科学期刊看重效应量(effect size)、统计显著性检验、以及**预先注册的假设**(这正是本专题L3要展开的话题)。

## 1. 三类落差分别长什么样

### 落差① 默认假设——两边都觉得"这不是明摆着的吗"

默认假设最难被发现, 恰恰因为它**从不会被说出口**——它藏在"我们当然应该这么做"这种潜台词里。几个典型对撞:

```
学科                     默认假设(常常自己都没意识到在假设)
──────────────         ──────────────────────────────────
ML/NLP研究者            规模化(scale)本身是有效的解释——"参数更多/数据更多/
                       算力更多, 效果通常会更好", 不需要先给出一个因果机制
认知科学/心理学研究者    一个现象必须能被一个精心设计、排除了混淆变量的对照
                       实验证明, 否则"效果好"不能证明"机制对"
经济学研究者            个体行为可以用理性决策模型(rational agent)近似, 
                       即便知道现实中人不完全理性, 这仍是默认的建模起点
生物学研究者            一个结论需要有可解释的分子/细胞机制支撑, 单纯"预测准确率
                       高"不能替代"机制被理解"这件事
```

当一个ML研究者和一个认知科学家合作, 前者习惯的论证方式是"我们的模型在某个任务上比基线高5个点, 所以这个设计是有效的", 后者第一反应往往是"高5个点说明了什么因果机制? 有没有排除是不是训练数据分布碰巧更接近测试集这种混淆因素?"——**这不是抬杠, 是两个学科对"什么才算解释清楚了一件事"有完全不同的默认门槛**, 如果不主动挑明, 双方很容易把对方的追问理解成"不信任我的工作"或者把对方的论证理解成"证据不够扎实"。

### 落差② 术语——同一个词, 不同的所指

除了开篇的"attention"例子, 类似的坑还有很多: "significant"(显著)在统计学里有严格的假设检验含义(p值小于某个阈值), 但在日常英语和很多非统计背景学科的口语用法里就是"重要/明显"的意思, 一个跨学科合作者说"这个结果不显著"如果没说清楚是统计学意义还是口语意义, 极容易被误读; "model"(模型)在ML里通常指一个可以被训练、参数化的函数(比如一个神经网络), 在物理学/生物学里"理论模型"往往指一套用于解释因果机制的数学结构, 未必需要"训练"这个动作; "robust"(鲁棒)在ML语境里常指"对输入扰动/分布偏移不敏感", 但在统计学里"稳健统计(robust statistics)"特指对异常值/偏离正态假设不敏感的一整套具体方法, 两者部分相关但绝不等同。

### 落差③ 评审标准——"这个结论算不算站得住脚"

这一条往往是合作论文写成之后才集中爆发的问题——你和合作者辛苦跑完了实验, 到了要投稿的阶段才发现, ML会议的审稿人想看的是"有没有和最强基线(SOTA)比较、有没有做消融实验(ablation)拆解每个组件的贡献", 而心理学/认知科学期刊的审稿人想看的是"假设是不是**在数据收集之前**就已经写清楚了(呼应本专题L3要讲的预注册)、效应量报告是否完整、有没有多重比较校正"。如果两边合作者从一开始没有谈拢"这篇论文最终打算投给哪个共同体的评审标准", 很可能出现"稿子按ML标准写完, 拿去投心理学期刊被批评方法论不够严谨"或者反过来"稿子按心理学标准写得四平八稳, 投ML会议被批评缺乏技术新意"的两难局面。

## 2. 解法: 共建一张双向术语对照表, 而不是单方面迁就

面对上面三类落差, 真正管用的解法不是"开会时多解释几句"(解释一次, 下次遇到新词还是会卡壳), 而是**从合作一开始就共同维护一张双向术语对照表**——注意"双向"两个字: 不是让认知科学家单方面学会ML黑话, 也不是让ML研究者单方面迁就对方的表达习惯, 而是把双方各自的核心术语都摆到桌面上, 写清楚对方领域里最接近的等价说法、以及"这两个说法并不完全对等"的具体备注:

```
NLP方术语                认知科学方等价/近似说法              备注(是否完全对等)
────────────           ──────────────────────           ────────────────────────
attention(注意力机制)     visual/selective attention        不对等: 前者是softmax权重矩阵,
                        (视觉/选择性注意力构念)              后者是行为学可测量的认知资源分配
significant(显著)        p < .05 假设检验意义上的"显著"       半对等: 双方都可以用统计显著性,
                                                          但ML论文常省略效应量报告
robust(鲁棒)             稳健统计(robust statistics,         部分重叠: 都关心"对异常/扰动
                        对异常值/偏态分布不敏感)              不敏感", 但具体方法完全不同
benchmark(基准数据集)     stimulus set/task battery         近似对等: 都是"标准化的测试材料",
                        (标准化刺激集/任务组)                但ML benchmark通常规模大得多
```

这张表的价值不在于"翻译得多精确"(有些概念本来就不完全对等, "备注"栏正是用来标注这种不对等程度, 而不是假装两边说的是同一件事), 而在于给双方一个**从第一次讨论就开始逐步积累、随时可查的共同锚点**——遇到新词就补一行, 而不是每次都临场重新解释一遍。这也是这份表和"多解释几遍"最本质的区别: 后者的效果会随对话次数线性重复消耗, 前者是一次性建设、长期复用的共享基础设施。

## 3. 完整走查示例

场景延续开篇: 你(NLP方向)和一位认知科学教授合作, 研究"大语言模型的注意力分布和人类阅读注意力是否存在结构相似性"。

```
第一次讨论会暴露的落差:
  假设落差: 你默认"模型表现好=某种机制成立"; 对方默认"必须有对照实验排除
            混淆因素才能下结论"。
  术语落差: "attention"在两边指向完全不同的对象(见0节)。
  评审标准落差: 你打算投ML会议(看重和基线比较+消融); 对方打算投认知科学
              期刊(看重预注册假设+效应量)。

用共享词汇表 + 提前谈拢评审标准重建:
  第一步: 建立术语表, 明确区分"attention weights"(模型侧, 你来定义)和
          "selective attention"(人类侧, 对方来定义), 后续讨论各自用精确
          的英文术语, 不再笼统说"attention"。
  第二步: 谈拢这篇论文最终投稿目标——决定投一个同时认可两种证据类型的
          交叉学科venue(比如某些认知科学与AI交叉的workshop), 事先约定
          论文里既要有ML式的量化比较, 也要有认知科学式的预注册假设检验,
          而不是等写完了才发现两边审稿人根本看不到自己认可的那部分证据。
  第三步: 每次讨论会结束前留5分钟, 检查这次讨论里有没有出现新词需要
          补进术语表——这一步最容易被省略, 却是让词汇表真正"活"起来、
          而不是建一次就搁置的关键动作。
                ↓
  结果: 三个月后论文投出, 两边合作者都能在稿子里认出"自己领域的证据
        标准被认真对待了", 而不是单方面按一套标准写完让另一边将就。
```

## 4. 常见误区

**误区①: 假设对方"应该"听得懂自己的黑话, 不主动定义术语就直接使用。** 某NLP博士生和一位社会学教授讨论"我们的模型有很强的bias(偏见)", 对方教授理解成了社会学语境下"结构性偏见/歧视"的严重指控, 而说话人本意只是统计学意义上的"系统性偏差(bias-variance分解里的bias)", 两边为此产生了不必要的误会和防御性对话, 花了半次会议时间才厘清根本不是在说同一件事。

**误区②: 把自己学科的评审标准当成唯一客观标准, 单方面要求对方接受。** 某跨学科合作项目里, ML一方坚持"我们只要benchmark数字好看就该发表", 完全不理会认知科学合作者提出的"需要报告效应量和置信区间"的要求, 认为这是"多此一举的形式主义"——结果论文投了两个不同共同体的venue都被拒: ML审稿人觉得缺乏技术新意, 认知科学审稿人觉得方法论证据不够扎实, 两头不讨好, 根源就是没有从一开始就谈拢"这篇论文打算符合谁的评审标准"。

**误区③: 术语表建一次就搁置, 不再更新。** 某团队在项目启动时认真做了一次术语对齐会议, 双方都觉得"以后应该没问题了", 但三个月后合作深入到新的子话题时, 新出现的专业词汇(比如"in-context learning"和认知科学"类比推理(analogical reasoning)"之间是否等价)完全没有被纳入讨论, 团队又退回到"各说各话、靠上下文硬猜"的老路上——一次性的对齐会议解决不了持续深化合作过程中不断冒出的新落差, 术语表需要被当成一份随合作进程持续更新的活文档, 而不是一次性交付物。

## 5. 经典参考

- **National Academies of Sciences, Engineering, and Medicine, 《Facilitating Interdisciplinary Research》**(National Academies Press, 2004/2005, DOI: 10.17226/11153): 这份由美国国家科学院/工程院/医学院联合委员会撰写的报告, 系统总结了跨学科研究(interdisciplinary research, IDR)的定义、驱动因素以及研究者/机构/资助方各自能采取的具体促进措施, 是理解"跨学科合作为什么天然更难"这一现象的权威参考, 本讲对"三类落差"的拆解正是在这份报告揭示的一般性困境基础上, 针对NLP/ML方向的具体场景做的进一步细化。
- 也见本仓库`research-team-operations`(9.13)专题L5「跨专业背景团队协作」——那一讲教的是研究者和工程师/PM/设计师这类**同一团队内不同职能角色**之间的术语对齐与决策边界划分(比如"这个功能什么时候能ready"这类工业界场景); 本讲教的是研究者和**另一个学术学科**的合作者之间, 从默认假设到评审标准的更深层落差, 两讲方法论上都用了"双向术语对照表"这个工具, 但应用场景和落差的根源完全不同, 不重复彼此内容。

## 6. 本讲小结 + 通往 L2

- 跨学科合作卡壳, 通常不是因为"沟通不够", 而是三类具体、可被识别的落差在起作用: 默认假设落差(彼此都觉得"显然如此"却互相矛盾)、术语落差(同一个词指向不同对象)、评审标准落差(什么样的证据才算站得住脚, 双方标准不同)。
- 解法是从合作一开始就共建一张**双向**的术语对照表, 并明确写出"备注"标注哪些概念并不完全对等, 而不是假装翻译总能做到一一对应。
- 这张表需要被当成随合作深入持续更新的活文档, 每次讨论会结束前花几分钟检查是否有新词要补入, 而不是开一次对齐会议就一劳永逸。

> **下一讲 L2「科学传播与公众沟通」**: 本讲教你怎么和另一个学科的专家对齐语言; 下一讲要面对一个门槛更高的听众——完全没有专业背景的公众和媒体。如果说跨学科合作至少还有"对方也是研究者"这个共同基础, 那么面向公众传播时, 你需要重新学习怎么在**不产生误导**的前提下, 把复杂的研究结论讲清楚。

**动手**: 想一个你现在(或未来可能)会遇到的跨学科合作场景(哪怕只是假想, 比如"如果要和心理学/经济学/生物学的研究者合作, 我的研究话题会涉及哪些容易混淆的词"), 写出至少5组"你的领域术语 ⇄ 对方领域等价说法 ⇄ 备注(是否完全对等)"，正式建立你自己的第一版术语对照表。
